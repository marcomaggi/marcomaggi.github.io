8.12 Convenience signal two

We can use the signal–2 exceptional–condition object–type to signal some special test condition. We can instantiate this type and also derive new types from it. To catch this exceptional condition we can do: 

cce_location_t        L[2];

if (cce_location(L)) {
  if (cctests_condition_is_signal_2(cce_condition(L))) {
    CCTESTS_PC(cctests_condition_signal_2_t, C, cce_condition(L));
    do_something_with(C);
  }
  cce_run_catch_handlers_final(L);
} else {
  do_something(L);
  cce_run_body_handlers(L);
}
Struct Typedef: cctests_descriptor_signal_2_t

Type of data structure representing the exceptional–condition descriptor. It has the following public fields:

cce_descriptor_t descriptor

The condition descriptor’s base values.

The parent of this type descriptor is the one referenced by cctests_descriptor_signal_ptr, Convenience signal.

Struct Typedef: cctests_condition_signal_2_t

Type of data structure representing the exceptional–condition object. It has the following public fields:

cctests_condition_signal_t signal

The condition object’s base values.

Function: void cce_descriptor_set_parent_to(cctests_condition_signal_2_t) (cce_descriptor_t * const D)

Set the parent of the descriptor referenced by D to the descriptor of the exceptional–condition object–type cctests_condition_signal_2_t.

Function: void cctests_condition_init_signal_2 (cctests_condition_signal_2_t * C)

Initialisation function for the condition object. This function is meant to be called from the initialisation function of sub–types of cctests_condition_signal_2_t.

Function: cce_condition_t const * cctests_condition_new_signal_2 (void)

Return a pointer to exceptional–condition object of type cctests_condition_signal_2_t made opaque in the type cce_condition_t.

Function: bool cctests_condition_is_signal_2 (cce_condition_t const * C)

Return true if C if of type cctests_condition_signal_2_t; otherwise return false.

Sub–typing example

To define a sub–type of cctests_condition_signal_2_t we can copy the code in the files: 

condition-signal-2-subtype.c
condition-signal-2-subtype-headear.h
condition-signal-2-subtype-body.c

under the tests directory of the source distribution; the code defines a new condition–object type my_condition_signal_2_subtype_t.
