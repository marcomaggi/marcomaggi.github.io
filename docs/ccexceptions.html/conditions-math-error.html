6.14.1 Error computing a mathematical expression

The mathematical error exceptional–condition object–type has the purpose of describing an exceptional condition caused by attempting to compute an invalid mathematical expression.

There should be only one mathematical error exceptional–condition object: the one predefined by CCExceptions and built into the library as statically allocated structure. Subtyping from this object–type is fine.

All the following definitions are accessible from the header file ccexceptions.h.

Struct Typedef: cce_descriptor_math_error_t

Structure type representing the mathematical error exceptional–condition object–type descriptor. This descriptor has the runtime error descriptor as parent. It has the following public fields:

cce_descriptor_t descriptor

Core values of the exceptional–condition object–type descriptor.

Struct Typedef: cce_condition_math_error_t

Structure type representing a mathematical error exceptional–condition object. It has the following public fields:

cce_condition_runtime_error_t runtime_error

Core values of the exceptional–condition object–type descriptor.

Function: void cce_condition_init_math_error (cce_condition_math_error_t * C)

Initialise an already allocated exceptional–condition object. This function exists to allow subtyping from cce_condition_math_error_t.

Function: cce_condition_t const * cce_condition_new_math_error (void)

Return a pointer to the statically allocated structure representing the predefined mathematical error exceptional–condition object.

Function: bool cce_condition_is_math_error (cce_condition_t const * C)

Return true if C references a mathematical error exceptional–condition object; otherwise return false.

Generic Macro: cce_condition_math_error_t const * cce_condition_math_error (cce_destination_t L)
Generic Macro: cce_condition_math_error_t const * cce_condition_math_error (cce_condition_t * C)
Generic Macro: cce_condition_math_error_t const * cce_condition_math_error (cce_condition_t const * C)

This generic macro dispatches its expansion according to its argument’s type. Cast a pointer to cce_condition_math_error_t.

This macro performs compile–time type–checking so that the cast operation is applied only to values of suitable types; we can apply this macro to pointers of all the types defined by CCExceptions for which it makes sense.

Dynamic Constant: cce_descriptor_math_error_t const * const cce_descriptor_math_error_ptr

Pointer to a statically allocated structure representing the mathematical error exceptional–condition object–type descriptor.

Dynamic Constant: cce_condition_math_error_t const * const cce_condition_math_error_ptr

Pointer to a statically allocated structure representing the mathematical error exceptional–condition object.
