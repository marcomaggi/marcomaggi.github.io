6.9 The logic error exceptional–condition object

The logic error exceptional–condition object–type has the purpose of describing an exceptional condition caused by a logic error; these are mostly synchronisation errors like failures to open a file or allocate memory.

There should be only one logic error exceptional–condition object: the one predefined by CCExceptions and built into the library as statically allocated structure. Subtyping from this object–type is fine.

All the following definitions are accessible from the header file ccexceptions.h.

Struct Typedef: cce_descriptor_logic_error_t

Structure type representing the logic error exceptional–condition object–type descriptor. This descriptor has the error descriptor as parent. It has the following public fields:

cce_descriptor_t descriptor

Core values of the exceptional–condition object–type descriptor.

Struct Typedef: cce_condition_logic_error_t

Structure type representing a logic error exceptional–condition object. It has the following public fields:

cce_condition_error_t error

Core values of the exceptional–condition object.

Function: void cce_condition_init_logic_error (cce_condition_logic_error_t * C)

Initialise an already allocated condition object. This function exists to allow subtyping from cce_condition_logic_error_t.

Function: cce_condition_t const * cce_condition_new_logic_error (void)

Return a pointer to the statically allocated structure representing the predefined logic error exceptional–condition object.

Function: bool cce_condition_is_logic_error (cce_condition_t const * C)

Return true if C references a logic error exceptional–condition object; otherwise return false.

Generic Macro: cce_condition_logic_error_t const * cce_condition_logic_error (cce_destination_t L)
Generic Macro: cce_condition_logic_error_t const * cce_condition_logic_error (cce_condition_t * C)
Generic Macro: cce_condition_logic_error_t const * cce_condition_logic_error (cce_condition_t const * C)

This generic macro dispatches its expansion according to its argument’s type. Cast a pointer to cce_condition_logic_error_t.

This macro performs compile–time type–checking so that the cast operation is applied only to values of suitable types; we can apply this macro to pointers of all the types defined by CCExceptions for which it makes sense.

Dynamic Constant: cce_descriptor_logic_error_t const * const cce_descriptor_logic_error_ptr

Pointer to a statically allocated structure representing the logic error exceptional–condition object–type descriptor.

Dynamic Constant: cce_condition_logic_error_t const * const cce_condition_logic_error_ptr

Pointer to a statically allocated structure representing the logic error exceptional–condition object.

Subtyping example

To define a subtype of cce_condition_logic_error_t we can copy the code in the files: 

condition-subtyping-logic-error.c
condition-subtyping-logic-error-header.h
condition-subtyping-logic-error-body.c

under the tests directory of the source distribution; the code defines a new exceptional–condition object–type my_condition_logic_error_subtype_t.
