5.1.3.2 Exception handlers for standalone buffer instances

A standalone instance is struct instance dynamically allocated on the heap. Exception handlers allow us to trigger deleteisation of buffer instances upon leaving a location context, (ccexceptions)The location handlers API.

The following definitions are accessible in the header file ccstrings.h.

Function: void ccstr_init_and_register_delete_buffer_clean_handler (cce_destination_t L, cce_clean_handler_t * H, ccstr_buffer_t const * B)

Associate the buffer referenced by B to the handler referenced by H in the context of the location L.

Function: void ccstr_init_and_register_delete_buffer_error_handler (cce_destination_t L, cce_error_handler_t * H, ccstr_buffer_t const * B)

Associate the buffer referenced by B to the handler referenced by H in the context of the location L.

Preprocessor Macro: void ccstr_init_and_register_delete_buffer_handler (cce_destination_t L, H, ccstr_buffer_t const * B)

Associate the buffer referenced by B to the handler referenced by H in the context of the location L. If the type of the pointer H is:

cce_clean_handler_t *

The macro use expands into a call to ccstr_init_and_register_delete_buffer_clean_handler().

cce_error_handler_t *

The macro use expands into a call to ccstr_init_and_register_delete_buffer_error_handler().
