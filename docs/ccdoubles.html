1 Overview of the package

This document describes version 0.1d0 of CCDoubles, a C99 language library implementing simple routines for vectors and matrices of double and double complex values. This library is meant to be used as companion for more sophisticated libraries such as CBLAS and LAPACKE.

CCDoubles installs the single header file ccdoubles.h. All the function names in the API are prefixed with ccdoubles_; all the preprocessor macro names are prefixed with CCDOUBLES_; all the type names are prefixed with ccdoubles_ and suffixed with _t.

  • overview linking: Linking code with the library.

Up: overview

1.1 Linking code with the library

This package installs a data file for pkg-config, so when searching for the installed library with the GNU Autotools, we can:

  • Install the file pkg.m4 from /usr/share/aclocal into the source tree of the package, for example under meta/autoconf/.
  • Include pkg.m4 in the template by adding the following line to acinclude.m4: 
              m4_include(meta/autoconf/pkg.m4)
  • Just add the following macro use to configure.ac: 
              PKG_CHECK_MODULES([CCDOUBLES],[ccdoubles >= 0.1])

    which will set the variables CCDOUBLES_LIBS and CCDOUBLES_CFLAGS.

Alternatively we can use the raw GNU Autoconf macros: 

     AC_CHECK_LIB([ccdoubles],[ccd_version_string],,
       [AC_MSG_FAILURE([test for CCDoubles library failed])])
     AC_CHECK_HEADERS([ccdoubles.h],,
       [AC_MSG_FAILURE([test for CCDoubles header failed])])

3.1.1 Functions acting on scalar complex values

The C99 language standard library defines common operations on scalar double complex numbers; in addition CCDoubles defines convenience functions to perform the same.

— Function: double complex ccdoubles_cplx_mul (double complex O1, double complex O2)

Even though C99 supports the multiplication operation as: 

          #include <ccdoubles.h>
          
          double complex  O1 = CCDOUBLES_CPLX(1.2, 3.4);
          double complex  O2 = CCDOUBLES_CPLX(5.6, 7.8);
          double complex  R  = O1 * O2;
          
          printf("R = %lf%+lfi\n", creal(R), cimag(R));
          -| R = -19.800000+28.400000i

this function computes and returns the product between the operands. Example: 

          #include <ccdoubles.h>
          
          double complex  O1 = CCDOUBLES_CPLX(1.2, 3.4);
          double complex  O2 = CCDOUBLES_CPLX(5.6, 7.8);
          double complex  R;
          
          R = ccdoubles_cplx_mul(O1, O2);
          
          printf("R = %lf%+lfi\n", creal(R), cimag(R));
          -| R = -19.800000+28.400000i
— Function: double complex ccdoubles_cplx_div (double complex O1, double complex O2)

Even though C99 supports the division operation as: 

          #include <ccdoubles.h>
          
          double complex  O1 = CCDOUBLES_CPLX(1.2, 3.4);
          double complex  O2 = CCDOUBLES_CPLX(5.6, 7.8);
          double complex  R  = O1 / O2;
          
          printf("R = %lf%+lfi\n", creal(R), cimag(R));
          -| R = 0.360521+0.104989i

this function computes and returns the division between the operands. Example: 

          #include <ccdoubles.h>

          double complex  O2 = CCDOUBLES_CPLX(5.6, 7.8);

          printf("R = %lf%+lfi\n", creal(R), cimag(R));

— Function: double complex ccdoubles_cplx_neg (double complex O)

Even though C99 supports the negation operation as: 

          #include <ccdoubles.h>

3.1.2 Preprocessor macros

— Preprocessor Macro: double complex CCDOUBLES_CPLX (double REAL, double IMAG)

Build and return a new double complex value. Expands into: 

          (((double)(REAL)) + ((double)(IMAG)) * _Complex_I)
NOTE The latest C Language Standard defines the same operation with the preprocessor macro CMPLX(REAL, IMAG).

— Preprocessor Macro: double complex Z (double real, double imag)

Return a new double complex.

This macro is an alias for CCDOUBLES_CPLX but it is defined only if the header ccdoubles.h is included with the C preprocessor symbol CCDOUBLES_ENABLE_SHORT_MACROS already defined.

— Preprocessor Macro: double Re (double complex Z)
— Preprocessor Macro: double Im (double complex Z)

Return the real or imaginary part of the complex number Z by applying creal or cimag to it.

This macro is defined only if the header ccdoubles.h is included with the C preprocessor symbol CCDOUBLES_ENABLE_SHORT_MACROS already defined.

— Preprocessor Macro: double * MREF (M)

Expand into a pointer to the first element of a bidimensional array: 

          MREF(M) ==> &M[0][0]

This macro is defined only if the header ccdoubles.h is included with the C preprocessor symbol CCDOUBLES_ENABLE_SHORT_MACROS already defined.

4.1 Introduction to vector operations

The vectors on which CCDoubles operates are meant to be contiguous arrays of double and double complex values, defined as follows: 

     #include <ccdoubles.h>
     #undef NSLOTS
     #define NSLOTS          2
     
     double          RV[NSLOTS] = { 1.2, 3.4 };
     double complex  CV[NSLOTS] = {
       CCDOUBLES_CPLX(1.2, 3.4),
       CCDOUBLES_CPLX(5.6, 7.8)
     };

or as follows with dynamic size (which forbids hard–coded initialisation of slots): 

     unsigned        nslots = 1;
     double          RV[nslots];
     double complex  CV[nslots];

The memory layout of a vector is equal to the memory layout of a matrix with a single row defined as follows: 

     unsigned        nrows = 1;
     unsigned        ncols = 3;
     double          RM[nrows][ncols];
     double complex  CM[nrows][ncols];

and equal to the memory layout of a matrix with a single column defined as follows: 

     unsigned        nrows = 3;
     unsigned        ncols = 1;
     double          RM[nrows][ncols];
     double complex  CM[nrows][ncols];

All the functions have simple implementation involving an iteration over the vector's slots using restrict pointers; examples for real valued vectors: 

     void
     ccdoubles_real_vector_add (unsigned nslots,
                                double * restrict result,
                                double * restrict operand1,
                                double * restrict operand2)
     {
       for (unsigned i=0; i<nslots; ++i) {
         result[i] = operand1[i] + operand2[i];
       }
     }
     void
     ccdoubles_real_vector_sin (unsigned nslots,
                                double * restrict result,
                                double * restrict operand)
     {
       for (unsigned i=0; i<nslots; ++i) {
         result[i] = sin(operand[i]);
       }
     }

examples for complex valued vectors: 

     void
     ccdoubles_cplx_vector_add (unsigned nslots,
                                double complex * restrict result,
                                double complex * restrict operand1,
                                double complex * restrict operand2)
     {
       for (unsigned i=0; i<nslots; ++i) {
         result[i] = operand1[i] + operand2[i];
       }
     }
     void
     ccdoubles_cplx_vector_sin (unsigned nslots,
                                double complex * restrict result,
                                double complex * restrict operand)
     {
       for (unsigned i=0; i<nslots; ++i) {
         result[i] = csin(operand[i]);
       }
     }

CCDoubles functions can be called as follows, see how the arrays are passed as arguments: 

     #include <ccdoubles.h>
     #undef NSLOTS
     #define NSLOTS         2
     
     double  V[NSLOTS] = { 1.2, 3.4 };
     double  R[NSLOTS];
     
     ccdoubles_real_vector_sin(NROWS, NCOLS, R, V);

the result is stored the array referenced by R, overwriting the old contents.

4.2.6 Special real vector operations

— Function: double ccdoubles_real_vector_scalar_product (unsigned nslots, const double * operand1, const double * operand2)

Perform the scalar product between the operands and return the result. The implementation is: 

          double        result = 0.0;
          for (unsigned i=0; i<nslots; ++i) {
            result += operand1[i] * operand2[i];
          }
— Function: void ccdoubles_real_vector_scalar_mul (unsigned nslots, double * result, double lambda, const double * operand)

Multiply the scalar lambda with all the slots in the operand.

— Function: void ccdoubles_real_vector_linear_combination (unsigned nslots, double * result, double alpha, double * operand1, double beta, double * operand2)

Perform the linear combination of the vector operands and the given scalars. The implementation is: 

          for (unsigned i=0; i<nslots; ++i) {
            result[i] = alpha * operand1[i] + beta * operand2[i];
          }
— Function: void ccdoubles_real_vector_linspace (unsigned nslots, double * result, double start, double past)

Fill the result vector with nslots evenly–spaced steps from start to past. It is implemented as follows: 

          double        step = (past - start) / ((double)nslots);
          for (unsigned i=0; i<nslots; ++i) {
            result[i] = ((double)i) * step + start;
          }

As example, the code: 

          unsigned    nslots = 10;
          double      R[nslots];
          double      start = 0.0;
          double      past  = 10.0;
          ccdoubles_real_vector_linspace (nslots, R, start, past);
          ccdoubles_real_vector_print_brackets(stdout, nslots, R);

prints (edited to add newlines): 

          [+0.000000 +1.000000 +2.000000 +3.000000
           +4.000000 +5.000000 +6.000000 +7.000000
           +8.000000 +9.000000]
— Function: void ccdoubles_real_vector_logspace (unsigned nslots, double * result, double start, double past)

Fill the result vector with nslots logarithmically spaced steps from pow(10.0, start) to pow(10.0, past). It is implemented as follows: 

          double        step = (past - start) / ((double)nslots);
          for (unsigned i=0; i<nslots; ++i) {
            result[i] = pow(10.0, ((double)i) * step + start);
          }

As example, the code: 

          unsigned    nslots = 10;
          double      R[nslots];
          double      start = 0.0;
          double      past  = 10.0;
          ccdoubles_real_vector_logspace (nslots, R, start, past);
          ccdoubles_real_vector_print_brackets(stdout, nslots, R);

prints (edited to add newlines): 

          [+1.000000
           +10.000000
           +100.000000
           +1000.000000
           +10000.000000
           +100000.000000
           +1000000.000000
           +10000000.000000
           +100000000.000000

5.1 Introduction to vector operations

The matrices on which CCDoubles operates are meant to be contiguous arrays of double and double complex stored in row–major order, defined as follows: 

     #include <ccdoubles.h>
     #undef NROWS
     #undef NCOLS
     #define NROWS          2
     #define NCOLS          2
     
     double          RM[NROWS][NCOLS] = {
       { 1.2, 3.4 },  /* this is 1st row */
       { 5.6, 7.8 }   /* this is 2nd row*/
     };
     double complex  CM[NROWS][NCOLS] = {
       { CCDOUBLES_CPLX(1.2, 2.3), CCDOUBLES_CPLX(3.4, 4.5) },
       { CCDOUBLES_CPLX(5.6, 6.7), CCDOUBLES_CPLX(7.8, 8.9) }
     };

or as follows with dynamic size (which forbids hard–coded initialisation of slots): 

     unsigned          nrows = 3;
     unsigned          ncols = 4;
     double          RM[nrows][ncols];
     double complex  CM[nrows][ncols];

All the matrix functions have simple implementation based on the corresponding vector function; examples for real valued matrices: 

     void
     ccdoubles_real_matrix_add (unsigned nrows, unsigned ncols,
                                double * restrict result,
                                double * restrict operand1,
                                double * restrict operand2)
     {
       ccdoubles_real_vector_add (nrows * ncols, result,
                                  operand1, operand2);
     }
     void
     ccdoubles_real_matrix_sin (unsigned nrows, unsigned ncols,
                                double * restrict result,
                                double * restrict operand)
     {
       ccdoubles_real_vector_sin (nrows * ncols, result, operand);
     }

examples for complex valued matrices: 

     void
     ccdoubles_cplx_matrix_add (unsigned nrows, unsigned ncols,
                                double complex * restrict result,
                                double complex * restrict operand1,
                                double complex * restrict operand2)
     {
       ccdoubles_cplx_vector_add (nrows * ncols,
                                  result, operand1, operand2);
     }
     void
     ccdoubles_cplx_matrix_sin (unsigned nrows, unsigned ncols,
                                double complex * restrict result,
                                double complex * restrict operand)
     {
       ccdoubles_cplx_vector_sin (nrows * ncols, result, operand);
     }

CCDoubles functions can be called as follows, see how the arrays are passed as arguments: 

     #include <ccdoubles.h>
     #undef NROWS
     #undef NCOLS
     #define NROWS          2
     #define NCOLS          2
     
     double  M[NROWS][NCOLS] = {
       { 1.2, 3.4 },
       { 5.6, 7.8 }
     };
     double  R[NROWS][NCOLS];
     
     ccdoubles_real_matrix_sin(NROWS, NCOLS, &R[0][0], &M[0][0]);

the result is stored the array referenced by R, overwriting the old contents.

— Function: void ccdoubles_real_matrix_linspace (unsigned nrows, unsigned ncols, double * result, double start, double row_past, double col_past)

Fill the result matrix with nrows times ncols evenly–spaced steps from start to row_past in the horizontal direction and from start to col_past in the vertical direction. It is implemented as follows: 

          double        row_step = (row_past - start) / ((double)nrows);
          double        col_step = (col_past - start) / ((double)ncols);
          for (unsigned i=0; i<nrows; ++i) {
            double      row_first = ((double)i) * row_step + start;
            for (unsigned j=0; j<ncols; ++j) {
              result[i * ncols + j] = ((double)j) * col_step + row_first;
            }
          }

          unsigned    nrows = 10;

          double      col_past = 10.0;

                                          start, row_past, col_past);
          ccdoubles_real_matrix_print_brackets(stdout, nrows, ncols,
                                               &R[0][0]);
