1 Overview of the package

Nausicaa/cURL is a distribution of libraries for R6RS Scheme implementations; it provides bindings to the LibcURL library: A tool to transfer data from or to a server, using one of the supported protocols (HTTP, HTTPS, FTP, FTPS, TFTP, DICT, TELNET, LDAP or FILE).

The installed libraries are:

(foreign net curl shared-object)
Exports the single binding curl-shared-object which references cURL's shared library.
(foreign net curl sizeof)
Exports bindings for constant values and cURL data types inspection.
(foreign net curl platform)
Exports a binding for each function in cURL. It implements a one–to–one map between foreign functions and Scheme functions. Most of the Scheme functions have the same name of the foreign ones, so curl_global_init() is available as curl_global_init; consult this documentation for the differences. Details on the platform library.

This library makes the original cURL documentation relevant for this Scheme package; the documentation should be at (URL last verified Mon Nov 16, 2009):

http://curl.haxx.se/libcurl/c/

(foreign net curl conditions)
Defines condition types used to signal Nausicaa/cURL errors.
(foreign net curl enumerations)
Defines enumeration sets of symbols and conversion functions between constant values and Scheme symbols.
(foreign net curl record-types)
Defines record types that match cURL structures and functions to convert between foreign structures and Scheme records.
(foreign net curl primitives)
Defines a high–level API for (foreign net curl platform), implementing, for most functions, marshaling of arguments and return values; the Scheme functions have the same name of the foreign ones, but with underscores turned into dashes; so curl_global_init() is available as curl-global-init.
(foreign net curl)
Compound library; reexports bindings from the following libraries: 
          (foreign net curl sizeof)
          (foreign net curl conditions)
          (foreign net curl enumerations)
          (foreign net curl primitives)

all the bindings from the ‘primitives’ library are reexported; only bindings to constant values are reexported from the ‘sizeof’ library.

(foreign net curl compensated)
Additional wrappers for constructors and destructors making use of the (compensations) library; this library is built on top of (foreign net curl). Compensation stacks.

7.1 Introduction to the easy interface

The “easy” session API provides simple one-at-a-time file transfer features; a program that requests index.html from the URL http://localhost:8080/ looks like this: 

     (import (rnrs)
       (compensations)
       (begin0)
       (only (foreign ffi pointers) pointer-null)
       (foreign net curl)
       (foreign net curl compensated))
     
     (define address "http://localhost:8080/index.html")
     
     (define document "")
     
     (define (write-callback buffer item-size item-count)
       (begin0-let ((len (* item-size item-count)))
         (set! out (string-append out (cstring->string buffer len)))))
     
     (curl-global-init CURL_GLOBAL_ALL)
     
     (with-compensations
       (let ((easy (curl-easy-init/c)))
         (curl-easy-setopt easy CURLOPT_URL address)
         (curl-easy-setopt easy CURLOPT_WRITEFUNCTION
                           (curl-make-write-callback write-callback))
         (curl-easy-setopt easy CURLOPT_WRITEDATA pointer-null)
         (curl-easy-perform easy)))
     
     (do-something-with document)

the callback write-callback may be called any number of times; if successful it must return the number of bytes consumed from the buffer, else it can return zero.

— Function: curl-make-write-callback scheme-function

Prepare a callback C function to be used as value for the ‘CURLOPT_WRITEFUNCTION’ option. It is defined as follows: 

          (define (curl-make-write-callback scheme-function)
            (make-c-callback*
               size_t
               (lambda (buffer item-size item-count custom)
                 (guard (E (else 0))
                   (scheme-function buffer item-size item-count)))
               (void* size_t size_t void*)))

          (define (curl-make-read-callback scheme-function)
            (make-c-callback*
               size_t

— Function: curl-make-ioctl-callback scheme-function

Prepare a callback C function to be used as value for the ‘CURLOPT_IOCTLFUNCTION’ option. It is defined as follows: 

          (define (curl-make-ioctl-callback scheme-function)
            (make-c-callback*
               curlioerr
               (lambda (handle action custom-data)
                 (guard (E (else -1))
                   (scheme-function handle size custom-data)))
               (void* int void*)))

The first argument to scheme-function is the cURL session handler; the second argument is one of the ‘CURLIOCMD_*’ constants; the third argument is the pointer previously set with the ‘CURLOPT_IOCTLDATA’ option.

At present (cURL version 7.19.7), the only meaningful requested action is ‘CURLIOCMD_RESTARTREAD’; when scheme-function receives this value as second argument, it should rewind to the start the read stream of the data to be sent. When successful, the callback must return zero; else it must return non–zero.

— Function: curl-make-seek-callback scheme-function

Prepare a callback C function to be used as value for the ‘CURLOPT_SEEKFUNCTION’ option. It is defined as follows: 

          (define (curl-make-seek-callback scheme-function)
            (make-c-callback*
              curlioerr
              (lambda (instream offset whence)
                (guard (E (else CURL_SEEKFUNC_FAIL))
                  (scheme-function intream offset whence)))
              (void* curl_off_t int)))

scheme-function should seek instream as described by offset and whence; notice that ‘SEEK_CUR’, ‘SEEK_SET’ and ‘SEEK_END’ are provided as constants by (foreign net curl sizeof).

10.1 Introduction

The “multi” session API provides a way to perform multiple file transfers through a single handler; a program that requests index.html from the URL http://localhost:8080/ looks like this: 

     (import (rnrs)
       (foreign net curl)
       (foreign net curl compensated))
     
     (with-compensations
       (let ((multi   (curl-multi-init/c))
             (easy1   (curl-easy-init/c))
             (easy2   (curl-easy-init/c))
             (out1    "")
             (out2    ""))
     
         (define (doit multi easy url result-proc)
           (let ((cb (lambda (buffer item-size item-count)
                       (let ((len (* item-size item-count)))
                         (result-proc (cstring->string buffer len))
                         len))))
             (curl-easy-setopt easy CURLOPT_URL url)
             (curl-easy-setopt easy CURLOPT_WRITEFUNCTION
                               (curl-make-write-callback cb))
             (curl-easy-setopt easy CURLOPT_WRITEDATA pointer-null)
             (curl-multi-add-handle multi easy)))
     
         (doit multi easy1 "http://localhost:8080/index.html"
               (lambda (str)
                 (set! out1 (string-append out1 str))))
     
         (doit multi easy2 "http://localhost:8080/index.html"
               (lambda (str)
                 (set! out2 (string-append out2 str))))
     
         (let loop ((running 0))
           (let-values (((code running) (curl-multi-perform multi)))
             (when (or (= code CURLM_CALL_MULTI_PERFORM)
                       (not (= 0 running)))
               (loop running))))
     
         ---))

11 Composing forms data

The ‘CURLOPT_HTTPPOST’ option to curl-easy-setopt allows us to compose data to be posted with HTTP.

— Function: curl-formadd alist
— Function: curl-formadd alist first* last*

Transform the alist into a linked list of struct curl_httppost structures to compose a single section of multipart/formdata HTTP ‘POST’; each invocation to this function must add a full section, else an error is raised.

Return two values: a pointer to the first curl_httppost in the list, a pointer to the last curl_httppost in the list.

To build a new list, we have to invoke the function without first* and last* arguments; to add sections to the list we use the returned values.

alist must be a Scheme alist with CURLFORM_* constants as alist keys and the appropriate option values as alist values; there is no need to terminate each alist with the ‘CURLFORM_END’ code. Internally this function makes use of the ‘CURLFORM_ARRAY’ option to curl_formadd().

If an alist value is a Scheme string, it is automatically, internally, converted to a C string with string->cstring from (foreign cstrings) and its memory released before returning.

Example: 

          (receive (first* last*)
              ;; add the first section
              (curl-formadd `((,CURLFORM_COPYNAME     . "htmlcode")
                              (,CURLFORM_COPYCONTENTS . "<html></html>")
                              (,CURLFORM_CONTENTTYPE  . "text/html")))
            (receive (first* last*)
                ;; add the second section
                (curl-formadd `((,CURLFORM_COPYNAME     . "name")
                                (,CURLFORM_COPYCONTENTS . "content"))
                              first* last*)
              ---
              (curl-easy-setopt handle CURLOPT_HTTPPOST first*)
              ---
              (curl-formfree first*)))
— Function: curl-formfree first*

Releases the memory associated to a linked list of struct curl_httppost structures.

— Function: curl-formadd-strerror code

Return a Scheme string describing the code returned by curl-formadd.
