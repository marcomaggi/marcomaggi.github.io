5.5 Spawning processes and the like

— Function: system command

Interface to the C function system(), system. Execute a command through the system shell; command must be a string holding the command to execute. If successful return the termination status of the executed process, posix status for details; if an error occurs: an exception is raised. 

          (system "ls -l")
— Function: fork
— Function: fork parent-proc child-thunk

Interface to the C function fork(), fork.

When called with no arguments: create a new process by forking the current one; if successful return a non–negative fixnum representing the the return value of fork(), else raise an exception.

When called with two arguments: create a new process by forking the current one; if successful the parent process applies parent-proc to a fixnum representing the child process pid and return its return value; if successful: the child process evaluates the child-thunk and returns its return value; if an error occurs: an exception is raised. 

          (fork
            (lambda (child-pid)
              (printf "in parent, child pid = ~s\n" child-pid))
            (lambda ()
              (printf "in child\n")
              (exit)))
          
          (printf "here we are in the parent\n")
— Function: execv filename argv

Interface to the C function execv(), execv. filename must be the pathname of an executable file; argv must be a list of strings representing command line arguments. Execute the command; if the function returns: an error has occurred, an exception is raised.

— Function: execve filename argv env

Interface to the C function execve(), execve. filename must the pathname of an executable file; argv must be a list of strings representing command line arguments; env must be a list of strings representing environment variables assignments. Execute the command; if the function returns: an error has occurred, an exception is raised.

— Function: execvp filename argv

Interface to the C function execvp(), execvp. filename must be the pathname of an executable file reachable in the current PATH; argv must be a list of strings representing command line arguments. Execute the command; if the function returns: an error has occurred, an exception is raised.

— Function: execl filename arg0 arg ...

Like execv but allows the arguments to be given expliticly.

— Function: execle filename argv env ...

Like execve but allows the environment to be given expliticly.

— Function: execlp filename arg0 arg ...

Like execvp but allows the arguments to be given expliticly.
