13.2.11 Hard and symbolic links

— Function: ikptr ikrt_posix_link (ikptr old_pathname, ikptr new_pathname)

Interface to the C function link(), link. Create a hard link new_pathname to the source file system entry old_pathname. If successful return zero, else return an encoded errno value.

— Function: ikptr ikrt_posix_symlink (ikptr file_pathname, ikptr link_pathname)

Interface to the C function symlink(), symlink. Create a symbolic link link_pathname to the source file system entry file_pathname. If successful return zero, else return an encoded errno value.

— Function: ikptr ikrt_posix_readlink (ikptr link_pathname, ikpcb * pcb)

Interface to the C function readlink(), readlink. If pathname references a symbolic link: follow it once and return the referenced pathname as bytevector; if an error occurs: return an encoded errno value.

— Function: ikptr ikrt_posix_realpath (ikptr pathname)

Interface to the C function realpath(), realpath. If pathname references a symbolic link: follow it and return the real pathname as bytevector; if an error occurs: return an encoded errno value.

— Function: ikptr ikrt_posix_unlink (ikptr pathname)

Interface to the C function unlink(), unlink. Delete the file system entry referenced by pathname, which must be a bytevector and referencing a file. If successful return zero, else return an encoded errno value.

— Function: ikptr ikrt_posix_remove (ikptr pathname)

Interface to the C function remove(), remove. Delete the file system entry referenced by pathname, which must be a bytevector and referencing a file or directory. If successful return zero, else return an encoded errno value.

— Function: ikptr ikrt_posix_rename (ikptr old_pathname, ikptr new_pathname)

Interface to the C function rename(), rename. Rename the file system entry old_pathname to new_pathname, both of which must be bytevectors. If successful return zero, else return an encoded errno value.
