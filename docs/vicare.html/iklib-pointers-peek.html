3.24.3 Peeking values

With all the following functions: the pointer argument must be a valid pointer object; the offset argument must be a fixnum. When adding an offset to a pointer: all the following functions do not scale the offset to the size of the poked value; pointer arithmetics is performed with byte offsets.

The following bindings are exported by the libraries (vicare), (ikarus) and (ikarus system $foreign) and reexported by the library (vicare ffi).

Peeking bitwise sized exact integers

The following functions retrieve signed and unsigned exact integers from raw memory location selecting the size with the number of bits required to represent the value.

— Function: pointer-ref-c-sint8 pointer offset
— Function: pointer-ref-c-uint8 pointer offset

Peek the 8 bits located at offset from pointer and return them as signed or unsigned exact integer.

— Function: pointer-ref-c-sint16 pointer offset
— Function: pointer-ref-c-uint16 pointer offset

Peek the 16 bits located at offset from pointer and return them as signed or unsigned exact integer.

— Function: pointer-ref-c-sint32 pointer offset
— Function: pointer-ref-c-uint32 pointer offset

Peek the 32 bits located at offset from pointer and return them as signed or unsigned exact integer.

— Function: pointer-ref-c-sint64 pointer offset
— Function: pointer-ref-c-uint64 pointer offset

Peek the 64 bits located at offset from pointer and return them as signed or unsigned exact integer.

Peeking C language exact integers

The following functions store signed and unsigned exact integers from raw memory locations selecting the size according to the one used by the C language implementation.

— Function: pointer-ref-c-signed-char pointer offset
— Function: pointer-ref-c-signed-char pointer offset

Peek the signed char or unsigned char integer located at offset from pointer and return it as signed or unsigned exact integer.

— Function: pointer-ref-c-signed-short pointer offset
— Function: pointer-ref-c-unsigned-short pointer offset

Peek the signed short int or unsigned short int integer located at offset from pointer and return it as signed or unsigned exact integer.

— Function: pointer-ref-c-signed-int pointer offset
— Function: pointer-ref-c-unsigned-int pointer offset

Peek the signed int or unsigned int integer located at offset from pointer and return it as signed or unsigned exact integer.

— Function: pointer-ref-c-signed-long pointer offset
— Function: pointer-ref-c-unsigned-long pointer offset

Peek the signed long or unsigned long integer located at offset from pointer and return it as signed or unsigned exact integer.

— Function: pointer-ref-c-signed-long-long pointer offset
— Function: pointer-ref-c-unsigned-long-long pointer offset

Peek the signed long long or unsigned long long integer located at offset from pointer and return it as signed or unsigned exact integer.

Peeking floating point numbers
— Function: pointer-ref-c-float pointer offset

Return the four–byte float (represented as IEEE 754 single precision floating point number) stored at offset from pointer. The value is extended to an IEEE 754 double precision floating point number that Vicare uses to represent inexact numbers.

— Function: pointer-ref-c-double pointer offset

Return the eight–byte float (represented as IEEE 754 double precision floating point number) stored at offset from pointer.

Peeking pointers
— Function: pointer-ref-c-pointer pointer offset

Return the pointer stored at offset from pointer. The size of the pointer (also the number of bytes loaded) depends on the architecture: it is 4 bytes on 32-bit systems and 8 bytes on 64-bit systems.
