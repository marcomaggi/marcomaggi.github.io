16.17.2 Constructors and destructors

The following bindings are exported by the library (vicare net channels).

— Function: open-binary-input-channel in-port

Build and return a new instance of binary-channel to be used to receive messages; the returned channel is inactive. The channel wraps the binary input or input/output port in-port.

— Function: open-binary-output-channel ou-port

Build and return a new instance of binary-channel to be used to send messages; the returned channel is inactive. The channel wraps the binary output or input/output port ou-port.

— Function: open-binary-input/output-channel port
— Function: open-binary-input/output-channel in-port ou-port

Build and return a new instance of binary-channel to be used to both receive and send messages; the returned channel is inactive.

The channel wraps the binary input or input/output port in-port, used to receive messages, and the binary output or input/output port ou-port, used to send messages. in-port and ou-port can be the same port.

When a single argument is given: port is used for both input and output.

— Function: open-textual-input-channel in-port

Build and return a new instance of textual-channel to be used to receive messages; the returned channel is inactive. The channel wraps the textual input or input/output port in-port.

— Function: open-textual-output-channel ou-port

Build and return a new instance of textual-channel to be used to send messages; the returned channel is inactive. The channel wraps the textual output or input/output port ou-port.

— Function: open-textual-input/output-channel port
— Function: open-textual-input/output-channel in-port ou-port

Build and return a new instance of textual-channel to be used to both receive and send messages; the returned channel is inactive.

The channel wraps the textual input or input/output port in-port, used to receive messages, and the textual output or input/output port ou-port, used to send messages. in-port and ou-port can be the same port.

When a single argument is given: port is used for both input and output.

— Function: close-channel chan

Finalise a channel closing its connection port; return unspecified values. A pending message delivery, if any, is aborted.

— Function: channel-abort! chan

Abort the current operation and reset the channel to inactive; return unspecified values.
