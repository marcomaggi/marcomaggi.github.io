12.25 Continuation objects

Continuation objects are memory blocks referenced by machine words tagged as vectors, whose first word is tagged as continuation: 

     |----------------|----------| reference to continuation
       heap pointer    vector tag
     
     |----------------|----------| first word of continuation
      all set to zero  vector tag

the layout of a continuation memory block is: 

     |-----|-----|-----|-----|
       tag   top  size  next

To understand the contents of a continuation object, we have to recall how the Scheme stack is used: 

      high memory
     |            |
     |------------|
     |            | <-- pcb->frame_base
     |------------|
     | used word  | <-- pcb->frame_base - wordsize
     |------------|
     |     .      |
     |     .      |
     |     .      |
     |------------|
     | free word  | <-- pcb->frame_pointer
     |------------|
     |     .      |
     |     .      |
     |     .      |
     |------------|
     | free word  | <-- pcb->stack_base
     |------------|
     |            |
       low memory

and remember that pcb->frame_base references a word that is one–off the end of the stack segment; so the first word in the stack is: 

     pcb->frame_base - wordsize

A continuation references the portion of the Scheme stack memory segment between frame_base and frame_pointer.

There are two kinds of continuation objects:

Scheme continuations
Represent a Scheme continuation. They are expliticly created by call/cc, but also used to represent a previous Scheme code execution when the allocated Scheme stack runs out of space.
system continuations
Represent a C continuation. They are used by the FFI to save the C execution context just before calling back to Scheme from C code.

The fields of a Scheme continuation are:

top
Untagged pointer referencing the first free word on the Scheme stack.
size
Number of bytes used on the Scheme stack, not counting the free word referenced by the frame pointer.
next
Tagged pointer referencing a continuation object; it represents the continuation to go back to after the execution of tthis continuation is finished. This “going back” is performed by the C function ik_exec_code().

The fields of a Scheme continuation are:

top
Untagged memory pointer referencing the C language stack last saved by Scheme code; it is the content of the field pcb->system_stack at the time this system continuation object was created.
size
Unused.
next
Tagged memory pointer referencing the next continuation; it is the content of the field pcb->next_k at the time this system continuation object was created.

Whatever the kind, usually a continuation is created and then saved in the field pcb->next_k in such a way that continuation objects create a linked list: 

             next_k
     |------|------|-----| PCB
               |
               v     next
            |-------|----| continuation
                       |
                       v     next
                    |-------|----| continuation
                               |
                                --> NULL
Basic operations
— Preprocessor Symbol: continuation_tag
— Preprocessor Symbol: system_continuation_tag

Integer values used to tag and recognise the first word in continuation memory blocks.

— Preprocessor Symbol: continuation_size
— Preprocessor Symbol: system_continuation_size

The number of bytes needed to hold a continuation memory block.

— Preprocessor Symbol: disp_continuation_tag
— Preprocessor Symbol: disp_continuation_top
— Preprocessor Symbol: disp_continuation_size
— Preprocessor Symbol: disp_continuation_next

Scheme continuation field displacements. The number of bytes to add to an untagged pointer to Scheme continuations to get the pointer to the first byte in the word holding the specified field.

— Preprocessor Symbol: disp_system_continuation_tag
— Preprocessor Symbol: disp_system_continuation_top
— Preprocessor Symbol: disp_system_continuation_size
— Preprocessor Symbol: disp_system_continuation_next

System continuation field displacements. The number of bytes to add to an untagged pointer to system continuations to get the pointer to the first byte in the word holding the specified field.

— Preprocessor Symbol: off_continuation_tag
— Preprocessor Symbol: off_continuation_top
— Preprocessor Symbol: off_continuation_size
— Preprocessor Symbol: off_continuation_next

Scheme continuation field offsets. The number of bytes to add to a tagged pointer to Scheme continuation to get the pointer to the first byte in the word holding the specified field.

— Preprocessor Symbol: off_system_continuation_tag
— Preprocessor Symbol: off_system_continuation_top
— Preprocessor Symbol: off_system_continuation_size
— Preprocessor Symbol: off_system_continuation_next

System continuation field offsets. The number of bytes to add to a tagged pointer to system continuation to get the pointer to the first byte in the word holding the specified field.

A Scheme continuation object is usually allocated as follows: 

     ikpcb *     pcb = ...;
     ikptr       s_kont;
     
     s_kont = ik_unsafe_alloc(pcb, IK_ALIGN(continuation_size)) \
        | vector_tag;
     IK_REF(s_kont, off_continuation_tag)  = continuation_tag;
     IK_REF(s_kont, off_continuation_top)  = pcb->frame_pointer;
     IK_REF(s_kont, off_continuation_size) = \
        pcb->frame_base - pcb->frame_pointer - wordsize;
     IK_REF(s_kont, off_continuation_next) = pcb->next_k;
     pcb->next_k = s_kont;
