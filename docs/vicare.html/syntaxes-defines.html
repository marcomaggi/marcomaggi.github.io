10.12.1 Special definitions

The following bindings are exported by the library (vicare syntactic-extensions).

— Macro: Syntax define-inline (?name ?arg ... . ?rest) ?form0 ?form ...

Wrapper for syntax-rules, expand to: 

          (define-syntax ?name
            (syntax-rules ()
              ((_ ?arg ... . ?rest)
               (begin ?form0 ?form ...))))
— Macro: define-constant ?name ?expr

Wrapper for identifier-syntax, expand to: 

          (begin
            (define ghost ?expr)
            (define-syntax ?name
              (identifier-syntax ghost)))

?expr is evaluated once and the identifier syntax ?name can be used to access the result; it is impossible to modify the value by acting upon ?name.

— Macro: define-inline-constant ?name ?value

Wrapper for identifier-syntax, expand to: 

          (define-syntax ?name
            (identifier-syntax ?value))

?value is evaluated every time ?name is referenced; it is impossible to modify the ?value by acting upon ?name. This syntax is useful to defined constants resulting from GNU Autoconf expansion: 

          (define-inline-constant SEEK_SET @VALUEOF_SEEK_SET@)
— Macro: define-syntax* (?who ?stx) . ?body

Wrapper for define-syntax, expand to: 

          (define-syntax ?who
            (lambda (?stx) . ?body))
— Macro: define-auxiliary-syntaxes ?name0 ?name ...

Define auxiliary syntaxes to be used as literal identifiers by syntax-rules, syntax-case and similar macros; expand to: 

          (define-syntax ?name0 (syntax-rules ()))
          (define-syntax ?name  (syntax-rules ()))
          ...
— Macro: define-exact-integer->symbol-function ?who (?code ...)

Define a function used to convert exact intgers to the corresponding symbolic name. Expand to: 

          (define (?who code)
            (define who '?who)
            (with-arguments-validation (who)
                ((exact-integer      code))
              (case-integers code
                ((?code)     '?code)
                ...
                (else #f))))
