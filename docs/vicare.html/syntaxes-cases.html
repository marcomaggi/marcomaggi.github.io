10.12.5 Special cases

The following bindings are exported by the library (vicare syntactic-extensions).

— Macro: case-word-size ((32) . ?body-32) ((64) . ?body-64)

If the platform word size is 32-bit expand to ?body-32, else expand to ?body-64.

— Macro: case-endianness (?who ?endianness) ((little) . ?lit-body) ((big) . ?big-body)
— Macro: case-endianness (?who ?endianness) ((big) . ?big-body) ((little) . ?lit-body)
— Auxiliary Syntax: big
— Auxiliary Syntax: little

If the platform endianness is little expand to ?lit-body, else expand to ?big-body.

— Macro: case-fixnums ?expr ((?fixnum0 ?fixnum ...) ?fx-body0 ?fx-body ...) ...
— Macro: case-fixnums ?expr ((?fixnum0 ?fixnum ...) ?fx-body0 ?fx-body ...) ... (else ?else-body0 ?else-body ...)
— Auxiliary Syntax: else

Special implementation of case expeting only fixnums as datums; for maximum speed, it makes use of unsafe operations on fixnums. else must be the binding exported by (rnrs) and (vicare).

?expr must be an expression evaluating to a fixnum, else the behaviour is undefined (and you do not want it).

— Macro: case-integers ?expr ((?int0 ?int ...) ?int-body0 ?int-body ...) ...
— Macro: case-integers ?expr ((?int0 ?int ...) ?int-body0 ?int-body ...) ... (else ?else-body0 ?else-body ...)
— Auxiliary Syntax: else

Special implementation of case expecting only exact integers as datums. else must be the binding exported by (rnrs) and (vicare).

?expr must be an expression evaluating to an exact integer, else the behaviour is undefined (and you do not want it).
