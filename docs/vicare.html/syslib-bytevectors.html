12.13 Low level bytevector operations

The arguments bv to these operations must be references to bytevectors, the arguments fx must be fixnums in the appropriate range, the arguments fl must be flonums. The following bindings are exported by the libraries (ikarus system $bytevectors) and (vicare system $bytevectors).

— Unsafe Operation: $make-bytevector fx

Allocate a new bytevector capable of holding fx bytes and return a reference to it.

— Unsafe Operation: $bytevector-length bv

Return a fixnum representing the number of bytes in the operand.

— Unsafe Operation: $bytevector-u8-ref bv fx

Return a fixnum representing the octet at offset fx in bv.

— Unsafe Operation: $bytevector-s8-ref bv fx

Return a fixnum representing the byte at offset fx in bv.

— Unsafe Operation: $bytevector-set! bv fx-index fx-value

Store the octet or byte fx-value at index fx-index of bv.

Double–precision IEEE flonums
— Unsafe Operation: $bytevector-ieee-double-native-ref bv fx

Return a flonum representing the value at index fx of bv.

— Unsafe Operation: $bytevector-ieee-double-native-set! bv fx fl

Store fl at index fx of bv.

— Unsafe Operation: $bytevector-ieee-double-nonnative-ref bv fx

Return a flonum representing the value at index fx of bv.

— Unsafe Operation: $bytevector-ieee-double-nonnative-set! bv fx fl

Store fl at index fx of bv.

Single–precision IEEE flonums
— Unsafe Operation: $bytevector-ieee-single-native-ref bv fx

Return a flonum representing the value at index fx of bv.

— Unsafe Operation: $bytevector-ieee-single-native-set! bv fx fl

Store fl at index fx of bv.

— Unsafe Operation: $bytevector-ieee-single-nonnative-ref bv fx

Return a flonum representing the value at index fx of bv.

— Unsafe Operation: $bytevector-ieee-single-nonnative-set! bv fx fl

Store fl at index fx of bv.

Other unsafe operations
— Function: $bytevector= bv1 bv2

Return #t if the two arguments are equal.

— Function: $bytevector-total-length init-len bvs

Given the list of bytevectors bvs: compute the total length of the bytevectors, add it to ini-len and return the result. If init-len is zero: the returned value is the total length of the bytevectors in bvs. The returned value may or may not be in the range of the maximum bytevector size.

— Function: $bytevector-concatenate total-len bvs

Concatenate the bytevectors in the list bvs, return the result. The resulting bytevector must have length total-len. Assume the arguments have been already validated.

— Function: $bytevector-reverse-and-concatenate total-len bvs

Reverse the list of bytevectors bvs and concatenate its items; return the result. The resulting bytevector must have length total-len. Assume the arguments have been already validated.
