3.27.2 Poking values

With all the following functions: the pointer argument must be a valid pointer object; the offset argument must be a fixnum. When adding an offset to a pointer: all the following functions do not scale the offset to the size of the poked value; pointer arithmetics is performed with byte offsets.

The following bindings are exported by the libraries (vicare), (ikarus) and (ikarus system $foreign) and reexported by the library (vicare ffi).

Poking bitwise sized exact integers

The following functions store signed and unsigned exact integers in raw memory location selecting the size with the number of bits required to represent the value.

— Function: pointer-set-c-sint8! pointer offset value
— Function: pointer-set-c-uint8! pointer offset value

Set to value the single byte of memory located at offset from pointer. VALUE must be an exact integer in the range representable by 8 bits.

— Function: pointer-set-c-sint16! pointer offset value
— Function: pointer-set-c-uint16! pointer offset value

Set to value the two bytes of memory located at offset from pointer. VALUE must be an exact integer in the range representable by 16 bits.

— Function: pointer-set-c-sint32! pointer offset value
— Function: pointer-set-c-uint32! pointer offset value

Set to value the two bytes of memory located at offset from pointer. VALUE must be an exact integer in the range representable by 32 bits.

— Function: pointer-set-c-sint64! pointer offset value
— Function: pointer-set-c-uint64! pointer offset value

Set to value the two bytes of memory located at offset from pointer. VALUE must be an exact integer in the range representable by 64 bits.

Poking C language exact integers

The following functions store signed and unsigned exact integers in raw memory locations selecting the size according to the one used by the C language implementation.

— Function: pointer-set-c-signed-char! pointer offset value
— Function: pointer-set-c-unsigned-char! pointer offset value

Set to value a number of bytes corresponding to the signed char or unsigned char type located at offset from pointer.

— Function: pointer-set-c-signed-short! pointer offset value
— Function: pointer-set-c-unsigned-short! pointer offset value

Set to value a number of bytes corresponding to the signed short int or unsigned short int type located at offset from pointer.

— Function: pointer-set-c-signed-int! pointer offset value
— Function: pointer-set-c-unsigned-int! pointer offset value

Set to value a number of bytes corresponding to the signed int or unsigned int type located at offset from pointer.

— Function: pointer-set-c-signed-long! pointer offset value
— Function: pointer-set-c-unsigned-long! pointer offset value

Set to value a number of bytes corresponding to the signed long or unsigned long type located at offset from pointer.

— Function: pointer-set-c-signed-long-long! pointer offset value
— Function: pointer-set-c-unsigned-long-long! pointer offset value

Set to value a number of bytes corresponding to the signed long long or unsigned long long type located at offset from pointer.

Poking floating point numbers
— Function: pointer-set-c-float! pointer offset flonum

Convert the Scheme floating point number flonum (represented in Vicare as an IEEE 754 double precision floating point number) to a float (an IEEE 754 single precision floating point number) and store the result in the four bytes at offset from pointer.

— Function: pointer-set-c-double! pointer offset flonum

Store the double precision IEEE 754 floating point value of the Scheme flonum in the eight bytes at offset from pointer.

Poking pointers
— Function: pointer-set-c-pointer! pointer offset value

Store address of the raw memory location referenced by the pointer value at offset from pointer.
