14.3.2 Function arguments on the stack

Right after the first function call that enters Scheme code from assembly code, the stack segment looks as follows: 

           high memory
     |                      |
     |----------------------|
     |                      | <-- pcb->frame_base
     |----------------------|
     | ik_underflow_handler | <-- FPR
     |----------------------|
                ...
     |----------------------|
     |                      | <-- pcb->stack_base
     |----------------------|
     |                      |
            low memory

If the called Scheme function needs some local variables, it allocates them on the stack: 

           high memory
     |                      |
     |----------------------|                         --
     | ik_underflow_handler | <-- FPR                 .
     |----------------------|                         .
     |   local variable 0   | <-- FPR - 1 * wordsize  . first
     |----------------------|                         . called
     |   local variable 1   | <-- FPR - 2 * wordsize  . function
     |----------------------|                         . frame
     |   local variable 2   | <-- FPR - 3 * wordsize  .
     |----------------------|                         --
     |                      |
            low memory

notice that local values are “moved” on the stack, not “pushed”: executing the assembly instruction push would decrement the FPR, we do not want this; rather we reserve frame space for a function call and never mutate the FPR until the function returns or calls another function.

If the Scheme function needs to call another Scheme function, as in: 

     (sub-func arg0 arg1 arg2)

it moves the arguments on the stack, leaving an empty machine word: 

           high memory
     |                      |
     |----------------------|         --
     | ik_underflow_handler | <-- FPR .
     |----------------------|         .
     |   local variable 0   |         . first
     |----------------------|         . called
     |   local variable 1   |         . function
     |----------------------|         . frame
     |   local variable 2   |         .
     |----------------------|         --
     |                      |
     |----------------------|
     |      argument 0      |
     |----------------------|
     |      argument 1      |
     |----------------------|
     |      argument 2      |
     |----------------------|
     |                      |
            low memory

it adjusts the FPR by explicitly subtracting the appropriate offset from it: 

           high memory
     |                      |
     |----------------------|         --
     | ik_underflow_handler |         .
     |----------------------|         .
     |   local variable 0   |         . first
     |----------------------|         . called
     |   local variable 1   |         . function
     |----------------------|         . frame
     |   local variable 2   | <-- FPR .
     |----------------------|         --
     |                      |
     |----------------------|
     |      argument 0      |
     |----------------------|
     |      argument 1      |
     |----------------------|
     |      argument 2      |
     |----------------------|
     |                      |
            low memory

and finally it performs a call instruction, which pushes a return address on the stack right in the empty machine word: 

           high memory
     |                      |
     |----------------------|         --
     | ik_underflow_handler |         .
     |----------------------|         .
     |   local variable 0   |         . first
     |----------------------|         . called
     |   local variable 1   |         . function
     |----------------------|         . frame
     |   local variable 2   |         .
     |----------------------|         --
     |    return address    | <-- FPR .
     |----------------------|         .
     |      argument 0      |         . sub-func
     |----------------------|         . called
     |      argument 1      |         . function
     |----------------------|         . frame
     |      argument 2      |         .
     |----------------------|         --
     |                      |
            low memory

When the function sub-func executes a ret instruction: the execution flow returns to return address and the FPR is incremented by a machine word: 

           high memory
     |                      |
     |----------------------|         --
     | ik_underflow_handler |         .
     |----------------------|         .
     |   local variable 0   |         . first
     |----------------------|         . called
     |   local variable 1   |         . function
     |----------------------|         . frame
     |   local variable 2   | <-- FPR .
     |----------------------|         --
     |                      |
            low memory

now the first called function adjusts the FPR by explicitly adding the appropriate offset to it: 

           high memory
     |                      |
     |----------------------|         --
     | ik_underflow_handler | <-- FPR .
     |----------------------|         .
     |   local variable 0   |         . first
     |----------------------|         . called
     |   local variable 1   |         . function
     |----------------------|         . frame
     |   local variable 2   |         .
     |----------------------|         --
     |                      |
            low memory

and is finally ready to go on with the execution.
