16.17.3 Configuration parameters

The following bindings are exported by the library (vicare net channels).

— Function: channel-set-maximum-message-size! chan maximum-message-size

Configure the maximum message size; return unspecified values.

maximum-message-size must be a positive exact integer representing the inclusive maximum message size; if the size of the message exceeds this value: message delivery will fail.

When not configured: the maximum message size defaults to 4096.

The maximum message size is not reset when a message delivery is completed.

— Function: channel-set-expiration-time! chan expiration-time

Configure the message delivery expiration time; return unspecified values.

If expiration-time is a time object (as defined by the library (vicare)): it represents the limit of time since the Epoch to complete message delivery; if the allotted time expires: message delivery will fail. Typically we build such time objects using time-from-now.

If expiration-time is #f: there is not time limit.

When not configured: the expiration time defaults to no time limit.

The expiration time is not reset when a message delivery is completed.

— Function: channel-set-message-terminators! chan terminators

Configure the message terminators; return unspecified values.

If chan is a binary channel: terminators must be a non–empty list of non–empty bytevectors representing possible message terminators. When not configured: the list of terminators defaults to: 

          (#ve(ascii "\r\n\r\n") #ve(ascii "\n\n"))

If chan is a textual channel: terminators must be a non–empty list of non–empty strings representing possible message terminators. When not configured: the list of terminators defaults to: 

          ("\r\n\r\n" "\n\n")

The terminators are not reset when a message delivery is completed.

— Function: channel-set-maximum-message-portion-size! chan max-portion-size

Configure the maximum message portion size; return unspecified values.

max-portion-size must be a positive fixnum representing the inclusive maximum message portion size in octets or characters; when receiving a message portion: this value is used as second argument to get-bytevector-n or get-string-n.

When not configured: the maximum portion size defaults to 4096 for binary channels and to 1024 for textual channels. This value can be reconfigured any number of times, even between message portion receptions.

The maximum message portion size is not reset when a message delivery is completed.
