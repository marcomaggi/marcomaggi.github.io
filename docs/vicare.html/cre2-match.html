H.4 Matching regular expressions

Basic regular expressions matching goes as follows: 

     #!r6rs
     (import (vicare)
       (prefix (vicare cre2) cre2.))
     
     (let ((rex (cre2.make-regexp "ciao|hello")))
       (cre2.match rex "ciao" #f #f 'unanchored))
     ⇒ #((0 . 4))
     
     (let ((rex (cre2.make-regexp "ciao|hello")))
       (cre2.match rex "hello" #f #f 'unanchored))
     ⇒ #((0 . 5))
     
     (let ((rex (cre2.make-regexp "ci(ao)|hello")))
       (cre2.match rex "ciao" #f #f 'unanchored))
     ⇒ #((0 . 4)
          (2 . 4))
     
     (let ((rex (cre2.make-regexp "c(i(ao))|hello")))
       (cre2.match rex "ciao" #f #f 'unanchored))
     ⇒ #((0 . 4)
          (1 . 4)
          (2 . 4))

The following bindings are exported by the (vicare cre2) library.

— Function: match rex text start end anchor

Match a substring of text against the regular expression object rex. Return false if there is no match, else return a vector representing the matching portions.

text must be a UTF-8 bytevector or a Scheme string which is internally transformed to a UTF-8 bytevector. All the indices in text used by this function are relative to the bytevector, not the Scheme string. Only when the string contains only characters in the ASCII range the indices for the UTF-8 bytevector equal the indices for the string.

The zero–based indices start (inclusive) and end (exclusive) select the range of text to be examined. As special case: if start is #f, the start index is set to zero; if end is #f, the end index is set to the length of the bytevector.

anchor selects the anchor point for the matching operation, it must be one of the symbols: unanchored, start, both.

Data about the matching groups is returned in a vector of pairs, which will have a number of slots equal to the number of parenthetical subexpressions in rex plus one. Each pair selects a range of bytes in text: the car is a non–negative fixnum representing the inclusive start index, the cdr is a non–negative fixnum representing the exclusive end index.

The first element of the match vector (index 0) references the full portion of the substring of text matching the pattern; the second element of the match vector (index 1) references the portion of text matching the first parenthetical subexpression, the third element of the match vector (index 2) references the portion of text matching the second parenthetical subexpression; and so on.
