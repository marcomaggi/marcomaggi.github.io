4.1 destructor interface: type definitions

Struct Typedef: ccstructs_dtor_I

Type of the destructor interface struct. Instances of this type are meant to be handed by value to function calls.

Struct Typedef: ccname_iface_table_type(ccstructs_dtor_I)

Type of methods table for the destructor interface. It has the following fields:

ccname_iface_method_type(ccstructs_dtor_I, final) * final

Pointer to function implementing the final() method.

ccname_iface_method_type(ccstructs_dtor_I, release) * release

Pointer to function implementing the release() method.

Function Prototype: void ccname_iface_method_type(ccstructs_dtor_I, final) (ccstructs_dtor_I I)

Type of functions implementing the final() method. Release all the asynchronous resources owned by the fields of the struct, if any. The struct itself is left alone.

Function Prototype: void ccname_iface_method_type(ccstructs_dtor_I, release) (ccstructs_dtor_I I)

Type of functions implementing the release() method. This method releases the memory allocated for the struct itself, if needed; if a struct is allocated on the stack or embedded into an enclosing struct: this method should do nothing.
