4.3 Exception handlers

The best way to use the destructor interface is through the exception handlers API.

Struct Typedef: ccstructs_clean_handler_t

Type of struct usable has clean handler with the facilities of the CCExceptions library.

Struct Typedef: ccstructs_error_handler_t

Type of struct usable has error handler with the facilities of the CCExceptions library.

Function: void ccstructs_clean_handler_init (cce_destination_t L, ccstructs_clean_handler_t * H, ccstructs_dtor_I I)

Initialise the exception handler H to apply ccstructs_dtor_delete() to I, in the context of the location L.

Function: void ccstructs_error_handler_init (cce_destination_t L, ccstructs_error_handler_t * H, ccstructs_dtor_I I)

Initialise the exception handler H to apply ccstructs_dtor_delete() to I, in the context of the location L.

Preprocessor Macro: void ccstructs_handler_init (cce_destination_t L, H, ccstructs_dtor_I I)

Initialise the exception handler H to apply ccstructs_dtor_delete() to I, in the context of the location L. If H is a pointer of type:

ccstructs_clean_handler_t

The macro use expands to a call to ccstructs_clean_handler_init().

ccstructs_error_handler_t

The macro use expands to a call to ccstructs_error_handler_init().
