31.2.9 Memory buffers

The following bindings are exported by (nausicaa ffi memory membuffers); the library is built on top of (nausicaa language classes).

— Class: <membuffer>

Record type of byte buffers; it is used as a FIFO for bytes or multi–byte data types.

— Function: make-<membuffer>
— Function: make-<membuffer> cache

Build and return a new <membuffer> record. cache must be a memory block cache interface function returned by make-block-cache; when not given, it defaults to page-blocks-cache.

The cache function must raise an &out-of-memory condition if memory allocation fails.

— Function: <membuffer>? obj

Return #t if obj is a record of type <membuffer>.

— Function: membuffer-push-memblock! mb blk
— Function: membuffer-push-memblock! mb blk blk.start
— Function: membuffer-push-memblock! mb blk blk.start blk.past

Enqueue in the <membuffer> record mb bytes from the <memblock> record blk; bytes are read from blk starting at offset blk.start and stopping before offset blk.past.

If no error occurs allocating memory, all the bytes in the selected range of blk are enqueued in mb. If an error occurs allocating memory, mb is left in a consistent state and an exception is raised with a condition object of type &membuffer-incomplete-push.

— Function: membuffer-pop-memblock! mb blk
— Function: membuffer-pop-memblock! mb blk blk.start
— Function: membuffer-pop-memblock! mb blk blk.start blk.past

Fill the <memblock> record blk, starting at offset blk.start and stopping before offset blk.past, with bytes dequeued from the <membuffer> record mb. Return the offset in blk of the position past the last filled byte. If blk is filled to blk.past, the return value is blk.past.

When blk.past is not given, it defaults to the size of blk. When blk.start is not given, it defaults to zero.

— Function: membuffer-push-bytevector! mb bv
— Function: membuffer-push-bytevector! mb bv bv.start
— Function: membuffer-push-bytevector! mb bv bv.start bv.past

Enqueue in the <membuffer> record mb bytes from the bytevector bv; bytes are read from bv starting at offset bv.start and stopping before offset bv.past.

If no error occurs allocating memory, all the bytes in the selected range of bv are enqueued in mb. If an error occurs allocating memory, mb is left in a consistent state and an exception is raised with a condition object of type &membuffer-incomplete-push.

— Function: membuffer-pop-bytevector! mb bv
— Function: membuffer-pop-bytevector! mb bv bv.start
— Function: membuffer-pop-bytevector! mb bv bv.start bv.past

Fill the bytevector bv, starting at offset bv.start and stopping before offset bv.past, with bytes from the <membuffer> record mb. Return the offset in bv of the position past the last filled byte. If bv is filled to bv.past, the return value is bv.past.

When bv.past is not given, it defaults to the length of the bytevector. When bv.start is not given, it defaults to zero.

Incomplete push condition object
— Condition Type: &membuffer-incomplete-push

Type of condition describing an incomplete push operation on a <membuffer> record. It has the following fields:

‘membuffer’
The <membuffer> record for which the push was incomplete.
‘source’
The source object from which data was read. It is a bytevector or a <memblock> record.
‘source-start-offset’
The offset in the source object from which data should have been read.
‘source-past-offset’
The exclusive upper limit offset of data in the source object.

— Function: make-membuffer-incomplete-push-condition mb source source-offset

Build and return a condition object of type &membuffer-incomplete-push.

— Function: membuffer-incomplete-push-condition? obj

Return true if obj is a condition object of type &membuffer-incomplete-push.

— Function: membuffer-incomplete-push/membuffer obj
— Function: membuffer-incomplete-push/source obj
— Function: membuffer-incomplete-push/source-start-offset obj
— Function: membuffer-incomplete-push/source-past-offset obj

Field accessors for condition objects of type &membuffer-incomplete-push.
