68.2.4 Condition objects and error handlers

The following bindings are exported by the (nausicaa net ipv6-addresses) library.

— Condition Type: &ipv6-address-parser-error

Tags compound condition objects as raised by the IPv6 parser or lexer. It is derived from &condition.

— Function: make-ipv6-address-parser-error-condition

Return a new condition object of type &ipv6-address-parser-error.

— Function: ipv6-address-parser-error-condition? obj

Return true if obj is a condtion object of type &ipv6-address-parser-error.

— Function: make-ipv6-address-parser-error-handler who irritants
— Function: make-ipv6-address-parser-error-handler who irritants condition-maker

Build and return a procedure to be used as error handler by the IPv6 address and address prefix parsers. It raises an exception with compound condition object of types: 

          &ipv4-address-parser-error
          &who
          &message
          &irritants

The optional condition-maker must be a condition object constructor and defaults to make-error; its result is added to the raised compound object. condition-maker is meant to be used to tag the compound object as &error or &assertion.
