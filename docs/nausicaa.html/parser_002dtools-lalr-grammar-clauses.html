12.3.5.3 Writing semantic clauses

Semantic clauses are Scheme forms that can do anything Scheme allows us to do. They are allowed to access the local bindings ‘$1’, ‘$2’, ... bound to values in the same number of the symbols of the production rule.

An important limit is the environment in which the semantic clauses are evaluated:

  • If we have instructed lalr-parser to generate a define form, we can take the form and place it in our code; in this case the environment is the one active at that position.
  • If we have instructed lalr-parser to generate a proper Scheme library, we can add libraries specifications to the import list of the generated library using the library-imports: clause of lalr-parser.
  • If we have instructed lalr-parser to evaluate the generated parser maker using the (rnrs eval (6)) library, we can add libraries specifications to the evaluation environment using the library-imports: clause of lalr-parser.

The following local bindings are accessible in the semantic clauses.

— Function: yypushback

If evaluated, tell the parser closure to reuse the last token (as lookahead) rather than to retrieve a new one from the lexer. This may be useful to handle some corner cases in the grammar definition.

— Variable: yycustom

This is the custom value we give as yycustom argument to the parser function. It can be any value, from a closure to a constant, to a continuation.
