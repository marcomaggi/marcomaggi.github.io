26.7.4 Setters and getters

Setters and getters have the purpose of providing a shorter syntax to access storage locations; they are especially useful with vector objects and similar: 

     (let/with-class (((o <vector>) '#(0 1 2 3 4)))
       (setf (o 2) #\c)
       (getf (o 2)))
     ⇒ #\c
     
     (let/with-class (((o <hashtable>) (make-eq-hashtable)))
       (setf (o 'ciao) 10)
       (list (getf (o 'ciao))
             (getf (o 'hello))
             (getf (o 'hello 'salut))))
     ⇒ (10 #f salut)

In this document: class specific setters and getters are described as setf and getf “methods”.

— Macro: setf place value
— Macro: setf identifier value

Apply a class setter to the specified place with the purpose of storing there value. The place argument must have the form: 

          (variable-name key0 key ...)

where variable-name must be an identifier bound to the class instance, the key values should select a storage location. The following meaningless example shows the mechanism: 

          (define-class <alpha>
            (fields (mutable a))
            (setter <alpha>-setf))
          
          (define/with-class (<alpha>-setf (o <alpha>) key0 key1 value)
            (set! o.a (list key0 key1 value)))
          
          (let/with-class (((o <alpha>) (make <alpha> 1)))
            (setf (o 2 3) 4)
            o.a)
          ⇒ (2 3 4)

The number of keys to be used by the setter is unspecified: it is enforced by the function or macro selected in the setter clause of define-class.

When used with the identifier argument: setf behaves as an alias of set! (notice, though, that renaming setf to set! when importing the library, causes problems when using the identifier-syntax macro).

— Macro: getf place

Apply a class getter to the specified place with the purpose of accessing the value stored there. The place argument must have the form: 

          (variable-name key0 key ...)

where variable-name must be an identifier bound to the class instance, the key values should select a storage location. The following meaningless example shows the mechanism: 

          (define-class <alpha>
            (fields (mutable a))
            (setter <alpha>-setf)
            (getter <alpha>-getf))
          
          (define/with-class (<alpha>-setf (o <alpha>) key0 key1 value)
            (set! o.a (list key0 key1 value)))
          
          (define/with-class (<alpha>-getf (o <alpha>) key0 key1)
            (list o.a key0 key1))
          
          (let/with-class (((o <alpha>) (make <alpha> 1)))
            (setf (o 2 3) 4)
            (getf (o 5 6)))
          ⇒ ((2 3 4) 5 6)

The number of keys to be used by the getter is unspecified: it is enforced by the function or macro selected in the getter clause of define-class.
