45.8.2.2 Utilities for seconds and subseconds units counts

The following bindings are exported by the (nausicaa times-and-dates seconds) library. The following naming conventions hold:

  • All the bindings starting with sn- take as arguments a seconds and nanoseconds counts.
  • All the bindings starting with su- take as arguments a seconds and microseconds counts.
  • All the bindings starting with sm- take as arguments a seconds and milliseconds counts.
  • All the bindings ending with ->sn return two values: seconds and nanoseconds counts.
  • All the bindings ending with ->su return two values: seconds and microseconds counts.
  • All the bindings ending with ->sm return two values: seconds and milliseconds counts.

Some of the bindings are macros: this is to allow a level of inline expansion for small expressions and save a function call. When the documentation refers to normalised values, it means that the seconds and subseconds counts are both non–negative or both non–positive and:

  • The milliseconds count M is in the range -1e-3 < M < 1e3.
  • The microseconds count U is in the range -1e-6 < U < 1e6.
  • The nanoseconds count N is in the range -1e-9 < N < 1e9.

The counts in seconds and subseconds units must be given as exact numbers when arguments and are always returned as exact numbers.

— Syntax: sn-normalise seconds nanoseconds
— Syntax: su-normalise seconds microseconds
— Syntax: sm-normalise seconds milliseconds

Return two values: a normalised seconds count and a normalised subseconds count. The returned values are the arguments normalised to be in such a way that the time from zero is left unchanged. Examples: 

          (sn-normalise +1 +1)    ⇒ +1 +1
          (sn-normalise -1 -1)    ⇒ -1 -1
          (sn-normalise +1 -1)    ⇒  0 +999999999
          (sn-normalise -1 +1)    ⇒  0 -999999999
          
          (sn-normalise +10 #e+1.001e9)     ⇒ +11 +1e6
          (sn-normalise -10 #e-1.001e9)     ⇒ -11 -1e6
— Syntax: sn->seconds seconds nanoseconds
— Syntax: su->seconds seconds microseconds
— Syntax: sm->seconds seconds milliseconds

Return an exact rational number representing the arguments in seconds units.

— Syntax: sn->milliseconds seconds nanoseconds
— Syntax: su->milliseconds seconds microseconds
— Syntax: sm->milliseconds seconds milliseconds

Return an exact rational number representing the arguments in milliseconds units.

— Syntax: sn->microseconds seconds nanoseconds
— Syntax: su->microseconds seconds microseconds
— Syntax: sm->microseconds seconds milliseconds

Return an exact rational number representing the arguments in microseconds units.

— Syntax: sn->nanoseconds seconds nanoseconds
— Syntax: su->nanoseconds seconds microseconds
— Syntax: sm->nanoseconds seconds milliseconds

Return an exact rational number representing the arguments in nanoseconds units.

— Syntax: seconds->sn seconds
— Syntax: seconds->su seconds
— Syntax: seconds->sm seconds

Return two values: seconds itself and zero. The returned values represents the count of seconds and subseconds corresponding to seconds.

— Syntax: milliseconds->sn milliseconds
— Syntax: milliseconds->su milliseconds
— Syntax: milliseconds->sm milliseconds

Return two values: the count of seconds and the count of subseconds corresponding to the count of milliseconds. The returned values are normalised.

— Syntax: microseconds->sn microseconds
— Syntax: microseconds->su microseconds
— Syntax: microseconds->sm microseconds

Return two values: the count of seconds and the count of subseconds corresponding to the count of microseconds. The returned values are normalised.

— Syntax: nanoseconds->sn nanoseconds
— Syntax: nanoseconds->su nanoseconds
— Syntax: nanoseconds->sm nanoseconds

Return two values: the count of seconds and the count of subseconds corresponding to the count of nanoseconds. The returned values are normalised.

— Function: smun->sn seconds millliseconds microseconds nanoseconds

Return two values: the count of seconds and the count of nanoseconds corresponding to the arguments. The returned values are normalised.

— Function: sn->smun seconds nanoseconds

Return four values: the counts of seconds, milliseconds, microseconds, nanoseconds corresponding to the arguments. The returned values are normalised.

— Function: sn-add seconds1 nanoseconds1 seconds2 nanoseconds2

Sum the two couples of counts and return two values: the resulting normalised counts of seconds and nanoseconds.

— Function: sn-sub seconds1 nanoseconds1 seconds2 nanoseconds2

Subtract the seconds couple of counts from the first and return two values: the resulting normalised counts of seconds and nanoseconds.

— Function: sn< seconds1 nanoseconds1 seconds2 nanoseconds2
— Function: sn> seconds1 nanoseconds1 seconds2 nanoseconds2
— Function: sn<= seconds1 nanoseconds1 seconds2 nanoseconds2
— Function: sn>= seconds1 nanoseconds1 seconds2 nanoseconds2

Return true if the first couple of counts is, respectively: less than, greater than, less than or equal to, greater than or equal to the second couple of counts. The comparison is performed on normalised counts.

— Function: utc-seconds->tai-seconds utc-seconds
— Function: tai-seconds->utc-seconds tai-seconds

Convert a count of seconds since the Unix Epoch from UTC to TAI and vice versa.
