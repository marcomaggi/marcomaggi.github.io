10.5 Bindings parsers

For all the following functions the synner argument is a function used to report syntax violations; we can think of it as: 

     (define synner
       (case-lambda
         ((message)
          (synner message #f))
         ((message subform)
          (syntax-violation 'who message
            (syntax->datum stx)
            (syntax->datum subform)))))

where who is the name of the calling syntax and stx is the syntax object holding the input bindings form; this is exactly the function defined by define-syntax* from (nausicaa language extensions).

— Function: parse-tagged-single-binding tagged-binding top-tag synner

Parse a single possibly–tagged binding form, which must come as argument in a syntax object with one of the forms: 

          (?id ?init)
          ((?id) ?init)
          ((?id ?tag0 ?tag ...) ?init)

tagged-binding must be the syntax object holding the binding form. top-tag must be the identifier to use as conventional class for untagged bindings, it is usually <top> from (nausicaa language classes). SYNNER must be a closure used to raise syntax violations.

Return three values: a syntax object representing the corresponding untagged binding, a syntax object representing the corresponding with-class clause, an identifier being the binding name (to be used, for example, to search for duplicates).

Examples: 

          (parse-tagged-single-binding #'(a 1) #'<top> synner)
          ⇒ #'(a 1) #'(a <top>) #'a
          
          (parse-tagged-single-binding #'((a) 1) #'<top> synner)
          ⇒ #'(a 1) #'(a <top>) #'a
          
          (parse-tagged-single-binding #'((a <red> <blu>) 1) #'<top> synner)
          ⇒ #'(a 1) #'(a <red> <blu>) #'a
— Function: parse-tagged-bindings tagged-bindings top-tag allow-duplicates synner

Parse a list of possibly–tagged bindings each definining a single value. Make use of parse-tagged-single-binding.

Return three values: a syntax object representing the corresponding list of untagged bindings; a syntax object representing the corresponding list of with-class clauses; a syntax object holding a list of symbols being all the binding names.

If allow-duplicates is #f: it is a syntax violation if an identifier appears multiple times as variable name.

tagged-bindings must be a syntax object holding the list of bindings. top-tag must be the identifier to use as conventional class for untagged bindings, it is usually <top> from (nausicaa language classes). synner must be a closure used to raise syntax violations.

Example: 

          (parse-tagged-bindings #'((a 1)) #'<top> #f synner)
          ⇒ #'((a 1)) #'((a <top>)) #'(a)
          
          (parse-tagged-bindings #'(((a) 1)) #'<top> #f synner)
          ⇒ #'((a 1)) #'((a <top>)) #'(a)
          
          (parse-tagged-bindings
            #'(((a <red> <blu>) 1))
            #'<top>
            #f synner)
          ⇒ #'((a 1)) #'((a <red> <blu>)) #'(a)
          
          (parse-tagged-bindings
            #'(((a <n>) 1)
               (b 2)
               ((c <p> <q>) 3))
            #'<top> #f synner)
          ⇒ #'((a 1) (b 2) (c 3))
             #'((a <n>) (b <top>) (c <p> <q>))
             #'(a b c)
— Function: parse-multiple-values-tagged-single-binding tagged-binding top-tag synner

Parse a single possibly–tagged binding form, defining multiple values, which must come as argument in a syntax object with one of the forms: 

          (?id ?init)
          ((?id0 ?id ...) ?init)
          (((?id0 ?tag0 ...) (?id ?tag ...) ...) ?init)

It is a syntax violation if an identifier appears multiple times as variable name.

tagged-binding must be the syntax object holding the binding form. top-tag must be the identifier to use as conventional class for untagged bindings, it is usually <top> from (nausicaa language classes). synner must be a closure used to raise syntax violations.

Return three values: a syntax object representing the corresponding untagged binding, a syntax object representing the corresponding list of with-class clauses, a syntax object holding a list of symbols being all the binding names (to be used, for example, to search for duplicates).

Examples: 

          (parse-multiple-values-tagged-single-binding
            #'(a (values 1 2))
            #'<top>
            synner)
          ⇒ #'(a (values 1 2)) #'(a <top>) #'(a)
          
          (parse-multiple-values-tagged-single-binding
            #'((a) 1)
            #'<top>
            synner)
          ⇒ #'((a) 1) #'(a <top>) #'(a)
          
          (parse-multiple-values-tagged-single-binding
            #'(((a <red> <blu>)) 1)
            #'<top>
            synner)
          ⇒ #'((a) 1) #'(a <red> <blu>) #'a
          
          (parse-multiple-values-tagged-single-binding
            #'((a b c) (values 1 2 3))
            #'<top>
            synner)
          ⇒ #'((a b c) (values 1 2 3))
             #'((a <top>) (b <top>) (c <top>))
             #'(a b c)
          
          (parse-multiple-values-tagged-single-binding
            #'(((a <red>) (b <blu>) (c <yel>)) (values 1 2 3))
            #'<top>
            synner)
          ⇒ #'((a b c) (values 1 2 3))
             #'((a <red>) (b <blu>) (c <yel>))
             #'(a b c)
— Function: parse-multiple-values-tagged-bindings tagged-bindings top-tag synner

Parse a list of possibly–tagged bindings each definining multiple values. Make use of PARSE-MULTIPLE-VALUES-TAGGED-SINGLE-BINDING.

Return three values: a syntax object representing the corresponding list of untagged bindings; a syntax object representing the corresponding list of with-class clauses; a syntax object holding a list of symbols being all the binding names.

It is a syntax violation if an identifier appears multiple times as variable name.

tagged-bindings must be a syntax object holding the list of bindings. top-tag must be the identifier to use as conventional class for untagged bindings, it is usually <top> from (nausicaa language classes). synner must be a closure used to raise syntax violations.

Example: 

          (parse-multiple-values-tagged-bindings
            #'((((a <n>)) 1)
               ((d) 4)
               ((g (h <n>) (i <t> <n>)) (values 7 8 9))
            #'<top>
            synner)
          ⇒ #'(((a) 1) ((d) 4) ((g h i) (values 7 8 9)))
              #'((a <n>) (d <top>) (g <top>) (h <h>) (i <t> <n>))
              #'(a d g h i)
