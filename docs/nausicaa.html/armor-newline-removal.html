63.8.2 Removing sequences of bytes

— Class: <newlines-decode-ctx>

Context record for newline removal.

— Immutable Field of <newlines-decode-ctx>: sequence

A bytevector representing the newline sequence of characters.

— Function: make-<newlines-decode-ctx> sequence
— Function: <newlines-decode-ctx>? obj

Constructor and predicate for records of type <newlines-decode-ctx>.

— Function: newlines-decode-update! ctx dst-bv dst-start src-bv src-start src-past

Copy bytes from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store the output in the bytevector dst-bv starting at dst-start (included); newline sequences are removed according to the specification in the context ctx, which must be a record of type <newlines-decode-ctx>.

Return three values:

  1. A boolean always false. This value is here to make this API equal to the one of the base64 decoder.
  2. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.

Bytes are read from src-bv until their number is less than the sequence length.

— Function: newlines-decode-final! ctx dst-bv dst-start src-bv src-start src-past

Copy bytes from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store the output in the bytevector dst-bv starting at DST-START (included); newline sequences are removed according to the the specification in the context ctx, which must be a record of type <newlines-decode-ctx>.

Return three values:

  1. A boolean: true if all the data from src-bv has been processed, false otherwise.
  2. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.
