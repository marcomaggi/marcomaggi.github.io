B.1.3 About writing patterns

At least one element in a subpattern with ellipsis

When we want to specify a pattern or subpattern that matches at least one element from the input form we have to use ?elm0 ?elm ...; this is because the ?elm ... subpattern matches zero or more elements. So the following correctly returns error when zero elements are used: 

     (import (rnrs) (nausicaa checks))
     (check-set-mode! 'report-failed)
     
     (define-syntax doit
       (syntax-rules ()
         ((_ ?elm0 ?elm ...)
          '(?elm0 ?elm ...))
         ((_ . ?rest)
          'error)))
     
     (check (doit 1)         => '(1))
     (check (doit 1 2 3)     => '(1 2 3))
     (check (doit)           => 'error)
     
     (check-report)

while the following happily matches zero arguments to the macro: 

     (import (rnrs) (nausicaa checks))
     (check-set-mode! 'report-failed)
     
     (define-syntax doit
       (syntax-rules ()
         ((_ ?elm ...)
          '(?elm ...))
         ((_ . ?rest)
          'error)))
     
     (check (doit 1)         => '(1))
     (check (doit 1 2 3)     => '(1 2 3))
     (check (doit)           => '())
     
     (check-report)

here the (_ . ?rest) is never matched.

Matching proper lists but not improper lists

When the ?arg ... and . ?args subpatterns appear at the end of a list, they both match all the other elements (the cdr), which can be zero or more. There is one difference: while ?arg ... matches only a proper list, . ?args matches both proper and improper lists. So the following works always: 

     (import (rnrs) (nausicaa checks))
     (check-set-mode! 'report-failed)
     
     (define-syntax doit
       (syntax-rules ()
         ((_ . ?args)
          '?args)))
     
     (check (doit 1)         => '(1))
     (check (doit 1 2 3)     => '(1 2 3))
     (check (doit 1 . 2)     => '(1 . 2))
     
     (check-report)

but the following raises an “invalid syntax” error in the improper list check: 

     (import (rnrs) (nausicaa checks))
     (check-set-mode! 'report-failed)
     
     (define-syntax doit
       (syntax-rules ()
         ((_ ?arg ...)
          '(?arg ...))))
     
     (check (doit 1)         => '(1))
     (check (doit 1 2 3)     => '(1 2 3))
     (check (doit 1 . 2)     => '(1 . 2))
     
     (check-report)

Notice that the <formals> argument of a lambda can be an improper list when we use the (arg1 arg2 . args) form to bind the rest arguments to args. So to properly match <formals> the general <formals> we have to use the ?arg . ?args subpattern.
