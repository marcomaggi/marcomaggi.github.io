77.1.3 Full lexer table token–makers

The bindings documented in this section are exported by the (nausicaa parser-tools r6rs lexer) library; the functions are the default token makers, used to build <lexical-token> records representing lexemes from the input (see lexical-tokens).

All the functions build and return instances of <lexical-token> records; all the functions make use of the current-input-source parameter to fill the input field of <source-location> records.

When a function cannot return a Scheme token, because the input string is incorrect: it makes use of the lexical-error-token-maker parameter to build and return a <lexical-token> record representing a lexer error. This happens, for example, if the hex character string holds a hex number out of range for Unicode.

All the functions have the following behaviour: 

     (lambda (yygetc yyungetc yytext yyline yycolumn yyoffset)
       (make <lexical-token> ---))

where the arguments are the bindings handed by (nausicaa parser-tools silex lexer) to the semantic action forms (see silex semantics action).
