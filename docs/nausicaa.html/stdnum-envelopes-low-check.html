72.1.4.2 Generating check digits

The following bindings are exported by the (nausicaa standard-numbers envelopes low) library. For all the functions, the meaning of the common arguments is the following:

sequence->check-digit
A function accepting a thunk as argument and returning the check digit as fixnum; the thunk must return the digits from the sequence one at a time and finally #f when no more input digits are available.
value->digit
A function mapping a value extracted from the a sequence to a fixnum digit in the range required by an algorithm.
— Function: list->left-fold-check-digit ell sequence->check-digit value->digit

Process the list sequence ell left–to–right and return the check digit as fixnum.

— Function: string->left-fold-check-digit str sequence->check-digit value->digit

Process the string sequence str left–to–right and return the check digit as fixnum.

— Function: string->right-fold-check-digit str sequence->check-digit value->digit

Process the string sequence str right–to–left and return the check digit as fixnum.

— Function: vector->left-fold-check-digit vec sequence->check-digit value->digit

Process the vector sequence vec left–to–right and return the check digit as fixnum.

— Function: vector->right-fold-check-digit vec sequence->check-digit value->digit

Process the vector sequence vec right–to–left and return the check digit as fixnum.

— Function: bytevector-u8->left-fold-check-digit bv sequence->check-digit value->digit

Process the unsigned bytes bytevector sequence bv left–to–right and return the check digit as fixnum.

— Function: bytevector-u8->right-fold-check-digit bv sequence->check-digit value->digit

Process the unsigned bytes bytevector sequence bv right–to–left and return the check digit as fixnum.
