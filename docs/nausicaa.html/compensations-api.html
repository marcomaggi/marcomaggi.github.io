26.2 Compensations API

— Syntax: compensate alloc-form0 alloc-form ... (with release-form0 release-form ...)
— Auxiliary Syntax: with

Evaluate all the alloc-forms and, if no error occurs, push a closure holding the release-forms on the current compensations stack. Return the return value of the last alloc-form.

— Syntax: with-compensations form ...
— Syntax: with-compensations/on-error form ...

Initialise a new local compensations stack, then evaluate all the forms. If no error occurs, return the return value of the last form.

with-compensations always invokes run-compensations after the forms have been evaluated.

with-compensations/on-error invokes run-compensations after the forms have been evaluated only if an exception was raised.

If the forms raise an exception: the exception is intercepted, the compensations are run, the exception is raised again as non–continuable exception. It is not possible to return to the forms with a continuable exception.

— Function: run-compensation

Evaluate all the compensation forms that are in the current stack, in last in/first out order. The compensations are evaluated in the body of a defer-exceptions, so exceptions raised are handled as described in deferred exceptions.

— Syntax: push-compensation release-form0 release-form ...

Push a closure holding the release-forms on the current compensations stack.
