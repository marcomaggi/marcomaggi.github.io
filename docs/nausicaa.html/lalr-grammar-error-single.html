75.5.4.2 Dummy example: Single token parser

Let's consider the following parser which accepts as input only the sequence: ‘A’, ‘*eoi*’; it is a corner case which is useful to understand. The correct way to handle this case is to have an error handler that does not return; in the following the error handler returns, leaving the parser to handle everything. 

     (define (example . tokens)
       (let* ((lexer        (make-lexer tokens))
              (make-parser  (lalr-parser
                              (output-value: #t)
                              (terminals: '(A))
                              (rules: '((e (A) : $1)))))
              (parser       (make-parser)))
         (parser lexer error-handler)))

We have already examined the correct usage of this grammar (lalr grammar examples single); we can now understand what happens in the following test: 

     (example (make-lexical-token 'A #f 1)
              (make-lexical-token 'A #f 2)
              (make-lexical-token 'A #f 3)))
     ⇒ ("unexpected end of input" . (eof-object))

following the reasoning exposed in the introduction (see lalr intro), we can imagine the input tokens on the following stacks: 

     #(#f  1  2  3  #<eof>)
     #(S   A  A  A   *eoi*)
       |
       p
  1. The first lookahead returns ‘A’ and the action is a shift: 
              #(#f  1  2  3  #<eof>)
          #(S   A  A  A   *eoi*)
                |
                p
  2. The second lookahead returns ‘A’ and the action is a reduce, which changes nothing because there is only one token and the semantic action is ‘$1’: 
              #(#f  1  2  3  #<eof>)
          #(S   A  A  A   *eoi*)
                |
                p
  3. The third lookahead returns ‘A’ and the action is error because only ‘*eoi*’ is acceptable now. The error handler is invoked and its return value discarded.

    The parser attempts to recover from the error; we already know that the recovery will fail, because we have not included any ‘error’ right–hand side rule in the grammar.

  4. The first operation of recovery is to reset the current parser state removing everything related to this right–hand side rule: 
              #(#f  2  3  #<eof>  #f)
          #(S   A  A   *eoi*  #f)
            |
            p
  5. The parser detects the fact that the stack has been fully rewind without finding a non–terminal with an ‘error’ directive, so it just stops reading tokens from the lexer and imposes a ‘*eoi*’ token as lookahead.
  6. The end–of–input token is invalid as lookahead from the initial state, so an “unexpected end–of–input error” is generated; the error handler is invoked and its return value becomes the return value of the parser closure.
