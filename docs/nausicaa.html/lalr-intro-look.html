60.1.4 Why the lookahead

Why do we need to look ahead at the next token's category? When the stacks are: 

     #(#f  1  +  2  #<eof>)
     #( S  N  A  N   *eoi*)
                 |
                 p

we can recognise the subvector ‘N A N’ to be a reducible one without looking a the next token's category; the same goes with: 

     #(#f  1  +  2  -  3  #<eof>)
     #( S  N  A  N  A  N   *eoi*)
                 |
                 p

But let's look at: 

     #(#f  1  +  2  *  3  #<eof>)
     #( S  N  A  N  M  N   *eoi*)
                 |
                 p

it is an error to reduce ‘N A N’, because the following multiplication takes precedence over addition; we can detect the need to give precedence to what follows only by looking at the next token's category.

This is not always the case. Let's make a change to our parser, let's say we want to parse scripts in which every line is an arithmetic expression. We add a new token with category ‘T’, as in terminator, and value ‘#\newline’.

Let's look at this example: 

     #(#f  1  +  2  *  3  #\newline  4  /  5  #\newline  #<eof>)
     #( S  N  A  N  M  N  T          N  M  N  T           *eoi*)
                       |
                       p

once the parser has performed the lookahead and detected the ‘T’ category, the grammar can tell it to reduce twice the stack, without doing another lookahead: 

     #(#f  1  +  6  #\newline  4  /  5  #\newline  #<eof>  #f #f)
     #( S  N  A  N  T          N  M  N  T           *eoi*  #f #f)
                 |
                 p
     
     #(#f  7  #\newline  4  /  5  #\newline  #<eof>  #f #f #f #f)
     #( S  N  T          N  M  N  T           *eoi*  #f #f #f #f)
           |
           p

Notice that, with a well written grammar, the parser generator can understand that the result of the first line is no more needed and remove it with a reduce action. Starting from the above stacks, the lookahead returns ‘T’ and the action is shift: 

     #(#f  7  #\newline  4  /  5  #\newline  #<eof>  #f #f #f #f)
     #( S  N  T          N  M  N  T           *eoi*  #f #f #f #f)
              |
              p

now the lookahead returns ‘N’ and the action can be a reduction which puts the stacks in the following state: 

     #(#f  4  /  5  #\newline  #<eof>  #f #f #f #f #f #f)
     #( S  N  M  N  T           *eoi*  #f #f #f #f #f #f)
        |
        p

parser is ready to start a new line.
