61.4.1 Expecting a token category

One of the most common operations is to verify that the next token belongs to a specific category; in the end, the state of the parser is advanced when a combinator matches one or more tokens with the specified category. This is used also to check that a <state> record represents the end of input, that is if it has category ‘*eoi*’.

Basically, a combinator matching a token with category ‘NUMBER’ should look like this: 

     (lambda (state)
       (let ((token (<state>-lookahead state)))
         (if (eq? 'NUMBER (<lexical-token>-category token))
             (make-<success> (next-state state)
                             (<lexical-token>-value token))
           (make-<error> state "expected NUMBER token"))))

notice how the returned <result> record holds the next state if it is a <success>, and the current state if it is an <error>. We want a method to generate such combinators with the category as parameter.

— Function: make-terminal-combinator category acceptor

Return a combinator which, if the next token has category eq? to category, applies the procedure acceptor to its semantic value. The result of this application must be a combinator, which is applied to the <state> record representing the remainder of the input stream.

The following example defines a combinator which checks that the next token has category ‘NUMBER’ and, if successful, returns a ‘success’ record (holding the next <state> record). The acceptor discards the semantic value and returns a combinator which just builds a <success> record. 

     (define comb
       (make-terminal-combinator 'NUMBER
         (lambda (semantic-value)
           (lambda (state)
             (make-<success> state semantic-value)))))

The following example defines combinators which check that the next two tokens have categories ‘A’ and ‘B’ and, if successful, return a ‘success’ record holding the semantic value of the second. 

     (define comb-a
       (make-terminal-combinator 'A
         (lambda (semantic-value)
           comb-b)))
     
     (define comb-b
       (make-terminal-combinator 'B
         (lambda (semantic-value)
           (lambda (state)
             (make-<success> state semantic-value)))))

The following example defines a combinator which checks that the next two tokens have category ‘NUMBER’ and, if successful, returns a ‘success’ record holding the sum of the semantic values; the second combinator is dynamically generated, so that it can reference the semantic value of the first result: 

     (define comb
       (make-terminal-combinator 'NUMBER
         (lambda (a)
           (make-terminal-combinator 'NUMBER
             (lambda (b)
               (lambda (state)
                 (make-<success> state (+ a b))))))))
