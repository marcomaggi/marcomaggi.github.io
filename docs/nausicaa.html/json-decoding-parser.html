67.2.3 Parsing JSON text

The following bindings are exported by the (nausicaa json) library.

— Function: make-json-parser

Build and return a new JSON parser using (nausicaa lalr lr-driver); the returned value is a parser function as described in lalr parser. The parser function returns the S–expression representing the JSON text; notice that strings in the S–expression are not processed by json-decode-string.

— Syntax: make-json-event-parser clause ...

Build and return a new JSON parser which calls provided functions when parsing elements. The returned parser function matches the interface of the parser functions based upon (nausicaa lalr lr-driver).

Event parsing is driven by the clauses described below.

— Auxiliary Syntax: begin-object: handler

Used when the parser receives a BEGIN_OBJECT token. handler is called with the symbol begin-object as single argument.

— Auxiliary Syntax: end-object: handler

Used when the parser receives a END_OBJECT token. handler is called with the symbol end-object as single argument.

— Auxiliary Syntax: begin-array: handler

Used when the parser receives a BEGIN_ARRAY token. handler is called with the symbol begin-array as single argument.

— Auxiliary Syntax: end-array: handler

Used when the parser receives a END_ARRAY token. handler is called with the symbol end-array as single argument.

— Auxiliary Syntax: begin-pair: handler

Used when the parser processes the name of a pair. handler is called with the symbol begin-pair as first argument and the string representing the pair's name as second argument.

— Auxiliary Syntax: end-pair: handler

Used when the parser terminates processing of a pair. handler is called with the symbol end-pair as single argument.

— Auxiliary Syntax: atom: handler

Used when the parser parses the value of a pair or an array element being a string, a literal or a number. When the value is:

A number
handler is called with the symbol number as first argument and the number itself as second argument.
A string
handler is called with the symbol string as first argument and the string itself as second argument. The string is not processed with json-decode-string.
The literal true
handler is called with the symbol true as first argument and #t as second argument.
The literal false
handler is called with the symbol false as first argument and #f as second argument.
The literal null
handler is called with the symbol null as first argument and () as second argument.
