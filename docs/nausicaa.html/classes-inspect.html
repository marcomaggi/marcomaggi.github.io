24.4 Class and instance inspection

Type descriptors
— Macro: class-type-descriptor class-name

Expand to the class descriptor associated to class-name.

— Macro: class-record-type-descriptor class-name

Expand to the record type descriptor associated to class-name. It is a replacement of record-type-descriptor for record types defined by define-class.

Constructor descriptors
— Macro: class-public-constructor-descriptor class-name

Expand to the public constructor descriptor associated to class-name.

— Macro: class-superclass-constructor-descriptor class-name

Expand to the superclass constructor descriptor associated to class-name.

— Macro: class-from-fields-constructor-descriptor class-name

Expand to the from–fields constructor descriptor associated to class-name.

Unique class identifiers
— Macro: class-type-uid class-name

Expand to the quoted symbol representing the class' UID.

— Macro: class-uid-list class-name

Expand into the list of type UIDs of the parents of class-name. The first element is the UID of the class itself, then comes the UID of the parent, then the UID of the parent's parent, etc. When inspecting a type whose parents are all classes (rather than records), the last element is always the UID of <top>.

— Function: class-uid-list-of obj

Return a list of record type UIDs representing the inheritance hierarchy of the record type associated to obj. This function supports both records and conventional builtin types.

— Function: class-uid-equal-or-parent? maybe-parent-uid-list maybe-child-uid-list

Accept as arguments lists of record type UID like those returned by class-uid-list and class-uid-list-of; return true if maybe-parent-uid-list represents a record type who is equivalent or parent of the record type represented by maybe-child-uid-list.

Instance identification
— Function: record-type-of obj

Return the record type descriptor associated to obj. This function supports both regular and conventional record type descriptors.

Class hierarchy
— Macro: class-parent-rtd-list class-name

Expand into the list of record type descriptors which are parents of class-name. The first element of the list is the RTD of class-name itself, then comes the RTD of the parent, then the RTD of the parent's parent, etc. When inspecting a type whose parents are all classes (rather than records), the last element is always the RTD of <top>.

— Function: record-type-parent? maybe-parent-rtd maybe-child-rtd

Return true if maybe-parent-rtd and maybe-child-rtd describe the same type, or maybe-parent-rtd is an ancestor of maybe-child-rtd.

— Function: Function record-parent-list rtd

Return the list of record type descriptors which are ancestors of rtd; the first element of the list is rtd itself, followed by its parent, then the parent of the parent and so on. This function supports both regular and conventional record type descriptors.
