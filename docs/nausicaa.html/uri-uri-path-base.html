11.2.5.3 Base URI path objects

The following bindings are exported by the (nausicaa uri uri) library.

— Abstract Class: <path>

Base class representing path components of URIs.

Constructors and makers
— Super constructor on <path>: constructor (obj <list-of-segments>)

Build and return a new instance of <path>. The argument must be a list of objects satisfying the requirements of <list-of-segments>.

— Function: make-path-object (path-type <symbol>) path

Utility function returning a subclass of <path>. path-type must be a symbol among: path-empty, path-abempty, path-absolute, path-noscheme, path-rootless. path must be, a possibly empty, list of <segment> objects.

Depending on the value of path-type the returned value is of class: <path-empty>, <path-abempty>, <path-absolute>, <path-rootless>, <path-noscheme>.

The arguments of this function are modeled after the return values of parse-path from (nausicaa parser-tools uri), parse-path.

Fields
— Immutable virtual field of <path>: <list-of-segments> path

A possibly empty list of <segment> instances.

— Immutable virtual field of <path>: <ascii-bytevector> bytevector

A bytevector representing the path component in the format requested by RFC 3986.

— Immutable virtual field of <path>: <ascii-string> string

A string representing the path component in the format requested by RFC 3986.

— Immutable virtual fields of <path>: <symbol> type

A symbol among: path-empty, path-abempty, path-absolute, path-noscheme, path-rootless.

Methods
— Method on <path>: put-bytevector (port <binary-output-port>)

Write to port a bytevector representing the path component in the format requested by RFC 3986.
