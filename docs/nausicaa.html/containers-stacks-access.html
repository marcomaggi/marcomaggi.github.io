10.5.3 Stack accessors and mutators

The following bindings are exported by the library (vicare containers stacks). The bindings whose name is prefixed with $ are unsafe operations: they do not validate their arguments before accessing them.

— Method on <stack>: top
— Method on <stack>: $top

Return the object at the top of the stack. Raise an assertion violation if the instance is empty.

— Method on <stack>: push! obj
— Method on <stack>: $push! obj

Push obj on the <stack>.

— Method on <stack>: pop!
— Method on <stack>: $pop!

Remove the object at the top of the <stack> and return it. Raise an assertion violation if stack is empty.
