24.7.2.2 Tagged bindings with let-values semantics
— Syntax: let-values/with-class ((vars init) ...) . body

Like let-values but automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (let-values/with-class
              ((((a <red>)   (b <blue>))   (values 1 2))
               (((c <green>) (d <white>))  (values 3 4)))
            (list a b c d))

is expanded to: 

          (let-values (((a b)  (values 1 2))
                       ((c d)  (values 3 4)))
            (with-class ((a <red>)
                         (b <blue>)
                         (c <green>)
                         (d <white>))
              (list a b c d)))

Each vars element can be one among: 

          id
          (id0 id ...)
          ((id0 tag0 ...)
           (id  tag  ...)
           ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class or label names.

If all the vars arguments are in the first or second form (untagged identifier): let-values/with-class defaults to let-values as defined by R6RS. If at least one vars has tags: the untagged ones default to bindings with <top> as conventional class name.

Notice the case of vars being a single identifier id, for example: 

          (let-values/with-class ((all (values 1 2 3)))
            all)
          ⇒ (1 2 3)

exactly as in let-values from (rnrs base (6)); in this case id can appear only untagged.

— Syntax: let*-values/with-class ((vars init) ...) . body

Like let*-values but, binding by binding, automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (let*-values/with-class
              ((((a <red>)   (b <blue>))   (values 1 2))
               (((c <green>) (d <white>))  (values 3 4)))
            (list a b c d))

is expanded to: 

          (let-values/with-class
              ((((a <red>) (b <blue>)) (values 1 2)))
            (let-values/with-class
                ((((c <green>) (d <white>)) (values 3 4)))
              (list a b c d)))

and in the end to: 

          (let-values (((a b) (values 1 2)))
            (with-class ((a <red>)
                         (b <blue>))
              (let-values (((c d) (values 3 4)))
                (with-class ((c <green>)
                             (d <white>)))
                  (list a b c d))))

Each vars element can be one among: 

          id
          (id0 id ...)
          ((id0 tag0 ...)
           (id  tag  ...)
           ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class or label names.

If all the vars arguments are in the first or second form (untagged identifier): let*-values/with-class defaults to let*-values as defined by R6RS. If at least one vars has tags: the untagged ones default to bindings with <top> as conventional class name.

Notice the case of vars being a single identifier id, for example: 

          (let*-values/with-class ((all (values 1 2 3)))
            all)
          ⇒ (1 2 3)

exactly as in let*-values from (rnrs base (6)); in this case id can appear only untagged.
