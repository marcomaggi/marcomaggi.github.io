30.2.7 Bytevectors conversion

The following bindings are exported by (nausicaa ffi memory bytevectors) and re–exported by (nausicaa ffi memory). The API requires the (nausicaa ffi memory memblocks) library when <memblock> records are used.

In the implementation of the following functions:

  • When copying from a bytevector to a memory block, bytes are peeked with bytevector-u8-ref and poked with pointer-set-c-uint8!.
  • When copying from a memory block to a bytevector bytes are peeked with pointer-ref-c-uint8 and poked with bytevector-u8-set!.

Remember that we can compare two bytevectors with bytevector=?, but to compare raw blocks of memory we can use the memcpy interface function.

— Function: bytevector->pointer bvector malloc
— Function: bytevector->pointer bvector malloc number-of-bytes
— Function: bytevector->pointer bvector malloc number-of-bytes offset

Allocate a memory block using malloc and copy into it the bytes in bvector. Return the pointer.

If number-of-bytes is given: That number of bytes is taken as size of the memory block, and only that number of bytes is copied.

If both number-of-bytes and offset are given: That number of bytes is taken as size of the memory block, and only that number of bytes is copied starting at offset from the beginning of the bytevector.

malloc must accept as single argument the number of bytes to allocate, and must return a pointer to the allocated memory block. If an error occurs allocating memory: malloc must raise an exception and not return to the caller.

— Function: bytevector->memblock bvector malloc
— Function: bytevector->memblock bvector malloc number-of-bytes
— Function: bytevector->memblock bvector malloc number-of-bytes offset

Allocate a memory block using malloc and copy into it the bytes in bvector. Return a <memblock> record.

If number-of-bytes is given: That number of bytes is taken as size of the memory block, and only that number of bytes is copied.

If both number-of-bytes and offset are given: That number of bytes is taken as size of the memory block, and only that number of bytes is copied starting at offset from the beginning of the bytevector.

malloc must accept as single argument the number of bytes to allocate, and must return a pointer to the allocated memory block. If an error occurs allocating memory: malloc must raise an exception and not return to the caller.

— Function: pointer->bytevector pointer number-of-bytes
— Function: pointer->bytevector pointer number-of-bytes offset

Build a bytevector of length number-of-bytes and copy into it the bytes from the block of memory referenced by pointer. If offset is given: Start to read bytes at offset from pointer.

— Function: memblock->bytevector memblock
— Function: memblock->bytevector memblock number-of-bytes
— Function: memblock->bytevector memblock number-of-bytes offset

Duplicate a <memblock> record into a bytevector.

If number-of-bytes is given: That number of bytes is taken as size of the bytevector, and only that number of bytes is copied.

If both number-of-bytes and offset are given: That number of bytes is taken as size of the bytevector, and only that number of bytes is copied starting at offset from the beginning of the block.
