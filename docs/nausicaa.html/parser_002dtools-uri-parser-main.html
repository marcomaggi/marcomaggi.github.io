12.5.4.5 Main components parser functions

The following functions can be used to parse the URI, absolute-URI, relative-ref and path components; the relevant portion of the grammar is: 

     URI           = scheme ":" hier-part [ "?" query ] [ "#" fragment ]
     
     relative-ref  = relative-part [ "?" query ] [ "#" fragment ]
     
     absolute-URI  = scheme ":" hier-part [ "?" query ]
     
     path          = path-abempty    ; begins with "/" or is empty
                   | path-absolute   ; begins with "/" but not "//"
                   | path-noscheme   ; begins with a non-colon segment
                   | path-rootless   ; begins with a segment
                   | path-empty      ; zero characters
     
     hier-part     = "//" authority path-abempty
                   | path-absolute
                   | path-rootless
                   | path-empty
     
     relative-part = "//" authority path-abempty
                   | path-absolute
                   | path-noscheme
                   | path-empty

The following bindings are exported by the library (nausicaa parser-tools uri). All the functions accept a port argument: it must be a binary input port supporting the port position.

— Function: parse-uri port

Read bytes from PORT expecting to get, from the first byte to the EOF, a URI component; parse the input decomposing it into its subcomponents. This function does not decode the percent–encoded bytes.

Return multiple values being:

scheme
A bytevector representing the scheme component; #f if the scheme is not present. According to the RFC: the scheme component is mandatory, but this function accepts its absence without raising exceptions.
authority
A bytevector representing the authority component, not including the leading slashes; #f if the authority is unspecified. The authority component is specified only if the hier-part component starts with ‘//’.
userinfo
A bytevector representing the userinfo component, not including the ending at–sign; #f if the userinfo is not present.
host.type
One of the symbols: reg-name, ipv4-address, ipv6-address, ipvfuture; when the host is empty: this value is reg-name. When the authority component is unspecified: this value is #f; notice that when the authority component is specified, the host component is specified too.
host.bv
host.data
Host data represented as the second and third return value from parse-host and associated to the symbol returned as host.type. If host.type is #f: these values are meaningless.
port-number
A bytevector representing the port component; #f if the port is not present.
path-type
One of the symbols: path-abempty, path-empty, path-absolute, path-rootless. When the authority is specified: this value is always path-abempty.
path
A possibly empty list of bytevectors representing the path segments.
query
A bytevector representing the query component; #f when the query is not present.
fragment
A bytevector representing the fragment component; #f when the fragment is not present.

If the host cannot be classified in reg-name, ip-literal or ipvfuture: an exception is raised with condition components &uri-parser-error, &who, &message, &irritants, the irritants being the input port.

If the path cannot be classified: an exception is raised with condition components &uri-parser-error, &who, &message, &irritants, the irritants being the input port.

— Function: parse-relative-ref port

Read bytes from port expecting to get, from the first byte to the EOF, a relative-ref component; parse the input decomposing it into its subcomponents. This function does not decode the percent–encoded bytes.

Return multiple values being:

authority
A bytevector representing the authority component; #f if the authority is not present. The authority component is specified only if the relative-part component starts with ‘//’.
userinfo
A bytevector representing the userinfo component, not including the ending at–sign; #f if the userinfo is not present.
host.type
One of the symbols: reg-name, ipv4-address, ipv6-address, ipvfuture; when the host is empty: this value is reg-name. When the authority component is unspecified: this value is #f; notice that when the authority component is specified, the host component is specified too.
host.bv
host.data
Host data represented as the second and third return value from parse-host and associated to the symbol above. If host.type is #f: these values are meaningless.
port-number
A bytevector representing the port component; #f if the port is not present.
path-type
One of the symbols: path-abempty, path-empty, path-absolute, path-noscheme. When the authority is present: this value is always path-abempty.
path
A possibly empty list of bytevectors representing the path segments.
query
A bytevector representing the query component; #f when the query is not present.
fragment
A bytevector representing the fragment component; #f when the fragment is not present.

If the host cannot be classified in reg-name, ip-literal or ipvfuture: an exception is raised with condition components &uri-parser-error, &who, &message, &irritants, the irritants being the input port.

If the path cannot be classified: an exception is raised with condition components &uri-parser-error, &who, &message, &irritants, the irritants being the input port.

— Function: parse-path port

Parse from port a path component. Return two values: #f or one of the symbols: path-empty, path-absolute, path-abempty, path-noscheme, path-rootless; the list of bytevectors representing the segments, possibly null.

If successful: leave the port position to the byte after the last read byte; if an error occurs: rewind the port position to the one before this function call.

Notice that the path component can be followed only by EOF; if the path cannot be classified: an exception is raised with condition components &uri-parser-error, &who, &message, &irritants, the irritants being the input port. 

          (import (vicare)
            (prefix (nausicaa parser-tools uri) uri.))
          
          (define (mkport S)
            (open-bytevector-input-port (string->ascii S)))
          
          (uri.parse-path (mkport ""))
          ⇒ path-empty ()
          
          (uri.parse-path (mkport "?query"))
          error--> &uri-parser-error
          (uri.parse-path (mkport "#fragment"))
          error--> &uri-parser-error
          
          (uri.parse-path (mkport "/"))
          ⇒ path-absolute (#vu8())
          
          (let*-values
              (((P)          (mkport "/ciao/hello/salut"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-absolute
            (map ascii->string path))    ⇒ ("ciao" "hello" "salut")
          
          (let*-values
              (((P)          (mkport "//"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-abempty
            (map ascii->string path))    ⇒ ("" "")
          
          (let*-values
              (((P)          (mkport "///"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-abempty
            (map ascii->string path))    ⇒ ("" "" "")
          
          (let*-values
              (((P)          (mkport "//ciao/"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-abempty
            (map ascii->string path))    ⇒ ("" "ciao" "")
          
          (let*-values
              (((P)          (mkport "ciao/hello"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-noscheme
            (map ascii->string path))    ⇒ ("ciao" "hello")
          
          (let*-values
              (((P)          (mkport "ci:ao/hello"))
               ((type path)  (uri.parse-path P)))
            type                         ⇒ path-rootless
            (map ascii->string path))    ⇒ ("ci:ao" "hello")
