34.1 Basic iterators API

The following bindings are exported by the (nausicaa iterators) library.

— Abstract Class: <iterator>

Base type for all the iterators.

— Immutable field of <iterator>: subject

The container being the subject of the iteration. This field is untagged; classes derived from <iterator> can shadow it with a tagged virtual field.

— Mutable field of <iterator>: current

The last value extracted from the subject of the iteration. Concrete implementations of the method next must update this field every time they extract a new value.

— Superclass constructor on <iterator>: constructor subject

Build a new instance of <iterator> whose subject field is initialised with subject and whose current field is initialised with the sentinel value, sentinel. Return the constructor function for the subclass which invoked this protocol.

— Virtual method on <iterator>: next

Advance the iteration, return the value extracted from the subject; memoize the extracted value in the current field.

If the iteration is over: raise a non–continuable exception with condition object of type &stop-iteration, iterators conditions; once the iteration is over, calling this method multiple times always causes &stop-iteration to be raised.

— Virtual method on <iterator>: more?

Return #t if more values can be extracted from the container. Whenever possible, concrete implementations of this method must only test the availability of more values, they must not actually perform the operation.

The following is an example definition for a simple left–to–right Scheme string iterator: 

     #!r6rs
     (import (nausicaa)
       (nausicaa iterators))
     
     (define-class <simple-string-iterator>
       (nongenerative example:<simple-string-iterator>)
       (inherit <iterator>)
       (fields (mutable (index <integer>)))
       (virtual-fields (immutable (subject <string>))
                       (mutable   (current <char>)))
       (protocol
        (lambda (make-iterator)
          (lambda (string)
            (assert (string? string))
            ((make-iterator string) 0))))
       (maker ()
              (subject:    sentinel))
       (methods more? next))
     
     ;; accessor for the virtual field SUBJECT
     (define (<simple-string-iterator>-subject (I <iterator>))
       I.subject)
     
     ;; accessor and mutator for the virtual field CURRENT
     (define (<simple-string-iterator>-current (I <iterator>))
       I.current)
     (define (<simple-string-iterator>-current-set! (I <iterator>) value)
       (set! I.current value))
     
     ;;; virtual methods implementations
     
     (define-virtual-method <simple-string-iterator> (more? (I <simple-string-iterator>))
       (< I.index I.subject.length))
     
     (define (%string-iterator-step (I <simple-string-iterator>))
       (if (< I.index I.subject.length)
           (begin0-let ((retval (getf (I.subject I.index))))
             (set!  I.current retval)
             (incr! I.index))
         (raise (make &stop-iteration I))))
     
     (define-virtual-method <simple-string-iterator> next
       %string-iterator-step)
     
     ;;; tests
     
     (let* (((S <string>) "ciao")
            ((J <simple-string-iterator>)
             (make <simple-string-iterator>
               (subject: S)))
            ((I <iterator>) J))
     
       (is-a? I <iterator>)                  ⇒ #t
       (is-a? J <iterator>)                  ⇒ #t
       (is-a? J <simple-string-iterator>)    ⇒ #t
     
       (sentinel? I.current)      ⇒ #t
       I.subject                  ⇒ "ciao"
       J.subject.length           ⇒ 4
     
       (I.more?)     ⇒ #t
       (I.next)      ⇒ #\c
     
       (I.more?)     ⇒ #t
       (I.next)      ⇒ #\i
     
       (I.more?)     ⇒ #t
       (I.next)      ⇒ #\a
     
       (I.more?)     ⇒ #t
       (I.next)      ⇒ #\o
     
       (I.more?)     ⇒ #f
       (guard (E ((is-a? &stop-iteration)
                  E.iterator)
                 (else E))
         (I.next))   ⇒ I
     
       ;;once it is over, it is over forever
       (I.more?)     ⇒ #f
       (guard (E ((is-a? &stop-iteration)
                  E.iterator)
                 (else E))
         (I.next))   ⇒ I
     
       #f)
