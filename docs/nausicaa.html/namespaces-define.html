19.2 Defining namespaces

The following bindings are exported by the (nausicaa language namespaces) library. inherit and mixins are the same keywords exported by the (nausicaa language classes) library.

— Syntax: define-namespace ?name ?clause ...
— Auxiliary Syntax: inherit ?super-name
— Auxiliary Syntax: mixins ?mixin-name ...
— Auxiliary Syntax: namespaces ?sub-spec ...
— Auxiliary Syntax: functions ?func-spec ...
— Auxiliary Syntax: syntaxes ?syntax-spec ...
— Auxiliary Syntax: constants ?const-spec ...

Define a namespace referenced by the identifier ?name.

— Auxiliary Syntax: functions ?func-spec ...

Include function identifiers in a namespace. The ?func-spec arguments must have one of the forms: 

          ?func-name
          (?func-name ?func-identifier)

where ?func-name is the identifier to be used as suffix in dot notation, ?func-identifier is the actual identifier in the function binding; when ?func-identifier is not given, it defaults to ?func-name. 

          (define-namespace <<alpha>>
            (functions a b (c c-fun)))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c-fun) 'c)
          
          (with-namespace ((ns <<alpha>>))
            (list (ns.a) (ns.b) (ns.c)))
          ⇒ (a b c)
— Auxiliary Syntax: syntaxes ?syntax-spec ...

Include syntax identifiers in a namespace. The ?syntax-spec arguments must have one of the forms: 

          ?syntax-name
          (?syntax-name ?syntax-identifier)

where ?syntax-name is the identifier to be used as suffix in dot notation, ?syntax-identifier is the actual identifier in the syntax binding; when ?syntax-identifier is not given, it defaults to ?syntax-name. 

          (define-namespace <<alpha>>
            (functions a b (c c-syn)))
          
          (define-syntax a
            (syntax-rules ()
              ((_ 'a))))
          (define-syntax b
            (syntax-rules ()
              ((_ 'b))))
          (define-syntax c-syn
            (syntax-rules ()
              ((_ 'c))))
          
          (with-namespace ((ns <<alpha>>))
            (list (ns.a) (ns.b) (ns.c)))
          ⇒ (a b c)
— Auxiliary Syntax: constants ?const-spec ...

Include constant identifiers in a namespace. The ?const-spec arguments must have one of the forms: 

          ?const-name
          (?const-name ?const-identifier)

where ?const-name is the identifier to be used as suffix in dot notation, ?const-identifier is the actual identifier in the constant binding; when ?const-identifier is not given, it defaults to ?const-name. 

          (define-namespace <<alpha>>
            (constants a b (c c-const)))
          
          (define-constant a 'a)
          (define-constant b 'b)
          (define-constant c-const 'c)
          
          (with-namespace ((ns <<alpha>>))
            (list ns.a ns.b ns.c))
          ⇒ (a b c)
— Auxiliary Syntax: inherit ?super-name

The auxiliary syntax inherit allows a namespace to inherit bindings from another namespace and also to shadow them; the ?super-name argument must be the identifier name of the super–namespace. Inheritance example: 

          (define-namespace <<alpha>>
            (functions a b))
          
          (define-namespace <<beta>>
            (inherit <<alpha>>)
            (functions c d))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c) 'c)
          (define (d) 'd)
          
          (with-namespace ((ns <<beta>>))
            (list (ns.a) (ns.b) (ns.c) (ns.d)))
          ⇒ (a b c d)

another inheritance example with shadowing: 

          (define-namespace <<alpha>>
            (functions a b))
          
          (define-namespace <<beta>>
            (inherit <<alpha>>)
            (functions (b d) c))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c) 'c)
          (define (d) 'D)
          
          (with-namespace ((ns <<beta>>))
            (list (ns.a) (ns.b) (ns.c)))
          ⇒ (a D c)
— Auxiliary Syntax: mixins ?mixin-name ...

The auxiliary syntax mixins allows namespace definition clauses to be shared between namespaces; the ?mixin-name arguments must be namespace identifier names. Selecting a namespace a to be mixed in namespace b is equivalent to including all the definition clauses of a into the definition of b. 

          (define-namespace <<alpha>>
            (functions a b))
          
          (define-namespace <<beta>>
            (functions c d))
          
          (define-namespace <<delta>>
            (mixins <<alpha>> <<beta>>)
            (functions e f))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c) 'c)
          (define (d) 'd)
          (define (e) 'e)
          (define (f) 'f)
          
          (with-namespace ((ns <<delta>>))
            (list (ns.a) (ns.b) (ns.c) (ns.d) (ns.e) (ns.f)))
          ⇒ (a b c d e f)
— Auxiliary Syntax: namespaces ?sub-spec ...

Include namespaces as sub-namespaces of this namespace. The ?sub-spec arguments must have one of the forms: 

          ?sub-name
          (?sub-name ?sub-identifier)

where ?sub-name is the identifier to be used as suffix in dot notation, ?sub-identifier is the actual identifier referencing the namespace; when ?sub-identifier is not given, it defaults to ?sub-name.

Sub–namespace clauses examples: 

          (define-namespace <<alpha>>
            (functions a b))
          
          (define-namespace <<beta>>
            (functions c d))
          
          (define-namespace <<delta>>
            (namespaces (A <<alpha>>)
                        (B <<beta>>))
            (functions e f))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c) 'c)
          (define (d) 'd)
          (define (e) 'e)
          (define (f) 'f)
          
          (with-namespace ((ns <<delta>>))
            (list (ns.A.a) (ns.A.b)
                  (ns.B.c) (ns.B.d)
                  (ns.e)   (ns.f)))
          ⇒ (a b c d e f)

another example with deep nesting: 

          (define-namespace <<alpha>>
            (functions a b))
          
          (define-namespace <<beta>>
            (namespaces (A <<alpha>>))
            (functions c d))
          
          (define-namespace <<delta>>
            (namespaces (B <<beta>>))
            (functions e f))
          
          (define (a) 'a)
          (define (b) 'b)
          (define (c) 'c)
          (define (d) 'd)
          (define (e) 'e)
          (define (f) 'f)
          
          (with-namespace ((ns <<delta>>))
            (list (ns.B.A.a) (ns.B.A.b)
                  (ns.B.c)   (ns.B.d)
                  (ns.e)     (ns.f)))
          ⇒ (a b c d e f)
