24.8.8 Input/output port objects types

The binary/textual attributes are not mututally exclusive with the input/output attributes.

— Foreign Class: <port>

Conventional type for objects satisfying port?.

— Immutable Virtual Field of <port>: transcoder

Accessing this field is like applying port-transcoder to the port.

— Immutable Virtual Field of <port>: textual?

Accessing this field is like applying textual-port? to the port.

— Immutable Virtual Field of <port>: binary?

Accessing this field is like applying binary-port? to the port.

— Immutable Virtual Field of <port>: has-port-position?

Accessing this field is like applying port-has-port-position? to the port.

— Immutable Virtual Field of <port>: has-set-port-position?

Accessing this field is like applying port-has-set-port-position? to the port.

— Immutable Virtual Field of <port>: port-position

Accessing this field is like applying port-position to the port; mutating this field is like applying set-port-position! to the port.

— Immutable Virtual Field of <port>: eof?

Accessing this field is like applying port-eof? to the port.

— Immutable Virtual Field of <port>: input?

Accessing this field is like applying input-port? to the port.

— Immutable Virtual Field of <port>: output?

Accessing this field is like applying output-port? to the port.

— Foreign Class: <input-output-port>

Conventional type for objects satisfying input/output-port?.

— Foreign Class: <input-port>

Conventional type for objects satisfying input-port?.

— Foreign Class: <output-port>

Conventional type for objects satisfying output-port?.

— Foreign Class: <textual-port>

Conventional type for objects satisfying textual-port?.

— Foreign Class: <binary-port>

Conventional type for objects satisfying binary-port?.
