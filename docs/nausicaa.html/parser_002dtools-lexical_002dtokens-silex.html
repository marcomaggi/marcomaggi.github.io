12.2.5 SILex default semantic actions

The following bindings are exported by the library (nausicaa parser-tools silex default-error-handler). The library is meant to be included in generated SILex table libraries using the library-imports: clause of lex, A lexical analyser generator.

— Syntax: silex-default-error-handler
— Syntax: silex-default-error-handler ?yytext

Expand to a custom semantic action for the ‘<<ERROR>>’ rule; it is meant to be used in SILex table files as: 

          <<ERROR>>       (silex-default-error-handler)

The semantic action builds and returns a record of type <lexical-token> with category ‘*lexer-error*’ and semantic value set to an error string describing the input that caused the error; the default yytext binding is used to build the error string.

When the optional ?yytex argument is present: this value is used in place of yytext to build the error message string.

— Syntax: silex-default-eof-handler

Expand to a custom semantic action for the ‘<<EOF>>’ rule; it is meant to be used in SILex table files as: 

          <<EOF>>         (silex-default-eof-handler)

The semantic action builds and returns a record of type <lexical-token> with category ‘*eoi*’, semantic value set to (eof-object) and length set to zero.
