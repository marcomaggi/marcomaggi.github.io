5.13.2 Bytevector labels: 8-bit words

— Builtin Label: <bytevector-u8>
— Builtin Label: <bytevector-s8>

Conventional type for objects satisfying bytevector? whose contents must be accessed as octets or bytes.

— Builtin Label: <ascii-bytevector>

Conventional type for objects satisfying bytevector? whose contents must be accesses as octets representing ASCII characters; it is derived from <bytevector-u8>. The predicate of this label returns #t if all the octets in the bytevector are in the range [0, 127]; long bytevectors may need significant time to be validated this way.

— Builtin Label: <latin1-bytevector>

Conventional type for objects satisfying bytevector? whose contents must be accesses as octets representing Latin1 characters; it is derived from <bytevector-u8>. The predicate of this label returns #t if all the octets in the bytevector are in one of the ranges [#x20, #x7E] and [#xA0, #xFF]; long bytevectors may need significant time to be validated this way.

— Builtin Label: <percent-encoded-bytevector>

Conventional type for objects satisfying bytevector? whose contents represent a percent–encoded text as defined by RFC 3986; it is derived from <bytevector-u8>. The predicate of this label involves applying uri-encoded-bytevector?.

OOPP syntaxes
— Getter on <bytevector-u8>: instance (?index)
— Getter on <bytevector-s8>: instance (?index)

Octet or byte getter. ?index is the zero–based index of the selected byte or octet.

— Setter on <bytevector-u8>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-s8>: set!/tags (instance (?index)) ?item

Octet or byte setter. ?index is the zero–based index of the selected byte or octet.
