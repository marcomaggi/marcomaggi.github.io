24.1.2 Defining hierarchies of classes

Inheritance (or subclassing) is driven by the following clauses:

inherit
Makes a class a subtype of another class.
parent
Makes a class a subtype of a record type.
parent-rtd
Makes a class a subtype of a record type.

only one among inherit, parent and parent-rtd can be used at once.

— Auxiliary Syntax: inherit class-name
— Auxiliary Syntax: inherit class-name (option ...)

Make the class type of the enclosing definition a subtype of class-name; it is not possible to mimic what inherit does using parent or parent-rtd.

The concrete fields of the superclass are always included in the class type, making the underlying superclass record type a parent of the new class record type. The optional list of options allows us to select which members of the superclass are accessible through the dot notation implemented by the with-class syntax.

A missing or empty list of options means expose everything: concrete fields, virtual fields, methods, setter and getter; each option argument can be one among:

all
everything
Expose everything: concrete fields, virtual fields, methods, setter and getter.
dry
nothing
Expose nothing: only the members of the new class type are accessible through dot notation.
concrete-fields
no-concrete-fields
Include or exclude the concrete fields.
virtual-fields
no-virtual-fields
Include or exclude the virtual fields.
fields
no-fields
Include or exclude both the concrete and virtual fields.
methods
no-methods
Include or exclude the methods.
setter-and-getter
no-setter-and-getter
Include or exclude the setter and getter.

Options state is accumulated from the options list in left to right order, starting from the “all exposed” state; so the following lists of options are equivalent: 

          (nothing virtual-fields methods)
          (nothing fields methods no-concrete-fields)
          (all no-concrete-fields no-setter-and-getter)
