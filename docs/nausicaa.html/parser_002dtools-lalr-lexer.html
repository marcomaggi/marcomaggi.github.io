11.3.3 Describing tokens

The parser closure accepts as argument a lexer closure which, invoked with no arguments, must return the next token from the input. Tokens are described using objects of type <lexical-token>, as defined by the library (nausicaa parser-tools lexical-tokens); Describing lexical tokens.

Once the lexer closure finds the end of input, it must return a token with category ‘*eoi*’; it must continue to return such a token if invoked multiple times.

If the lexer closure finds a lexer error in the input, it must return a token with category ‘*lexer-error*’; lexer errors are unrecoverable.
