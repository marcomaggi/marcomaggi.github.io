25.1.7.5 Low level interface

The following bindings are exported by the (nausicaa ffi compat) library. This interface adapts the Scheme implementation FFI to the requirements of the Nausicaa distribution.

— Function: open-shared-object library-name

Open a shared library and return its reference object. library-name must be a Scheme string representing the file name or pathname to be opened. If the foreign symbol is not found: raise a continuable exception with raise-shared-object-opening-error.

— Function: lookup-shared-object library-reference foreign-symbol

Search the shared object referenced by library-reference for an exported symbol matching foreign-symbol, which must be a Scheme string, and return its address as pointer object. If the operation fails: raise a continuable exception with components &who, &message, &shared-object-lookup-error, &foreign-symbol.

— Function: make-c-callout ret-type address arg-types
— Function: make-c-callout/with-errno ret-type address arg-types

Build and return a new callout closure wrapping a foreign function referenced by the pointer object address.

ret-type must be a Scheme symbol in the enumeration enum-clang-types, representing the C language data type of the foreign function's return value.

arg-types must be a list of Scheme symbols in the enumeration enum-clang-types, representing the C language data types of the foreign function's arguments. To specify a foreign function accepting no arguments, arg-types must be (void).

Variadic functions are not supported.

— Function: make-c-callback ret-type scheme-function arg-types

Build and return a new pointer value referencing a callback object; the callback, when invoked, applies the scheme function

ret-type must be a Scheme symbol in the enumeration enum-clang-types, representing the C language data type of the foreign function's return value.

arg-types must be a list of Scheme symbols in the enumeration enum-clang-types, representing the C language data types of the foreign function's arguments. To specify a foreign function accepting no arguments, arg-types must be (void).

Variadic functions are not supported.

— Function: free-c-callback callback-pointer

Release all the resources associated to the referenced callback. This operation is supported by some Sceme implementations, and unsupported by other ones; in the latter case, calling this function does nothing.
