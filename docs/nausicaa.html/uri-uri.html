11.2 Uniform Resource Identifier objects

The library (nausicaa uri) exports utilities to handle Uniform Resource Identifiers (URI) specifications, according to RFC 3986: “Uniform Resource Identifier (URI): Generic Syntax”. The RFC defines URIs as follows:

A Uniform Resource Identifier (URI) is a compact sequence of characters that identifies an abstract or physical resource.

In the context of the Nausicaa libraries: the proper representation of a URI is the Scheme bytevector. Such representation can be parsed to produce objects of type <uri> or <relative-ref>; the OOPP API to URI parsing and producing is really basic; this is because advanced handling of URIs should be merged with requirements of specific URI schemes, like http: and file:.

The library (nausicaa uri) reexports all the bindings from (nausicaa uri ip). To parse a representation of URIs we can use the library (nausicaa parser-tools uri) and its simplification trough the library (nausicaa parser-tools uri utilities).

  • uri uri scheme: URI scheme objects.
  • uri uri userinfo: URI userinfo objects.
  • uri uri host: URI host objects.
  • uri uri port-number: URI port-number objects.
  • uri uri path: URI path objects.
  • uri uri query: URI query objects.
  • uri uri fragment: URI fragment objects.
  • uri uri uri: Uniform Resource Identifiers objects.
  • uri uri ref: Relative references objects.
