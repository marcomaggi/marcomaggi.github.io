64.1.3 Splitting an address into tokens

The lexer for the (nausicaa email addresses) library is built upon the (nausicaa silex) lexer. A lexical analyser generator

— Function: make-address-lexer IS

Return an email address lexer closure drawing tokens from the input system IS. The input system must have been created using the (nausicaa silex) library.

— Parameter: address-lexer-allows-comments

If set to true when make-address-lexer is invoked, the lexer closure returns also comment tokens; else comment tokens are silently discarded.

— Function: address->tokens IS

Wrapper for make-address-lexer which reads all the tokens and returns them as a list.

The lexer closure is a thunk returning <lexical-token> records with the following categories:

‘DOT’
‘COMMA’
‘COLON’
‘SEMICOLON’
‘AT’
Represent the named character; the semantic value is the character itself.
‘ANGLE-OPEN’
‘ANGLE-CLOSE’
Represent the ‘#\<’ and ‘#\>’ characters; the semantic value is the character itself.
‘ATOM’
The component of local parts and domains; the semantic value is the string. See RFC 2822 section “3.2.4 Atom”.
‘DOMAIN-LITERAL-OPEN’
The opening of a domain literal; the semantic value is a string containing the single character ‘[’.
‘DOMAIN-LITERAL-CLOSE’
The closing of a domain literal; the semantic value is a string containing the single character ‘]’.
‘DOMAIN-LITERAL-INTEGER’
A component of domain literal; the semantic value is an integer number between 0 included and 256 excluded.
‘QUOTED-TEXT’
A quoted text element; the semantic value is the string.
