12.2.2 Lexical token objects

The following bindings are exported by the library (nausicaa parser-tools lexical-tokens).

— Class: <lexical-token>

Hold the informations about a token from the input stream.

— Common constructor on <lexical-token>: constructor category location value length

Build and return a new instance of <lexical-token> using the given arguments as field values.

— Maker syntax on <lexical-token>: <lexical-token> (?clause ...)
— Auxiliary Syntax on <lexical-token>: category: ?category
— Auxiliary Syntax on <lexical-token>: location: ?location
— Auxiliary Syntax on <lexical-token>: value: ?value
— Auxiliary Syntax on <lexical-token>: length: ?length

Build and return a new instance of <lexical-token> using the default constructor.

The clause category: is mandatory: its value must be a symbol representing the token's category.

?location defaults to the return value of unspecified-source-location from the library (nausicaa parser-tools source-locations).

?value can be any object and should represent the token's value; it defaults to #f.

?length must be a non–negative fixnum representing the length in characters of the input string representing the token; it defaults to zero.

— Immutable field of <lexical-token>: <symbol> category

Represents the grammar's terminal symbol to which this token belongs. Every parser API must provide an interface to specify the list of allowed terminal symbols.

By convention: a token representing the end–of–input must have category set to ‘*eoi*’; a token representing a lexer error must have category set to ‘*lexer-error*’;

— Immutable field of <lexical-token>: <source-location> location

Represents the location in the input from which the token was built.

— Immutable field of <lexical-token>: value

Represents the semantic value associated with the token; it can be any Scheme value. When the token represents the end–of–input: this field is set to the return value of (eof-object).

— Immutable Field of <lexical-token>: <nonnegative-fixnum> length

It is meant to be the length of the string which generated the token. When the token represents the end–of–input: this field is set to zero.

— Method on <lexical-token>: <boolean> end-of-input?

Return #t if the token has category ‘*eoi*’, otherwise return #f.

— Method on <lexical-token>: <boolean> lexer-error?

Return #t if the token has category ‘*lexer-error*’, otherwise return #f.

— Method on <lexical-token>: <boolean> special?

Return #t if the token has category ‘*eoi*’ or ‘*lexer-error*’, otherwise return #f.
