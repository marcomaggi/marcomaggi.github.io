5.8 Spine label

A spine is the head of a proper list: null or a pair whose cdr is null or a pair. It can be used to iterate over a proper list without fully validating it first.

— Builtin Label: <spine>

Conventional type of objects satisfying null? or pair? and, in the case of a pair, having null or a pair as cdr.

— Common constructor on <spine>: <spine> constructor car cdr

Return the result of applying cons to car and cdr. cdr must be a <spine>.

Safe accessors
— Immutable virtual field of <spine>: <top> car

To be accessed only if the instance is a pair. Access the car of the pair using car.

— Immutable virtual field of <spine>: <spine> cdr

To be accessed only if the instance is a pair. Access the cdr of the pair using cdr.

— Immutable virtual field of <spine>: <exact-integer> length

Access the length of the list.

— Immutable virtual field of <spine>: <boolean> null?

True if the spine is null.

— Immutable virtual field of <spine>: <boolean> pair?

True if the spine is a pair.

Unsafe accessors
— Immutable virtual field of <spine>: <top> $car

To be accessed only if the instance is a pair. Access the car of the pair using the unsafe operation $car from (vicare system $pairs).

— Immutable virtual field of <spine>: <spine> $cdr

To be accessed only if the instance is a pair. Access the cdr of the pair using the unsafe operation $cdr from (vicare system $pairs).

OOPP syntaxes
— Maker on <spine>: <spine> ()
— Maker on <spine>: <spine> (?car ?cdr)

When the first form is used: return null. When the second form is used: build and return a new <spine> object using the common protocol.
