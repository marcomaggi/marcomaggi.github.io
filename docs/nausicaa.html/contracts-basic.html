20.2 Basic contracts definition

— Syntax: define-contract keyword subject contract
— Auxiliary Syntax: ->

Define an identifier syntax keyword which can be used in place of the identifier subject to reference a variable, function or macro. contract can be one among: 

          (arg-predicate ... -> ret-predicate ...)
          (arg-predicate ... ->)
          (arg-predicate ...)
          (-> ret-predicate ...)
          predicate

where predicate, arg-predicate and ret-predicate are identifiers; in the last case the contract is for a variable, in the other cases it is for a function or macro.

The contract for a variable validates the value using the selected predicate whenever the variable is accessed or mutated. When a variable is accessed: subject is referenced only once. When a variable is mutated: its new value's expression is evaluated only once. Notice that the contract is not enforced for the initialisation value bound to subject. Example of variable with contract: 

          (import (rnrs) (nausicaa language contracts))
          
          (define-contract alpha %alpha integer?)
          (define %alpha 123)
          
          alpha ⇒ 123
          
          (begin
            (set! alpha 456)
            alpha) ⇒ 456
          
          (set! alpha #\a) error--> assertion violation

The contract for a function or macro validates the arguments with the function or macro bound to arg-predicate and validates the return values with the function or macro bound to ret-predicate; the function or macro bound to subject must accept a number of arguments equal to the number of arg-predicate identifiers and return a number of values equal to the number of ret-predicate identifiers; a function can have no arguments and no return values; notice that a function referenced through an identifier syntax cannot take a “rest” argument. When applying the function or macro, subject is referenced only once. Example of function with contract: 

          (import (rnrs) (nausicaa language contracts))
          
          (define (%doit a b c)
            (list a b c))
          
          (define-contract doit
              %doit
            (integer? string? symbol? -> list?))
          
          (doit 1 "two" 'three) ⇒ (1 "two" 'three)

If a predicate returns false: a non–continuable exception is raised with condition types &contract-violation, &message and &irritants. The &irritants condition object holds a list with a single item: the offending value.

Notice that it is possible to define a contract for a function, then use it or not in its own body, recursively: 

          (define-contract alpha
              %alpha
            (integer? -> integer?))
          
          (define (%alpha n)
            ;; recursive use without contract
            (if (zero? n)
                1
              (%alpha (- n 1))))
          
          (define-contract beta
              %beta
            (integer? -> integer?))
          
          (define (%beta n)
            ;; recursive use with contract
            (if (zero? n)
                1
              (beta (- n 1))))

The fact that a function with a contract does not support the “rest” argument is a bit annoying; we can overcome this limitation as follows: 

          (import (rnrs) (nausicaa language contracts))
          
          (define-contract doit
              %doit
            (integer? integers? -> vector?))
          
          (define (integers? ell)
            (for-all integer? ell))
          
          (define (%doit n ell)
            (vector n ell))
          
          (doit 1 (list 2 3 4))
          ⇒ #(1 (2 3 4))
          
          (apply (lambda (n . args)
                   (doit n args))
                 '(1 2 3 4))
          ⇒ #(1 (2 3 4))

it involves an ugly notation, but it works.
