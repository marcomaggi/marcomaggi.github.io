19.3 Derived contract–related syntaxes

— Syntax: define/contract (keyword . args) contract body0 body ...
— Syntax: define/contract keyword contract expression
— Syntax: define/contract keyword contract
— Auxiliary Syntax: ->

Define a new variable and a contract for its value, or a function and a contract for its arguments and return values. contract must be as described in the documentation of define-contract. The first form is equivalent to: 

          (define-contract keyword
              the-subject
            contract)
          
          (define/with-class (the-subject . args)
            body0 body ...)

where define/with-class is the binding from (nausicaa language classes); notice that the defined function can reference itself recursively with the keyword identifier, so, internally, it has to satisfy its contract. The second form is equivalent to: 

          (define-contract keyword
              the-subject
            contract)
          
          (define the-subject expression)

and the third to: 

          (define-contract keyword
              the-subject
            contract)
          
          (define the-subject)
— Syntax: let-contract ((keyword subject contract) ...) body0 body ...
— Auxiliary Syntax: ->

Does the same job of define-contract but with a syntax similar to let-syntax; the relation between let-contract and define-contract is the same as the relation between let-syntax and define-syntax.

Notice that let-contract expands into a single let-syntax form.

— Syntax: with-outer-contracts ((keyword contract) ...) body0 body ...
— Auxiliary Syntax: ->

Expand to a begin form whose forms are a sequence of contract definitions: 

          (define-contract keyword the-subject contract)
          ...

followed by the body forms in which the occurrences of the identifiers keyword have been substituted with the automatically generated identifiers the-subject.

This allows functions defined in the body forms to call each other without enforcing the contracts, while functions referencing keyword, defined outside with-outer-contracts, will be subjected to the contracts.
