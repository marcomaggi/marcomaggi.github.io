72.2 Type definitions for code evaluation

— Class: <environment>

Hold the context required to evaluate expressions.

— Immutable Field of <environment>: imports

The list of import specifications, in the same format accepted by import in the library form. This field is tagged as <list>.

— Immutable Field of <environment>: bindings

An alist of symbols and constants, represents the bindings to create before the evaluation of expressions. This field is tagged as <list>.

— Syntax: make <environment> clause ...
— Auxiliary Syntax: bindings: binds
— Auxiliary Syntax: imports: specs

Build and return a new <environment> instance; clause must be an auxiliary syntax among bindings: and imports:.

— Auxiliary Syntax: bindings: binds

Auxiliary syntax for make. Select an initial set of bindings to define before evaluating an expression, in addition to the bindings from imported libraries. When not used defaults to null, which means no additional bindings.

binds must be an alist of symbols and constants: the symbols will become binding identifiers, the constants will become the bindng values. Notice that a closure is a constant.

— Auxiliary Syntax: imports: specs

Auxiliary syntax for make. Select the library to import to build the environment for expression evaluation. specs must be null or a list of library import specifications as accepted by import in the library form. When imports: is not used, the import specifications default to ((rnrs)).

— Method on <environment>: make* <environment> imports bindings

Build and return a new <environment> instance; imports must be as described for the imports: auxiliary syntax, bindings must be as described for the bindings: auxiliary syntax.

— Method on <environment>: eval expression

Evaluate expression in the augmented environment of the instance. Return the return values of expression. expression is evaluated as if it is the first form in the body of a let form, so it can contain definitions.

— Method on <environment>: eval-for-bindings expression result-identifiers

Evaluate expression in the augmented environment of the instance; this method is used to acquired the list of values bound to the identifiers selected by the list of symbols result-identifiers. The return value is an alist having symbols from result-identifiers as keys and the associated constants as values; the order of pairs in the alist is the same as the order of symbols in result-identifiers.

expression is evaluated as if it is the first form in the body of a let form, so it can contain definitions.

— Method on <environment>: augment new-bindings

Build and return a new <environment> instance having: the same import specifications of the caller instance; the same list of bindings with the given bindings prepended.

If the addition of new binding would cause a duplication an assertion violation is raised.

— Method on <environment>: eval-to-augment! expression new-identifiers

Evaluate expression in the augmented environment of the instance; this method is used to acquired the list of values bound to the identifiers selected by the list of symbols result-identifiers and add them to the bindings of the instance.

expression is evaluated as if it is the first form in the body of a let form, so it can contain definitions.

This method mutates the internal list of bindings; if augmenting the bindings would lead to duplication: an assertion violation is raised before evaluating the expression. All the <environment> records derived from this one with the augment method will see their list of bindings mutated.
