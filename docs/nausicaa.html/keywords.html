59 Keyword values

The (nausicaa keywords) library implements a type of value, keywords, disjoint from all the other types. Keywords can be treated much like symbols, but they must be compared with keyword=? not eq?.

— Syntax: keyword ?symbol
— Function: %keyword symbol

Both the macro and the function return a new keyword value associated to the Scheme symbol. The macro is a wrapper for the function, so the following two forms are equivalent (notice the quoting): 

          (keyword   alpha)
          (%keyword 'alpha)

When this function is applied to the same symbol multiple times, it returns the same keyword value; so those keywords are eq? to each other.

— Syntax: define-keywords ?symbol ...

Build new keyword values using keyword upon ?symbol arguments, then bind them to ?symbol. Keyword values defined by define-keywords can be, and are meant to be, exported by a library if there is the need.

— Function: keyword? obj

Return true if obj is a keyword value.

— Function: keyword=? obj1 obj2

Return true if the arguments are equal keywords; two keywords are equal if keyword->symbol applied to them returns the same symbol according to eq?.

— Function: keyword->symbol key
— Function: symbol->keyword str

Convert a keyword to and from a symbol.

— Function: keyword->string key
— Function: string->keyword str

Convert a keyword to and from a string.

— Syntax: with-keywords keywords form0 form ...

Define a set of keywords while the forms are evaluated. The following: 

          (with-keywords (:a :b :c :d)
            ---)

expands to: 

          (let ((:a (keyword :a))
                (:b (keyword :b))
                (:c (keyword :c))
                (:d (keyword :d)))
            ---)
— Syntax: let-keywords ?options ?allow-unknown ((?name ?key ?default) ...) ?form0 ?form ...
— Syntax: let-keywords* ?options ?allow-unknown ((?name ?key ?default) ...) ?form0 ?form ...

These are let and let* like macros that allow parsing a list of keyword/value couples in ?options. All the ?key arguments must be keywords identifier; they are created by this form to be available in the context of the ?form.

  1. First all the ?default expressions are evaluated and their results bound to the ?name identifiers, like with let or let*.
  2. Then ?options is parsed and the values associated to the ?key keywords are assigned to the ?name variables.
  3. Finally the ?form arguments are evaluated like in a begin form. The ?form arguments are the region of the ?name bindings.

If ?allow-unknown is true: ?options can contain keywords not included in the ?key list; else an error is raised. Each keyword in ?options must be followed by a value, else an error is raised.

Example: 

          (define (doit . options)
            (let-keywords options #f
                 ((a :a #\a)
                  (b :b #\b)
                  (c :c #\c)
                  (d :d #\d))
              (list a b c d)))
          
          (with-keywords (:a :b :c :d)
            (doit :a 1 :b 2 :d 4))
          ⇒ (1 2 #\c 4)
