69.2.1 Tokenising an IPv6 address string

The following bindings are exported by the (nausicaa net helpers ipv6address-lexer) library.

— Constant: ipv6-address-lexer-table

A lexer table to be used by the library (nausicaa silex lexer) to tokenise textual IPv6 address or IPv6 address prefix representations.

To tokenise an IPv6 address string we can do: 

     (import (nausicaa)
       (nausicaa net helpers ipv6address-lexer)
       (prefix (nausicaa silex lexer) lex.)
       (nausicaa parser-tools))
     
     (define (make-address-lexer address-string)
       (let ((IS (lex.make-IS (string: address-string))))
         (lex.make-lexer ipv6-address-lexer-table IS)))

every time the lexer thunk is evaluated, it returns an object of type <lexical-token>.

The token categories are: COLON, DOT, SLASH, NUMBER. The tokens of category COLON, DOT and SLASH hold, as semantic value, the characters #\:, #\. and #\/ respectively; the tokens of category NUMBER hold as semantic value the string representing a value in the input IPv6 address. The interpretation of numbers, especially their base, cannot be determined by the lexer.

The longest IPv6 address string has: 8 numbers as address components, 7 separators between address components, a final slash and number to specify the address prefix length; the total is 8 + 7 + 2 = 17 tokens. We may want to use make-max-count-lexer from (nausicaa silex utilities) to limit the number of returned tokens.
