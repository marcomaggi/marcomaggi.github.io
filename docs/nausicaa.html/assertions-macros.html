17.1 Assertion macros

The following bindings are exported by the (assertions) library.

— Macro: assert expression

This is a replacement for assert from (rnrs base (6)); it behaves in the same way, but expands to expression itself if the parameter enable-assertions? from the library (configuration) is set to false.

— Syntax: arguments clause ...
— Auxiliary Syntax: who who
— Auxiliary Syntax: number-of-arguments who
— Auxiliary Syntax: argument arg-clauses
— Auxiliary Syntax: formal arg-name
— Auxiliary Syntax: ordinal arg-index
— Auxiliary Syntax: predicate arg-predicate
— Auxiliary Syntax: description arg-description

Expand into a set of assertions which are meant to test the arguments of a function; despite being a set of assertions, the expansion of this macro is a definition, so it can be used at the beginning of a body without switching its context from definitions to expressions.

The who and number-of-arguments clauses select the argument for the &who condition object and the number of mandatory arguments expected by the function (zero or more).

The argument clause defines the expected argument through the following subclauses:

formal
Specify the formal name of the argument.
ordinal
Specify the 1–based index of the argument in the list of mandatory function arguments.
predicate
Select the predicate to be applied to validate the value.
description
A description string used in the error message. 
          (import (rnrs)
            (prefix (assertions) ass.))
          
          (define (doit a b c)
            (ass.arguments (ass.who 'doit)
                           (ass.number-of-arguments 3)
               (ass.argument (ass.formal a)
                             (ass.ordinal 1)
                             (ass.predicate integer?)
                             (ass.description "value of a"))
               (ass.argument (ass.formal b)
                             (ass.ordinal 2)
                             (ass.predicate string?)
                             (ass.description "value of b"))
               (ass.argument (ass.formal c)
                             (ass.ordinal 3)
                             (ass.predicate symbol?)
                             (ass.description "value of c")))
            (list a b c))
          
          (doit 1 "two" 'three)   ⇒ (1 "two" three)
          
          (doit #\a "two" 'three))
          error--> expected value of a as argument number 1
          
          (doit 1 2 'three))
          error--> expected value of b as argument number 2
          
          (doit 1 "two" 3))
          error--> expected value of c as argument number 3
