25.1.1 Installed libraries

Here is a list of the most important libraries:

(nausicaa ffi clang-data-types)
(nausicaa ffi clang-data-types compat)
Export functions to convert Scheme symbols representing external C language data types to Scheme symbols representing C language data types recognised by the underlying Scheme implementation; it also provides enumerations to inspect the sets of supported types.

The generic library is processed by GNU Autoconf to gather values from the package configuration step; the compat library is specific to the underlying implementation.

(nausicaa ffi sizeof)
Exports macros used to inspect constant values like the size of basic C language data types. The library is processed by GNU Autoconf to gather values from the package configuration step.
(nausicaa ffi peekers-and-pokers)
(nausicaa ffi peekers-and-pokers compat)
Export an API for raw memory access through peeker and poker functions and user–friendly macros; the compat library is specific to the underlying implementation.
(nausicaa ffi conditions)
Definition of condition object related to the foreign–functions interface.
(nausicaa ffi pointers)
Exports an interface to memory pointer values as disjoint Scheme object type.
(nausicaa ffi compat)
Exports an adapter interface to the foreign–functions interfaces of the underlying Scheme implementation. This library is not meant to be used directly by client code.
(nausicaa ffi)
Exports both a low level function–based API and a high level user–friendly macro–based API. It provides loading and inspection of shared libraries and constructors of callout and callback functions.
