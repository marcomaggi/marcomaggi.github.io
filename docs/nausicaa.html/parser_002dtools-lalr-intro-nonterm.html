12.3.1.7 Non–terminal categories

We have seen the graph representation of the grammar with tokens belonging to terminal categories. But how many nodes are there in such a graph? There are an infinite number of expressions, and consequently an infinite number of terminal sequences is required to represent them. This would make it impossible to code a parser, unless we take a different approach.

We start by noticing that, if the following sequences of terminals represent expressions: 

     N
     A N
     N A N
     N M N

then the following sequences of terminals represent expressions with subexpression: 

     N A  N
     N A  A N
     N A  N A N
     N A  N M N
     
     N M  N
     N M  A N
     N M  N A N
     N M  N M N

Let's define a new category, indicated by the Scheme symbol ‘E’, which represents the basic expressions: 

     E == N
       == A N
       == N A N
       == N M N

then we can represent the composite sequences written above with: 

     N A E
     N M E

and more generally: 

     E A E
     E M E

Let's suppose in defining the category ‘E’ we can use ‘E’ itself, recursively; then we can write: 

     E == N
       == A E
       == E A E
       == E M E
       == O E C

in which we have included a rule for the parentheses; with this definition involving a few symbols we can represent the whole set of possible expressions. The library (nausicaa parser-tools lalr) allows us to do exactly this, using the following S–expression: 

     (E (N)
        (A E)
        (E A E)
        (E M E)
        (O E C))

Categories defined this way, as sequences of the same and other categories, are called non–terminals for reasons we can imagine. The definition of non–terminals changes nothing in the way the parser itself works, but it is the fundamental tool to describe the grammar. We can use any number of non–terminals to describe a grammar.
