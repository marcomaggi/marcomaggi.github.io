22.1.1 Defining class types

— Macro: define-class name-spec class-clause ...

Define a new record type satisfying the constraints of a class definition; expand to a regular record type definition using (rnrs records procedural (6)) and some additional syntax definitions used by the (nausicaa language classes) library.

The arguments are compatible with the ones of define-record-type: every regular record type definition can be turned into a class definition simply by changing define-record-type into define-class. In addition to all the clauses of define-record-type, the class-clause arguments can be the auxiliary syntaxes described in this section.

If none of the class-clause arguments is a inherit or parent or parent-rtd syntax, the clause (inherit <top>) is added to the record definition. Basic conventional RTDs.

If none of the class-clause arguments is a nongenerative syntax, the clause (nongenerative) is added to the record definition; however, notice that it is unsafe to use automatically generated UIDs, it is better and useful to explicitly select a unique identifier.
