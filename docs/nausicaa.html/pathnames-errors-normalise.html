41.5.4 Pathname normalisation error condition type

The following bindings are exported by the (nausicaa pathname conditions) library; it is suggested to add a prefix when importing the library.

— Condition Type: &normalisation

Condition type representing a pathname normalisation error: it is used when an error occurs normalising a list of bytevectors representing pathname segments; it is derived from &pathname.

It has the following fields:

absolute?
A boolean value, #t if the list of bytevectors represents an absolute pathname.
original-segments
A list of bytevectors representing pathname segments which originated the error; it is the list before any normalisation.

— Function: make-normalisation-condition absolute? original-segments

Build and return a new &normalisation condition value.

— Function: normalisation-condition? obj

Return #t if obj is a condition value of type &normalisation.

— Function: condition-normalisation/absolute? obj
— Function: condition-normalisation/original-segments obj

Accessors for the fields of a condition value of type &normalisation.

— Syntax: raise-normalisation-error who message absolute? original-segments

Raise an exception with compound condition object of types &normalisation, &who, &message.
