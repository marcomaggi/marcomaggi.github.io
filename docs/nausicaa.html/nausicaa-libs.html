2.1 Component libraries

Unless otherwise specified, the library (nausicaa) reexports the bindings from the following libraries:

(vicare)
Almost all the bindings are reexported with the exception of the following ones: 
          define define-values
          let let* letrec letrec*
          let-values let*-values
          lambda case-lambda set!
          do begin
          receive receive-and-return

(nausicaa language oopp)
All the bindings are reexported, with the following renamings: 
          set!/tags               == set!
          define/tags             == define
          define-values/tags      == define-values
          lambda/tags             == lambda
          let/tags                == let
          let*/tags               == let*
          letrec/tags             == letrec
          letrec*/tags            == letrec*
          let-values/tags         == let-values
          let*-values/tags        == let*-values
          receive/tags            == receive
          receive-and-return/tags == receive-and-return
          begin/tags              == begin
          do/tags                 == do
          do*/tags                == do*
          case-lambda/tags        == case-lambda

(nausicaa language builtins)
All the bindings are reexported.
(nausicaa language multimethods)
All the bindings are reexported.
(nausicaa language conditions)
Provides basic condition types and utilities for condition types definition (see Condition object types).
((nausicaa language increments))
Provides syntaxes to increment and decrement numeric values (see Numeric increment syntaxes).
(vicare language-extension namespaces)
All the bindings are reexported.
(vicare language-extensions sentinels)
Sentinel values (The sentinel value).
(nausicaa language simple-match)
Simple pattern matching syntax (Simple destructuring match syntax).
(nausicaa language infix)
Syntax for infix–to–prefix notation conversion (Infix to prefix transformer.
