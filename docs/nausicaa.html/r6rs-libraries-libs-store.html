77.4.6.2 Collecting library forms

The bindings documented in this section are exported by the (nausicaa parser-tools r6rs libraries) library.

— Class: <library-store>

Record type describing collected <library> records.

— Immutable field of <library-store>: <hashtable> registry

Map full library name symbolic expressions to <library> records.

— Immutable field of <library-store>: <hashtable> ids->names

Map symbolic expressions representing library reference identifiers to a list of matching <library-name> records, which in turn reference <library> records in the registry.

— Common constructor on <library-store>: constructor
— Common constructor on <library-store>: constructor list-of-libraries

Build a new instance of <library-store>. If list-of-libraries is used: it must be a list of <library> records which is registered in the store; a typical candidate for this argument is the return value of rnrs-6-libraries.

— Method on <library-store>: <list> names (L <library-reference>)

Return a list of <library-name> records matching the given library reference; the list references libraries in the store. If no registered library names match: return null.

— Method on <library-store>: <library> library (N <library-name>)

Return a <library> record from the registry associated to the given library name; if no registered librarires match: return false.

— Method on <library-store>: register-library (L <library>)

Add L to the internal registry. Also add an association between the library name and the L record. If a library with the same name is already in the registry: it is silently overwritten.

— Method on <library-store>: register-ids->names (R <library-reference>) (names <list>)

Add the association between the identifiers of R and names, which must be a list of <library-name> records, to the already registered associations for that list of identifers. The operation fails with an assertion violation if one of the names is not already present in the internal registry.

— Method on <library-store>: <list> matching-libraries (R <library-reference>)

Return a possibly null list of <library> records from internal the registry, whose name matches the given library reference. The list is sorted from higher version number to lower version number.
