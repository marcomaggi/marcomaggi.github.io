72.1.9 Reading line comments

The following bindings are exported by the (nausicaa r6rs lexer) library.

— Constant: r6rs-line-comment-lexer-table

A lexer table to read R6RS compliant line comments. Both comments with end–of–line and comments spanning until the end–of–input are accepted. This table needs an input system configured with (counters: 'all). See the documentation of read-line-comment for a usage example. A lexer thunk using this table returns:

  • The comment as a string.
  • The return value of the function referenced by the parameter eoi-token-maker when the end of input is found.
  • The return value of the function referenced by the parameter lexical-error-token-maker when a lexical error is detected in the input system.

— Function: read-line-comment IS

Given an input system, read characters composing an R6RS line comment; return the string representing the comment. This function expects the functions with default behaviour to be in the parameters of (nausicaa r6rs lexer).

If an error occurs reading the line comment: a condition object is raised with components &lexical, &message, &who, &irritants; the single value in the &irritants list is the string that caused the error.

If end of input is found reading the line comment: a condition object is raised with components &lexical, &message, &who, &irritants. The single value in the &irritants list is the end–of–file object.

The implementation of this function is a useful example of usage for r6rs-line-comment-lexer-table: 

          (define (read-line-comment IS)
            (let (((T <lexical-token>)
                  ((lex.make-lexer r6rs-line-comment-lexer-table IS))))
              (define (%error message)
                (raise
                 (condition (make-lexical-violation)
                            (make-message-condition message)
                            (make-who-condition 'read-line-comment)
                            (make-irritants-condition (list T.value)))))
              (cond (T.end-of-input?
                     (%error
                      "end of input found while reading line comment"))
                    (T.lexer-error?
                     (%error
                     "lexical violation while reading line comment"))
                    (else T))))
