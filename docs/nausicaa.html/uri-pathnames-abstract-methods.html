11.3.2.4 Extending class methods

The following bindings are exported by the library (nausicaa uri pathnames abstract).

— Multimethod: <bytevector> pathname-bytevector ptn

Return a bytevector representation of the pathname; the returned value must not be mutated.

— Multimethod: <string> pathname-string ptn

Return a string representation of the pathname; the returned value must not be mutated.

— Multimethod: <bytevector> pathname-uri-representation ptn

Return a bytevector representation of the pathname in the format specified by RFC 3986; the returned value must not be mutated.

— Multimethod: <pathname> pathname-extension ptn

Return a bytevector representing the extension of the last segment of the instance pathname, dot separator not included.

— Multimethod: <pathname> pathname-dirname ptn

Return a pathname object representing the directory part of the instance pathname.

— Multimethod: <pathname> pathname-tailname ptn

Return a pathname object representing the last segment of the instance pathname.

— Multimethod: <pathname> pathname-rootname ptn

Return a pathname object representing the instance pathname with the extension of the last segment stripped, dot separator not included.

— Multimethod: pathname-split ptn

Return 2 values: a boolean, #t if the instance pathname is absolute; a list of bytevectors each representing a segment of the instance pathname.

— Multimethod: <boolean> pathname-prefix? ptn1 ptn2

Return #t if the pathname ptn1 is a prefix of the pathname ptn2, otherwise return #f.

— Multimethod: <boolean> pathname-suffix? ptn1 ptn2

Return #t if the pathname ptn1 is a suffix of the pathname ptn2, otherwise return #f.

— Multimethod: <pathname> pathname-prepend ptn1 ptn2

Return a new pathname object representing the pathname ptn1 prepended to the pathname ptn2.

— Multimethod: <pathname> pathname-append ptn1 ptn2

Return a new pathname object representing the pathname ptn1 appended to the pathname ptn2.

— Multimethod: <pathname> pathname replace-extension ptn ext

Return a new pathname object representing the pathname ptn with the extension of the last segment replaced by ext.

— Multimethod: <boolean> pathname=? ptn1 ptn2
— Multimethod: <boolean> pathname=? (ptn1 <absolute-pathname>) (ptn2 <relative-pathname>)
— Multimethod: <boolean> pathname=? (ptn1 <relative-pathname>) (ptn2 <absolute-pathname>)

Return #t if the pathname arguments are equal, otherwise return #f. The two predefined implementations always return #f because, obviously, absolute pathnames cannot be equal to relative pathnames.
