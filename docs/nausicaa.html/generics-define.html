24.4 Defining generic functions and methods

— Syntax: define-generic <name>

Define a new generic function (a macro) and bind it to <name>. The number of arguments is unspecified: it will be established by the first define-method or add-method applied to <name>.

— Syntax: define-generic/merge <name> <generic0> <generic> ...

Define a new generic function and bind it to <name>. The internal collection of methods holds the union of the method collections from the <generic> arguments, which must be generic function identifiers.

The union is performed visiting arguments from left to right; when two methods have the same signature, the one from the leftmost generic function has precedence.

Merging methods is useful when two different libraries export generic functions bound to the same identifier.

— Syntax: define-method <generic> (<arg-spec> ...) <body>
— Syntax: define-method (<generic> <arg-spec> ...) <body>

Add a new method to an already existent generic function, <generic>.

The list of <arg-spec> specifies the class of the arguments for which this method specialises the generic function; an <arg-spec> can be:

(<arg> <class name>)
Where <arg> is the formal name of the argument and <class name> the (possibly conventional) class name of expected values.
<arg>
Where <arg> is the formal name of the argument. In this case the class defaults to the conventional <top>, which is interpreted as parent of every other class (and so it has the least specificity).

If a method is defined with the same signature of an already registered method: the old method is overwritten by the new one.

— Syntax: add-method <generic> <classes> <closure>

Add a new method to an already existent generic function, <generic>. <classes> must be a list of class names for which this method specialises the generic function. <closure> must be the method's closure.
