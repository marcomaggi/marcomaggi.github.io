25.4 Defining generic functions and methods

The following syntaxes allow the definition and merging of generic functions; merging generics is useful when two different libraries export generic functions bound to the same identifier.

— Syntax: define-generic ?name ?formals ?clause ...
— Auxiliary Syntax: uid-list-of ?func
— Auxiliary Syntax: merge ?generic ...

Define a new ordinary generic function (a macro) and bind it to ?name. The number of arguments is specified by ?formals: a list of identifiers, one for each argument; the list is used only to determine the number of arguments (it can be used to document the intended meaning of arguments).

When the merge clause is used: the internal method table is initialised to the union of the method tables from the ?generic arguments, which must be ordinary generic function identifiers. The union is performed by visiting arguments from left to right; when two methods have the same signature, the one from the leftmost generic function takes precedence.

See below for the documentation of the uid-list-of clause.

— Syntax: define-generic* ?name ?formals ?clause ...
— Auxiliary Syntax: uid-list-of ?func
— Auxiliary Syntax: merge ?generic ...
— Auxiliary Syntax: reverse-before-methods ?boolean

Define a new starred generic function (a macro) and bind it to ?name. The number of arguments is specified by ?formals: a list of identifiers, one for each argument; the list is used only to determine the number of arguments (it can be used to document the intended meaning of arguments).

When the merge clause is used: the internal method tables are initialised to the unions of the method tables from the ?generic arguments, which must be starred generic function identifiers. The union is performed by visiting arguments from left to right; when two methods have the same signature, the one from the leftmost generic function takes precedence.

The default methods composition protocol states that :before and :around methods are evaluated from the most specific to the least specific, while :after methods are evaluated from the least specific to the most specific; the protocol can be customised in the following ways:

  • When the reverse-before-methods clause is used and ?boolean is #t: the order of :before methods is reversed to “from the least specific to the most specific” prior to evaluating them.

generics methods for an example of why these customisations are useful.

See below for the documentation of the uid-list-of clause.

— Auxiliary Syntax: uid-list-of ?func

The optional clause uid-list-of allows the specification of a custom function to extract a unique symbol representing the type of the arguments in calls to the generic functions; the default value is class-uid-list-of exported by nausicaa language classes. ?func can be any expression evaluating to a function.

When merging generic functions: the type inspector function is class-uid-list-of if we do not use this clause.

Usually the uid-list-of clause selects a wrapper for class-uid-list-of; for example: 

          (define (new-uid-list-of obj)
            (if (pointer? obj)
                '(nausicaa:ffi:pointer <top>)
             (class-uid-list-of obj)))
— Syntax: define-method (?generic ?arg-spec ...) ?body
— Syntax: define-method ?generic (?arg-spec ...) ?body
— Syntax: define-method ?kind ?generic (?arg-spec ...) ?body
— Syntax: define-method ?generic ?kind (?arg-spec ...) ?body
— Auxiliary Syntax: :primary
— Auxiliary Syntax: :around
— Auxiliary Syntax: :before
— Auxiliary Syntax: :after

Add a new method to an already existent generic function ?generic either ordinary or starred.

When the ?kind argument is not present or it is the :primary syntax: ?generic can be either an ordinary or starred generic function.

When the ?kind argument is the :before, :after or :around syntax: ?generic must be a starred generic function.

The list of ?arg-spec specifies the class of the arguments for which this method specialises the generic function; an ?arg-spec can be:

(?arg ?class-name)
Where ?arg is the formal name of the argument and ?class-name the (possibly conventional) class name of expected values.
?arg
Where ?arg is the formal name of the argument. In this case the class defaults to the conventional <top>, which is interpreted as parent of every other class (and so it has the least specificity).

If a method is defined with the same signature of an already registered method: the old method is overwritten by the new one.

— Syntax: add-method ?generic ?types ?closure
— Syntax: add-method ?generic ?kind ?types ?closure
— Auxiliary Syntax: :primary
— Auxiliary Syntax: :around
— Auxiliary Syntax: :before
— Auxiliary Syntax: :after

Add a new method to an already existent generic function ?generic either ordinary or starred.

When the ?kind argument is not present or it is the :primary syntax: ?generic can be either an ordinary or starred generic function.

When the ?kind argument is the :before, :after or :around syntax: ?generic must be a starred generic function.

?types must be a list of class names for which this method specialises the generic function. ?closure must be the method's closure.
