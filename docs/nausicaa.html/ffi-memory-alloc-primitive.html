31.2.2.3 Primitive functions

The following functions perform allocation through customisable functions, which allow allocation logging and validation of error reactions.

— Function: primitive-malloc number-of-bytes
— Parameter: primitive-malloc-function

primitive-malloc allocates a block of memory and returns a pointer to it, or #f if an error occurs; the allocation is performed using the current value of primitive-malloc-function, which must be a function.

The parameter is preset to platform-malloc*; if a custom function is assigned to the parameter, it must match its behaviour.

— Function: primitive-realloc pointer new-number-of-bytes
— Parameter: primitive-realloc-function

primitive-realloc reallocates a block of memory with a new size and returns a pointer to the new block, or #f if an error occurs; the allocation is performed using the current value of primitive-realloc-function, which must be a function.

The parameter is preset to platform-realloc*; if a custom function is assigned to the parameter, it must match its behaviour.

— Function: primitive-calloc count element-size
— Parameter: primitive-calloc-function

primitive-calloc allocates a block of memory (big enough to hold count elements of element-size each) and returns a pointer to it, or #f if an error occurs; the allocation is performed using the current value of primitive-calloc-function, which must be a function.

The parameter is preset to platform-calloc*; if a custom function is assigned to the parameter, it must match its behaviour.

— Function: primitive-free pointer
— Parameter: primitive-free-function

primitive-free releases a block of memory; the release is performed using the current value of primitive-free-function, which must be a function.

The parameter is preset to platform-free.
