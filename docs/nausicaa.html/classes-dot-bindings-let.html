24.7.2.1 Tagged bindings with let semantics
— Syntax: let/with-class ((var init) ...) . body
— Syntax: let/with-class name ((var init) ...) . body

Like let (both named and unnamed) but automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (let/with-class (((a <red>)  1)
                           ((b <blue>) 2))
            (list a b))

is expanded to: 

          (let ((a 1)
                (b 2))
            (with-class ((a <red>)
                         (b <blue>))
              (list a b)))

Each var element can be one among: 

          id
          (id tag ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class or label names.

If all the var arguments are in the first form (untagged identifier): let/with-class defaults to let as defined by R6RS. If at least one var has tags: the untagged ones default to bindings with <top> as conventional class name. 

          #!r6rs
          (import (nausicaa))
          
          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (immutable numerator)
                            (immutable denominator)))
          
          (define (<fraction>-numerator o)
            (numerator (<fraction>-number o)))
          
          (define (<fraction>-denominator o)
            (denominator (<fraction>-number o)))
          
          (let/with-class (((a <fraction>) (make <fraction> 2/3))
                           ((b <fraction>) (make <fraction> 4/5)))
            (list a.numerator b.denominator))
          ⇒ (2 5)
— Syntax: let*/with-class ((var init) ...) . body

Like let* but, binding by binding, automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (let*/with-class (((a <red>)  1)
                            ((b <blue>) 2))
            (list a b))

is expanded to an equivalent of: 

          (let/with-class (((a <red>) 1))
            (let/with-class (((b <blue>) 2))
              (list a b)))

and in the end to: 

          (let ((a 1))
            (with-class ((a <red>))
              (let ((b 2))
                (with-class ((b <blue>))
                  (list a b)))))

Each var element can be one among: 

          id
          (id tag ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class names.

If all the var arguments are in the first form (untagged identifier): let*/with-class defaults to let* as defined by R6RS. If at least one var has tags: the untagged ones default to bindings with <top> as conventional class name.

Given two bindings var1, init1 and var2, init2 defined in sequence: init2 is evaluated in the region of the identifier syntaxes used to access the fields of var1. 

          #!r6rs
          (import (nausicaa))
          
          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (immutable numerator)
                            (immutable denominator)))
          
          (define (<fraction>-numerator o)
            (numerator (<fraction>-number o)))
          
          (define (<fraction>-denominator o)
            (denominator (<fraction>-number o)))
          
          (let*/with-class (((a <fraction>) (make <fraction> 2/3))
                            ((b <fraction>) (make <fraction>
                                              (/ a.numerator 5))))
            b.number)
          ⇒ 2/5
— Syntax: letrec/with-class ((var init) ...) . body

Like letrec but automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (letrec/with-class (((a <red>)  1)
                              ((b <blue>) 2))
            (list a b))

is expanded to an equivalent of: 

          (let ((a #<undefined>)
                (b #<undefined>))
            (with-class ((a <red>))
                         (b <blue>))
              ;; do not enforce the order of evaluation
              ;; of the init expressions
              (let ((T1 1)
                    (T2 2))
                (set! a T1)
                (set! b T2)
                (list a b)))

Each var element can be one among: 

          id
          (id tag ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class or label names.

If all the var arguments are in the first form (untagged identifier): letrec/with-class defaults to letrec as defined by R6RS. If at least one var has tags: the untagged ones default to bindings with <top> as conventional class name.

The init expressions can contain references to both the ID identifiers and the identifier syntaxes defined by with-class, as long as they are not resolved. 

          (define-class <alpha>
            (fields (immutable value)
                    (immutable proc)))
          
          (define-class <beta>
            (fields (immutable value)
                    (immutable proc)))
          
          (letrec/with-class
               (((a <alpha>) (make <alpha>
                               1 (lambda ()
                                   (cons a.value b.value))))
                ((b <beta>)  (make <beta>
                               2 (lambda ()
                                   (cons a.value b.value)))))
            (list (a.proc) (b.proc)))
          ⇒ ((1 . 2) (1 . 2))
— Syntax: letrec*/with-class ((var init) ...) . body

Like letrec* but automatically enclose the body in a with-class syntax using the specified tags. For example, the form: 

          (letrec*/with-class (((a <red>)  1)
                               ((b <blue>) 2))
            (list a b))

is expanded to an equivalent of: 

          (let ((a #<undefined>)
                (b #<undefined>))
            (with-class ((a <red>))
                         (b <blue>))
              ;; enforce the order of evaluation
              ;; of the init expressions
              (set! a 1)
              (set! b 2)
              (list a b))

Each var element can be one among: 

          id
          (id tag ...)

in which: id must be an identifier representing the variable name, tag must be identifiers representing class or label names.

If all the var arguments are in the first form (untagged identifier): letrec*/with-class defaults to letrec* as defined by R6RS. If at least one var has tags: the untagged ones default to bindings with <top> as conventional class name.

The init expressions can contain references to the identifier syntaxes, as long as they are not resolved.

The only difference between letrec/with-class and letrec*/with-class is that the latter enforces the order of evaluation of the init expressions. 

          (define-class <alpha>
            (fields (immutable value)
                    (immutable proc)))
          
          (define-class <beta>
            (fields (immutable value)
                    (immutable proc)))
          
          (letrec*/with-class
              (((a <alpha>) (make <alpha>
                              1 (lambda ()
                                  (cons a.value b.value))))
               ((b <beta>)  (make <beta>
                              2 (lambda ()
                                  (cons a.value b.value))))
               ((c <top>)   (list (a.proc) (b.proc))))
            c)
          ⇒ ((1 . 2) (1 . 2))
