6.3.5 Unimplemented error condition

The following bindings are exported by the (nausicaa language conditions) library.

— Condition Type Label: &unimplemented

The type of condition used to signal unimplemented features. It is derived from &error.

— Function: make-unimplemented-condition

Return a new instance of &unimplemented.

— Function: unimplemented-condition? obj

Return #t if obj is of type &unimplemented.

— Function: raise-unimplemented-error who
— Function: raise-unimplemented-error who message
— Function: raise-unimplemented-error who message irritants

Raise a composite condition object with component types: &unimplemented, &who, &message, &irritants.
