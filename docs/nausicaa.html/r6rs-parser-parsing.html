77.2.1 Parsing source code

The following bindings are exported by the (nausicaa parser-tools r6rs parser) library.

— Function: make-r6rs-parser

Build and return a parser function exposing the API of LALR parsers (see lalr parser).

The returned parser function accepts 3 arguments: a lexer thunk returning <lexical-token> instances; an error handler function; a custom value which is made available to the parser semantic actions bound to yycustom. Its return value is the list of datums parsed from the lexer.

The semantic actions of the parser table make use of the functions referenced by the parameters exported by the (nausicaa parser-tools r6rs parser) library (see r6rs parser params). The default parametrised functions build an S–expression almost as the read function from (rnrs io simple (6)) does.

The value returned by the parser function with the default parametrised functions is a list of datums read from the input system; notice the following special cases:

  • If the input is empty (the first read operation from the input system returns end–of-input): the returned value is null.
  • If the input is the unquoted empty list ‘()’: the returned value is the symbolic expression ‘(())’.
  • If the input is the quoted empty list ‘'()’ or the equivalent ‘(quote ())’: the returned value is the symbolic expression ‘((quote ()))’.

Notice that the returned lexer function is not equivalent to read or get-datum as defined by R6RS: the returned lexer consumes characters from its input system until the end–of–input is found, read and get-datum return the first datum and leave the rest untouched.

The following example shows: the libraries to import; a simple error handler function, making no use of the source location input field; the definition of a function that directly parses a string: 

     #!r6rs
     (import (nausicaa)
       (nausicaa parser-tools lexical-tokens)
       (prefix (nausicaa parser-tools silex lexer) lex.)
       (prefix (nausicaa parser-tools r6rs lexer)  r6.)
       (prefix (nausicaa parser-tools r6rs parser) r6.))
     
     (define (error-handler message (T <lexical-token>))
       (raise
        (condition (make-lexical-violation)
                   (make-message-condition
                    (string-append message
                     " line "   (number->string T.location.line)
                     " column " (number->string T.location.column)))
                   (make-irritants-condition `(,T.value)))))
     
     (define (parse string)
       (let* ((IS      (lex.make-IS
                          (lex.string: string)
                          (lex.counters: 'all)))
              (lexer   (r6.make-token-lexer IS))
              (parser  (r6.make-r6rs-parser)))
         (parser lexer error-handler #f)))
     
     ;; empty input
     (parse "")              ⇒ ()
     
     ;; null input
     (parse "()")            ⇒ (())
     
     ;; quoted null
     (parse "'()")           ⇒ ((quote ()))
     (parse "(quote ())")    ⇒ ((quote ()))
     
     ;; single datums
     (parse "ciao")          ⇒ (ciao)
     (parse "\"ciao\"")      ⇒ ("ciao")
     (parse "123")           ⇒ (123)
     (parse "#t")            ⇒ (#t)
     (parse "#f")            ⇒ (#f)
     (parse "#\\newline")    ⇒ (#\newline)
     (parse "#\\A")          ⇒ (#\A)
     
     ;; multiple datums
     (parse "ciao 123 #\\A \"bau\"")         ⇒ (ciao 123 #\A "bau")
     
     ;; datums in compound datums
     (parse "(ciao \"bau\" #\\A)")     ⇒ ((ciao "bau" #\A))
     (parse "[ciao \"bau\" #\\A]")     ⇒ ((ciao "bau" #\A))
     (parse "#(ciao \"bau\" #\\A)")    ⇒ (#(ciao "bau" #\A))
     (parse "#vu8(1 2 3)")             ⇒ (#vu8(1 2 3))
     
     (parse "(1 2 3) (4 5 6) (7 8 9)")
     ⇒ ((1 2 3) (4 5 6) (7 8 9))
     
     (parse "(1 (4 5 6) 2 (7 8 9) 3)")
     ⇒ ((1 (4 5 6) 2 (7 8 9) 3))
