31.1.7.3 Calling Scheme functions

The following bindings are exported by the (nausicaa ffi) library. As example, the following form defines a callback accepting three ‘int’ as arguments and returning an ‘int’: 

     (define (fn a b c)
       (* a b c))
     
     (define cb
       (make-c-callback 'int fn '(int int int)))
— Function: make-c-callback ret-type scheme-proc arg-types

Build and return a callback object to be used as argument to a callout procedure. The callback invokes the Scheme function scheme-proc.

ret-type must be a symbol in the enum-clang-external-types enumeration; it represents the type of the callback return value. arg-types must be a list of symbols in the enum-clang-external-types enumeration; they represent the types of the callback arguments.

When a foreign function invokes the C language callback, the callback:

  1. Accepts as many C language arguments as there are symbols in arg-types; to specify an empty list of arguments, we can use (void).
  2. Does marshaling from C language arguments to Scheme objects as specified by the arg-type symbols.
  3. Applies the Scheme function to the Scheme values.
  4. Returns the return value of the Scheme function, marshaled to a C language value as specified by the ret-type symbol.

Every time a callback is built, a small amount of memory is allocated; such memory cannot be subject to garbage collection, so it is leaked unless we apply free-c-callback to the callback object.

— Function: free-c-callback callback

Release the memory associated to callback. On some Scheme implementations this call does nothing, because they do not allow releasing such a resource.
