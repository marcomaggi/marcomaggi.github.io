26.6 Defining and composing mixins

— Syntax: define-mixin name clause0 clause ...

Associate a set of mixin clauses to the identifier name, which can be later referenced by a mixins clause in the body of a class or label definition.

Right or wrong define-mixin imposes no constraints on the given clauses; it is our responsibility to select the ones which are convenient for the application model.

The mixins clause can be used in the body of a mixin definition; the result is that the clauses of the selected mixins are added to the definition of the current mixin.

The following is an example of mixin usage in which the maker of the receiving class accepts a value for a field in the mixin: 

     #!r6rs
     (import (nausicaa))
     
     (define-mixin <stuff>
       (fields c)
       (method (doit (o <stuff>))
         (+ 1 o.c)))
     
     (define-class <base>
       (fields a))
     
     (define-class <one>
       (inherit <base>)
       (maker ()
              (a:  #f)
              (p:  #f)
              (c:  #f))
       (fields p)
       (mixins <stuff>))
     
     (define-class <two>
       (inherit <base>)
       (maker ()
              (a:  #f)
              (q:  #f)
              (c:  #f))
       (fields q)
       (mixins <stuff>))
     
     (let (((o <one>)
            (make <one>
              (a: 10)
              (p: 20)
              (c: 30))))
       (o.doit))
     ⇒ 31

the following is an example of multiple mixins attached to the same class definition: 

     #!r6rs
     (import (nausicaa))
     
     (define-mixin <stuff>
       (fields c)
       (method (doit (o <stuff>))
         (+ 1 o.c)))
     
     (define-mixin <other-stuff>
       (fields p))
     
     (define-class <base>
       (fields a))
     
     (define-class <one>
       (inherit <base>)
       (maker ()
              (a:  #f)
              (p:  #f)
              (c:  #f))
       (mixins <stuff> <other-stuff>))
     
     (let (((o <one>)
            (make <one>
              (a: 10)
              (p: 20)
              (c: 30))))
       (o.doit))
     ⇒ 31

the following is an example of mixin accessing the fields of the receiving class: 

     #!r6rs
     (import (nausicaa))
     
     (define-mixin <stuff>
       (fields c)
       (method (doit (o <stuff>))
         (+ o.a o.c)))
     
     (define-class <alpha>
       (fields a)
       (mixins <stuff>))
     
     (let-make (((o <alpha>) (make* <alpha> 1 2)))
       (o.doit))
     ⇒ 3
