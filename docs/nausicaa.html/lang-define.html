9.5 Macro definition helpers

— Syntax: define-inline (?name ?arg ...) ?form0 ?form ...

Define a macro with with a definition resembling a function. It is defined as: 

          (define-syntax define-inline
            (syntax-rules ()
              ((_ (?name ?arg ...) ?form0 ?form ...)
               (define-syntax ?name
                 (syntax-rules ()
                   ((_ ?arg ...)
                    (begin ?form0 ?form ...)))))))
— Syntax: define-identifier-accessor-mutator ?name ?thing ?accessor
— Syntax: define-identifier-accessor-mutator ?name ?thing ?accessor ?mutator

Helper for the definition of an identifier syntax. It is defined as: 

          (define-syntax define-identifier-accessor-mutator
            (syntax-rules ()
              ((_ ?name ?thing ?accessor ?mutator)
               (define-syntax ?name
                 (identifier-syntax-accessor-mutator ?thing ?accessor ?mutator)))
              ((_ ?name ?thing ?accessor)
               (define-syntax ?name
                 (identifier-syntax-accessor-mutator ?thing ?accessor)))))
— Syntax: identifier-syntax-accessor-mutator ?thing ?accessor
— Syntax: identifier-syntax-accessor-mutator ?thing ?accessor ?mutator

Helper for the definition of an identifier syntax. It is defined as: 

          (define-syntax identifier-syntax-accessor-mutator
            (syntax-rules ()
              ((_ ?thing ?accessor ?mutator)
               (identifier-syntax
                (_                (?accessor ?thing))
                ((set! _ expr)    (?mutator  ?thing expr))))
              ((_ ?thing ?accessor)
               (identifier-syntax (?accessor ?thing)))))
— Syntax: with-accessor-and-mutator (?spec0 ?spec ...) ?body0 ?body ...

Expand to a let-syntax form used to define identifier syntaxes; ?spec and subsequent forms are expected to be in one of the forms: 

          (?name ?thing ?accessor)
          (?name ?thing ?accessor ?mutator)

in which the symbol ?name is defined as identifier syntax that makes use of ?accessor and ?mutator to access and mutate ?thing. Examples: 

          (let ((a 1))
            (with-accessor-and-mutator ((alpha a (lambda (x) x) set!))
              alpha))
          ⇒ 1
          
          (let ((a 1))
            (with-accessor-and-mutator ((alpha a (lambda (x) x) set!))
              (set! alpha 2)
              alpha))
          ⇒ 2
          
          (let ((a 1))
            (with-accessor-and-mutator ((alpha a (lambda (x) x)))
               alpha))
          ⇒ 1
          
          (let ((a 1)
                (b 2))
            (with-accessor-and-mutator ((alpha a (lambda (x) x) set!)
                                        (beta  b (lambda (x) x) set!))
              (list alpha beta)))
          ⇒ (1 2)
          
          (let ((a 1)
                (b 2))
            (with-accessor-and-mutator ((alpha a (lambda (x) x) set!)
                                        (beta  b (lambda (x) x) set!))
              (set! alpha 3)
              (set! beta  4)
              (list alpha beta)))
          ⇒ (3 4)
          
          (let ((a 1)
                (b 2))
             (with-accessor-and-mutator ((alpha a (lambda (x) x))
                                         (beta  b (lambda (x) x)))
               (list alpha beta)))
          ⇒ (1 2)
— Syntax: define-constant name expression

Bind name to the result of expression as define would do, but additionally make name impossible to be the first argument to set! from (rnrs base (6)).

— Syntax: define-syntax* (name stx) body0 body ...

Like define-syntax but allows a more function-like definition and introduces some bindings in the body. The following macro use: 

          (define-syntax* (name stx)
            body0 body ...)

expands to: 

          (define-syntax name
            (lambda (stx)
              bindings ...
              body0 body ...))

currently bindings is: 

          (define synner
            (case-lambda
             ((message)
              (synner message #f))
             ((message subform)
              (syntax-violation name message
                (syntax->datum stx) (syntax->datum subform)))))

that is: a function called synner which can be used to raise a syntax violation.

— Macro: define-auxiliary-syntax symbol ...
— Macro: define-auxiliary-syntaxes symbol ...

Define one or more auxiliary syntaxs whose names are the symbol arguments. The defined syntaxes are meant to be used as keywords in definitions, like fields and protocol for define-record-type.
