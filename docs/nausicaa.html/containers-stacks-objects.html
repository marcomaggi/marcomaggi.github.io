10.5.1 Stack objects

The following bindings are exported by the library (vicare containers stacks).

— Label Type: <stack>

Label type to be used to handle stack objects as defined by (vicare containers stacks). In this documentation <stack> object arguments to functions are indicated as stack.

— Default constructor on <stack>: constructor obj ...

Build and return a <stack> object holding the given objects, which are pushed on the stack from right to left.

— Syntax: <stack> (obj ...)

Build and return a <stack> object using the default constructor.

Object properties
— Method on <stack>: putprop key value
— Method on <stack>: $putprop key value

Add a new property key to the property list of the <stack>; key must be a symbol. If key is already set: the old entry is mutated to reference the new value.

— Method on <stack>: getprop stack key
— Method on <stack>: $getprop stack key

Return the value of the property key in the property list of the <stack>; if key is not set: return #f. key must be a symbol.

— Method on <stack>: remprop stack key
— Method on <stack>: $remprop stack key

Remove the property key from the property list of the <stack>; if key is not set: nothing happens. key must be a symbol.

— Method on <stack>: property-list stack
— Method on <stack>: $property-list stack

Return a new association list representing the property list of the <stack>. The order of the entries is the same as the property creation order.

Other operations
— Method on <stack>: hash stack
— Method on <stack>: $hash stack

Return an exact integer to be used as hashtable key for the <stack>. Hashtables having a <stack> as key can be instantiated as follows: 

          (make-hashtable (lambda ((S <stack>))
                            (S hash))
                          eq?)
Arguments validation

The following bindings are meant to be used with the facilities of the library (vicare arguments validation).

— Validation Clause: <stack> obj

Succeed if obj is a <stack> object.
