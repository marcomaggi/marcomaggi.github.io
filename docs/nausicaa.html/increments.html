7 Numeric increment syntaxes

The library (nausicaa language increments) exports syntaxes to ease incrementing and decrementing of numeric values. The bindings whose name is prefixed with $ are unsafe operations: they do not validate their arguments before accessing them.

— Syntax: incr! ?expr
— Syntax: incr! ?expr ?step
— Syntax: $incr! ?expr
— Syntax: $incr! ?expr ?step

Increment ?expr by ?step and return the result. ?expr must be an identifier or OOPP syntax usable to access and mutate a number object, otherwise the behaviour is undefined. ?step must be an expression evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

The syntax expands as follows: 

          (incr! ?expr ?step)
          ==> (begin
                (set! ?expr (+ ?expr ?step))
                ?expr)
— Syntax: pre-incr! ?expr
— Syntax: pre-incr! ?expr ?step
— Syntax: $pre-incr! ?expr
— Syntax: $pre-incr! ?expr ?step

Aliases for incr! and $incr!.

— Syntax: post-incr! ?expr
— Syntax: post-incr! ?expr ?step
— Syntax: $post-incr! ?expr
— Syntax: $post-incr! ?expr ?step

Increment ?expr by ?step and return the value of ?expr before the operation. ?expr must be an identifier or OOPP syntax usable to access and mutate a number object, otherwise the behaviour is undefined. ?step must be an expression evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

The syntax expands as follows: 

          (post-incr! ?expr ?step)
          ==> (receive-and-return (v)
                  ?expr
                (set! ?expr (+ ?expr ?step)))
— Syntax: decr! ?expr
— Syntax: decr! ?expr ?step
— Syntax: $decr! ?expr
— Syntax: $decr! ?expr ?step

Decrement ?expr by ?step and return the result. ?expr must be an identifier or OOPP syntax usable to access and mutate a number object, otherwise the behaviour is undefined. ?step must be an expression evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

The syntax expands as follows: 

          (decr! ?expr ?step)
          ==> (begin
                (set! ?expr (- ?expr ?step))
                ?expr)
— Syntax: pre-decr! ?expr
— Syntax: pre-decr! ?expr ?step
— Syntax: $pre-decr! ?expr
— Syntax: $pre-decr! ?expr ?step

Aliases for decr! and $decr!.

— Syntax: post-decr! ?expr
— Syntax: post-decr! ?expr ?step
— Syntax: $post-decr! ?expr
— Syntax: $post-decr! ?expr ?step

Decrement ?expr by ?step and return the value of ?expr before the operation. ?expr must be an identifier or OOPP syntax usable to access and mutate a number object, otherwise the behaviour is undefined. ?step must be an expression evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

The syntax expands as follows: 

          (post-decr! ?expr ?step)
