41.1 Overview of the pathname model

In the context of the (nausicaa pathname ---) libraries, a pathname is a bytevector representing a file or directory on a file system. A bytevector must adhere to the following conventions, else it is not a pathname:

  • Pathnames can be absolute or relative to other pathnames.
  • A pathname can be split, using any separator, in a list of segment bytevectors representing its components.
  • The list of segments alone does not specify if the pathname is absolute or relative.
  • Some additional information can be needed along with the segments and the absolute/realtive flag to fully specify a pathname (for example: drives on Windows(tm) platforms).
  • A segment holding a single byte representing a dot in ASCII coding represents the current directory.
  • A segment holding two bytes representing a dot in ASCII coding represents the parent directory.
