10.8.4 Iterating over strings

The following bindings are exported by the library (nausicaa containers iterators).

— Class: <string-iterator>

An iterator for Scheme strings; it is derived from <iterator>.

— Common constructor on <string-iterator>: constructor the-string start past stride

Build and return a new instance of <string-iterator> using the-string as string upon which to iterate, start as index of the first character, past as index of the character past the last one to return and stride as stride.

The stride is the number of characters in the string to jump forward or backward to when iterating; it must be +1 or -1 when we want to visit all the characters.

start and past must be fixnums greater than, or equal to, -1; stride must be a non–zero fixnum. If stride is positive, it must be: 

          0 <= start <= past <= (string-length the-string)

if stride is negative, it must be: 

          -1 <= past <= start < (string-length the-string)
— Maker on <string-iterator>: <string-iterator> (?clauses ...)
— Mandatory clause on <string-iterator>: subject: ?the-string
— Optional clause on <string-iterator>: start: ?start
— Optional clause on <string-iterator>: past: ?past
— Optional clause on <string-iterator>: stride: ?stride

Build and return a new <string-iterator> instance using the common constructor.

?stride defaults to +1. If the stride is positive: ?start defaults to zero and ?past defaults to the length of the string. If the stride is negative: ?start defaults one less than the length of the string and ?past defaults to -1.

— Immutable field of <string-iterator>: <char> current

Shadows the current member of <iterator> adding the <char> tag.
