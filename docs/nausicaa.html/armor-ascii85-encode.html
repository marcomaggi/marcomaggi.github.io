68.6.2 Encoding byte streams with ASCII85

— Class: <ascii85-encode-ctx>

Context descriptor for ASCII85 encoding.

— Function: make-<ascii85-encode-ctx>
— Function: <ascii85-encode-ctx>? obj

Constructor and predicate for records of type <ascii85-encode-ctx>.

— Function: ascii85-encode-update-length number-of-bytes

Return the minimum number of bytes required in the output bytevector when ascii85-encode-update! is applied to number-of-bytes bytes of binary input data. Notice that the function will consume input in 4-bytes blocks only and produce output in 5-bytes blocks only, so the return value is always zero or an exact multiple of 5.

— Function: ascii85-encode-final-length number-of-bytes

Return the minimum number of bytes required in the output vector when ascii85-encode-final! is applied to number-of-bytes < 4 bytes of binary input data. The returned value does not take into account the ending ‘~>’ sequence. If number-of-bytes is invalid: the return value is #f.

— Function: ascii85-encode-length number-of-bytes

Return the minimum number of bytes required in the output vector when ascii85-encode-final! is applied to number-of-bytes bytes of binary input data. This is also the number required when both ascii85-encode-update! and ascii85-encode-final! are used.

— Constant: ascii85-encode-block-length

The length of the input block: 4.

— Function: ascii85-encode-update! ctx dst-bv dst-start src-bv src-start src-past

Encode binary data from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store ASCII characters in the bytevector dst-bv starting at dst-start (included); encoding is performed according to the specification in the context ctx, which must be a record of type <ascii85-encode-ctx>.

This function is meant to encode bytes from the input stream when we know that more input bytes will be available in the future.

Return two values:

  1. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  2. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.

— Function: ascii85-encode-final! ctx dst-bv dst-start src-bv src-start src-past

Encode binary data from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store ASCII characters in the bytevector dst-bv starting at dst-start (included); encoding is performed according to the specification in the context ctx, which must be a record of type <ascii85-encode-ctx>.

This function is meant to encode the last bytes from the input stream and produce the last bytes to the output stream. This function does not output the trailing ‘~>’ sequence.

Return three values:

  1. A boolean: true if success, false if the output bytevector was filled before flushing all the bytes.
  2. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.
