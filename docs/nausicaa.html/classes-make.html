24.2 Record makers

— Macro: make* class-name expression ...

Expand to the application of the public constructor of class-name to the results of evaluating the expression arguments, which can be any expression and are evaluated only once. For example, given: 

          (define-class <alpha>
            (fields x s c)
            (protocol (lambda (make-top)
                        (lambda (x)
                          ((make-top) x (sin x) (cos x))))))

the statement: 

          (define a
            (make* <alpha> 1.2))

is equivalent to: 

          (define a
            (make-<alpha> 1.2))

Notice that, using this macro, there is no need to export the class constructor out of a library: it is enough to export the class name. Example: 

          (library (alpha)
            (export <alpha>)
            (import (nausicaa))
          
            (define-class <alpha>
              (fields x s c)
              (protocol (lambda (make-top)
                          (lambda (x)
                            ((make-top) x (sin x) (cos x)))))))
          
          (library (usage)
            (export)
            (import (nausicaa)
              (alpha))
          
            (define a
              (make* <alpha> 1.2)))
— Macro: make class-name maker-clause ...

Expand to the application of the maker constructor of class-name to the results of parsing the maker-clause arguments.

This macro allows both mandatory, positional arguments and optional arguments as specified with the maker clause of define-class. The maker will use the maker–specific constructor if one was specified, else it will default to the public constructor.

If no maker is defined for the class: make is equivalent to make*.

— Macro: make-from-fields class-name expression ...

Like make* but the expression arguments must evaluate to the raw field values; it uses the default constructor function. This macro is useful especially for debugging purposes and for use with the custom maker; classes define custom-maker for details.
