11.4.2 IPv6 address lexer and parser

IPv6 addresses are defined by RFC 4291 “IP Version 6 Addressing Architecture” as 128-bit words commonly represented as 8 components of 16 bits each in base 16: 

     0001:0002:0003:0004:0005:0006:0A07:0B08

the rightmost 2 components can optionally be represented as an IPv4 dotted quad, with numbers in base 10: 

     1:2:3:4:5:6:10.7.11.8

a single double colon sequence ‘::’ can be present in the address and it represents the number of zeros required to make the address numeric components of length 8; so this address: 

     1:2:3::6:0A07:0B08

is equivalent to: 

     1:2:3:0:0:6:0A07:0B08

the double colon sequence represents at least 1 zero component, so the following address is wrong because it would represent 9 components: 

     1:2:3::4:5:6:7:8
