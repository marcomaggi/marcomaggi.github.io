72.1.2.6 Special token–maker parameters

The following parameters are not used directly by the semantic action forms of r6rs-lexer-table: this table directly processes neither strings nor nested comments; instead it produces lexical tokens for double quote characters (which open strings) and nested comment opening sequences, ‘#|’. A full R6RS lexer must recognise such opening tokens and switch table to process strings and nested comments, for example using read-string and read-nested-comment.

The following bindings are exported by the library (nausicaa r6rs lexer).

— Parameter: string-token-maker

Set to a function returning a <lexical-token> record representing a string. The default is make-string-token.

— Parameter: nested-comment-token-maker

Set to a function returning a <lexical-token> record representing a nested comment. The default is make-nested-comment-token.
