24.1.7 Defining class predicates

— Auxiliary Syntax: predicate ?predicate-identifier
— Auxiliary Syntax: predicate ?predicate-expression

Select a custom predicate function or syntax to be used to identify instances of classes with the is-a? macro. When not given in a class definition, the predicate of the underlying record type is used.

This clause is useful especially to select custom predicates for the builtin class types; example: 

          (define-label <pair>
            (predicate pair?)
            (virtual-fields (immutable car car)
                            (immutable cdr cdr)))

another example: 

          (define-label <alpha>
            (predicate (lambda (o)
                         (and (integer? o) (exact? o)))))
          
          (is-a? 123   <alpha>)   ⇒ #t
          (is-a? 123.4 <alpha>)   ⇒ #f

When the form with ?predicate-identifier is used, the argument must be an identifier referencing a function or macro.

When the non–identifier ?predicate-expression form is used: it must be an expression evaluating to the predicate function. The predicate clause with a non–identifier argument is equivalent to the clause: 

          (predicate predicate-name)

and the associated definition: 

          (define predicate-name ?predicate-expression)

where predicate-name is an automatically generated identifier built by appending a dash and custom-predicate to the class or label name.

— Auxiliary Syntax: predicate-syntax ?predicate-identifier
— Auxiliary Syntax: predicate-syntax ?predicate-expression

This clause works exactly like predicate, but when the argument is a non–identifier the associated definition is: 

          (define-syntax predicate-name ?predicate-expression)

that is: it uses define-syntax rather than define.
