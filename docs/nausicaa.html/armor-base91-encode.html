68.5.2 Encoding byte streams with base91

— Class: <base91-encode-ctx>

Context descriptor for base91 encoding. The fields of this record depend on the input data: records of this type cannot be reused for different streams.

— Function: make-<base91-encode-ctx>
— Function: <base91-encode-ctx>? obj

Constructor and predicate for records of type <base91-encode-ctx>.

— Function: base91-encode-update-length number-of-bytes

Return the minimum number of bytes required in the output bytevector when base91-encode-update! is applied to number-of-bytes of binary input data.

— Function: base91-encode-final-length unused

Return the minimum number of bytes required in the output bytevector when base91-encode-final! is applied to an encoding context to flush the buffered bits. unused is ignored: it is here only to provide an API equal to the one of the base64 encoder.

— Function: base91-encode-length number-of-bytes

Return the minimum number of bytes required in the output bytevector when base91-encode-final! is applied to number-of-bytes of binary input data. This is also the number required when both base91-encode-update! and base91-encode-final! are used.

— Function: base91-encode-update! ctx dst-bv dst-start src-bv src-start src-past

Encode binary data from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store ASCII characters in the bytevector dst-bv starting at dst-start (included); encoding is performed according to the specification in the context ctx, which must be a record of type <base91-encode-ctx>.

This function is meant to encode bytes from the input stream when we know that more input bytes will be available in the future.

Return two values:

  1. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  2. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.

— Function: base91-encode-final! ctx dst-bv dst-start src-bv src-start src-past

Encode binary data from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store ASCII characters in the bytevector dst-bv starting at dst-start (included); encoding is performed according to the specification in the context ctx, which must be a record of type <base91-encode-ctx>.

This function is meant to encode the last bytes from the input stream and produce the last bytes to the output stream.

After all the bytes from the source bytevector have been consumed, the bits buffered in the context are flushed to the output buffer.

Return three values:

  1. A boolean: true if success, false if the output bytevector was filled before flushing all the bytes.
  2. The index of the next non–written byte in dst-bv; if dst-bv is left full, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.
