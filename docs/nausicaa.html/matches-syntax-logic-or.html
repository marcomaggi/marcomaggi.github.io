6.3.10.2 Logical disjunction

A logical OR pattern usage looks like this: 

     (match 123
       ((:or (:predicate number? x)
             (:predicate symbol? x))
        x))
     ⇒ 123
     
     (match 'alpha
       ((:or (:predicate number? x)
             (:predicate symbol? x))
        x))
     ⇒ alpha

we can think of it as expanding to: 

     (let ((expr 123))
       (let ((success (lambda (x) x)))
         (if (number? expr)
             (let ((x expr))
               (success x))
           (if (symbol? expr)
               (let ((x expr))
                 (success x))
             (match-mismatch-error 'match expr)))))

the ‘success’ procedure is the “success continuation”; it takes as arguments one value for each of the pattern variables in each of the :or alternative patterns. All the pattern variables defined in an :or alternative must be defined in all the other, else an error will occur. (The ‘success’ identifier is introduced by match while expanding, so it cannot be referenced directly in the body of a clause.)

The above example is correct because ‘x’ is defined in both the alternative patterns, the following example is wrong: 

     (match 123
       ((:or (:predicate integer? x)
             (:predicate symbol?  y))
        y))
     error--> unbound identifier

because it expands to: 

     (let ((expr 123))
       (let ((success (lambda (x y) y)))
         (if (integer? expr)
             (let ((x expr))
               (success x y))        ;undefined y
           (if (symbol? expr)
               (let ((y expr))
                 (success x y))      ;undefined x
             (match-mismatch-error 'match expr)))))

An empty :or pattern matches nothing, it always fails: 

     (match 123
       ((:or) 'fail))
     error--> no matching pattern
