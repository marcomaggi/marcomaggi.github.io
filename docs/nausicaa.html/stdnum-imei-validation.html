72.5.2 Validating IMEI identifiers

The following bindings are exported by the (nausicaa standard-numbers imei) library. The following functions attempt validation of IMEI identifiers; the 16 digits IMEI do not have a check digit, so validation is problematic. Given a 16 digits IMEI: 

     35686800-004141-20

if we mistype it as: 

     35686800-0041412-0

it is impossible to distinguish between a 16 digits IMEI with the last hyphen in the wrong position and a 15 digits IMEI with an additional digit in the second field; in this case the functions below will interpret the identifier as a valid 16 digits IMEI; the position of the hyphens is never taken as an invalidating property.

— Function: imei15?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 15 digits IMEI identifier. Do not validate the field numbers, only the format of the string. Perform the following checks:

  • The first character is a digit.
  • The last character is a digit.
  • The string is composed of digits and hyphens.
  • No two consecutive hyphens are present.
  • The string stripped of all the hyphens has 15 characters.
  • The check digit must match the core digits according to the Luhn algorithm.

Notice that there is no check for the correct position of the hyphen separators.

— Function: imei16?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 16 digits IMEI identifier. Do not validate the field numbers, only the format of the string. Perform the following checks:

  • The first character is a digit.
  • The last character is a digit.
  • The string is composed of digits and hyphens.
  • No two consecutive hyphens are present.
  • The string stripped of all the hyphens and has 16 characters.

Notice that there is no check for the correct position of the hyphen separators.

— Function: imei?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 15 digits or 16 digits IMEI identifier. Do not validate the field numbers, only the format of the string.

This function makes use of imei15?/well-formed and imei16?/well-formed.

— Function: imei-type imei-string

Validate imei-string as a string representing an IMEI identifier; return the number of digits 15 or 16 or #f if the argument is invalid. Do not validate the field numbers, only the format of the string.

This function makes use of imei15?/well-formed and imei16?/well-formed. 

          (imei-type "35-417803-685978-9")        ⇒ 15
          (imei-type "35686800-004141-20")        ⇒ 16
          
          ;; wrong check digit
          (imei-type "35-417803-685978-1")        ⇒ #f
— Function: imei15-check-digit imei-core-string

Compute the check digit for a 10 digits IMEI, return the corresponding character. The string argument must not include the check digit. No validation is performed on the input argument. 

          ;; 35-417803-685978-9
          (imei15-check-digit "35-417803-685978") ⇒ #\9
          (imei15-check-digit "35417803685978")   ⇒ #\9
          
          ;; 350077-52-323751-3
          (imei15-check-digit "350077-52-323751") ⇒ #\3
          (imei15-check-digit "35007752323751")   ⇒ #\3
— Function: imei15? obj
— Function: imei16? obj
— Function: imei? obj

Validate the argument as an IMEI identifier. At present these bindings are aliases for imei15?/well-formed, imei16?/well-formed and imei?/well-formed.
