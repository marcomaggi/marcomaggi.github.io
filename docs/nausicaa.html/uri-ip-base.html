11.1.2 Generic IP address objects

The following bindings are exported by the library (nausicaa uri ip).

— Abstract Class: <ip-address>

Base class for all the address types representing IP addresses.

Constructors and makers
— Super constructor on <ip-address>: constructor
— Super constructor on <ip-address>: constructor option ...

Supertype constructor for types derived from <ip-address>. When called with some arguments it supports the following keyword values:

#:string-rep obj
The argument obj must be #f or a string object representation of the address as defined by RFC 3986.
#:bytevector-rep obj
The argument obj must be #f or a bytevector object representation of the address as defined by RFC 3986.

Fields
— Immutable virtual field of <ip-address>: <ascii-string> string

String object representation of the address. It must represent the address string as defined by RFC 3986; every character in the string can be directly converted to an ASCII encoded character, some sequences of characters may represent a percent–encoded character.

The value of this field is set by the constructor with the keyword #:string-rep or computed by the multimethod ip-address->string and memoized in the object.

Notice that some subclasses of <ip-address> have string representation that is invalid as percent–encoded string as defined by RFC 3986.

— Immutable virtual field of <ip-address>: <ascii-bytevector> bytevector

Bytevector object representation of the address as defined by RFC 3986; every octet in the bytevector represents an ASCII encoded character, some sequences of octets may represent a percent–encoded character.

The value of this field is set by the constructor with the keyword #:bytevector-rep or computed by the multimethod ip-address->bytevector and memoized in the object.

Notice that some subclasses of <ip-address> have bytevector representation that is invalid as percent–encoded bytevector as defined by RFC 3986.

Multimethod implementations
— Multimethod: ip-address->bytevector (addr <ip-address>)

Build and return a bytevector representation of the address from its string representation; raise an error if a string representation cannot be built. Expect all the characters in the field string of addr to be directly convertible to the corresponding ASCII encoding.
