5.12 Vector type

— Builtin Label: <vector>

Conventional type for objects satisfying vector?.

— Common constructor on <vector>: <vector> constructor item ...

Return the result of applying vector to the item arguments.

Safe accessors
— Immutable virtual field of <vector>: <fixnum> length

Access the length of the vector.

— Immutable virtual field of <vector>: <boolean> empty?

True if the vector length is zero.

Unsafe accessors
— Immutable virtual field of <vector>: <fixnum> $length

Return the result of applying $vector-length from (vicare system $vectors) to the vector.

— Immutable virtual field of <vector>: <boolean> $empty?

Return #t if the vector length is zero. Make use of unsafe operations from (vicare system $vectors).

Methods
— Method on <vector>: <vector> subvector (begin <fixnum>)
— Method on <vector>: <vector> subvector (begin <fixnum>) (end <fixnum>)

Build and return a new vector holding the selected subvector of the instance. When end is not used: it defaults to the length of the vector.

When begin and end are zero or positive: this method behaves like subvector from (nausicaa language common). If either begin and/or end are negative: they are first normalised with: 

          (+ begin/end (vector-length instance))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selects the penultimate value and so on.

— Method on <vector>: map proc . vectors

Apply vector-map to the instance vector and the optional vectors.

— Method on <vector>: for-each proc . vectors

Apply vector-for-each to the instance vector and the optional vectors.

— Method on <vector>: for-all proc . vectors

Apply vector-for-all to the instance vector and the optional vectors.

— Method on <vector>: exists proc . vectors

Apply vector-exists to the instance vector and the optional vectors.

— Method on <vector>: fill! fill

Apply vector-fill! to the instance vector and fill.

— Method on <vector>: <top> ref (index <fixnum>)

When index is zero or positive: this method behaves like vector-ref from (rnrs base (6)). If index is negative: it is first normalised with: 

          (+ index (vector-length vector))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selects the penultimate value and so on.

— Method on <vector>: <top> set! (index <fixnum>) (value <top>)

Store value at index in the vector. When index is zero or positive: this method behaves like vector-set! from (rnrs vectors (6)). If index is negative: it is first normalised with: 

          (+ index (vector-length vector))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selects the penultimate value and so on.

OOPP syntaxes
— Maker on <vector>: <vector> (?item ...)

Return the result of applying vector to the results of evaluating the expressions ?item.

— Getter on <vector>: instance (index)

Item getter; it behaves like the ref method.

— Setter on <mutable-vector>: set!/tags (instance (index)) ?item

Item setter; it behaves like the set! method.
