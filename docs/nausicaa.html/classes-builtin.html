25.8 Conventional type descriptors

Builtin Scheme object types can be treated as instances of class types by accepting some limitations; this allows us to use the facilities of the (nausicaa language classes) library with builtin objects.

Builtin Scheme values have no RTD, so (nausicaa language classes) assigns a conventional one to them. Conventional RTDs have two purposes: to provide an interface for identifier macros defined by with-class; to be used in multimethod dispatching by the (nausicaa language generics) library. The conventional RTDs can be used neither to instantiate builtin objects nor as parents for other class types; for example: make-<string> does not return a builtin Scheme string object.

A record is described by its record type descriptor (RTD); unfortunately we cannot rely on RTDs describing the same record type to be eq? to each other, so we have to rely on the UIDs of non–generative record types. When dispatching multimethods, it is convenient to use those UIDs to represent the “class” of a record.

  • classes builtin chart: A chart for predefined builtin types.
  • classes builtin base: Basic conventional types.
  • classes builtin list: Pairs and lists types.
  • classes builtin compound: Non--list compound types.
  • classes builtin number: Numeric objects types.
  • classes builtin port: Input/output port objects types.
  • classes builtin string: Character and string types.
  • classes builtin misc: Miscellaneous conventional types.
