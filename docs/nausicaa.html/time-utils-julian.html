50.8.4 Julian calendar utilities

The following bindings are exported by the (nausicaa times-and-dates julian-calendar) library. The arguments to the functions are supposed to be in their valid type and range:

  • Nanoseconds as exact integers in [0, 999999999].
  • Seconds as exact integers in [0, 59] when fractions of a minute, as exact integers when counts since the Epoch on the TAI or UTC scale.
  • Minutes as exact integers in [0, 59].
  • Hours as exact integers in [0, 23].
  • Days as exact integers in [1, 28], [1, 29], [1, 30] or [1, 31] according to the month number.
  • Months as exact integers in [1, 12].
  • Years as exact integers.
  • Julian Dates as exact integers or exact rationals.
  • Julian Day Numbers and Modified Julian Day Numbers as exact integers.
Conversion to/from seconds and nanoseconds
— Function: utc-seconds-and-nanoseconds->julian-day-number utc-seconds nanoseconds

Return the Julian Day Number representing the given counts of UTC seconds (time zone zero) since the Unix Epoch and nanoseconds.

— Function: tai-seconds-and-nanoseconds->julian-day-number tai-seconds nanoseconds

Return the Julian Day Number representing the given counts of TAI seconds (time zone zero) since the Unix Epoch and nanoseconds.

— Function: julian-day-number->utc-seconds-and-nanoseconds jdn

Return two values: the UTC seconds count (time zone zero) since the Unix Epoch and nanoseconds counts representing jdn.

— Function: julian-day-number->tai-seconds-and-nanoseconds jdn

Return two values: the TAI seconds count (time zone zero) since the Unix Epoch and nanoseconds counts representing JDN.

— Function: utc-seconds-and-nanoseconds->julian-date utc-seconds nanoseconds

Return the Julian Date representing the given counts of UTC seconds (time zone zero) since the Unix Epoch and nanoseconds.

— Function: tai-seconds-and-nanoseconds->julian-date tai-seconds nanoseconds

Return the Julian Date representing the given counts of TAI seconds (time zone zero) since the Unix Epoch and nanoseconds.

— Function: julian-date->utc-seconds-and-nanoseconds jd

Return two values: the UTC seconds count (time zone zero) since the Unix Epoch and nanoseconds counts representing jd.

— Function: julian-date->tai-seconds-and-nanoseconds jd

Return two values: the TAI seconds count (time zone zero) since the Unix Epoch and nanoseconds counts representing jd.

Conversion to/from Gregorian calendar
— Function: julian-day-encode-number year month day

Return an exact integer representing the Julian Day Number at the given date on the proleptic Gregorian calendar at 12:00:00Z, that is: starting at noon, UTC scale, time zone zero. This function assumes the given date is correct.

When converting a full time point to a Julian Day Number: the counts of hours, minutes, seconds and nanoseconds are discarded.

From the Calendar FAQ, section 2.16.1.

— Function: julian-day-decode-number jdn

Return 3 values: year, month, day in the proleptic Gregorian calendar corresponding to the Julian Day Number jdn.

When converting a Julian Day Number to a full time point: the count of hours must be set to 12 and the counts of minutes, seconds and nanoseconds must be set to zero.

From the Calendar FAQ, section 2.16.1.

— Function: time-point->julian-date year month day hours minutes seconds nanoseconds

Convert the given time point into a Julian Date value.

— Function: julian-date->time-point jd

Convert a Julian Date value to a time point tuple; return 7 values: year, month, day, hours, minutes, seconds, nanoseconds.

Conversion between JDN and MJDN
— Function: julian-day-number->modified-julian-day-number jdn

Return the Modified Julian Day Number corresponding to the Julian Day Number jdn.

— Function: modified-julian-day-number->julian-day-number mjdn

Return the Julian Day Number corresponding to the Modified Julian Day Number mjdn.
