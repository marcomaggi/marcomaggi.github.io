5.14 Hashtable labels

— Builtin Label: <hashtable>

Conventional type for objects satisfying hashtable?.

— Builtin Label: <hashtable-eq>

Conventional type for objects satisfying hashtable? and built to compare keys with eq?.

— Builtin Label: <hashtable-eqv>

Conventional type for objects satisfying hashtable? and built to compare keys with eqv?.

— Builtin Label: <string-hashtable>
— Builtin Label: <string-ci-hashtable>

Conventional types for objects satisfying hashtable? and built to use strings as keys.

— Builtin Label: <symbol-hashtable>

Conventional type for objects satisfying hashtable? and built to use symbols as keys.

Safe accessors
— Immutable virtual field of <hashtable>: size

Access the size of the table.

— Immutable virtual field of <hashtable>: keys

Access the keys. Retrieving this field is like applying hashtable-keys to the hashtable.

— Immutable virtual field of <hashtable>: <boolean> mutable?

Retrieving this field is like applying hashtable-mutable? to the hashtable.

Methods
— Method on <hashtable>: entries

Access the keys and values. Calling this method is like applying hashtable-entries to the hashtable.

— Method on <hashtable>: delete! key

Equivalent to applying hashtable-delete! to the instance.

— Method on <hashtable>: <boolean> contains? key

Equivalent to applying hashtable-contains? to the instance.

— Method on <hashtable>: clear!

Equivalent to applying hashtable-clear! to the instance.

— Method on <hashtable>: update! key proc default

Equivalent to applying hashtable-update! to the instance.

— Method on <hashtable>: copy
— Method on <hashtable>: copy mutable?

Equivalent to applying hashtable-copy to the instance.

OOPP syntaxes
— Maker on <hashtable-eq>: <hashtable-eq> ((?key ?val) ...)

Build and return a new hashtable object using make-eq-hashtable. Initialise the table with the given key/value entries.

— Maker on <hashtable-eqv>: <hashtable-eqv> ((?key ?val) ...)

Build and return a new hashtable object using make-eqv-hashtable. Initialise the table with the given key/value entries.

— Maker on <string-hashtable>: <string-hashtable> ((?key ?val) ...)
— Maker on <string-ci-hashtable>: <string-ci-hashtable> ((?key ?val) ...)

Build and return a new hashtable object using make-hashtable, the hash functions string-hash and string-ci-hash (respectively) and the equivalence functions string=? and string-ci=? (respectively). Initialise the table with the given key/value entries.

— Maker on <symbol-hashtable>: <symbol-hashtable> ((?key ?val) ...)

Build and return a new hashtable object using make-hashtable, the hash function symbol-hash and the equivalence function eq?. Initialise the table with the given key/value entries.

— Getter on <hashtable>: instance (?key)
— Getter on <hashtable>: instance (?key) (?default)

Value getter. Equivalent to applying hashtable-ref to instance; return the value associated to ?key. If ?default is used: it must be an expression whose value is returned when the requested key is not in the table.

— Setter on <hashtable>: set!/tags (instance (?key)) ?item

Value setter. Equivalent to applying hashtable-set! to instance.
