58 Defining storage locations

The (nausicaa variables) library provides a way to define simple storage locations similar to the variables used in many languages. The variable type of value is disjoint from the other types.

— Function: make-variable
— Function: make-variable obj

Build and return a new variable value. The variable is initialised to obj, which defaults to the sentinel. The sentinel value

— Function: variable? obj

Return #t if obj is a variable value.

— Function: variable-set! variable obj
— Function: variable-ref variable

Setter and getter for variables.

It is possible to access variables' slots using identifier syntaxes: 

     (define-syntax define-variable
       (syntax-rules ()
         ((_ ?name)
          (define-variable/test ?name sentinel))
         ((_ ?name ?value)
          (define ?name (make-variable ?value)))))
     
     (define-syntax v
       (identifier-syntax
          (_
           (variable-ref  the-var))
          ((set! _ ?e)
           (variable-set! the-var ?e))))
     
     (define-variable the-var 123)
     
     v ⇒ 123
     (set! v 456)
     v ⇒ 456

a more sophisticated of this is the define-variable syntax exported by (nausicaa variables).

— Syntax: define-variable name
— Syntax: define-variable name obj
— Syntax: define-variable (name . <formals>) <body>

Like define, but stores the value in a newly built variable which is then bound to name. Example: 

          (define-variable the-var 123)
          
          v ⇒ 123
          (set! v 456)
          v ⇒ 456

The difference between define-syntax and define, is that define only defines a binding which cannot be modified, while define-syntax allocates a storage location, so the value bound to name can be modified at will.
