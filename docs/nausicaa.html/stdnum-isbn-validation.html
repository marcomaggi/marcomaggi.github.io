76.4.2 Validating ISBN identifiers

The following bindings are exported by the (nausicaa standard-numbers isbn) library.

— Function: isbn10?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 10 digits ISBN identifier. Do not validate the field numbers, only the format of the string.

Perform the following checks:

  • obj is a Scheme string.
  • The first character is a digit.
  • The last character is a digit or the upper case letter ‘X’.
  • The string is composed of digits and hyphens or digits and white spaces with the last character possibly being the upper case letter ‘X’.
  • No two consecutive hyphens or white spaces are present.
  • The string stripped of all the hyphens and white spaces has 10 characters.
  • The check digit is recomputed and it must match the core digits.

— Function: isbn13?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 13 digits ISBN identifier. Do not validate the field numbers, only the format of the string.

Perform the following checks:

  • obj is a Scheme string.
  • The string begins with the prefix ‘978’ or ‘979’.
  • The last character is a digit.
  • The string is composed of digits and hyphens or digits and white spaces.
  • No two consecutive hyphens or white spaces are present.
  • The string stripped of all the hyphens and white spaces has 13 characters.
  • The check digit is recomputed and it must match the core digits.

— Function: isbn?/well-formed obj

Return a boolean, true if the argument can be interpreted as a well formatted 10 digits or 13 digits ISBN identifier. Do not validate the field numbers, only the format of the string.

This function makes use of isbn10?/well-formed and isbn13?/well-formed.

— Function: isbn-type isbn-string

Validate isbn-string as a string representing an ISBN identifier; return the number of digits 10 or 13 or #f if the argument is invalid. Do not validate the field numbers, only the format of the string.

This function makes use of isbn10?/well-formed and isbn13?/well-formed. 

          (isbn-type "0-201-63361-2")     ⇒ 10
          (isbn-type "978-0-201-63361-0") ⇒ 13
          
          ;; wrong check digit
          (isbn-type "0-201-63361-8")     ⇒ #f
— Function: isbn10-check-digit isbn-core-string

Compute the check digit for a 10 digits ISBN, return the corresponding character. The string argument must not include the check digit. No validation is performed on the input argument. 

          ;; 3-16-148410-X
          (isbn10-check-digit "3-16-148410") ⇒ #\X
          
          ;; 0 201 63361 2
          (isbn10-check-digit "0 201 63361") ⇒ #\2
          
          ;; 013490012X
          (isbn10-check-digit "013490012")   ⇒ #\X
— Function: isbn13-check-digit isbn-core-string

Compute the check digit for a 13 digits ISBN, return the corresponding character. The string argument must not include the check digit. No validation is performed on the input argument. 

          ;; 978-3-16-148410-0
          (isbn13-check-digit "978-3-16-148410") ⇒ #\0
          
          ;; 978 0 201 63361 0
          (isbn13-check-digit "978 0 201 63361") ⇒ #\0
          
          ;; 9780134900124
          (isbn13-check-digit "978013490012")    ⇒ #\4
