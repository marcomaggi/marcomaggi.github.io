30.1.7.2 Calling foreign functions

The following bindings are exported by the (nausicaa ffi) library.

— Function: make-c-callout ret-type address arg-types
— Function: make-c-callout/with-errno ret-type address arg-types

Build and return a new Scheme callout closure for a C language foreign function whose address is specified by the address pointer object.

ret-type must be a symbol in the enum-clang-external-types enumeration; it represents the type of the foreign function's return value. arg-types must be a list of symbols in the enum-clang-external-types enumeration; they represent the types of the foreign function's arguments.

The ‘/with-errno’ variant returns a callout closure which, when evaluated, returns two values: the return value from the foreign function and the value of the C standard variable errno right after the foreign function call.

NOTE Some POSIX functions return an integer representing success or failure and only when the returned value represents failure the ‘errno’ value is meaningful.

The returned closure:

  1. Accepts as many Scheme object arguments as there are symbols in arg-types; to specify an empty list of arguments, we can use (void).
  2. Does marshaling from Scheme arguments to C language values as specified by the arg-types symbols.
  3. Applies the foreign function to the C values.
  4. Returns the return value of the foreign function, marshaled to a Scheme value as specified by the ret-type symbol.

— Function: make-c-function shared-object ret-type funcname arg-types
— Function: make-c-function/with-errno shared-object ret-type funcname arg-types

Wrappers for the make-c-callout functions which determine the address of the function by looking for the funcname symbol in the shared-object using lookup-shared-object*.

Beware that, sometimes, the exported symbols are different from the ones we write in C language code to call a function. If the foreign symbol is not found, a continuable exception is raised with raise-shared-object-lookup-error; returning to the continuation must return the address of the function. Shared objects lookup error.

For Unix One way to discover the names of symbols exported by a shared object, is to apply the strings command to the shared object itself. For example, to discover the names of deflate functions in Zlib: 
          $ strings /usr/lib/libz.so | grep deflate
          deflateInit_
          deflate
          deflateEnd
          ...
