71.5.1 Introduction to IMEI identifiers

For an introduction on IMEI identifiers see (URL last verified Jun 1, 2011):

http://en.wikipedia.org/wiki/Imei

for the IMEI Allocation and Approval Guidelines see (URL last verified Jun 1, 2011):

http://www.gsmworld.com/documents/DG06_v5.pdf

The (nausicaa standard-numbers imei) library handles IMEI identifiers as Scheme strings, stripped of any blank character before and after them; IMEI identifiers often appear with the string IMEI in front of them, but the library assumes that this tag has already been stripped.

The following is an example of IMEI identifiers: 

     35-417803-685978-9
     35-686800-004141-20

there are two formats: the 15 digits one is properly named IMEI and it has a Luhn check digit as last character, the 16 digits one is named IMEISV and has a software version (SV) as last two characters (no check digit in this one).

In the context of Nausicaa/Scheme, we refer to IMEI and IMEISV identifiers as 15 digits IMEI identifiers and as 16 digits IMEI identifiers. The corresponding library bindings have names prefixed with imei15 and imei16; the prefix imei without digits specification is for functions accepting both 15 and 16 digits IMEI identifiers.

The identifier can be split into 4 fields of fixed length for both formats. Fields can be contiguous or separated by a single hyphen.
