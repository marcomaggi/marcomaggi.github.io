6.1 Generic functions and their customisation

There are two variants of “generic functions” in (nausicaa mehve), both of them are in truth syntaxes. The first kind is the one that generalises a function with fixed arity (fixed number of arguments); a function like: 

     (define (rnrs.sin x)
       ---)

is generalised with: 

     (define-generic sin (x))
     
     (add-method sin (<number>) rnrs.sin)

and allows us to specialise it for non–<number> arguments, for example: 

     (define-method (sin (x <list>))
       (x.map sin))
     
     (define-method (sin (x <vector>))
       (x.map sin))

The other kind is a generalisation for variadic functions (which take an undefined number of arguments); a function like: 

     (define (rnrs.+ . operands)
       ---)

is redefined with this infrastructure: 

     (define-syntax +
       (lambda (stx)
         (syntax-case stx ()
           (?id
            (identifier? #'?id)
            #'addition)
           ((_)
            #'(addition-0))
           ((_ ?a)
            #'(addition-1 ?a))
           ((_ ?a ?b)
            #'(addition-2 ?a ?b))
           ((_ ?a ?b ?c)
            #'(addition-2 (addition-2 ?a ?b) ?c))
           ((_ . ?operands)
            #'(addition . ?operands)))))
     
     (define (addition . operands)
       (if (null? operands)
           (addition-0)
         (let loop ((result   (car operands))
                    (operands (cdr operands)))
           (if (null? operands)
               result
             (loop (addition-2 result (car operands))
                   (cdr operands))))))
     
     (define-generic addition-0 ())
     (define-generic addition-1 (a))
     (define-generic addition-2 (a b))
     
     (define-method (addition-0)
       0)
     
     (define-method (addition-1 (a <number>))
       a)
     
     (add-method addition-2 (<fixnum> <fixnum>) fx+)
     (add-method addition-2 (<flonum> <flonum>) fl+)
     (add-method addition-2 (<number> <number>) rnrs.+)

and specialised methods can be added to the generic functions addition-0, addition-1 and addition-2. In this case (nausicaa mehve) exports the binding for +, but the bindings for addition-0, addition-1 and addition-2 are exported by the auxiliary library (nausicaa mehve arithmetics).

If a binding F is not generalised by (nausicaa mehve), it can only be generalised by writing a custom library which imports (nausicaa mehve) and reexports all its bindings and also redefines the binding F; the faster way to do this is to just copy and customise the source code of the (nausicaa mehve) library.

This is annoying, but it is the only way of doing it because:

  • R6RS forbids to mutate an imported binding.
  • It makes no sense to have all the bindings to functions in (nausicaa) redefined as generics.

In any case requests to generalise a binding to function can be submitted to the Nausicaa/Scheme maintainer.
