70.5.3 Path segment component parser functions

The following functions can be used to parse the segment, segment-nz and segment-nz-nc components of an URI; the relevant portion of the grammar is: 

     segment       = *pchar
     segment-nz    = 1*pchar
     segment-nz-nc = 1*( unreserved | pct-encoded | sub-delims | "@" )
     
     pchar         = unreserved | pct-encoded | sub-delims | ":" | "@"
     
     unreserved    = ALPHA | DIGIT | "-" | "." | "_" | "~"
     pct-encoded   = "%" HEXDIG HEXDIG
     sub-delims    = "!" | "$" | "&" | "'" | "(" | ")"
                   | "*" | "+" | "," | ";" | "="

The following bindings are exported by the (nausicaa uri low) library; it is suggested to prefix them when importing this library. All the functions accept a port argument: it must be a binary input port drawing characters from a bytevector.

— Function: parse-segment port

Accumulate bytes from port while they are valid for a segment component; notice that an empty segment is valid.

If EOF or a byte not valid for a segment is read: return a bytevector holding the bytes accumulated so far, invalid byte excluded; the port position is left pointing to the byte after the last accumulated one.

If an invalid percent–encoded sequence is read, an exception is raised with type &parser-error; the port position is rewind to the one before this function call.

— Function: parse-segment-nz port

Accumulate bytes from port while they are valid for a segment-nz component; notice that an empty segment-nz is not valid: nz stands for “non–zero”.

If the first read operation returns EOF or an invalid byte: the port position is restored to the one before this function call and the return value is false.

If, after at least one valid byte is read, EOF or an invalid byte is read: return a bytevector holding the bytes accumulated so far, invalid byte excluded; the port position is left pointing to the byte after the last accumulated one.

If an invalid percent–encoded sequence is read, an exception is raised with type &parser-error; the port position is rewind to the one before this function call.

— Function: parse-segment-nz-nc port

Like parse-segment-nz, but parses the segment-nz-nc component which cannot be empty and cannot hold bytes representing colon characters in ASCII encoding: nz-nc stands for “non–zero no–colon”.

— Function: parse-slash-and-segment port

Attempt to read from port the sequence slash character plus segment component; notice that an empty segment is valid.

If these components are successfully read: return a bytevector (possibly empty) holding the accumulated segment bytes; the port position is left pointing to the byte after the last accumulated byte from the segment.

If EOF or a byte different from slash is read as first byte: return false; the port position is rewind to the one before this function call.

If an invalid percent.-encoded sequence is read, an exception is raised with type &parser-error; the port position is rewind to the one before this function call.
