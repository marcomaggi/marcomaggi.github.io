66.6.3 Decoding byte streams with ASCII85

None of the following functions will take care to recognise and parse the opening ‘<~’ or closing ‘~>’ sequences of the ASCII85 encoding.

— Class: <ascii85-decode-ctx>

Context descriptor for ASCII85 decoding.

— Function: make-<ascii85-decode-ctx>
— Function: <ascii85-decode-ctx>? obj

Constructor and predicate for records of type <ascii85-decode-ctx>.

— Function: ascii85-decode-update-length number-of-chars

Return the minimum number of bytes required in the output vector when ascii85-decode-update! is applied to number-of-chars ASCII characters of input. Notice that the function will consume input in 5-bytes blocks only and produce output in 4-bytes blocks only, so the return value is always zero or an exact multiple of 4.

This function does not take into account that an input character could be ‘#\z’, corresponding to 4 output bytes.

— Function: ascii85-decode-final-length number-of-chars

Return the minimum number of bytes required in the output vector when ascii85-decode-final! is applied to number-of-chars < 5 ASCII characters of input. All the arguments between 0 and 4 are valid; if the argument is not valid the return value is #f. If the argument is 1, the return value is 4: this is to allow a single ‘#\z’ character to be processed.

— Function: ascii85-decode-length number-of-chars
— Function: ascii85-decode-length number-of-chars padding?

Return the minimum number of bytes required in the output vector when ascii85-decode-final! is applied to number-of-chars ASCII characters of input. This is also the number required when both ascii85-decode-update! and ascii85-decode-final! are used.

This function does not take into account that an input character could be ‘#\z’, corresponding to 4 output bytes.

— Constant: ascii85-decode-block-length

The length of the input block: 5.

— Function: ascii85-decode-update! ctx dst-bv dst-start src-bv src-start src-past

Decode ASCII characters from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store binary data bytes in the bytevector dst-bv starting at dst-start (included); decoding is performed according to the specification in the context ctx, which must be a record of type <ascii85-decode-ctx>.

Return three values:

  1. A boolean always false. This return value is used to make the ASCII85 API equal to the one of the base64 decoder.
  2. The index of the next non–written byte in dst-bv; if dst-bv is filled to the end, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the characters from src-bv are consumed, this value is src-past.

ASCII characters are read from src-bv in chunks of 5 bytes; binary data bytes are written to dst-bv in chunks of 4 bytes. If not enough room exists in dst-bv to store a block, available input data is not consumed. As a special exception: if the first character to be read in a chunk is ‘#\z’, it is decoded to a sequence of 4 zero bytes.

— Function: ascii85-decode-final! ctx dst-bv dst-start src-bv src-start src-past

Decode ASCII characters from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store binary data bytes in the bytevector dst-bv starting at dst-start (included); decoding is performed according to the specification in the context ctx, which must be a record of type <ascii85-decode-ctx>.

Return three values:

  1. A boolean: true if success, false if the output bytevector was filled before flushing all the bytes.
  2. The index of the next non–written byte in dst-bv; if dst-bv is filled to the end, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.

ASCII characters are read from src-bv in chunks of 5 bytes; binary data bytes are written to dst-bv in chunks of 4 bytes. After that the trailing characters are decoded. If not enough room exists in dst-bv to store a block, available input data is not consumed. As a special exception: if the first character to be read in a chunk is ‘#\z’, it is decoded to a sequence of 4 zero bytes.
