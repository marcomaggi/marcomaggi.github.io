12.1.1 Source location objects

— Class: <source-location>

Represent a source location in a source of characters used as input for a lexer. It is meant to reference the position of the next character to parse from the input, which is the position of the first char in the next token produced by the lexer.

— Common constructor on <source-location>: constructor specified? line column offset

Build and return a new <source-location> object initialised with the given field values.

— Maker syntax on <source-location>: <source-location> (?clause ...)
— Maker syntax on <source-location>: <source-location> (unspecified)
— Auxiliary Syntax on <source-location>: line: ?line
— Auxiliary Syntax on <source-location>: column: ?column
— Auxiliary Syntax on <source-location>: offset: ?offset

Build and return a new instance of <source-location> using the common constructor and the given arguments. When a ?clause argument is not used, its default value is such that the returned object represents the conventional starting position, with line number 1, column number 1 and offset 0.

When the unspecified argument is used: it must be an identifier whose symbol name is unspecified; the return value is an object whose specified? field is set to #f.

— Immutable field of <source-location>: <boolean> specified?
— Immutable virtual field of <source-location>: <boolean> unspecified?

The field specified? is #t if this object represents a specified source location; #f otherwise. The field unspecified? is #t if this object represents an unspecified source location; #f otherwise.

— Immutable field of <source-location>: <positive-fixnum> line
— Immutable field of <source-location>: <positive-fixnum> column
— Immutable field of <source-location>: <nonnegative-fixnum> offset

Counters for the line number, column number and character offset from the start of the input. By convention: line and column numbers start from 1, the offset starts from 0. The value of these fields is meaningful only if the field specified? is set to #t.

These fields are modeled to match the local bindings ‘yyline’, ‘yycolumn’ and ‘yyoffset’ available to the lexer actions of SILex; Evaluation of the actions.

— Method on <source-location>: <boolean> start?

Return #t if the instance is specified and represents the conventional starting position: line number 1, column number 1 and offset 0; otherwise return #f.
