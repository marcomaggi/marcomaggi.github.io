11.2 Describing lexical tokens

It is an accepted design pattern to split the interpretation of a stream of text into a lexing step and a parsing step. In such scenario a lexer function fragments the input into tokens each of which is assigned a category, the parser associates a semantic meaning to a sequence of token categories.

The library (nausicaa parser-tools lexical-tokens) defines the class <lexical-token> which is meant to be used by all the lexers and parsers distributed with Vicare Scheme. The library must be used along with (nausicaa parser-tools source-locations), which defines the class <source-location>.

  • parser-tools lexical-tokens intro: Overview of lexical tokens.
  • parser-tools lexical-tokens token: Lexical token objects.
  • parser-tools lexical-tokens eoi: End-of-input objects.
  • parser-tools lexical-tokens error: Lexical error objects.
  • parser-tools lexical-tokens silex: SILex default semantic actions.
