78.1 Converting strings

— Function: infix-string->sexp expr-string

Return the prefix S–expression corresponding to expr-string. The S–expression can be used directly for some symbolic interpretation or evaluated with eval. Evaluation

Examples: 

          (infix-string->sexpr "1*2-3")   ⇒ (- (* 1 2) 3)
          (infix-string->sexpr "1//3")    ⇒ (div 1 3)
          (infix-string->sexpr "1%3")     ⇒ (mod 1 3)
          (infix-string->sexpr "1^3")     ⇒ (expt 1 3)
          
          (infix-string->sexpr "cos(sin(1.1)+4)")
          ⇒ (cos (+ (sin 1.1) 4))
          
          (infix-string->sexpr "fun(1, 2, 3, 4)")
          ⇒ (fun 1 2 3 4)
          
          (infix-string->sexpr "cos(a) * (tan(b) / c)")
          ⇒ (* (cos a) (/ (tan b) c))
          
          (infix-string->sexp "a * (b / a ? b : c)")
          ⇒ (* a (if (/ b a) b c))

Some interpretation rules:

  • Blank characters are ignored.
  • Symbols in expr-string must match the following (nausicaa parser-tools silex) regular expression: 
              initial         [a-zA-Z_]
          subsequent      {initial}|[0-9\.@!$&:<=>\?~\-]
          symbol          {initial}{subsequent}*

    Syntax of the specification.

  • The following infix operators are supported, in order of increasing precedence from left to right: 
              + - * / // % ^ < > <= >= =

    remember that the Scheme comparison operators are meant to return a boolean value, rather than a number object. ‘//’ stands for div, ‘%’ stands for mod, ‘^’ stands for expt.

  • The if–then–else statement involves the special symbols ? and : which cannot be variable symbols; this statement has the least precedence.
  • Every symbol which is not an operator is interpreted as variable reference or function call. If the first non–blank character following the symbol is an open parenthesis, it is interpreted as function call.
