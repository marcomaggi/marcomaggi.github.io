11.2.5.2 URI list of segment objects

The path component of URIs is made of segment components, which are stored in lists by (nausicaa uri): 

     #!vicare
     (import (nausicaa)
       (prefix (nausicaa uri) uri.))
     
     (define ell
       '(#ve(ascii "home")
         #ve(ascii "marco")
         #ve(ascii "src")
         #ve(ascii "devel")))
     
     (uri.<list-of-segments> O
       (<> (ell)))
     
     (O bytevector)
     ⇒ #ve(ascii "home/marco/src/devel")
     
     (O string)
     ⇒ "home/marco/src/devel"
     
     (receive (port getter)
         (open-bytevector-output-port)
       (O put-bytevector port)
       (getter))
     ⇒ #ve(ascii "home/marco/src/devel")

The following bindings are exported by the (nausicaa uri uri) library.

— Label: <list-of-segments>

Label used to tag list of bytevectors representing path segment components of URIs; it is derived from <list>.

Constructors, makers, predicates
— Constructor on <list-of-segments>: constructor obj

Just return the given argument. If tagged argument validation is enabled: validate the argument with the predicate of <list-of-segments>.

— Maker on <list-of-segments>: <list-of-segments> (?obj)

Filter the given argument using the label constructor.

— Predicate on <list-of-segments>: predicate obj

Return #t if obj is a, possibly empty, list of bytevectors in which every octet can be interpreted as character in ASCII encoding and the sequence of characters is valid for a segment URI component.

An empty bytevector is allowed only as last component and only if it is preceded by another component; in this case the empty bytevector represents a segment with string representation ‘.’.

Fields
— Immutable virtual field of <list-of-segments>: <ascii-bytevector> bytevector

A bytevector representing the segment components in the format requested by RFC 3986, with slash characters as separators; notice that no leading slash is inserted.

— Immutable virtual field of <list-of-segments>: <ascii-string> string

A string representing the segment components in the format requested by RFC 3986, with slash characters as separators; notice that no leading slash is inserted.

Methods
— Method on <list-of-segments>: put-bytevector (port <binary-output-port>)

Write to port a bytevector representing the segment components in the format requested by RFC 3986.
