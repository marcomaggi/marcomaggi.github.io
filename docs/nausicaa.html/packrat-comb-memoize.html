76.4.5 Memoizing combinators

It is possible to memoize the result of applying a combinator into the <state> record itself; simplifying, it goes like this: 

     (define comb
       (lambda (state)
         ---))
     
     (lambda (state)
       (cond ((memoize-assq 'keyword state) => cdr)
             (else
              (begin0-let ((result (comb state))
                (memoize-acons 'keyword result state)))))))

if a result has been already computed for the state and associated to ‘keyword’ in the internal alist, then that result is returned; else the combinator is applied. We want a method to generate such a combinator with the keyword and combinator as parameters.

— Function: make-memoize-combinator combinator
— Function: make-memoize-combinator combinator keyword

Return a new combinator which first searches keyword between the memoized results of the state:

  • If it is found, return the associated <result> record.
  • Else applies combinator to the state, memoizes the result, then returns it.

keyword is the key associated to the result in the <state> record, it must be a value comparable with eq?; if not supplied, it defaults to an automatically generated, eq? comparable value (actually a sentinel value, The sentinel value).
