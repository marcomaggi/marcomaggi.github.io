77.4.2.2 Low level functions handling library names

The bindings documented in this section are exported by the (nausicaa parser-tools r6rs libraries names low) library.

— Function: library-name? obj

Return #t if obj is a valid library name as specified by R6RS.

— Function: version-numbers? obj

Return #t if obj is a list of version numbers, else return #f, this includes obj being null.

— Function: version-number? obj

Return #t if obj is a version number, else return #f.

— Function: library-name-decompose obj

Interpret obj as a symbolic expression representing a library name as specified by R6RS. Return two values: the list of identifiers, the version specification. If obj is not a valid library name: return #f and #f. The returned values may share structure with obj.

— Function: library-name->identifiers obj

If obj is a valid library name as specified by R6RS, return the list of identifiers; else return #f.

— Function: library-name->version obj

If obj is a valid library name as specified by R6RS, return the version specification; else return #f.

— Function: library-name-identifiers=? name1 name2

Return #t if the arguments are library names having equal list of identifiers; else return #f.

The arguments are validated with assert from (nausicaa).

— Function: library-name=? name1 name2
— Function: library-name<? name1 name2
— Function: library-name<=? name1 name2

Return #t if the arguments are library names having equal list of identifiers and version specifications satisfying the selected predicate; else return #f. Version numbers are compared using the predicates library-version=?, library-version<? and library-version<=?.

The arguments are validated with assert from (nausicaa).

— Function: library-version=? version1 version2
— Function: library-version<? version1 version2
— Function: library-version<=? version1 version2

Return #t if the arguments are library versions satisfying the selected predicate; else return #f.

The arguments are validated with assert from (nausicaa).

If one of the lists is longer and the elements up to the end of the shortest are equal: the lists are “equal” if the tail of the longest is made of zeros.

Examples: 

          (library-version=? '(1 2 3)
                             '(1 2 3))            ⇒ #t
          
          (library-version=? '(1 2 3)
                             '(1 2 3 4))          ⇒ #f
          
          (library-version=? '(1 2 3)
                             '(1 2 3 0 0 0))      ⇒ #t
          
          
          (library-version<? '(1 2 3)
                             '(1 2 3))            ⇒ #f
          
          (library-version<? '(1 2 4)
                             '(1 2 3))            ⇒ #f
          
          (library-version<? '(1 2 3)
                             '(1 2 4))            ⇒ #t
          
          (library-version<? '(1 2 3)
                             '(1 2 3 4))          ⇒ #t
          
          (library-version<? '(1 2 3)
                             '(1 2 3 0 0 0))      ⇒ #f
          
          
          (library-version<=? '(1 2 3)
                              '(1 2 3))           ⇒ #t
          
          (library-version<=? '(1 2 4)
                              '(1 2 3))           ⇒ #f
          
          (library-version<=? '(1 2 3)
                              '(1 2 4))           ⇒ #t
          
          (library-version<=? '(1 2 3)
                              '(1 2 3 4))         ⇒ #t
          
          (library-version<=? '(1 2 3)
                              '(1 2 3 0 0 0))     ⇒ #t
