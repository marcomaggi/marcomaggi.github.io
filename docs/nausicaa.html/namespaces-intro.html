19.1 Introduction to namespaces

Namespaces are syntaxes to be used as “gates” to access a set of bindings; each namespace is referenced by an identifier name which, by loose convention, is surrounded by double angular parentheses <<name>>.

Here is an example showing the basic usage: 

     (library (alpha)
       (export <<alpha>>)
       (import (nausicaa))
       (define (one) 1)
       (define (two) 2)
       (define-namespace <<alpha>>
         (functions one two)))
     
     (library (beta)
       (export three)
       (import (nausicaa) (alpha))
       (define (three)
         (with-namespace ((A <<alpha>>))
           (+ (A.one) (B.two)))))

the syntax with-namespace allows access to the bindings of the namespace <<alpha>> even though they are not exported by the library (alpha); the prefix ‘A’ is a dummy identifier which is bound to nothing, it only provides a local namespace for bindings.

The most interesting usage of namespaces is as means of composition for expressions at expand time. Let's consider this example libraries: 

     (library (alpha)
       (export doit)
       (import (rnrs))
       (define (doit x y z)
         (+ x (* y z))))
     
     (library (beta)
       (export)
       (import (rnrs) (alpha))
       (display (doit 1 2 3)))

if we desire to parametrise the operations + and * we could do the following for run time: 

     (library (alpha)
       (export doit)
       (import (rnrs))
       (define (doit + * x y z)
         (+ x (* y z))))
     
     (library (beta)
       (export)
       (import (rnrs) (alpha))
       (display (doit + * 1 2 3)))

or the following for expand time: 

     (library (alpha)
       (export doit)
       (import (rnrs))
       (define-syntax doit
         (syntax-rules ()
           ((_ ?+ ?* ?x ?y ?z)
            (?+ ?x (?* ?y ?z))))))
     
     (library (beta)
       (export)
       (import (rnrs) (alpha))
       (display (doit + * 1 2 3)))

there are situations in which we do not want to expose details about the parametrised functions at the call site, rather we desire to propose a set of bindings from which the syntax doit picks the needed ones; we can do it with namespaces: 

     (library (alpha)
       (export doit)
       (import (nausicaa))
       (define-syntax doit
         (syntax-rules ()
           ((_ ?x ?y ?z ?operations)
            (with-namespace ((N ?operations))
              (N.+ ?x (N.* ?y ?z)))))))
     
     (library (one)
       (export <<one>>)
       (import (nausicaa))
       (define-namespace <<one>>
         (functions + *)))
     
     (library (two)
       (export <<two>>)
       (import (nausicaa))
       (define-namespace <<two>>
         (functions (+ add)
                    (* mul)))
       (define (add a b)
         ---)
       (define (mul a b)
         ---))
     
     (library (beta)
       (export)
       (import (rnrs) (alpha) (one) (two))
       (display (doit 1 2 3 <<one>>))
       (display (doit 1 2 3 <<two>>)))

expressions are parametrised through the namespace identifier.
