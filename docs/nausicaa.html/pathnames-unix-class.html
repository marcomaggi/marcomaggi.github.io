41.4.2 Class interface

— Class: <absolute-pathname>

Type of absolute Unix pathnames; it is derived from <absolute-pathname> exported by (nausicaa pathnames abstract).

— Class: <relative-pathname>

Type of relative Unix pathnames; it is derived from <relative-pathname> exported by (nausicaa pathnames abstract).

— Common constructor on <absolute-pathname>: constructor obj
— Common constructor on <relative-pathname>: constructor obj

Build and return a new Unix pathname record from obj, which must be a string or bytevector representing the pathname.

An attempt to instantiate <absolute-pathname> from an obj representing a relative pathname or to instantiate <relative-pathname> from an obj representing an absolute pathname results in an assertion violation.

If obj is empty an exception is raised using raise-parser-error from (nausicaa pathnames conditions). After parsing the input is normalised with normalise-pathname from (nausicaa pathnames unix low); if normalisation fails: an exception is raised using raise-normalisation-error from (nausicaa pathnames conditions).

The class fields are initialised with newly built values.

— Maker: make <absolute-pathname> . ?clauses
— Maker: make <relative-pathname> . ?clauses
— Clause: string: ?str
— Clause: bytevector: ?bv
— Clause: segments: ?segs

Build and return a new Unix pathname record. These makers allow the direct specification of the segments, already validated and normalised.

The clauses string:, bytevector: and segments: are mutually exclusive.

When the string: or bytevector: clause is present: this maker reverts to the use of the common constructor.

When the segments: clause is present: ?segs must be a list of bytevectors, already validated and normalised, representing the pathname; the list is not validated.

We must use with care the segments: clause, because no validation is performed on its argument; on the other hand, it is often needed to create a new pathname record by decomposing and recomposing other pathnames and in this case the list of validated and normalised segments is already available.

— Function: pathname obj

Build and return a new Unix pathname record from obj, which must be a string or bytevector representing the pathname. Everything works as documented for the class constructors. In general, we can think of the returned value as an instance of <pathname>.
