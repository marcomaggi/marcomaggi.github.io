34.6 Iterating over bytevectors

The following bindings are exported by the (nausicaa iterators) library.

— Class: <bytevector-iterator>

An iterator for Scheme bytevectors of unsigned bytes; it is derived from <iterator>.

— Immutable field of <string-iterator>: <fixnum> current

Shadows the current field of <iterator> adding the <fixnum> tag.

— Common constructor on <bytevector-iterator>: constructor the-bytevector start past stride

Build a new instance of <bytevector-iterator> using the-bytevector as bytevector upon which to iterate, start as index of the first character, past as index of the character past the last one to return and stride as stride.

start, past and stride must be exact integers. start must be between 0 and one less than the bytevector length. past must be between -1 and the bytevector length. stride cannot be zero and it must have the same sign of the difference past minus start.

The stride is the number of characters in the bytevector to jump forward to when iterating; it must be 1 when we want to visit all the characters.

— Syntax on <bytevector-iterator>: make <bytevector-iterator> . ?clauses
— Auxiliary syntax on <bytevector-iterator>: subject: ?the-bytevector
— Auxiliary syntax on <bytevector-iterator>: start: ?start
— Auxiliary syntax on <bytevector-iterator>: past: ?past
— Auxiliary syntax on <bytevector-iterator>: stride: ?stride

Build and return a new class instance using the common constructor.

If the start: clause is not used: the start index defaults to zero. If the past: clause is not used: the past index defaults to the length of the bytevector. If the stride: clause is not used: the stride defaults to 1.
