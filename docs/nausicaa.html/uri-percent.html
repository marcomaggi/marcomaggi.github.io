70.4 Percent encoding of characters

A bytevector, whose bytes are interpreted as ASCII characters, is valid as URI component only if its characters match the following (nausicaa silex) specification (see silex syntax for the syntax of the specification): 

     ALPHA           [a-zA-Z]
     DIGIT           [0-9]
     HEXDIG          [0-9a-fA-F]
     
     pct-encoded     %{HEXDIG}{HEXDIG}
     
     gen-delims      [:/\?#\[\]@]
     sub-delims      [\!\$&\'()\*\+,\;=]
     
     reserved        {gen-delims}|{sub-delims}
     unreserved      {ALPHA}|{DIGIT}|[\-\._~]

all the characters which are not unreserved are meant to be percent–encoded in a URI string; in particular the following must be encoded: 

     : / ? # [ ]  \ ! $ & ' ( ) * + , ; =

the characters in the unreserved set can, but should not, be encoded.

The following bindings are exported by the (nausicaa uri low) library; it is suggested to prefix them when importing this library.

— Function: unreserved-char? obj

Return true if obj represents an unreserved character according to RFC 3986. obj can be either a character or an integer representing a character according to char->integer.

— Syntax: percent-encode ?obj ?clause ...
— Auxiliary Syntax: char-selector pred
— Auxiliary Syntax: string-result? bool

This syntax is a maker, Maker syntaxes. Percent–encode the given object, return the result; characters outside a selected range are percent–encoded.

obj must be an expression evaluating to any character, or string, or bytevector; the character is converted to a string and the string is converted to a bytevector with string->utf8.

The following clauses are supported:

char-selector pred
pred must be a predicate function which, applied to an integer representing a character according to char->integer, returns true if the character must be percent encoded. The default is to encode all the character not unreserved.
string-result? bool
If bool is true: return a string, else return a bytevector; the default is to return a bytevector.

— Syntax: percent-decode ?obj ?clause ...
— Auxiliary Syntax: string-result? bool

This syntax is a maker, Maker syntaxes. Percent–decode the given obj, return the result.

obj must be an expression evaluating to any string or bytevector; the string is converted to a bytevector with string->utf8.

The following clauses are supported:

string-result? bool
If bool is true: return a string, else return a bytevector; the default is to return a bytevector.

— Function: normalise-percent-encoded-bytevector bv

Normalise the given percent–encoded bytevector; bytes that are encoded but should not are decoded; return the normalised bytevector, in which percent–encoded characters are displayed in upper case. Assume that bv is composed by bytes in the valid range for URIs.

Notice that this function does not percent–encode bytes which are already not so.

— Function: normalise-percent-encoded-string str

Normalise the given percent–encoded string; characters that are encoded but should not are decoded; return the normalised string, in which percent–encoded characters are displayed in upper case. Assume that str is composed by characters in the valid range for URIs.

Notice that this function does not percent–encode characters which are already not so.
