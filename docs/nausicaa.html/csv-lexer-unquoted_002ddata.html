78.1.2 Tokenising unquoted data

The (nausicaa parser-tools csv) library defines unquoted data as everything that is not inside a string. The lexer built around ‘csv-unquoted-data-table’ will return:

  • The symbol ‘eol’ to represent the end of a line. Every sequence of any number of ‘#\newline’ and ‘#\return’ characters, in any order, is collapsed to a single end–of–line.
  • The symbol ‘string’ to represent the start of a string. It is returned when the lexer finds a ‘"’ character.
  • In any other case the next character from the input system.

The ‘csv-unquoted-data-table/comma’ works the same, with the following difference in the return values:

  • The symbol ‘field’ when the token is a comma character.

We can take advantage of this to tokenise unquoted data as follows: 

     #!r6rs
     (import (rnrs)
       (nausicaa parser-tools silex lexer)
       (nausicaa parser-tools csv unquoted-data-lexer))
     
     (define (tokenise string)
       (let* ((IS    (lex.make-IS (lex.string: string)))
              (lexer (lex.make-lexer csv-unquoted-data-table IS)))
         (do ((token (lexer) (lexer))
              (ell   '()))
             ((or (not token) (eq? token 'string))
              (reverse ell))
           (set! ell (cons token ell)))))
     
     (tokenise "alpha, beta")
     ⇒ (#\a #\l #\p #\h #\a #\, #\space #\b #\e #\t #\a)
     
     ; End of line
     (tokenise "alpha\nbeta")
     ⇒ (#\a #\l #\p #\h #\a eol #\b #\e #\t #\a)
     
     ; End of line
     (tokenise "alpha\n\rbeta")
     ⇒ (#\a #\l #\p #\h #\a eol #\b #\e #\t #\a)
     
     ; End of line
     (tokenise "alpha\n\r\n\rbeta")
     ⇒ (#\a #\l #\p #\h #\a eol #\b #\e #\t #\a)
     
     ; Read until the string opening.
     (tokenise "alpha \"beta")
     ⇒ (#\a #\l #\p #\h #\a #\space)
