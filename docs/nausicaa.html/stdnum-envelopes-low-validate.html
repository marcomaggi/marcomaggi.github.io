76.1.4.4 Validating envelopes

The following bindings are exported by the (nausicaa standard-numbers envelopes low) library. For all the functions, the meaning of the common arguments is the following:

check-digit
A fixnum representing the check digit for the given sequence.
value->digit
A function mapping a value extracted from the a sequence to a fixnum digit in the range required by an algorithm; it is applied to the check digit encoded in the sequence.
— Function: left-flap-list-envelope? ell check-digit value->digit

Return #t if ell is a list and it represents a left–flap envelope having check digit equal to check-digit according to fx=?.

— Function: left-flap-string-envelope? str check-digit value->digit

Return #t if str is a string and it represents a left–flap envelope having check digit equal to check-digit according to fx=?.

— Function: right-flap-string-envelope? str check-digit value->digit

Return #t if str is a string and it represents a right–flap envelope having check digit equal to check-digit according to fx=?.

— Function: left-flap-vector-envelope? vec check-digit value->digit

Return #t if vec is a vector and it represents a left–flap envelope having check digit equal to check-digit according to fx=?.

— Function: right-flap-vector-envelope? vec check-digit value->digit

Return #t if vec is a vector and it represents a right–flap envelope having check digit equal to check-digit according to fx=?.

— Function: left-flap-bytevector-u8-envelope? bv check-digit value->digit

Return #t if bv is an unsigned bytes bytevector and it represents a left–flap envelope having check digit equal to check-digit according to fx=?.

— Function: right-flap-bytevector-u8-envelope? bv check-digit value->digit

Return #t if bv is an unsigned bytes bytevector and it represents a right–flap envelope having check digit equal to check-digit according to fx=?.
