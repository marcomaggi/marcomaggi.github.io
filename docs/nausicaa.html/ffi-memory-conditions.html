30.2.1 Out of memory errors

The following bindings are exported by (nausicaa ffi memory conditions) and re–exported by (nausicaa ffi memory).

— Condition: &out-of-memory

Type of condition objects representing a memory allocation error; it is derived from &error. It has the following fields:

number-of-bytes
An exact non–negative integer representing the number of bytes requested in the failed memory allocation; it can be set to #f if the number is not known (for example when the error is raised by a foreign library).

This condition type is for failed memory allocations not meant to be recoverable by an exception handler.

— Condition: &memory-request

Type of condition objects representing a memory allocation error; it is derived from &out-of-memory. It has the following fields:

clean
A boolean value, #t if the requested memory was meant to be reset to zero.

This condition type is for failed memory allocations meant to be recoverable by an exception handler.

— Condition Constructor: make-out-of-memory-condition number-of-bytes
— Condition Predicate: out-of-memory-condition? object
— Field Accessor: condition-out-of-memory/number-of-bytes object

Functions to handle non–recoverable out of memory condition objects. number-of-bytes is the size of the requested allocation that could not be satisfied.

— Condition Constructor: make-memory-request-condition number-of-bytes clean
— Condition Predicate: memory-request-condition? object
— Field Accessor: condition-memory-request/number-of-bytes object
— Field Accessor: condition-memory-request/clean? object

Functions to handle recoverable out of memory condition objects. number-of-bytes is the size of the requested allocation that could not be satisfied. clean must be a boolean value: #t if the requested memory was meant to be reset to zero.

Examples: 

     (define (malloc size)
       (or (primitive-malloc size)
           (raise
              (condition (make-who-condition 'malloc)
                         (make-message-condition "out of memory")
                         (make-out-of-memory-condition size)))))
     
     (define (malloc* size)
       (or (primitive-malloc size)
           (raise-continuable
              (condition (make-who-condition 'malloc)
                         (make-message-condition "out of memory")
                         (make-memory-request-condition size #f)))))
     
     (define (calloc* size)
       (or (primitive-calloc size)
           (raise-continuable
              (condition (make-who-condition 'malloc)
                         (make-message-condition "out of memory")
                         (make-memory-request-condition size #t)))))
— Function: raise-out-of-memory who number-of-bytes

Raise a non–continuable exception representing a non–recoverable memory allocation failure. The raised condition object is a compound of &who, &message and &out-of-memory.

— Function: raise-memory-request who number-of-bytes clean

Raise a continuable exception representing a recoverable memory allocation failure. The raised condition object is a compound of &who, &message and &memory-request. Functions raising this condition should be ready to accept as return value the requested block of memory (allocated by the exception handler).
