51.1 Plain matching

Let's start with some example of sexp matching: 

     (let ((pattern '(a 1 #t))
           (form    '(a 1 #t))
       (sexp-match pattern form))
     ⇒ ()
     
     (sexp-match '(a b) '(a c))
     error--> mismatch between B and C
     
     (sexp-match `(a ,(sexp-pred integer?))
                 '(a 1))
     ⇒ ()
     
     (sexp-match `(a ,(sexp-pred integer?))
                 '(a b))
     error--> mismatch between INTEGER? and B

the (sexps) library defines several matching operators, and custom ones can be easily coded. Matching operators

— Function: sexp-match pattern form

Recursively match elements in the sexp template pattern with elements in the sexp form. Return an alist whose entries represent the bindings between variables from pattern and values from form, or null if no bindings where defined.

If a mismatch between pattern and form is detected, an error is raised with condition of type &sexp-mismatch.

pattern must be a sexp template with the following constraints and semantics meaning:

  • pattern must be null or a proper list. If it is null, it can match only a null form.
  • If an element of pattern is a pair, it must be the first pair in a proper list; the list is interpreted as a nested sexp pattern.
  • If an element of pattern is a procedure, it must be a matching operator. Matching operators
  • Every other element is compared to the elements of form using equal?.

form must be a sexp whose elements have the following constraints and semantics meaning:

  • form must be null or a proper list. If it is null, it can be matched both by a null pattern and by a pattern holding an appropriate matching operator.
  • If an element of form is a pair, it is not mandatory for it to be the first pair in a proper list. A proper list in form can be matched by a proper list in pattern. In general, whether a pair in form will be matched or not depends on the operators in pattern.
  • Every other element is compared to the elements of pattern using equal?. Notice that the behaviour of equal? with respect to procedures is undefined in R6RS; it appears there is no safe way to match a procedure in form.

— Function: sexp-match? pattern form

Like sexp-match but return #t when successfully matches and #f otherwise.

— Syntax: sexp-mismatch-error who pattern form

Raise a non–continuable exception with conditions of type &sexp-mismatch, &message, &who. All the arguments are evaluated only once.

— Condition Type: &sexp-mismatch

Type of condition used to signal a sexp mismatch. It is derived from &mismatch.

— Function: make-sexp-mismatch-condition pattern-token form-token

Return a new condition object of type &sexp-mismatch. pattern-token must be a value representing the portion of sexp pattern which failed to match. form-token must be a value representing the portion of sexp form which failed to match.

— Function: sexp-mismatch-condition? obj

Return #t if obj is a condition object of type &sexp-mismatch.

— Function: condition-sexp-mismatch/pattern obj
— Function: condition-sexp-mismatch/form obj

Accessors for the fields of a &sexp-mismatch condition object.
