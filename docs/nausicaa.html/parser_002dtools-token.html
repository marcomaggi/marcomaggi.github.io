61.2 Lexical token records

Lexical token records are meant to store informations about a token from the input stream; they should be produced by the lexer and consumed by the parser. For example, when the lexer tokenises the number 123 from the string ‘123’, it can build a record with: 

     (let ((position ---))
       (make* <lexical-token> 'NUMBER position 123 3))

while when tokenising a string it can do: 

     (let ((position ---))
       (make* <lexical-token> 'STRING position "some text" 9))

When the lexer has to signal the end–of–input, it must return a token with category ‘*eoi*’, and continue to return it if it is called again and again; it can do: 

     (let ((position ---))
       (make* <lexical-token> '*eoi* position (eof-object)))

When the lexer closure has to signal a lexer error (an invalid character from the input stream), it must return a token with category ‘*lexer-error*’.

The following bindings are exported by the (nausicaa parser-tools lexical-token) library; this library is built on top of (nausicaa language classes).

— Class: <lexical-token>

Hold the informations about a token from the input stream.

— Immutable Field of <lexical-token>: category

Represents the grammar's terminal symbol to which this token belongs. Every parser API must provide an interface to specify the list of allowed terminal symbols.

— Immutable Field of <lexical-token>: location

Represents the location in the input from which the token was built; it must be #f or an instance of <source-location> record.

— Immutable Field of <lexical-token>: value

Represents the semantic value associated with the token. It can be any Scheme value.

— Immutable Field of <lexical-token>: length

It is meant to be the length of the string which generated the token. When the token represents the end–of–input, this field can be set to zero.

— Immutable Virtual Field of <lexical-token>: end-of-input?

True if the token has category ‘*eoi*’.

— Immutable Virtual Field of <lexical-token>: lexer-error?

True if the token has category ‘*lexer-error*’.

— Immutable Virtual Field of <lexical-token>: special?

True if the token has category ‘*eoi*’ or ‘*lexer-error*’.

— Syntax: make* <lexical-token> category location value length

Build and return a new record of type <lexical-token>.

— Syntax: make <lexical-token> clause ...
— Auxiliary Syntax: category: category
— Auxiliary Syntax: location: location
— Auxiliary Syntax: value: value
— Auxiliary Syntax: length: length

Build and return a new record of type <lexical-token>. The category: clause is mandatory; location defaults to #f; value defaults to #f; length defaults to zero.

Traditional records API
— Function: make-<lexical-token> category location value length

Build and return a new record of type <lexical-token>.

— Function: make-<lexical-token>/end-of-input location

Build and return a new record of type <lexical-token> meant to represent the end–of–input. The field ‘category’ is set to the symbol ‘*eoi*’, the field ‘value’ is set to ‘(eof-object)’, the field ‘length’ is set to zero. The field ‘location’ is set to location, which must be a <source-location> record or #f.

— Function: make-<lexical-token>/lexer-error location value length

Build and return a new record of type <lexical-token> meant to represent a lexer error. The field ‘category’ is set to ‘*lexer-error*’, all the other fields are initialised with the arguments.

— Function: <lexical-token>? obj

Return true if obj is a record of type of <lexical-token>.

— Function: <lexical-token>? obj

Return true if obj is an instance of <lexical-token>.

— Function: <lexical-token>?/end-of-input obj

Return true if obj is an instance of <lexical-token> having category eq? to the symbol ‘*eoi*’.

— Function: <lexical-token>?/lexer-error obj

Return true if obj is an instance of <lexical-token> having category eq? to the symbol ‘*lexer-error*’.

— Function: <lexical-token>?/special obj

Return true if obj is an instance of <lexical-token> having category eq? to the symbol ‘*eoi*’ or ‘*lexer-error*’.

— Function: <lexical-token>-category tok
— Function: <lexical-token>-location tok
— Function: <lexical-token>-value tok
— Function: <lexical-token>-length tok

Accessors for the fields of <lexical-token> records.
