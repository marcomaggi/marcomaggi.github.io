31.1.3 Interface to C language types

Nausicaa's FFI accepts a number of C language type identifiers represented by Scheme symbols; these external types have redundant names to improve user friendlyness; when processing the definition of a binding to foreign functions, they are first normalised to internal types as described in the table below.

The <size_t-integer>, <ssize_t-integer>, <intptr_t-integer> and <uintptr_t-integer> type specifiers used in the tables, are the unsigned and signed identifiers used on the underlying platform to represent the size_t, ssize_t, intptr_t and uintptr_t C language data types; they are symbols among: 

     signed-int    signed-long    signed-long-long
     unsigned-int  unsigned-long  unsigned-long-long

We have to remember that C strings (zero terminated arrays of ‘char’) must be converted from/to Scheme strings with string->cstring and cstring->string from the (nausicaa ffi cstrings) library (or equivalent), and exchanged with foreign functions as pointer objects. The following bindings are exported by the (nausicaa ffi clang type-translation) library.

— Enumeration Type: enum-clang-external-types
— Enumeration Type: enum-clang-internal-types
— Enumeration Type: enum-clang-types

Enumeration type for the external, internal or implementation type symbols.

— Syntax: clang-external-types <symbol> ...
— Syntax: clang-internal-types <symbol> ...
— Syntax: clang-types <symbol> ...

Evaluate to an enumeration set of external, internal or implementation type symbols.

— Function: clang-external-type->clang-internal-type type-symbol
— Function: clang-internal-type->clang-type type-symbol
— Function: clang-external-type->clang-type type-symbol

Convert between type symbols of different sets; if the symbol is not recognised, an assertion violation is raised.

— Function: clang-foreign-type->clang-external-type type-symbol

Return type-symbol itself if it is a Scheme symbol in the enumeration enum-clang-external-types, otherwise return #f.

— Function: clang-maybe-foreign-type->clang-external-type type-symbol

Return type-symbol itself; this is a special function which can be used by the macros make-c-function* and similar, see their documentation for details.
