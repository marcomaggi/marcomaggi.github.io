22.7.2 Local bindings

The following macros integrate the facilities of the with-class macro with the binding facilities of the let, let*, letrec and letrec* syntaxes from (rnrs base (6)) and the do syntax from (rnrs control (6)).

NOTE It is possible to use the facilities of the (nausicaa contracts) library to assert that the type of values involved in the bindings is the one selected by the class identifiers. Merging the features of contracts along with the bindings–with–classes macros may result in inflexible code or bloated code after the expansion; so, for the time being, this merging is not implemented.
— Macro: let/with-class ((var init) ...) . body

Like let but automatically enclose the body in a with-class syntax using the specified class types. Each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names.

If all the var arguments are in the first form (untyped variable): let/with-class defaults to let as defined by R6RS. If at least one var has type: the untyped ones default to bindings with <top> as conventional class name. 

          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (immutable numerator)
                            (immutable denominator)))
          
          (define (<fraction>-numerator o)
            (numerator (<fraction>-number o)))
          
          (define (<fraction>-denominator o)
            (denominator (<fraction>-number o)))
          
          (let/with-class (((a <fraction>) (make <fraction> 2/3))
                           ((b <fraction>) (make <fraction> 4/5)))
            (list a.numerator b.denominator))
          ⇒ (2 5)
— Macro: let*/with-class ((var init) ...) . body

Like let* but automatically enclose the body in a with-class syntax using the specified class types. Each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names.

If all the var arguments are in the first form (untyped variable): let*/with-class defaults to let* as defined by R6RS. If at least one var has type: the untyped ones default to bindings with <top> as conventional class name.

Given two bindings var1, init1 and var2, init2 defined in sequence: init2 is evaluated in the region of the identifier syntaxes used to access the fields of var1. 

          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (immutable numerator)
                            (immutable denominator)))
          
          (define (<fraction>-numerator o)
            (numerator (<fraction>-number o)))
          
          (define (<fraction>-denominator o)
            (denominator (<fraction>-number o)))
          
          (let*/with-class (((a <fraction>) (make <fraction> 2/3))
                            ((b <fraction>) (make <fraction>
                                              (/ a.numerator 5))))
            b.number)
          ⇒ 2/5
— Macro: letrec/with-class ((var init) ...) . body

Like letrec but automatically enclose the body in a with-class syntax using the specified class types. Each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names.

If all the var arguments are in the first form (untyped variable): letrec/with-class defaults to letrec as defined by R6RS. If at least one var has type: the untyped ones default to bindings with <top> as conventional class name.

The init expressions can contain references to the identifier syntaxes, as long as they are not resolved. 

          (define-class <alpha>
            (fields (immutable value)
                    (immutable proc)))
          
          (define-class <beta>
            (fields (immutable value)
                    (immutable proc)))
          
          (letrec/with-class
               (((a <alpha>) (make <alpha>
                               1 (lambda ()
                                   (cons a.value b.value))))
                ((b <beta>)  (make <beta>
                               2 (lambda ()
                                   (cons a.value b.value)))))
            (list (a.proc) (b.proc)))
          ⇒ ((1 . 2) (1 . 2))
— Macro: letrec*/with-class ((var init) ...) . body

Like letrec* but automatically enclose the body in a with-class syntax using the specified class types. Each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names.

If all the var arguments are in the first form (untyped variable): letrec*/with-class defaults to letrec* as defined by R6RS. If at least one var has type: the untyped ones default to bindings with <top> as conventional class name.

The init expressions can contain references to the identifier syntaxes, as long as they are not resolved.

The only difference between letrec/with-class and letrec*/with-class is that the latter enforces the order of evaluation of the init expressions. 

          (define-class <alpha>
            (fields (immutable value)
                    (immutable proc)))
          
          (define-class <beta>
            (fields (immutable value)
                    (immutable proc)))
          
          (letrec*/with-class
              (((a <alpha>) (make <alpha>
                              1 (lambda ()
                                  (cons a.value b.value))))
               ((b <beta>)  (make <beta>
                              2 (lambda ()
                                  (cons a.value b.value))))
               ((c <top>)   (list (a.proc) (b.proc))))
            c)
          ⇒ ((1 . 2) (1 . 2))
— Macro: do/with-class ((var init step) ...) (test expression ...) command ...

Like do from (rnrs control (6)), but each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names. The step, expression and command expressions can use dot notation with id.

— Macro: do*/with-class ((var init step) ...) (test expression ...) command ...

Like do* from (nausicaa language extensions), but each var element can be one among: 

          id
          (id class ...)

in which: id must be an identifier representing the variable name, class must be identifiers representing class names. The step, expression and command expressions can use dot notation with id. When multiple variables are defined, each init expression can use dot notation with the previous bindings.

— Syntax: let-make ((var class arg ...) ...) body0 body ...
— Syntax: let-make* ((var class arg ...) ...) body0 body ...
— Syntax: let*-make ((var class arg ...) ...) body0 body ...
— Syntax: let*-make* ((var class arg ...) ...) body0 body ...
— Syntax: letrec-make ((var class arg ...) ...) body0 body ...
— Syntax: letrec-make* ((var class arg ...) ...) body0 body ...
— Syntax: letrec*-make ((var class arg ...) ...) body0 body ...
— Syntax: letrec*-make* ((var class arg ...) ...) body0 body ...

let like syntaxes to make it short to build a new class instance and bind it with a class tag; for example, in the following: 

          (define-class <alpha>
            (fields a b))
          
          (let-make ((o <alpha> 1 2))
            (list o.a o.b))

the use of let-make expands to: 

          (let/with-class (((o <alpha>) (make <alpha> 1 2)))
            (list o.a o.b))

All the syntaxes with -make as suffix expand to uses of make; all the syntaxes with -make* as suffix expand to uses of make*.
