58.1.1 Handling the parser state

The parser state is represented by a linked list of <state> records, terminated by a thunk which, if evaluated, returns the next <state> record in the list, already chained. The list is not a Scheme list. 

      -----------
     |  <state>  |
     |           |
     | lookahead +--> #<first lexical-token>
     |           |
     |   next    |
      -----+-----
           |
           v
      -----------
     |  <state>  |
     |           |
     | lookahead +--> #<second lexical-token>
     |           |
     |    next   |
      -----------
           |
           v
      -----------
     |  <state>  |
     |           |
     | lookahead +--> #<third lexical-token>
     |           |
     |    next   |
      -----------
           |
           v
      #<procedure>

Each <state> record represents the sequence of subsequent tokens drawn from the lexer. By keeping a referenct to a <state> record, we can reparse the same sequence any number of times; this is used to backtrack from a state to a previous one.
