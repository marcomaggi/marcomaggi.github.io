67.2.1 Type definitions

— Class: <library-name>

Class type representing library names as specified by R6RS.

— Immutable Field of <library-name>: identifiers

List of identifiers in the library reference specification.

— Immutable Field of <library-name>: version

The version specification as a, possibly null, list of exact, non–negative, integers.

— Function: make-<library-name> sexp

Parse sexp as an symbolic expression representing a library name as specified by R6RS; build and return a new <library-name> object representing it. If sexp is not valid: raise an assertion violation.

— Function: <library-name>? obj

Return #t if obj is an object of type <library-name>.

— Function: <library-name>-identifiers name
— Function: <library-name>-version name

Accessors for the fields of <library-name> objects.

— Method on <library-name>: <boolean> identifiers=? (name <library-name>)

Return #t if the arguments are library names having equal list of identifiers; else return #f.

— Method on <library-name>: <boolean> = (name <library-name>)
— Method on <library-name>: <boolean> < (name <library-name>)
— Method on <library-name>: <boolean> <= (name <library-name>)

Return #t if the arguments have equal list of identifiers and version specifications satisfying the selected predicate; else return #f.

Examples: 

     (let ((a (make-<library-name> '(x y (1 2))))
           (b (make-<library-name> '(x y (1 3))))
           (c (make-<library-name> '(x y (1 4))))
           (d (make-<library-name> '(x y (2 1)))))
     
       (list-sort (lambda/with ((a <library-name>) (b <library-name>))
                    (a.< b))
         (list a d c b))
       ⇒ (list a b c d)
     
       )
