5.24.1 Hashable and properties clauses

— Mixin: <hashable-and-properties-clauses>

Mixin type implementing hash function and properties list for classes; this mixin cannot be used by labels. When importing this mixin we must specialise the identifier <class> to the importing class type, as follows: 

          (define-class <mine>
            (mixins (<hashable-and-properties-clauses>
                     (<class>     <mine>))))

This mixin appends a single concrete field to the enclosing class; such field must be initialised to #f.

— Mutable field of <hashable-and-properties-clauses>: <symbol> %uid

Contains #f or a gensym. This field should never be accessed directly.

— Virtual immutable field of <hashable-and-properties-clauses>: <symbol> uid

Contains gensym. The symbol is unique for the instance object.

— Virtual immutable field of <hashable-and-properties-clauses>: <exact-integer> hash

An exact integer to be used as hashtable key for the instance. Hashtables having a <hashable-and-properties-clauses> as key can be instantiated as follows: 

          (make-hashtable
            (lambda ((V <hashable-and-properties-clauses>))
              (V hash))
            eq?)
Object properties
— Method on <hashable-and-properties-clauses>: putprop key value

Add a new property key to the property list of the instance; key must be a symbol. If key is already set: the old entry is mutated to reference the new value.

— Method on <hashable-and-properties-clauses>: getprop key

Return the value of the property key in the property list of the instance; if key is not set: return #f. key must be a symbol.

— Method on <hashable-and-properties-clauses>: remprop key

Remove the property key from the property list of the instance; if key is not set: nothing happens. key must be a symbol.

— Method on <hashable-and-properties-clauses>: property-list

Return a new association list representing the property list of the instance. The order of the entries is the same as the property creation order.
