64.3.1 The parser state

— Record Type: <state>

Represents the state of the parser at a specific point in the input stream. Records of this type are the argument for the combinator functions. Fields description follows:

‘lookahead’
A <lexical-token> record from the input stream. It is the first token examined by combinators. This field is immutable.
‘next-state’
A <state> record or a thunk returning a <state> record. It represents the next state generated by parsing the token in the ‘lookahead’ field. To generate a new state we must do: 
               (let* ((state ---)
                      (next  (<state>-next-state state)))
                 (if (<state>? next)
                     next
                   (next)))

or just call next-state.

‘memoized-results’
An alist mapping keywords to <result> records; it represents the result of applying a combinator to this record. Each memoizing combinator must have a unique symbol associated to it, to be used as keyword for this alist.

Only combinators generated by make-memoize-combinator access this field.

— Function: <state>? obj

Return #t if obj is a record of type <state>.

— Function: <state>-lookahead state
— Function: <state>-next-state state
— Function: <state>-memoized-results state

Field accessors.

— Function: initialise-state lexer

Return a <state> record representing the whole input stream generated by lexer. lexer must be a thunk returning <lexical-token> records.

— Function: next-state state

Return a <state> record representing the input stream after the token in state, which must be a <state> record.

— Function: pushback token state

Return a <state> record representing the input stream before the token in state, which must be a <state> record. It can be used to fake reading tokens from the input.

— Syntax: memoize-acons keyword result state

Memoize the <result> record result in the <state> record state, associating it to keyword which must be a value comparable with eq?.

— Syntax: memoize-assq keyword state

Search the memoized result associated to keyword in the <state> record state; return #f if not found, else a pair whose car is keyword and whose cdr is a <success> or <error> record.

— Syntax: memoize-set! keyword result state

Search the memoized result associated to keyword in the <state> record state, then store result in the cdr of the entry. It is an error if the state has no memoized result associated to keyword.
