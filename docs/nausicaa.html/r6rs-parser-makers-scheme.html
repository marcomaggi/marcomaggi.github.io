74.2.4.2 Scheme value datum makers

The following bindings are exported by the (nausicaa r6rs parser) library.

— Function: make-identifier-datum yypushback yycustom value

Default function for the identifier-datum-maker parameter. Accept as value the string representing the identifier; return the corresponding symbol.

— Function: make-boolean-datum yypushback yycustom value

Default function for the boolean-datum-maker parameter. Accept as value the boolean value itself; return the boolean value itself.

— Function: make-number-datum yypushback yycustom value

Default function for the number-datum-maker parameter. Accept as value the number itself; return the number itself.

— Function: make-string-datum yypushback yycustom value

Default function for the string-datum-maker parameter. Accept as value the string itself; return the string itself.

— Function: make-character-datum yypushback yycustom value

Default function for the character-datum-maker parameter. Accept as value the character itself; return the character itself.

— Function: make-pair-datum yypushback yycustom car cdr

Default function for the identifier-datum-maker parameter. Accept as car and cdr the car and cdr datums; return the corresponding pair.

— Function: make-list-datum yypushback yycustom value

Default function for the list-datum-maker parameter. Accept as value the list of datums; return the list itself.

— Function: make-vector-datum yypushback yycustom value

Default function for the vector-datum-maker parameter. Accept as value the list of datums; return the corresponding vector.

— Function: make-bytevector-datum yypushback yycustom value

Default function for the bytevector-datum-maker parameter. Accept as value the list of datums; return the corresponding bytevector.

If one of the datums is not an exact integer in the range [0, 255] an exception is raised with condition object having components: &lexical, &who, &message, &irritants. The single element in the irritants list is the offending value.

Instances of <interlexeme-space> are removed from the list before building the bytevector (see <interlexeme-space>).
