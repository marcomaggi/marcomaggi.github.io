10.1 Unwrapping syntax objects

— Function: unwrap-syntax-object stx
— Function: unwrap-syntax-object stx options

Given a syntax object stx decompose it and return the corresponding symbolic expression holding datums and identifiers. Take care of returning a proper list when the input is a syntax object holding a proper list.

options must be an enumeration set of type enum-unwrap-option; it drives this function as follows:

keep-general-quoted
Do not unwrap pairs whose car is one of the identifiers: quote, quasiquote, syntax, quasisyntax. Such pairs are left untouched as wrapped syntax objects.
This functions also provides a workaround for bugs in Ikarus (Vicare) and Mosh, which expand syntax objects holding a list into IMproper lists.

— Enumeration Type: enum-unwrap-options

Enumeration for the options of unwrap-syntax-object; the included symbols are: 

          keep-general-quoted
— Macro: unwrap-options symbol ...

Expand into an enumeration set of type enum-unwrap-options holding the given symbol arguments.

— Function: syntax->list stx
— Function: syntax->list stx tail

Given a syntax object stx holding a list, return a new proper list holding the component syntax objects. tail must be null or a proper list which will become the tail of the returned list; tail defaults to null.

— Function: syntax->vector stx

Unwrap the syntax object stx holding a Scheme vector; return the unwrapped vector. If stx does not hold a vector: raise a syntax violation exception.
