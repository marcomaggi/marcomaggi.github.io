57.1.1 Type, makers, accessors

— Class: <source-location>

References the position of the next character to parse in the input stream, which is the position of the first char in the next token produced by the lexer.

— Immutable Field of <source-location>: input

Describes the source of the input. It can be any Scheme value, for example a string representing the pathname of an input file. Set to #f if the source is unknown.

— Immutable Field of <source-location>: line
— Immutable Field of <source-location>: column
— Immutable Field of <source-location>: offset

Counters for the line number, column number and character offset from the start of the input. By convention: Line and column numbers start from 1, the offset starts from 0.

When the value of these counters is not known, by convention, they should be set to #f, so that they are no more numbers.

Notice that these values match the local bindings ‘yyline’, ‘yycolumn’ and ‘yyoffset’ available to the lexer actions of the (nausicaa silex) library. Evaluation of the actions

— Immutable Virtual Field of <source-location>: line-string
— Immutable Virtual Field of <source-location>: column-string
— Immutable Virtual Field of <source-location>: offset-string

Strings representing the counters for the line number, column number and character offset from the start of the input.

— Function: <source-location>-input loc
— Function: <source-location>-line loc
— Function: <source-location>-column loc
— Function: <source-location>-offset loc

Accessors for the fields of a <source-location> record.
