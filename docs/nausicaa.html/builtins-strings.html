5.11 String type

— Builtin Label: <string>

Conventional type for immutable objects satisfying string?.

— Builtin Label: <mutable-string>

Conventional type for mutable objects satisfying string?. It is a sublabel of <string>.

— Builtin Label: <ascii-string>

Conventional type for mutable objects satisfying string? and ascii-encoded-string?. It is a sublabel of <string>.

— Builtin Label: <latin1-string>

Conventional type for mutable objects satisfying string? and latin1-string?. It is a sublabel of <string>.

— Builtin Label: <percent-encoded-string>

Conventional type for mutable objects satisfying string? and percent-encoded-string?. It is a sublabel of <string>.

Constructors and makers
— Common constructor on <string>: <string> constructor ch ...
— Common constructor on <mutable-string>: <mutable-string> constructor ch ...

Return the result of applying string to the ch arguments.

Safe accessors
— Immutable virtual field of <string>: <fixnum> length

Accessing this field is like applying string-length to the string.

— Immutable virtual field of <string>: <boolean> empty?

It is #t if the string length is zero.

— Immutable virtual field of <string>: <string> upcase

Accessing this field is like applying string-upcase to the string.

— Immutable virtual field of <string>: <string> downcase

Accessing this field is like applying string-downcase to the string.

— Immutable virtual field of <string>: <string> titlecase

Accessing this field is like applying string-titlecase to the string.

— Immutable virtual field of <string>: <string> foldcase

Accessing this field is like applying string-foldcase to the string.

— Immutable virtual field of <string>: <bytevector-u8> utf8

Accessing this field is like applying string->utf8 to the string, Operations on strings.

— Immutable virtual field of <string>: <bytevector-u16n> utf16
— Immutable virtual field of <string>: <bytevector-u16n> utf16 endianness

Accessing this field is like applying string->utf16 to the string, Operations on strings.

— Immutable virtual field of <string>: <bytevector-u16l> utf16le
— Immutable virtual field of <string>: <bytevector-u16b> utf16be
— Immutable virtual field of <string>: <bytevector-u16n> utf16n

Accessing these fields is like applying string->utf16le, string->utf16be, string->utf16n to the string, Additional string functions.

— Immutable virtual field of <string>: <bytevector-u32> utf32
— Immutable virtual field of <string>: <bytevector-u32> utf32 endianness

Accessing this field is like applying string->utf32 to the string, Operations on strings.

— Immutable virtual field of <string>: <ascii-bytevector> ascii
— Immutable virtual field of <string>: <ascii-bytevector> $ascii

Accessing this field is like applying string->ascii to the string, Additional string functions.

— Immutable virtual field of <string>: <ascii-bytevector> latin1
— Immutable virtual field of <string>: <ascii-bytevector> $latin1

Accessing this field is like applying string->latin1 to the string, Additional string functions.

— Immutable virtual field of <string>: <percent-encoded-bytevector> percent-encoding

Accessing this field is like applying string->percent-encoding to the string, Additional string functions.

Unsafe accessors
— Immutable virtual field of <string>: <fixnum> $length

Return the result of applying $string-length from (vicare system $strings) to the string.

— Immutable virtual field of <string>: <boolean> $empty?

Return #t if the string length is zero. Make use of unsafe operations from (vicare system $strings).

Methods
— Method on <string>: <integer> hash

Return the result of applying string-hash to the string instance.

— Method on <string>: <string> substring (begin <fixnum>)
— Method on <string>: <string> substring (begin <fixnum>) (end <fixnum>)

Build and return a new string holding the selected substring of the subject. When end is not used: it defaults to the length of the string.

When begin and end are zero or positive: this method behaves like substring from (rnrs base (6)). If either begin and/or end are negative: they are first normalised with: 

          (+ begin/end (string-length string))

so that negative indices are counted from the end of the string: -1 selects the ultimate character, -2 selects the penultimate character and so on.

— Method on <string>: <string> append (string <string>) ...

Build and return a new string whose characters form the concatenation of the subject and given strings.

— Method on <string>: <list> list

Return a list holding all the characters in the string.

— Method on <string>: for-each proc

Apply the function proc to each character in the string.

— Method on <string>: <string> copy

Build and return a new string holding the same characters.

— Method on <string>: <char> ref (index <fixnum>)

When index is zero or positive: this method behaves like string-ref from (rnrs base (6)). If index is negative: it is first normalised with: 

          (+ index (string-length string))

so that negative indices are counted from the end of the string: -1 selects the ultimate character, -2 selects the penultimate character and so on.

— Method on <string>: <top> ref (index <fixnum>) (ch <char>)

When index is zero or positive: this method behaves like string-set! from (rnrs base (6)). If index is negative: it is first normalised with: 

          (+ index (string-length string))

so that negative indices are counted from the end of the string: -1 selects the ultimate character, -2 selects the penultimate character and so on.

OOPP syntaxes
— Maker on <string>: (?char ...)
— Maker on <mutable-string>: (?char ...)

Return the result of applying string to the results of evaluating the expressions ?char.

— Getter on <string>: instance (index)

Character getter; it behaves like the ref method.

— Setter on <mutable-string>: set!/tags (instance (index)) ?char

Character setter; it behaves like the set! method.
