28.1.9.8 Determining the offset of fields in data structures

It is often needed to know the offset of a field in a C language data structure. From a pointer to the first byte of a data structure and the offset of one of its fields, we can compute the pointer to the first byte of the fields by adding the two.

The following determines the offset of the field ‘l_start’ in the structure ‘struct flock’ (defined in fcntl.h) and stores it in the output variable ‘OFFSETOF_STRUCT_FLOCK_L_START’: 

     NAUSICAA_OFFSETOF_FIELD_TEST([OFFSETOF_STRUCT_FLOCK_L_START],
        [struct flock],
        [l_start])

the following does the same for the ‘l_whence’ field and stores it in the output variable ‘OFFSETOF_STRUCT_FLOCK_L_WHENCE’: 

     NAUSICAA_OFFSETOF_FIELD_TEST([OFFSETOF_STRUCT_FLOCK_L_WHENCE],
        [struct flock],
        [l_whence])
— Autoconf Macro: NAUSICAA_OFFSETOF_FIELD_TEST (suffix, struct, field, headers)

A wrapper for the Autoconf macro ‘AC_COMPUTE_INT’ that acquires the offset of a field in a C data structure.

struct is the type of the structure, field is the name of the field.

The output variable ‘OFFSETOF_<SUFFIX>’ is set to the result. If the test fails the value of the output variable will be #f.

The optional headers must be a chunk of code including header files. By default the expansion of ‘NAUSICAA_INCLUDES_DEFAULT’ is included in the test program.
