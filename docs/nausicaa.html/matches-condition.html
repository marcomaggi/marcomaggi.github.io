52.2 Mismatch error condition

When the expansion of match, or one of its derivatives, detects a mismatch between the expression and the patterns in all the clauses, an error is raised with condition type &match-mismatch.

— Condition Type: &match-mismatch

Raised to signal an expression/pattern mismatch while evaluating the expansion of match; it is derived from &mismatch from the (nausicaa langugae conditions) library. There is only one field, ‘expression’, which holds the mismatched expression.

— Function: make-match-mismatch-condition expression

Build and return a condition of type &match-mismatch.

— Function: match-mismatch-condition? obj

Return #t if obj is a condition of type &match-mismatch.

— Function: condition-match-mismatch/expression obj

Accessor for the ‘expression’ field of a &match-mismatch condition object.

— Syntax: match-mismatch-error <who> <expression>

Raise a non–continuable exception with condition types: &match-mismatch, &message, &who. All the arguments are evaluated only once.
