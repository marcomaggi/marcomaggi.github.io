12.5.5 Utility URI parser functions

The library (nausicaa parser-tools uri utilities) implements utility functions to parse URI components using the facilities of the library (nausicaa parser-tools uri) and represent the results with objects of the types defined by the library (nausicaa uri).

For all the functions: if an error occurs parsing the input, an exception is raised with compound condition object of types: &procedure-argument-violation, &who, &message, &irritants.

— Function: string->uri str

Parse the string str expecting it to represent a URI component. If successful return an instance of <uri> defined by (nausicaa uri).

— Function: bytevector->uri bv

Parse the bytevector bv expecting it to represent a URI component. If successful return an instance of <uri> defined by (nausicaa uri).

— Function: read-uri port

Read octets from the binary input port port expecting them to represent a URI component. If successful return an instance of <uri> defined by (nausicaa uri).

— Function: string->relative-ref str

Parse the string str expecting it to represent a relative-ref component. If successful return an instance of <relative-ref> defined by (nausicaa uri).

— Function: bytevector->relative-ref bv

Parse the bytevector bv expecting it to represent a relative-ref component. If successful return an instance of <relative-ref> defined by (nausicaa uri).

— Function: read-relative-ref port

Read octets from the binary input port port expecting them to represent a relative-ref component. If successful return an instance of <relative-ref> defined by (nausicaa uri).
