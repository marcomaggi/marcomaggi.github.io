73.1.3.4 Scheme–semantic–value token–makers

The following bindings are exported by the library (nausicaa r6rs lexer).

— Function: make-identifier-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing an identifier. The category is set to the symbol IDENTIFIER; the semantic value is set to the string representing the identifier.

— Function: make-boolean-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a boolean value. The category is set to the symbol BOOLEAN; the semantic value is set to the boolean value.

— Function: make-named-character-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a named character, for example ‘#\newline’. The category is set to the symbol CHARACTER; the semantic value is set to the character itself.

If the yytext string does not represent a named character compliant with R6RS: an assertion violation is raised. This should not happen if the lexer table is correct.

— Function: make-hex-character-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a hex character, for example ‘#\x005C’. The category is set to the symbol CHARACTER; the semantic value is set to the character itself.

— Function: make-literal-character-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a literal character, for example ‘#\A’. The category is set to the symbol CHARACTER; the semantic value is set to the character itself.

If yytext does not represent a hex character compliant with R6RS: the function referenced by the lexical-error-token-maker parameter is applied to the same arguments handed to this function.

— Function: make-number-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a number. The category is set to the symbol NUMBER; the semantic value is set to the number itself.
