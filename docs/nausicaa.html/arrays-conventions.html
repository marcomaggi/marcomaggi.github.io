40.1 Arguments conventions

In the documentation of (nausicaa arrays) we call:

items
The values collected in Scheme's builtin lists and vectors.
elements
The values collected in the arrays.

The following conventions are used in the documentation of arguments to the functions of (nausicaa arrays):

array
A proper array object.
shape
A proper shape object.
position
A proper position object.
position-step
A builtin Scheme vector used to represent the difference between two position values.
dimension
A strictly positive, exact integer representing the extension of a dimension in an array.
index
A non–negative, exact integer representing the coordinate of an element.
start
past
last
When specifying a range of items or elements: start is the left–included bound, past is the right–included bound, last is the right–excluded bound.
obj
Can be any value.
