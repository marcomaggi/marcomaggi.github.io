72.4.1 Introduction to ISBN identifiers

For an introduction on ISBN identifiers see (URL last verified May 18, 2011):

http://en.wikipedia.org/wiki/Isbn

for the ISBN User's Manual see (URL last verified May 18, 2011):

http://www.isbn.org/standards/home/isbn/international/ISBNmanual.asp

for more informations on the ‘978’ and ‘979’ prefixes see (URL last verified May 18, 2011):

http://en.wikipedia.org/wiki/Global_Trade_Item_Number

for more material about ISBN identifiers see (URL last verified May 18, 2011):

http://isbn.org/standards/home/education/index.html

in particular notice the online ISBN converter (URL last verified May 23, 2011):

http://www.isbn.org/converterpub.asp

The (nausicaa standard-numbers isbn) library handles ISBN identifiers as Scheme strings, stripped of any blank character before and after them; ISBN identifiers often appear with the string ISBN in front of them, but the library assumes that this tag has already been stripped.

The following are examples of ISBN identifiers for “Design Patterns”, Gamma, Helm, Johnson, Vlissides, Addison Wesley, 1995: 

     0-201-63361-2           10 digits, hyphen separator
     0 201 63361 2           10 digits, space separator
     0201633612              10 digits, no separator
     978-0-201-63361-0       13 digits, hyphen separator
     978 0 201 63361 0       13 digits, space separator
     9780201633610           13 digits, no separator

the first digit ‘0’ means that the book has been published in the English language; the number ‘201’ is assigned to the publisher; the number ‘63361’ is assigned to this specific title; the last digit ‘2’ in the 10 digit format and ‘0’ in the 13 digit format are check digits used to detect typing errors.

Each title has both a 10 digits and a 13 digits ISBN identifier. A 13 digits ISBN includes the prefix, and the check digits are different, but the core of the identifiers is the same.

The identifier can be split into fields identifying the language, the publisher, the specific item (the title) and a check digit used to help validating the correctness of the identifier itself. Fields can be contiguous or separated by a single white space or a single hyphen; either spaces or hyphens must be selected, mixed separators are forbidden.

While the number of digits in an identifier is fixed, 10 or 13, the number of digits in each field is variable; when splitting an ISBN without separators into its fields (or when validating an already split identifier) we must:

  1. If the identifier has 13 digits: strip the prefix. For example in ‘9780201633610’ the prefix ‘978’ must be stripped.
  2. Recognise the digits representing the language and strip them. For example in ‘0201633610’, ‘0’ represents the English language. We do this by consulting a table of registered language numbers.
  3. Recognise the digits representing the publisher and strip them. For example in ‘201633610’, ‘201’ represents the publisher. We do this by consulting a table of registered publisher numbers; each registered language has a table of publishers.
  4. Separate the last digit which is the check digit. For example in ‘633610’, the last ‘0’ represents the check digit.
  5. What is left represents the title. For example ‘63361’.

The tables needed to recognise registered languages and publishers are available in XML format from:

http://www.isbn-international.org/page/ranges

and they must be updated periodically to stay up to date; they were last updated into Nausicaa/Scheme with data for Wed, 20 Apr 2011 16:54:22 GMT. stdnum isbn tables format for details on the format of the file.
