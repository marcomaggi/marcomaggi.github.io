58.1.2 Combinator functions

A combinator is a function accepting a single <state> record as argument and returning a <success> or <error> record. The returned record is generated by analysing the tokens in the <state> records; a combinator consuming a single token looks like this: 

     (define (combinator state)
       (let* ((token (<state>-lookahead state))
              (value (some-operation token)))
         (if value
             (make-<success> (next-state state) value)
           (make-<error> state "an error description"))))

notice how the returned record holds the next state if it is a <success> and the current state if it is an <error>. A combinator analysing three tokens looks like this: 

     (define (combinator state-1)
       (let* ((state-2 (next-state state-1))
              (state-3 (next-state state-2))
              (token-1 (<state>-lookahead state-1))
              (token-2 (<state>-lookahead state-2))
              (token-3 (<state>-lookahead state-3))
              (value (some-operation token-1 token-2 token-3)))
         (if value
             (make-<success> (next-state state-3) value)
           (make-<error> state-1 "an error description"))))

here the successful return record keeps a reference to the state next to the last <state> record used to generate it; this means that given the result of a combinator application, we can access the deepest parser <state> and so continue parsing from there.

Often we chain combinator functions so the second combinator is applied only if the first one succeeds: 

     (define (first state)
       (let* ((token (<state>-lookahead state))
              (value (some-operation token)))
         (if value
             (second (next-state state))
           (make-<error> state "an error description"))))
     
     (define (second state)
       (let* ((token (<state>-lookahead state))
              (value (some-operation token)))
         (if value
             (make-<success> (next-state state) value)
           (make-<error> state "an error description"))))
     
     (first init-state)

notice how first takes care of generating a new <state> record with next-state.

There are many ways, and so much freedom, in composing combinators to parse the tokens. (nausicaa packrat) provides a handful of standard ways to compose client supplied combinators.
