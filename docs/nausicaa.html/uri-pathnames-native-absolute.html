11.3.4.1 Absolute pathname native class

The following bindings are exported by the library (nausicaa uri pathnames).

— Class: <absolute-native-pathname>

Type of absolute native pathnames; it is derived from <absolute-pathname>.

Makers, constructors, predicates
— Common constructor on <absolute-native-pathname>: constructor pathname

Build and return a new instance of <absolute-native-pathname> from pathname, which must be a string or bytevector representing an absolute native pathname.

— Maker on <absolute-native-pathname>: <absolute-native-pathname> (?pathname)

Build and return a new instance of <absolute-native-pathname> using the common constructor.

Methods
— Method on <absolute-native-pathname>: extension

Return a bytevector representing the extension of the last segment of the instance pathname, dot separator not included.

— Method on <absolute-native-pathname>: dirname

Return an <absolute-native-pathname> object representing the directory part of the instance pathname.

— Method on <absolute-native-pathname>: tailname

Return a <relative-native-pathname> object representing the last segment of the instance pathname.

— Method on <absolute-native-pathname>: rootname

Return an <absolute-native-pathname> object representing the instance pathname with the extension of the last segment stripped, dot separator not included.

— Method on <absolute-native-pathname>: split

Return 2 values: a boolean, #t if the instance pathname is absolute; a list of bytevectors each representing a segment of the instance pathname.

— Method on <absolute-native-pathname>: prefix? (ptn <absolute-native-pathname>)
— Method on <absolute-native-pathname>: prefix? (ptn <relative-native-pathname>)

Return #t if the instance pathname is a prefix of ptn, otherwise return #f.

— Method on <absolute-native-pathname>: suffix? (ptn <absolute-native-pathname>)
— Method on <absolute-native-pathname>: suffix? (ptn <relative-native-pathname>)

Return #t if the instance pathname is a suffix of ptn, otherwise return #f.

— Method on <absolute-native-pathname>: prepend (ptn <absolute-native-pathname>)

Return a new pathname object representing the instance pathname prepended to ptn.

— Method on <absolute-native-pathname>: replace-extension (ext <bytevector>)
— Method on <absolute-native-pathname>: replace-extension (ext <string>)

Return an <absolute-native-pathname> object representing the instance pathname with the extension of the last segment replaced by ext.
