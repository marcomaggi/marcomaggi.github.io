28.2.5 Compensations

The functions described here make use of compensation stacks to handle semi–automatice release of allocated memory. Compensation stacks

The following bindings are exported by (nausicaa ffi memory compensated) and re–exported by (nausicaa ffi memory). This library makes internal use of (nausicaa ffi memory caches); the API requires also the (nausicaa language compensations) and (nausicaa ffi memory memblocks) libraries.

— Function: malloc/compensated number-of-bytes
— Function: calloc/compensated count element-size
— Function: malloc/c number-of-bytes
— Function: calloc/c count element-size

Wrappers for malloc and calloc that allocate a block of memory and return a pointer to it. The appropriate invocation to primitive-free, to free the block, is added to the dynamic environment of the current compensations stack.

— Function: malloc-small/compensated
— Function: malloc-small/c

Acquire a small block of memory from small-blocks-cache and return a pointer to it. The appropriate invocation to small-blocks-cache, to release the block, is added to the dynamic environment of the current compensations stack.

— Function: malloc-page/compensated
— Function: malloc-page/c

Acquire a page of memory from page-blocks-cache and return a pointer to it. The appropriate invocation to page-blocks-cache, to release the block, is added to the dynamic environment of the current compensations stack.

— Function: malloc-block/compensated number-of-bytes
— Function: malloc-block/c number-of-bytes

Allocate a new memory block and return a pointer to it. The appropriate invocation to the release function is added to the dynamic environment of the current compensations stack.

Allocation and releasing of memory blocks is done, in this order:

  1. Relying on small-blocks-cache if the number of bytes is less than or equal to the size of a small block.
  2. Relying on page-blocks-cache if the number of bytes is less than or equal to the size of a page block.
  3. Relying on calloc and primitive-free in all the other cases.

— Function: malloc-memblock/compensated number-of-bytes
— Function: malloc-memblock/c number-of-bytes

Acquire a block of memory from memblocks-cache and return a record of type <memblock>. The appropriate invocation to memblocks-cache, to release the block, is added to the dynamic environment of the current compensations stack.
