30.3.4 Converting to and from cstrings

— Function: string->cstring string/symbol
— Function: string->cstring string/symbol malloc
— Function: string->cstring/c string/symbol

Build and return new C string representing the Scheme string/symbol and return a pointer value referencing it. The returned string is encoded in UTF8 format and terminated by a zero byte.

string->cstring allocates memory using the supplied malloc function, or malloc if malloc is not given. To use a compensated allocation: We can use malloc/c or malloc-block/c as malloc argument.

The conversion is performed as follows:

  1. If string/symbol is a Scheme symbol, its string name is acquired and becomes the input string.
  2. The function string->utf8 is used to convert the Scheme string into a bytevector holding the string representation in UTF8 format.
  3. Raw memory is allocated and, byte by byte, the bytevector is duplicated into the memory block; a zero terminating byte is appended.

string->cstring/c allocates memory using malloc-block/c.

— Function: cstring->string cstr
— Function: cstring->string cstr number-of-bytes

Build and return a new Scheme string representing cstr, which must be a pointer to a memory block holding a zero terminated, UTF8 encoded, C language string.

The conversion is performed as follows:

  1. If number-of-bytes is given, that number of bytes are taken from the memory block; else the length of the input string is determined using strlen.
  2. A bytevector is allocated and, byte by byte, the string is copied into it.
  3. The function utf8->string is used to convert the bytevector into a Scheme string.

— Function: string->memblock string/symbol
— Function: string->memblock string/symbol malloc
— Function: string->memblock/c string/symbol

Build and return new record of type <memblock> representing the Scheme string/symbol and return a pointer value referencing it. The returned memblock represents the string encoded in UTF8 format without the trailing zero byte.

string->memblock allocates memory using the supplied malloc function, or malloc if malloc is not given. To use a compensated allocation: We can use malloc/c or malloc-block/c as malloc argument.

The conversion is performed as follows:

  1. If string/symbol is a Scheme symbol, its string name is acquired and becomes the input string.
  2. The function string->utf8 is used to convert the Scheme string into a bytevector holding the string representation in UTF8 format.
  3. Raw memory is allocated and, byte by byte, the bytevector is duplicated into the memory block.

string->memblock/c allocates memory using malloc-block/c.

— Function: memblock->string mb
— Function: memblock->string mb number-of-bytes

Build and return a new Scheme string representing the <memblock> record mb, which must hold a UTF8 encoded, C language string with or without the trailing zero byte.

The conversion is performed as follows:

  1. If number-of-bytes is given, that number of bytes are taken from the memory block; else the length of the input string is the size of the <memblock> itself.
  2. A bytevector is allocated and, byte by byte, the string is copied into it.
  3. The function utf8->string is used to convert the bytevector into a Scheme string.
