22.1.5 Defining custom class makers

— Auxiliary Syntax: custom-maker identifier

Select a maker to be used by make; this clause is alternative to maker, it is an error to include both maker and custom-maker in a class definition.

When custom-maker is used: make expands to a macro use of identifier which must use any available method to build a new class instance: 

          (make class arg ...)
          ==> (identifier arg ...)

Custom makers do not make use of the transformer defined by maker-transformer and do not make use of the protocol function defined by maker-protocol.

The following example shows a usage of custom-maker: 

          (define-auxiliary-syntaxes
            a: b:)
          
          (define-maker make-alpha
            (make-from-fields <alpha>)
            ((a: 1)
             (b: 2)))
          
          (define-class <alpha>
            (fields a b)
            (custom-maker make-alpha))
