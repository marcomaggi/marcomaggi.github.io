3.7 Defining custom makers

— Syntax: make-from-fields ?class ?expression ...

Build and return an instance of ?class from raw field values; each ?expression is evaluated once and it must return a raw field value. The order of the ?expression arguments must match the order of the arguments to the from–fields constructor. From–fields constructor.

This macro cannot be used to build labeled values.

— Auxiliary Syntax: maker ?expression

Define the transformer function used by the maker syntax of classes and labels. ?expression is evaluated once at expand time and it must return a macro transformer function.

When the maker syntax of ?type: 

          (?type (?arg ...))

is expanded, the transformer is applied to a syntax object: 

          #'(?type (?arg ...))

and it must return a syntax object representing an expression that, evaluated at run time, must return an instance of ?type or raise an exception.

Example: 

          (define-class <rect-coords>
            (fields x y)
            (maker (lambda (stx)
                     (syntax-case stx (rec pol)
                       ((_ (pol ?rho ?theta))
                        #'(let ((rho   ?rho)
                                (theta ?theta))
                            (make-<rect-coords> (* rho (cos theta))
                                                (* rho (sin theta)))))
                       ((_ (rec ?x ?y))
                        #'(make-<rect-coords> ?x ?y))))))
          
          (<rect-coords> R (<> (rec 1.0 2.0)))
          (R x)                   ⇒ 1.0
          (R y)                   ⇒ 2.0
          
          
          (<rect-coords> Q (<> (pol 1.0 2.0)))
          (R x)                   ⇒ -0.4161468365471424
          (R y)                   ⇒ +0.9092974268256817
