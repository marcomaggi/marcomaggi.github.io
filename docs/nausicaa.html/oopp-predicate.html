3.9 Defining custom predicates

Predicate clauses allow the definition of custom predicates for labeled values; they cannot be used in the definition of class types.

— Auxiliary Syntax: predicate ?expression

Define a custom predicate function or syntax to be used to identify instances of labeled values with the predicate syntax or the is-a? macro. This clause can be used in the body of define-label and define-mixin.

?expression is evaluated once and it must return a predicate function.

Usage example: 

     (define-label <pair>
       (predicate pair?)
       (virtual-fields (immutable car car)
                       (immutable cdr cdr)))
     
     ((<pair>) '(1 . 2))             ⇒ #t
     ((<pair>) 123)                  ⇒ #f
     
     (is-a? '(1 . 2) <pair>)         ⇒ #t
     (is-a? 123 <pair>)              ⇒ #f

Custom predicates of sublabels are composed with the custom predicates of their superlabels, with the predicate of the superlabel applied first; example: 

     (define-label <list>
       (predicate list?)
       (virtual-fields (immutable car car)
                       (immutable cdr cdr)))
     
     (define-label <list-of-numbers>
       (parent <list>)
       (predicate (lambda (ell)
                    (for-all number? ell))))
     
     (define L '(1 2 3))
     (define M '(a b c))
     (define N "abc")
     
     ((<list>) L)                 ⇒ #t
     ((<list-of-numbers>) L)      ⇒ #t
     (<list>? L)                  ⇒ #t
     (<list-of-numbers>? L)       ⇒ #t
     
     ((<list>) M)                 ⇒ #t
     ((<list-of-numbers>) M)      ⇒ #f
     
     ((<list>) N)                 ⇒ #f
     ((<list-of-numbers>) N)      ⇒ #f

the predicate function of <list-of-numbers> is equivalent to: 

     (define <list-of-numbers>-private-predicate
       (lambda (ell)
         (for-all number? ell)))
     
     (define (<list-of-numbers>? obj)
       (and (list? obj)
            (<list-of-numbers>-private-predicate obj)))
