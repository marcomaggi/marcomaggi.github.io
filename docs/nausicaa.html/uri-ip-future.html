11.1.7 Future versions of IP address objects

RFC 3986 defines a syntax to specify future versions of address literals, beyond the one for registered names and numeric IPv4 and IPv6; the class <ipvfuture-address> is a container to represent such address representations. 

     #!vicare
     (import (nausicaa)
       (prefix (nausicaa uri ip) ip.))
     
     (ip.<ipvfuture-address> O
       (<> (10 '#ve(ascii "ci%3Fa%3Do"))))
     
     (O string)
     ⇒ "[vA.ci%3Fa%3Do]"
     
     (O bytevector)
     ⇒ #ve(ascii "[vA.ci%3Fa%3Do]")
     
     (O version-flag)        ⇒ 10
     (O literal)             ⇒ #ve(ascii "ci%3Fa%3Do")

The following bindings are exported by the library (nausicaa uri ip).

— Abstract Class: <ipvfuture-address>

Type for future versions of IP address literals. It is derived from <ip-address>.

Constructors and makers
— Common constructor on <ipvfuture-address>: constructor version-flag address

Common constructor for objects of type <ipvfuture-address>. version-flag must be a fixnum in the range [0, 15] representing the version number of the literal format, according to RFC 3986. address must be a bytevector object representation of the address as defined by RFC 3986.

— Maker on <ipvfuture-address>: <ipvfuture-address> (?version-flag ?address)

Build and return a new instance of <ipvfuture-address> using the common constructor.

Fields
— Immutable field of <ipvfuture-address>: <ipvfuture-version-flag> version-flag

A fixnum in the range [0, 15] representing the version of the literal format, according to RFC 3986.

— Immutable field of <ipvfuture-address>: <ascii-bytevector> literal

A bytevector object representation of the address literal as defined by RFC 3986. Every octet represents a character in ASCII encoding, some sequences of octets may represent a percent–encoded character.

Notice that the bytevector representation of the literal is, in general, invalid as percent–encoded bytevector as defined by RFC 3986.

Multimethod implementations
— Multimethod: ip-address->string (addr <ipvfuture-address>)

Build and return a string representation of the address in the format specified for URIs by RFC 3986. In general, the returned object is not a valid percent–encoded string.

— Multimethod: ip-address->bytevector (addr <ipvfuture-address>)

Build and return a bytevector representation of the address in the format specified for URIs by RFC 3986. In general, the returned object is not a valid percent–encoded bytevector.
