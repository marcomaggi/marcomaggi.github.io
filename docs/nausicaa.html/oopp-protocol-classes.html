3.6.1 Class construction protocols

Class types have explicitly defined construction protocols; the semantics of protocol functions is the same as the one defined by R6RS for record protocols. The following constructor functions are made available by define-class:

Public constructor
It is used when instantiating a class through the maker syntax. The public constructor of a class <B>, derived from class <A>, is built on top of the super constructor of <A>. It is defined with the public-protocol clause.
Super constructor
It is used when instantiating a subclass, it is the function handed as argument to the subclass' constructor protocol function. The super constructor of a class <B>, derived from class <A>, is built on top of the super constructor of <A>. It is defined with the super-protocol clause.
Common constructor
When the public constructor is not explicitly defined: the common constructor is used as public constructor. When the super constructor is not explicitly defined: the common constructor is used as super constructor. The common constructor of a class <B>, derived from class <A>, is built on top of the super constructor of <A>. It is defined with the protocol clause.

When no common constructor is explicitly defined: it defaults to the from–fields constructor.

From–fields constructor
It is always and automatically generated to be used by the macro make-from-fields and it builds instances from raw field values. The from–fields constructor of a class <B>, derived from class <A>, is built on top of the from–fields constructor of <A>.

This constructor is equivalent to the default constructor generated by define-record-type when no protocol clause is used.

— Auxiliary Syntax: protocol ?expression

Define the common constructor's protocol function. ?expression must evaluate to the protocol function.

— Auxiliary Syntax: public-protocol ?expression

Define the public constructor's protocol function. ?expression must evaluate to the protocol function.

— Auxiliary Syntax: super-protocol ?expression

Define the super constructor's protocol function. ?expression must evaluate to the protocol function.

Examples of building class instances

Here is an example of default constructor usage; notice that no protocol syntax is used, so the constructors accept the raw field values as arguments: 

     (define-class <alpha>
       (fields a b))
     
     (define-class <beta>
       (parent <alpha>)
       (fields c d))
     
     (<alpha> A (<> (1 2)))
     (A a)                   ⇒ 1
     (A b)                   ⇒ 2
     
     (<beta> B (<> (1 2 3 4)))
     (B a)                   ⇒ 1
     (B b)                   ⇒ 2
     (B c)                   ⇒ 3
     (B d)                   ⇒ 4

The following code makes use of explicitly defined common protocols completely equivalent to the default ones: 

     (define-class <alpha>
       (fields a b)
       (protocol (lambda (make-top)
                   (lambda (a b)
                     ((make-top) a b)))))
     
     (define-class <beta>
       (parent <alpha>)
       (fields c d)
       (protocol (lambda (make-alpha)
                   (lambda (a b c d)
                     ((make-alpha a b) c d)))))
     
     (<alpha> A (<> (1 2)))
     (A a)                   ⇒ 1
     (A b)                   ⇒ 2
     
     (<beta> B (<> (1 2 3 4)))
     (B a)                   ⇒ 1
     (B b)                   ⇒ 2
     (B c)                   ⇒ 3
     (B d)                   ⇒ 4

The following code makes use of the make-from-fields syntax: 

     (define-class <alpha>
       (fields a b))
     
     (define-class <beta>
       (parent <alpha>)
       (fields c d))
     
     (<alpha> A (make-from-fields <alpha> 1 2))
     (A a)                   ⇒ 1
     (A b)                   ⇒ 2
     
     (<beta> B (make-from-fields <beta> 1 2 3 4))
     (B a)                   ⇒ 1
     (B b)                   ⇒ 2
     (B c)                   ⇒ 3
     (B d)                   ⇒ 4

The following code defines a common protocol accepting only some of the fields' values: 

     (define-class <alpha>
       (fields a b)
       (protocol (lambda (make-top)
                   (lambda (a)
                     ((make-top) a 2)))))
     
     (define-class <beta>
       (parent <alpha>)
       (fields c d)
       (protocol (lambda (make-alpha)
                   (lambda (a c)
                     ((make-alpha a) c 4)))))
     
     (<alpha> A (<> (1)))
     (A a)                   ⇒ 1
     (A b)                   ⇒ 2
     
     (<beta> B (<> (1 3)))
     (B a)                   ⇒ 1
     (B b)                   ⇒ 2
     (B c)                   ⇒ 3
     (B d)                   ⇒ 4

The following code shows how to use the super and public protocols: 

     (define-class <alpha>
       (fields a b)
       (public-protocol (lambda (make-top)
                          (lambda ()
                            ((make-top) 1 2))))
       (super-protocol  (lambda (make-top)
                          (lambda (a b)
                            ((make-top) (* 10 a) (* 10 b))))))
     
     (define-class <beta>
       (parent <alpha>)
       (fields c d)
       (public-protocol (lambda (make-alpha)
                          (lambda (a b)
                            ((make-alpha a b) 3 4)))))
     
     (<alpha> A (<> ()))
     (A a)                   ⇒ 1
     (A b)                   ⇒ 2
     
     (<beta> B (<> (1 2)))
     (B a)                   ⇒ 10
     (B b)                   ⇒ 20
     (B c)                   ⇒ 3
     (B d)                   ⇒ 4
