37.2 Class interface

The following bindings are exported by the (nausicaa pathnames) library. The classes documented below must never be instantiated directly; we should instantiate derived platform–specific classes instead, like <unix-relative-pathname> and <unix-absolute-pathname>; see Manipulating Unix pathnames.

— Class: <pathname>

Type of pathnames.

— Class: <absolute-pathname>

Type of absolute pathnames; it is derived from <pathname>.

— Class: <relative-pathname>

Type of relative pathnames; it is derived from <pathname>.

— Immutable field of <pathname>: bytevector

Bytevector representing the pathname. The value itself must not be mutated.

— Immutable field of <pathname>: segments

Possibly empty list of bytevectors representing the pathname segments; when the list is empty: if the pathname is absolute, it is the root; if the pathname is relative, it is the single dot. The value itself must not be mutated.

— Immutable virtual field of <pathname>: string

Scheme string representing the value of the bytevector field.

— Method on <pathname>: absolute absolute-base

Return a record holding an absolute pathname representing the absolute version of this instance pathname with respect to absolute-base, which must be an absolute pathname record. If this instance is already absolute: the return value can be this instance itself.

This method is implemented by the pathname-absolute generic function from (nausicaa pathnames methods).

— Method on <relative-pathname>: prepend relative-base

Return a record holding a relative pathname representing the pathname of relative-base with the pathname of this instance appended; relative-base must be a relative pathname record.

This method is implemented by the pathname-compose generic function from (nausicaa pathnames methods).

The following methods extract components from an absolute pathname; they are not defined for relative pathnames because some operations are undefined. For example, what is the directory part of the relative pathname ‘a/../..’?

Some methods deal with the extension and the name; the extension of a pathname segment is the trailing subvector up to the first byte representing a dot in ASCII coding, dot byte included; the name of a segment is everything but the extension. A segment having no dot byte, has no extension; a segment having the first byte representing a dot, and no other dots, has no extension.

Here is a table of examples for Unix pathnames: 

     pathname        dirname  rootname    tail       extension  name
     ---------------------------------------------------------------
     /               /        /           .          #f         .
     /dir            /        /dir        dir        #f         dir
     /dir/file.ext   /dir     /dir/file   file.ext   .ext       file
     /dir/.file      /dir     /dir/.file  .file      #f         .file
     /dir/.file.ext  /dir     /dir/.file  .file.ext  .ext       .file
— Method on <absolute-pathname>: tail

Return a new <relative-pathname> object representing the last segment of this instance. If the instance represents the root directory: the returned value has a single segment representing the single–dot entry.

This method is implemented by the pathname-tail generic function from (nausicaa pathnames methods).

— Method on <absolute-pathname>: dirname

Return a new <absolute-pathname> object representing all the segments in this instance but the last. If the instance represents the root directory: the returned value represents the root directory, too.

This method is implemented by the pathname-dirname generic function from (nausicaa pathnames methods).

— Method on <absolute-pathname>: rootname

Return a new <absolute-pathname> object representing the same pathname of this instance, but having the last segment trimmed of the extension.

This method is implemented by the pathname-rootname generic function from (nausicaa pathnames methods).

— Method on <absolute-pathname>: extension

Return a new bytevector representing the extension of the last segment in this instance. If the last segment has no extension: the return value is false.

This method is implemented by the pathname-extension generic function from (nausicaa pathnames methods).

— Method on <absolute-pathname>: name

Return a new bytevector representing the name of the last segment in this instance. If the instance represents the root directory: the returned value represents a single dot in ASCII coding.

This method is implemented by the pathname-name generic function from (nausicaa pathnames methods).
