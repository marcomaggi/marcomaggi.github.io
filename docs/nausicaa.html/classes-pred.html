24.3 Record type predicates

— Syntax: is-a? expression class-name
— Syntax: is-a? <> class-name
— Auxiliary Syntax: <>

Expand to the application of the class predicate of class-name to the result of evaluating expression, which can be any expression and is evaluated only once.

This macro uses the predicate selected with the predicate clause of define-class or the default predicate when no explicit selection was made.

For example, given: 

          (define-class <alpha>
            (fields a b c))

the following statement: 

          (is-a? x <alpha>)

is equivalent to: 

          (<alpha>? x)

while given: 

          (define-class <alpha>
            (predicate alpha?)
            (fields a b c))
          
          (define (alpha? obj)
            (and (<alpha>? obj)
                 (< 0 (<alpha>-a obj))))

the following statements are equivalent: 

          (is-a? x <alpha>)
          (alpha? x)

Notice that, using this macro, there is no need to export the class predicate out of a library: it is enough to export the class name. Example: 

          (library (defs)
            (export <alpha>)
            (import (rnrs) (nausicaa language classes))
            (define-class <alpha>
              (fields a b c)))
          
          (library (tests)
            (export)
            (import (rnrs) (nausicaa language classes) (alpha))
            (let ((x (make <alpha>
                       1 2 3)))
              (is-a? x <alpha>)))

When the <> auxiliary syntax is used in place of expression: the form expands to the class predicate function, which can be used as argument; the following forms are equivalent: 

          (is-a? x <list>)
          ((is-a? <> <list>) x)

this is especially useful when using map or for-all: 

          (for-all (is-a? <> <list>) obj)

The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language classes).
