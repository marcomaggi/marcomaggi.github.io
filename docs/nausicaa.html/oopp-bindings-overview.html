3.19.1 Introduction to OOPP bindings

OOPP notation is available only for the members of tags defined by define-class and define-label, it cannot be used with types defined by define-record-type.

The binding syntaxes exported by (nausicaa language oopp) match the R6RS ones exported by (rnrs base (6)) and (rnrs control (6)) and some Vicare–specific ones exported by (vicare).

Whenever in the original syntaxes a ?variable identifier appears, in the OOPP syntaxes a ?tagged-variable form appears; a use of ?tagged-variable must have one of the formats: 

     ?variable
     (?variable ?tag)
     #(?variable ?tag)

where: ?variable is an identifier representing the binding name, ?tag is an identifier representing a class or label type.

Whenever in the original syntaxes a ?bindings form appears, in the OOPP syntaxes a ?tagged-bindings form appears; a use of ?tagged-bindings must have the format: 

     ((?tagged-variable ?init) ...)

Whenever in the original syntaxes a ?mv-bindings form appears, in the OOPP syntaxes a ?tagged-mv-bindings form appears; a use of ?tagged-mv-bindings must have the format: 

     ((?tagged-formals ?init) ...)

Whenever in the original syntaxes a ?formals form appears, in the OOPP syntaxes a ?tagged-formals form appears; a use of ?tagged-formals must have one of the formats: 

     ?variable
     #(?variable ?tag)
     (?tagged-variable ...)
     (?tagged-variable0 ?tagged-variable ... . ?variableN)
     (?tagged-variable0 ?tagged-variable ... . #(?variable ?tag))

notice that a tagged rest argument must be specified using the vector syntax #(?variable ?tag), the list syntax (?variable ?tag) would be a syntax error.
