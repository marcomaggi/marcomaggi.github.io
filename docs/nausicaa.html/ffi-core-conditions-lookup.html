30.1.2.5 Shared objects lookup error
— Condition Type: &shared-object-lookup-error

Tags a compound condition object as describing a failed attempt to retrieve the address of an exported entity from a shared library; it is derived from &error. It has no fields.

— Function: make-shared-object-lookup-error-condition
— Function: shared-object-lookup-error-condition? obj

Constructor and predicate for condition objects of type &shared-object-lookup-error.

— Syntax: raise-shared-object-lookup-error <who> <message> <shared-object> <foreign-symbol>

Raise a continuable exception using as exception value a composite condition object with components of type: &who, &message, &shared-object, &foreign-symbol.

It is meant to be used when retrieving the pointer of an entity exported by a shared object fails; when returning to the continuation, the returned value must be the pointer object referencing the entity.
