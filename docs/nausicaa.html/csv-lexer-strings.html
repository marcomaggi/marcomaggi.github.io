79.1.1 Tokenising strings

The (nausicaa parser-tools csv) library defines strings in a CSV file as sequences of characters enclosed in double–quotes ‘"’. Inside a string a double double–quote ‘""’ is interpreted as a single double–quote.

To tokenise a string means to convert a string into the sequence of characters in it; this is done by a lexer using ‘csv-strings-table’. This table is meant to be used after the opening double–quote has been tokenised, and it will consume all the characters up to, and including, the closing double–quote.

When the lexer finds the closing double–quote it returns #f, else it returns the next character. We can take advantage of this to tokenise strings as follows: 

     #!r6rs
     (import (rnrs)
       (prefix (nausicaa parser-tools silex lexer) lex.)
       (nausicaa parser-tools csv strings-lexer))
     
     (define (tokenise string)
       (let* ((IS    (lex.make-IS (lex.string: string)))
              (lexer (lex.make-lexer csv-strings-table IS)))
         (do ((token (lexer) (lexer))
              (ell   '()))
             ((not token)
              (reverse ell))
           (set! ell (cons token ell)))))
     
     ;;All the test strings must end with a \" to signal
     ;;the end-of-string to the strings lexer.
     
     (tokenise "\"")
     ⇒ ()
     
     (tokenise "abcd\"")
     ⇒ (#\a #\b #\c #\d)
     
     ; Quoted double-quote.
     (tokenise "ab\"\"cd\"")
     ⇒ (#\a #\b #\" #\c #\d)
     
     ; Nested string.
     (tokenise "ab \"\"ciao\"\" cd\"")
     ⇒ (#\a #\b #\space #\" #\c #\i #\a #\o #\" #\space #\c #\d)
     
     ; Stop reading at the ending double-quote.
     (tokenise "ab\"cd")
     ⇒ (#\a #\b)

notice that the input system has to be created with ‘counters: 'all’. If we want to rebuild the string, we can do it as follows: 

     #!r6rs
     (import (rnrs)
       (nausicaa parser-tools silex lexer)
       (nausicaa parser-tools csv strings-lexer))
     
     (define (tokenise string)
       (let* ((IS    (lex.make-IS (lex.string: string)))
              (lexer (lex.make-lexer csv-strings-table IS)))
         (let-values (((port the-string) (open-string-output-port)))
           (do ((token (lexer) (lexer)))
               ((not token)
                (the-string))
             (write-char token port)))))
     
     ;;All the test strings must end with a \" to signal
     ;;the end of string to the strings lexer.
     
     (tokenise "\"")
     ⇒ ""
     
     (tokenise "abcd\"")
     ⇒ "abcd"
     
     ; Quoted double-quote.
     (tokenise "ab\"\"cd\"")
     ⇒ "ab\"cd"
     
     ; Nested string.
     (tokenise "ab \"\"ciao\"\" cd\"")
     ⇒ "ab \"ciao\" cd"
     
     ; Stop reading at the ending double-quote.
     (tokenise "ab\"cd")
     ⇒ "ab"
