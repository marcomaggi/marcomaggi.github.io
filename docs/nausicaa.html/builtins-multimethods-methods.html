5.23.2 Predefined multimethods

Generic methods
— Generic Function: object->string obj

Return a string representation for obj. A non–specialised method is predefined to use the following implementation: 

          (call-with-string-output-port
             (lambda (port)
               (display obj port)))
Input ports methods

For all the following input methods: if EOF is found and no data is available: return the EOF object; if the underlying device is in non–blocking mode and no data is available: return the would–block object.

— Generic Function: get-single port

Read, consume and return a single data unit from the port.

— Generic Function: lookahead-single port

Read, without consuming and return a single data unit from the port.

— Generic Function: get-multi-n port count

Read, consume and return count data units from the port.

— Generic Function: get-multi-n! port storage start count

Read and consume count data units from the port and store them in the storage starting at offset start.

— Generic Function: get-multi-some port

Read, consume and return multiple data units from the port.

— Generic Function: get-multi-all port

Read, consume and return all the data from the port.

Output ports methods
— Generic Function: put-single port storage

Write storage to the port.

— Generic Function: put-multi-2 port storage

Write storage to the port.

— Generic Function: put-multi-3 port storage start

Write storage to the port starting at offset start.

— Generic Function: put-multi-4 port storage start count

Write count units of data from storage to the port, starting at offset start.
