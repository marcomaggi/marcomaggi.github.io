72.1.1 Introduction to envelopes

In the context of the (nausicaa standard-numbers envelopes) library, a checksum algorithm is a function accepting a sequence of fixnums as input and returning a single fixnum as output, being the check digit. Input fixnums usually must be in a specified range, often mapping to characters through ASCII or other encoding.

To produce the check digit, the sequence of fixnums stored in an aggregate data value can be processed “left to right” or “right to left”; a check digit generated by iterating left to right is called left–fold check digit, a check digit generated by iterating right to left is called right–fold check digit.

The check digit itself is then usually attached to the sequence to the left or right; a sequence with the check digit attached is called envelope and the check digit itself is called flap. An envelope with the check digit attached to the left is called left–flap envelope, an envelope with the check digit attached to the right is called right–flap envelope.

Example. Let's say the checksum algorithm is just the + function; given the sequence: 
     1 2 3 4 5

the left–fold check digit is: 

     ((((1 + 2) + 3) + 4) + 5) = 15

which happens to be equal to the right–fold check digit: 

     (1 + (2 + (3 + (4 + 5)))) = 15

the left–flap envelope is: 

     15 1 2 3 4 5

and the right–flap envelope is: 

     1 2 3 4 5 15

The following are examples of check digit computation and envelope generation using the mod 37, 36 algorithm, defined by ISO 7064:1983, applied to identifier strings compliant with the Global Release Identifier (GRid) specification (see GRid parsing and validation): 

     #!r6rs
     (import (nausicaa)
       (nausicaa standard-numbers envelopes)
       (nausicaa standard-numbers iso-7064-1983 mod-37-36))
     
     (object->check-digit "A12425GABC1234002"
                          <<left-fold/right-flap-string>>
                          <<mod-37-36>>)
     ⇒ 22
     
     (object->envelope "A12425GABC1234002"
                       <<left-fold/right-flap-string>>
                       <<mod-37-36>>)
     ⇒ "A12425GABC1234002M"
     
     (envelope? "A12425GABC1234002M"
                <<left-fold/right-flap-string>>
                <<mod-37-36>>)
     ⇒ #t
     
     (object->check-digit '(1 2 3 4 5)
                          <<left-fold/left-flap-list>>
                          <<mod-37-36/digits>>)
     ⇒ 34
     
     (object->envelope '(1 2 3 4 5)
                       <<left-fold/left-flap-list>>
                       <<mod-37-36/digits>>)
     ⇒ (34 1 2 3 4 5)
     
     (envelope? '(34 1 2 3 4 5)
                <<left-fold/left-flap-list>>
                <<mod-37-36/digits>>)
     ⇒ #t
