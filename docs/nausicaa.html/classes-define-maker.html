25.1.4 Defining class makers

— Auxiliary Syntax: maker (positional-arg ...) (keyword default . options) ...

Define the specification of the maker to be used by make; defining a maker for a class means defining a specialised macro with define-maker. The maker clause in the following definition: 

          (define-class <alpha>
            (fields a b c d e)
            (maker (a b c)
                   (d 1)
                   (e 2 (mandatory))))

can be thought as equivalent to: 

          (define-maker (the-maker a b c)
            (make* <alpha>)
            ((d 1)
             (e 2 (mandatory))))

and can be used as: 

          (make <alpha> #\a #\b #\c
            (e 4))

where the make macro will expand to an use of the the-maker macro. See the documentation of define-maker for details on the arguments of maker; Defining makers.

The maker will use the maker–specific constructor if one was specified with the maker-protocol clause, else it will default to the public constructor.

If a maker-transformer clause is present: the use of make will expand into a use of the specified maker transformer; the first argument to the maker transformer will be the maker constructor. If no maker-transformer clause is present: the use of make will expand into a call to the constructor.

When no maker is defined: the make macro can still be used to instantiate the class: it defaults to use the public protocol, so it becomes equivalent to a make* use.

— Auxiliary Syntax: maker-transformer transformer

Select a maker macro transformer to be used as expansion of make. transformer can be an identifier or an expression evaluating to a macro transformer. The transformer will receive the maker protocol constructor as first argument, and it is meant to use it in its output form.

This is an example of identifier transformer: 

          (define-class <alpha>
            (fields a b)
            (maker (a)
                   (b 2))
            (maker-transformer transformer))
          
          (define-syntax transformer
            (syntax-rules ()
              ((_ ?constructor ?a ?b)
               (?constructor (+ 1 ?a) (+ 2 ?b)))))
          
          (let (((o <alpha>) (make <alpha> 1)))
            (list o.a o.b)) ⇒ (2 4)
          
          (let (((o <alpha>) (make <alpha> 1 (b 20))))
            (list o.a o.b)) ⇒ (2 22)

and this is an example of expression transformer: 

          (define-class <alpha>
            (fields a b)
            (maker (a)
                   (b 2))
            (maker-transformer
             (lambda (stx)
               (syntax-case stx ()
                 ((_ ?constructor ?a ?b)
                  #'(?constructor (+ 1 ?a) (+ 2 ?b)))))))
          
          (let (((o <alpha>) (make <alpha> 1)))
            (list o.a o.b)) ⇒ (2 4)
          
          (let (((o <alpha>) (make <alpha> 1 (b 20))))
            (list o.a o.b)) ⇒ (2 22)
— Auxiliary Syntax: maker-protocol expression

Select the maker constructor's protocol function. When this clause is not used: the maker syntax expands to an application of the public constructor function.

Here is a meaningless usage example: 

          (define-class <alpha>
            (fields a b c)
            (maker (a) (b 2))
            (maker-protocol (lambda (make-top)
                              (lambda (a b)
                                ((make-top) a b 456)))))
          
          (let (((o <alpha>) (make <alpha> 1)))
            (list o.a o.b o.c))
          ⇒ (1 2 456)
          
          (let (((o <alpha>) (make <alpha> 1 (b 20))))
            (list o.a o.b o.c))
          ⇒ (1 20 456)
          
          (let (((o <alpha>) (make* <alpha> 1 2 3)))
            (list o.a o.b o.c))
          ⇒ (1 2 3)
