51.3 Transforming S–expressions

A sexp transformer is used to match a sexp with a pattern, extracting variable values from it. The position of the “variables” in the sexp form is marked by objects of type ‘sexp-variable’. Example: 

     (import (nausicaa)
       (sexps)
       (sentinel))
     
     (define trans
       (let-sexp-variables ((?a sentinel)
                            (?b sentinel))
         (make-sexp-transformer `(doit (alpha ,(sexp-var ?a))
                                       (beta  ,(sexp-var ?b)))
                                `(doit ,?a ,?b))))
     
     (trans '(doit (alpha 1) (beta 2)))
     ⇒ (doit 1 2)

the following example makes use of the default values for unbound variables: 

     (import (nausicaa)
       (sexps)
       (sentinel))
     
     (define trans
       (let-sexp-variables ((?a 10)
                            (?b 20))
         (make-sexp-transformer
           `(doit ,(sexp-any (sexp-or `(alpha ,(sexp-var ?a))
                                      `(beta  ,(sexp-var ?b)))))
           `(doit ,?a ,?b))))
     
     (trans '(doit (alpha 1) (beta 2)))
     ⇒ (doit 1 2)
     
     (trans '(doit (alpha 1)))
     ⇒ (doit 1 20)
     
     (trans '(doit (beta 2)))
     ⇒ (doit 10 2)
     
     (trans '(doit))
     ⇒ (doit 10 20)
— Function: make-sexp-transformer pattern output

Return a function that takes a sexp, matches it to the pattern sexp, extracts bindings for the variables, and returns a copy of the sexp output with the variables substituted by their values.

If a variable referenced by output is left undefined by the bindings:

  • If the variable has default value set to the sentinel value from (sentinel): an error is raised.
  • If the variable has default value different from the sentinel value from (sentinel): the default value is substituted.

See sexp-match for the syntax of pattern.

— Function: sexp-substitute-bindings output bindings-alist

Build and return a copy of the sexp output in which the occurrences of the keywords in bindings-alist are substituted with their values. The keywords are meant to be records of type ‘sexp-variable’.
