3.1 Overview of labels and classes

With the facilities of the library (rnrs records syntactic (6)) we can define a new record type with the syntax: 

     (define-record-type <alpha>
       (fields (mutable a)
               (immutable b)))
     
     (define (<alpha>-sum o)
       (+ (<alpha>-a o)
          (<alpha>-b o)))

and then use it with code like: 

     (define o (make-<alpha> 1 2))
     (<alpha>? o)            ⇒ #t
     (<alpha>-a o)           ⇒ 1
     (<alpha>-b o)           ⇒ 2
     (<alpha>-sum o)         ⇒ 3

we notice that:

  • If we define a type in a library and then we want to use it in another library, we have to export all the associated identifiers: <alpha>, <alpha>?, <alpha>-a, <alpha>-b, <alpha>-sum.
  • There is no way to define a function tied to this record type, neither a syntax to use such function: functions acting on R6RS records, like <alpha>-sum, are just ordinary Scheme functions.
  • If we use <alpha> in a chunk of code and later we change the record type to something else: we have to change all the function names; there is no syntax to define interfaces implemented by multiple types.

Now let's look at the same definition using the facilities of (nausicaa language oopp) to define a class: 

     (define-class <alpha>
       (fields (mutable a)
               (immutable b))
       (method (sum (o <alpha>))
          (+ (o a) (o b))))

it is pretty similar; we can use this type with code like: 

     (<alpha> o (<> (1 2)))
     ((<alpha>) o)           ⇒ #t
     (o a)                   ⇒ 1
     (o b)                   ⇒ 2
     (o sum)                 ⇒ 3

there are differences with the R6RS code:

  • If we define a type in a library and then we want to use it in another library: we need to export only the type identifier <alpha>.
  • It is possible to define methods associated to the class type and there is an OOPP syntax to use them.
  • If we change the code to use a different type: we need to change only the type name, which appears in few places; it is possible to define interfaces implemented by multiple types.

Also notice that the R6RS bindings defined by define-record-type are available also with the facilities of (nausicaa language oopp): 

     (define-class <alpha>
       (fields (mutable a)
               (immutable b))
       (method (sum (o <alpha>))
          (+ (o a) (o b))))
     
     (define o (make-<alpha> 1 2))
     (<alpha>? o)            ⇒ #t
     (<alpha>-a o)           ⇒ 1
     (<alpha>-b o)           ⇒ 2
     (<alpha>-sum o))        ⇒ 3

so it is easy to convert existing record type definitions to classes definitions; notice, however, that it is not possible to just substitute define-record-type with define-class: some clauses of the former are not supported by the latter.

So, classes can be used to define new record types when we would use define-record-type in plain R6RS code; (nausicaa language oopp) also allows us to define “labels”. Label definitions are similar to class definitions, but, while the latter defines a record type, the former only defines identifier syntaxes to handle any value.

The following example defines a tag to handle immutable pair values: 

     (define-label <pair>
       (protocol (lambda ()
                   cons))
       (predicate pair?)
       (virtual-fields (immutable car car)
                       (immutable cdr cdr)))
     
     (<pair> P '(1 . 2))
     ((<pair>) P)            ⇒ #t
     (P car)                 ⇒ 1
     (P cdr)                 ⇒ 2
