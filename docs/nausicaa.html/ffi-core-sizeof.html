31.1.4 Sizes and limits

The (nausicaa ffi sizeof) library exports a set of bindings to allow interface libraries to be aware of C language type sizes and limits. When using this library, it is suggested to import it with prefix ‘ffi.’, because other libraries in the Nausicaa distribution export bindings with the same name.

In all the following syntax descriptions: ?type-name must be a symbol representing a C language type in the enumeration enum-clang-external-types from the (nausicaa ffi clang type-translation) library.

— Syntax: c-sizeof ?type-name
— Syntax: c-sizeof ?type-name ?number-of-values

In the first form: expand to the number of bytes required to hold a single value of ?type-name.

In the second form: expand to the number of bytes required to hold an array of ?number-of-values of ?type-name; this form makes use of the correct type stride. ?number-of-values must be an expression which evaluates to an exact non–negative integer.

— Syntax: c-alignof ?type-name

Expand to the number of bytes of alignment in data structures for the specified type.

— Syntax: c-strideof ?type-name

Expand to the number of bytes of array stride for the specified type.

— Syntax: c-valueof ?thing

Expand to an exact integer representing the value of ?thing, which is usually a constant name; for example: the value of ‘int-max’ is the C language constant INT_MAX. ?thing can be one among: 

          char-max        char-min
          schar-max       schar-min       uchar-max
          short-max       short-min       ushort-max
          int-max         int-min         uint-max
          long-max        long-min
          wchar-max
          ssize_t-max     ssize_t-min
          size_t-max      size_t-min
          intptr_t-max    intptr_t-min
          uintptr_t-max   uintptr_t-min
— Syntax: c-inspect ?thing

Expand to a value representing the ?thing property or constant; ?thing can be one of the following:

pointer-integer
A symbol representing one of the unsigned integer types for callout arguments and return values; it is meant to be the integer type that is equal in size to a pointer, so it can be used in place of the ‘pointer’ type for Scheme implementations that do not have it.
words-bigendian
It is #t if the platform is big endian, #f otherwise; notice that this information is also available with the R6RS function native-endianness.
on-32-bits-system
on-64-bits-system
on-32-bits-systems
on-64-bits-systems
It is #t if the size of the platform's int type is 32 or 64 bits respectively.
