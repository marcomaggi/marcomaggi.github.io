3.11 Defining classes having methods

— Auxiliary Syntax: method (?method-spec . ?formals) . ?body
— Auxiliary Syntax: method ?method-spec ?expression

Add a method to the enclosing class, label or mixin definition; the method is applicable only through a syntax use in OOPP notation. This clause can be used multiple times.

When the first form is used, the argument ?method-spec can have one of the following formats: 

          ?method-name
          (?method-name ?rv-tag0 ?rv-tag ...)
          #(?method-name ?rv-tag0 ?rv-tag ...)

when the second form is used, the argument ?method-spec can have one of the following formats: 

          ?method-name
          #(?method-name ?rv-tag)

where: ?method-name must be an identifier representing the method name; ?rv-tag must be a tag identifier representing the type of a return value. When a single return value tag is specified: the method is available for nested method invocation according to OOPP syntax specifications. When the first form is used: the return value tags are used to validate the returned arguments as described for lambda/tags.

  • When the first form is used, the method definition is added to the methods list as if the following clause had been used: 
                   (methods (?method-name function-name))

    the function definition is expanded to: 

                   (define/tags (function-name . ?formals) . ?body)

    where function-name is an automatically generated identifier built by appending a dash and ?method-name to the type name. The first argument in ?formals must be the type instance.

  • When the second form is used, the method definition is added to the methods list as if the following clause had been used: 
                   (methods (?method-name function-name))

    the function definition is expanded to: 

                   (define function-name ?expression)

    where function-name is an automatically generated identifier built by appending a dash and ?method-name to the type name; the ?expression must evaluate to a function accepting at least one argument: the type instance.

Here is an example of the first form: 

          (define-class <stuff>
            (fields a b)
            (method (sum (S <stuff>))
              (+ (S a) (S b))))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3

and an example of the second form: 

          (define-class <stuff>
            (fields a b)
            (method sum
              (lambda/tags ((S <stuff>))
                (+ (S a) (S b)))))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3
— Auxiliary Syntax: method-syntax ?method-spec ?transformer

Add a method to the enclosing class, label or mixin definition; the method is applicable only through a syntax use in OOPP notation. This clause can be used multiple times.

The argument ?method-spec can have one of the following formats: 

          ?method-name
          (?method-name ?rv-tag)
          #(?method-name ?rv-tag)

where: ?method-name must be an identifier representing the method name; ?rv-tag must be a tag identifier representing the type of a return value. When the single return value tag is specified: the method is available for nested method invocation according to OOPP syntax specifications.

The method definition is added to the methods list as if the following clause had been used: 

          (methods (?method-name transformer-name))

the macro definition is expanded to: 

          (define-syntax transformer-name ?transformer)

where transformer-name is an automatically generated identifier built by appending a dash and ?method-name to the type name. ?transformer must evaluate to a macro transformer: a function accepting a syntax object as single argument and returning a syntax object as single value. The first argument to the macro will be the type instance.

Here is an example of method syntax: 

          (define-class <stuff>
            (fields a b)
            (method-syntax sum
              (lambda (stx)
                (syntax-case stx ()
                  ((_ ?instance)
                   #'(let (((S <stuff>) ?instance))
                       (+ (S a) (S b))))))))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3
— Auxiliary Syntax: methods ?method-spec ...

Add methods to the enclosing class, label or mixin definition; the methods are applicable only through a syntax use in OOPP notation. This clause can be used multiple times.

The ?method-spec arguments can be of the form: 

          ?method-name
          (?method-name)
          (?method-name ?invokable-name)
          ((?method-name ?rv-tag) ?invokable-name)
          (#(?method-name ?rv-tag) ?invokable-name)

where ?method-name and ?invokable-name must be identifiers; in the first two forms the invokable name is built in the same way the field accessor names are built: appending a dash and ?method-name to the type name. When ?rv-tag is used: it must be the tag identifier representing the type of a return value, the method is then available for nested method invocation according to OOPP syntax specifications.

?invokable-name must be an identifier bound to a Scheme function or syntax which will be invoked as method; it must accept as first argument the type instance which is the subject of the application.

Here is a usage example of the first form: 

          (define-class <stuff>
            (fields a b)
            (methods sum))
          
          (define/tags (<stuff>-sum (S <stuff>))
            (+ (S a) (S b)))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3

here is a usage example of the third form with a method function: 

          (define-class <stuff>
            (fields a b)
            (methods (sum <stuff>-sum)))
          
          (define/tags (<stuff>-sum (S <stuff>))
            (+ (S a) (S b)))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3

and here is a usage example of the third form with a method syntax: 

          (define-class <stuff>
            (fields a b)
            (methods (sum <stuff>-sum)))
          
          (define-syntax <stuff>-sum
            (lambda (stx)
              (syntax-case stx ()
                ((_ ?instance)
                 #'(let (((S <stuff>) ?instance))
                     (+ (S a) (S b)))))))
          
          (<stuff> S (<> (1 2)))
          (S sum)                 ⇒ 3
Method overloading

There is no support for methods overloading in the (nausicaa language oopp) library, but we can do it using the facilities of the (nausicaa language generics) library. Example: 

     #!r6rs
     (import (nausicaa))
     
     (define-class <alpha>
       (fields a)
       (methods doit))
     
     (define-generic <alpha>-doit (o v))
     
     (define-method (<alpha>-doit (o <alpha>) (v <char>))
       (cons 'char v))
     
     (define-method (<alpha>-doit (o <alpha>) (v <integer>))
       (cons 'int  v))
     
     (<alpha> o (<> (1)))
     (o doit #\a)                    ⇒ (char . #\a)
     (o doit 2)                      ⇒ (int . 2)
