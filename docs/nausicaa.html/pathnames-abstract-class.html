41.2.1 Abstract class interface

The classes documented below cannot be instantiated; we should instantiate platform–specific subclasses instead, like <relative-pathname> and <absolute-pathname> exported by (nausicaa pathnames unix), see Manipulating Unix pathnames.

The following bindings are exported by the (nausicaa pathnames abstract) library.

— Abstract Class: <pathname>

Base type for pathnames.

— Abstract Class: <absolute-pathname>

Base type for absolute pathnames; it is derived from <pathname>.

— Abstract Class: <relative-pathname>

Base type for relative pathnames; it is derived from <pathname>.

— Superclass constructor on <pathname>: constructor bytevector segments

Build a new pathname record using the bytevector and the associated list of segments already verified and normalised.

— Superclass constructor on <absolute-pathname>: constructor bytevector segments
— Superclass constructor on <relative-pathname>: constructor bytevector segments

Build a new absolute or relative pathname record; the arguments are the same as the ones of the <pathname> superclass constructor.

Public fields
— Immutable field of <pathname>: <bytevector> bytevector

Bytevector representing the pathname. The value itself must not be mutated.

— Immutable field of <pathname>: <list> segments

Possibly empty list of bytevectors representing the pathname segments; when the list is empty: if the pathname is absolute, it is the root; if the pathname is relative, it is the single dot. The value itself must not be mutated.

— Immutable field of <pathname>: <pair> last-pair

The last pair in the list of segments stored in the segments field, or #f if segments is null.

— Immutable field of <pathname>: <string> string

Scheme string representing the value of the bytevector field.

Methods
— Method on <pathname>: <absolute-pathname> absolute (prefix <absolute-pathname>)

Return a record holding an absolute pathname representing the absolute version of this instance's pathname with respect to prefix. If this instance is already absolute: the return value can be this instance itself.

This method is implemented by the absolute generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (absolute ptn)
            (let* (((P <pathname>)
                    (pathname ptn))
                   ((Q <absolute-pathname>)
                    (P.absolute (pathname "/a/b/c"))))
              Q.string))
          
          ;; relative pathnames
          (absolute ".")                  ⇒ "/a/b/c"
          (absolute "ciao")               ⇒ "/a/b/c/ciao"
          (absolute "./ciao")             ⇒ "/a/b/c/ciao"
          (absolute "../ciao")            ⇒ "/a/b/ciao"
          (absolute "../../../ciao")      ⇒ "/ciao"
          
          (absolute "../../../../ciao")   error--> &normalisation
          
          ;; already absolute pathnames
          (absolute "/")                  ⇒ "/"
          (absolute "/ciao/hello")        ⇒ "/ciao/hello"
— Method on <relative-pathname>: <relative-pathname> prepend (prefix <relative-pathname>)
— Method on <relative-pathname>: <absolute-pathname> prepend (prefix <absolute-pathname>)

Return a record holding a relative or absolute pathname representing the pathname of prefix with this instance's pathname appended.

This method is implemented by the prepend generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <relative-pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (prepend-absolute ptn)
            (let* (((P <pathname>)
                    (pathname ptn))
                   ((Q <absolute-pathname>)
                    (P.prepend (pathname "/a/b"))))
              Q.string))
          
          (prepend-absolute ".")          ⇒ "/a/b"
          (prepend-absolute "c")          ⇒ "/a/b/c"
          (prepend-absolute "c/d/e")      ⇒ "/a/b/c/d/e"
          (prepend-absolute "..")         ⇒ "/a"
          (prepend-absolute "../..")      ⇒ "/"
          
          (doit "../../../")              error--> &normalisation
          
          (define (prepend-relative ptn)
            (let* (((P <pathname>)
                    (pathname ptn))
                   ((Q <relative-pathname>)
                    (P.prepend (pathname "./a/b"))))
              Q.string))
          
          (prepend-relative ".")          ⇒ "a/b"
          (prepend-relative "c")          ⇒ "a/b/c"
          (prepend-relative "c/d/e")      ⇒ "a/b/c/d/e"
          (prepend-relative "..")         ⇒ "a"
          (prepend-relative "../..")      ⇒ "."
          (prepend-relative "../../..")   ⇒ ".."
          (prepend-relative "../../../..") ⇒ "../.."
— Method on <absolute-pathname>: <absolute-pathname> append (suffix <relative-pathname>)
— Method on <relative-pathname>: <relative-pathname> append (suffix <relative-pathname>)

Return a record holding an absolute or relative pathname representing this instance's pathname with suffix appended.

This method is implemented by the append generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <absolute-pathname> and <relative-pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (append-relative ptn)
            (let* (((P <pathname>)
                    (pathname ptn))
                   ((Q <relative-pathname>)
                    (P.append (pathname "./a/b"))))
              Q.string))
          
          (append-relative ".")           ⇒ "a/b"
          (append-relative "c")           ⇒ "c/a/b"
          (append-relative "c/d/e")       ⇒ "c/d/e/a/b"
          (append-relative "..")          ⇒ "../a/b"
          (append-relative "../..")       ⇒ "../../a/b"
          (append-relative "../../..")    ⇒ "../../../a/b"
          (append-relative "../../../..") ⇒ "../../../../a/b"
          
          (append-relative "/")           ⇒ "/d/e"
          (append-relative "/a/b")        ⇒ "/a/b/d/e"

The following methods extract components from an absolute pathname; they are not implemented for relative pathnames because some operations are undefined. For example, what is the directory part of the relative pathname ‘a/../..’?

Some methods deal with the extension and the name; the extension of a pathname segment is the trailing subvector up to the first byte representing a dot in ASCII coding, dot byte included; the name of a segment is everything but the extension. A segment having no dot byte, has no extension; a segment having the first byte representing a dot, and no other dots, has no extension.

Here is a table of examples for Unix pathnames: 

     pathname        dirname  rootname    tail       extension  name
     ---------------------------------------------------------------
     /               /        /           .          #f         .
     /dir            /        /dir        dir        #f         dir
     /dir/file.ext   /dir     /dir/file   file.ext   .ext       file
     /dir/.file      /dir     /dir/.file  .file      #f         .file
     /dir/.file.ext  /dir     /dir/.file  .file.ext  .ext       .file
— Method on <pathname>: <relative-pathname> tail

Return a new <relative-pathname> object representing the last segment of this instance. If the instance represents the root directory: the returned value has a single segment representing the single–dot entry.

This method is implemented by the tail generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (tail ptn)
            (let* (((P <absolute-pathname>) (pathname ptn))
                   ((Q <relative-pathname>) (P.tail)))
              Q.string))
          
          (tail "/")      ⇒ "."
          (tail "/a")     ⇒ "a"
          (tail "/a/b/c") ⇒ "c"
— Method on <absolute-pathname>: <absolute-pathname> dirname

Return a new <absolute-pathname> object representing all the segments in this instance but the last. If the instance represents the root directory: the returned value represents the root directory, too.

This method is implemented by the dirname generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <absolute-pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (dirname ptn)
            (let* (((P <absolute-pathname>) (pathname ptn))
                   ((Q <absolute-pathname>) (P.dirname)))
              Q.string))
          
          (dirname "/")           ⇒ "/"
          (dirname "/a")          ⇒ "/"
          (dirname "/a/b/c")      ⇒ "/a/b"
— Method on <absolute-pathname>: <absolute-pathname> rootname

Return a new <absolute-pathname> object representing the same pathname of this instance, but having the last segment trimmed of the extension.

This method is implemented by the rootname generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <absolute-pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (define (rootname ptn)
            (let* (((P <absolute-pathname>) (pathname ptn))
                   ((Q <absolute-pathname>) (P.rootname)))
              Q.string))
          
          (rootname "/")                  ⇒ "/"
          (rootname "/a")                 ⇒ "/a"
          (rootname "/a/b/c")             ⇒ "/a/b/c"
          
          (rootname "/.ciao")             ⇒ "/.ciao"
          (rootname "/a/.ciao")           ⇒ "/a/.ciao"
          (rootname "/a/b/c/.ciao")       ⇒ "/a/b/c/.ciao"
          
          (rootname "/the-file.ext")      ⇒ "/the-file"
          (rootname "/a/the-file.ext")    ⇒ "/a/the-file"
          
          (rootname "/.the-file.ext")     ⇒ "/.the-file"
          (rootname "/a/.the-file.ext")   ⇒ "/a/.the-file"
— Method on <absolute-pathname>: <bytevector> extension

Return a new bytevector representing the extension of the last segment in this instance. If the last segment has no extension: the return value is false.

This method is implemented by the extension generic function from (nausicaa pathnames methods); it has a concrete implementation for the <absolute-pathname> class. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix)
            (prefix (nausicaa pathnames unix low) ul.))
          
          (define (extension ptn)
            (let (((P <absolute-pathname>) (pathname ptn)))
              (ul.to-string (P.extension))))
          
          (extension "/")                 ⇒ #f
          (extension "/a")                ⇒ #f
          (extension "/a/b/c")            ⇒ #f
          
          (extension "/.ciao")            ⇒ #f
          (extension "/a/.ciao")          ⇒ #f
          (extension "/a/b/c/.ciao")      ⇒ #f
          
          (extension "/file.ext")         ⇒ ".ext"
          (extension "/a/file.ext")       ⇒ ".ext"
          (extension "/a/b/c/file.ext")   ⇒ ".ext"
          
          (extension "/.file.ext")        ⇒ ".ext"
          (extension "/a/.file.ext")      ⇒ ".ext"
          (extension "/a/b/c/.file.ext")  ⇒ ".ext"
— Method on <absolute-pathname>: <bytevector> name

Return a new bytevector representing the name of the last segment in this instance. If the instance represents the root directory: the returned value represents a single dot in ASCII coding.

This method is implemented by the name generic function from (nausicaa pathnames methods); it has a concrete implementation for the <absolute-pathname> class. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix)
            (prefix (nausicaa pathnames unix low) ul.))
          
          (define (name ptn)
            (let (((P <absolute-pathname>) (pathname ptn)))
              (ul.to-string (P.name))))
          
          (name "/")                      ⇒ "."
          (name "/a")                     ⇒ "a"
          (name "/a/b/c")                 ⇒ "c"
          
          (name "/.ciao")                 ⇒ ".ciao"
          (name "/a/.ciao")               ⇒ ".ciao"
          (name "/a/b/c/.ciao")           ⇒ ".ciao"
          
          (name "/file.ext")              ⇒ "file"
          (name "/a/file.ext")            ⇒ "file"
          (name "/a/b/c/file.ext")        ⇒ "file"
          
          (name "/.file.ext")             ⇒ ".file"
          (name "/a/.file.ext")           ⇒ ".file"
          (name "/a/b/c/.file.ext")       ⇒ ".file"
— Method on <absolute-pathname>: <absolute-pathname> replace-extension (ext <bytevector>)
— Method on <absolute-pathname>: <absolute-pathname> replace-extension (ext <string>)
— Method on <absolute-pathname>: <absolute-pathname> replace-extension (source <absolute-pathname>)

Return a pathname record representing the same pathname of this instance, but with the extension replaced with the one from source, which can be a bytevector, string or pathname. The bytevector and string ext arguments must represent only the extension, leading dot byte included.

When the instance represents the root directory: the returned value can be the instance itself or a new pathname record representing the root directory; in all the other cases the return value is a new pathname record.

If source has no extension: raise an exception violation with condition types: &pathname, &error, &who, &message, &irritants.

This method is implemented by the replace-extension generic function from (nausicaa pathnames methods); it must be implemented by concrete subclasses of <absolute-pathname>. 

          #!r6rs
          (import (nausicaa)
            (nausicaa pathnames unix))
          
          (let (((o <absolute-pathname>)
                (pathname "/a/b/file.ext")))
             (is-a? (o.replace-extension ".zip")
                    <absolute-pathname>))
          ⇒ #t
          
          (define (doit input)
            (let* (((o <absolute-pathname>)
                    (pathname input))
                   ((p <pathname>)
                    (o.replace-extension ".zip")))
              p.string))
          
          (doit "/a/b/file.ext")  ⇒ "/a/b/file.zip"
          (doit "/a/b/file")      ⇒ "/a/b/file.zip"
          (doit "/a/b/file.")     ⇒ "/a/b/file.zip"
          (doit "/a/b/.")         ⇒ "/a/b.zip"
          (doit "/a/b/..")        ⇒ "/a.zip"
          (doit "/")              ⇒ "/"
          (doit "/a/b/.file.ext") ⇒ "/a/b/.file.zip"
          (doit "/a/b/.file")     ⇒ "/a/b/.file.zip"
          
          (let* (((o <absolute-pathname>)
                  (pathname "/a/b/file.ext"))
                 ((p <pathname>)
                  (o.replace-extension (pathname "/d/e/other.zip"))))
            p.string)
          ⇒ "/a/b/file.zip"
