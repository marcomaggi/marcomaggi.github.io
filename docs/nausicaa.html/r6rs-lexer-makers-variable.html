74.1.3.3 Variable–semantic–value token–makers

The following bindings are exported by the library (nausicaa r6rs lexer).

— Function: make-line-comment-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a line comment with line–ending included, ‘; ---’. The category is set to the symbol LINECOMMENT; the semantic value is set to the string representing the comment, line ending included.

— Function: make-line-comment-noend-token yygetc yyungetc yytext yyline yycolumn yyoffset

The next character from the input system is peeked, if it is the end of file: return a <lexical-token> record representing a line comment without line–ending. The category is set to the symbol LINECOMMENT; the semantic value is set to the string representing the comment.

If the next character from the input system is not the end of file: apply the function referenced by the lexical-error-token-maker parameter to the same arguments given to this function and return its result.

If in the lexer table there is no error in the regular expression matching a comment without line ending: the next character from the input stream is always the end of file.

— Function: make-white-space-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a white space. The category is set to the symbol WHITESPACE; the semantic value is set to the string representing the blank characters.

— Function: make-line-ending-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a line ending. The category is set to the symbol LINEENDING; the semantic value is set to the string representing the line ending character or characters.

— Function: make-sharp-bang-token yygetc yyungetc yytext yyline yycolumn yyoffset

Return a <lexical-token> record representing a sharp–bang comment, ‘#!---’. The category is set to the symbol SHARPBANG; the semantic value is set to a string whose prefix is #! followed by the identifier.
