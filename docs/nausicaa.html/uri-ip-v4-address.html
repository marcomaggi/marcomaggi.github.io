11.1.4.1 IPv4 address objects class

The following bindings are exported by the library (nausicaa uri ip).

— Class: <ipv4-address>

Represents an IPv4 address. It is derived from <ip-numeric-address> as defined by (nausicaa uri ip), Numeric IP address object.

Constructors and makers
— Common constructor on <ipv4-address>: constructor numbers
— Common constructor on <ipv4-address>: constructor third second first zeroth

Build and return a new object of type <ipv4-address>. When applied to a single argument: numbers must be a vector of 4 fixnums in the range [0, 255]. When applied to 4 arguments: each one must be a fixnum in the range [0, 255].

— Maker on <ipv4-address>: <ipv4-address> (?numbers)
— Maker on <ipv4-address>: <ipv4-address> (?third ?second ?first ?zeroth)

Build and return a new object of type <ipv4-address> using the common constructor. 

          (<ipv4-address> O
            (<> ((parse-ipv4-address-only "1.2.3.4"))))
          
          (list (O third) (O second) (O first) (O zeroth))
          ⇒ (1 2 3 4)
Fields
— Immutable field of <ipv4-address>: <ipv4-address-fixnum> zeroth
— Immutable field of <ipv4-address>: <ipv4-address-fixnum> first
— Immutable field of <ipv4-address>: <ipv4-address-fixnum> second
— Immutable field of <ipv4-address>: <ipv4-address-fixnum> third

The numeric components of the address, each representing 8 bits. zeroth is the rightmost component, first is the second from the right, etc.

— Immutable virtual field of <ipv4-address>: <boolean> private?

True if the address' prefix is 10.0.0.0/8 or 172.16.0.0/12 or 192.168.0.0/16. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> loopback?

True if the address' prefix is 127.0.0.0/8. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> localhost?

True if the address is 127.0.0.1. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> link-local?

True if the address' prefix is 169.254.0.0/16. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> reserved?

True if the address' prefix is 192.0.0.0/24 or 240.0.0.0/4. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> test-net-1?

True if the address' prefix is 192.0.2.0/24. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> six-to-four-relay-anycast?

True if the address' prefix is 192.88.99.0/24. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> benchmark-tests?

True if the address' prefix is 198.18.0.0/15. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> test-net-2?

True if the address' prefix is 198.51.100.0/24. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> test-net-3?

True if the address' prefix is 203.0.113.0/24. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> multicast?

True if the address' prefix is 224.0.0.0/4. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable virtual field of <ipv4-address>: <boolean> limited-broadcast?

True if the address prefix is 255.255.255.255. See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

Multimethod implementations
— Multimethod: ip-address->string (addr <ipv4-address>)

Build and return a string representation of the address in the format specified for URIs by RFC 3986.

— Multimethod: ip-address->bytevector (addr <ipv4-address>)

Build and return a bytevector representation of the address in the format specified for URIs by RFC 3986.

— Multimethod: ip-address->bignum (addr <ipv4-address>)

Build and return an exact integer representation of the address.
