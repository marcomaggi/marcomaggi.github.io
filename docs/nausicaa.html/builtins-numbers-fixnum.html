5.15.10 Fixnum number objects

— Builtin Label: <fixnum>

Conventional type for objects satisfying fixnum?. It is derived from <exact-integer>.

— Builtin Label: <nonzero-fixnum>

Conventional type for objects satisfying fixnum? but not fxzero?. It is derived from <fixnum>.

— Builtin Label: <nonpositive-fixnum>

Conventional type for objects satisfying fixnum? and fxnonpositive?. It is derived from <fixnum>.

— Builtin Label: <nonnegative-fixnum>

Conventional type for objects satisfying fixnum? and fxnonnegative?. It is derived from <fixnum>.

— Builtin Label: <positive-fixnum>

Conventional type for objects satisfying fixnum? and fxpositive?. It is derived from <nonnegative-fixnum>.

— Builtin Label: <negative-fixnum>

Conventional type for objects satisfying fixnum? and fxnegative?. It is derived from <nonpositive-fixnum>.

Methods: conversion
— Method on <fixnum>: string
— Method on <fixnum>: string base
— Method on <fixnum>: $string
— Method on <fixnum>: $string base

Calling string is like applying to the instance number the function fixnum->string from (vicare). Calling $string does the same without validating the argument.

base must be one among 2, 8, 10, 16; when not given it defaults to 10.

— Method on <fixnum>: flonum
— Method on <fixnum>: $flonum

Calling flonum is like applying to the instance number the function fixnum->flonum from (rnrs arithmetic fixnums (6)). Calling $flonum does the same without validating the argument.
