24.1.4 Defining constructor protocols

The following constructor descriptors are be built by define-class: the public constructor is used when instantiating a class through the make* macro; the superclass constructor is used when instantiating a subclass, it is the function handed as argument to the subclass' protocol function; the from–fields constructor is automatically generated to be used by the make-from-fields macro and it builds instances from raw field values.

Additionally another constructor can be optionally defined: the maker constructor to be used by the maker specified with the maker clause and the high–level macro make; classes define maker for details on the maker constructor.

The superclass and public constructors are built using the parent's superclass constructor descriptor:

  • When inheritance is specified through the inherit clause: the superclass constructor is acquired with class-superclass-constructor-descriptor applied to the superclass name (or an equivalent operation).
  • When inheritance is specified through the parent clause: the superclass constructor is acquired with record-constructor-descriptor applied to the record type name.
  • When inheritance is specified through the parent-rtd clause: the superclass constructor is the one directly specified in the clause.
— Auxiliary Syntax: protocol expression

Select the default constructor protocol function for all the other constructors; when not used: the default protocol is the one automatically provided by make-record-constructor-descriptor when invoked with the protocol argument set to #f.

— Auxiliary Syntax: public-protocol expression

Select the public constructor's protocol function, overriding the default one.

— Auxiliary Syntax: superclass-protocol expression

Select the superclass constructor's protocol function, overriding the default one.
