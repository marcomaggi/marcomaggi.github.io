3.6.2 Labeled values construction protocols

Labeled values have explicitly defined construction protocols; the semantics of protocol functions is different from the one defined by R6RS for record protocols. The following constructor functions are built by define-label:

Public constructor
It is used when building a labeled value through the maker syntax. The public constructor is a standalone function: it does not invoke the constructor of the superlabel or superclass. It is defined with the public-protocol clause.
Common constructor
When the public constructor is not explicitly defined: the common constructor is used as public constructor. The common constructor is a standalone function: it does not invoke the constructor of the superlabel or superclass. It is defined with the protocol clause.

When no common constructor is explicitly defined: a default function is created accepting any number of arguments and always raising an &assertion exception.

Label protocol functions must accept no arguments and return a constructor function.

— Auxiliary Syntax: protocol ?expression

Define the common constructor's protocol function. ?expression is evaluated only if the common protocol is used and it must evaluate to the protocol function.

— Auxiliary Syntax: public-protocol ?expression

Define the public constructor's protocol function. ?expression must evaluate to the protocol function.

Examples of building labeled values

Here is an example of common protocol definition for a labeled value: 

     (define-label <the-pair>
       (protocol (lambda () cons))
       (virtual-fields (immutable car car)
                       (immutable cdr cdr)))
     
     (<the-pair> P (<> (1 2)))
     (P car)                 ⇒ 1
     (P cdr)                 ⇒ 2
