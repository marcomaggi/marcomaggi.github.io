28.1.9.3 Determining the value of expressions

It is often needed to know the value of a C language expression. The following acquires the value of ‘INT_MAX’ (defined in limits.h) and assigns it to the output variable ‘VALUEOF_INT_MAX’: 

     NAUSICAA_VALUEOF_TEST([INT_MAX],[INT_MAX])

while the following acquires the offset of the field ‘l_start’ in the structure ‘struct flock’ (defined in fcntl.h): 

     NAUSICAA_VALUEOF_TEST([OFFSETOF_STRUCT_FLOCK_L_START],
        [offsetof(struct flock,l_start)])

the ‘offsetof’ operator is a C preprocessor macro, which can be defined as: 

     #define offsetof(TYPE,FIELD)            \
                                             \
       ((void *) &(((TYPE) *) 0)->(FIELD))

when missing in the C library's headers.

— Autoconf Macro: NAUSICAA_VALUEOF_TEST (suffix, expr, default, headers)

A wrapper for the Autoconf macro ‘AC_COMPUTE_INT’ that acquires the value of a C language expression, which must be an integer.

The output variable ‘VALUEOF_<suffix>’ is set to the result. If the test fails: The value of the output variable will be default, or #f if default is not given.

The optional headers must be a chunk of code including header files. By default the expansion of ‘NAUSICAA_INCLUDES_DEFAULT’ is included in the test program.

— Autoconf Macro: NAUSICAA_ENUM_VALUE (symbol, headers)

Wrapper for NAUSICAA_VALUEOF_TEST which expands to: 

          NAUSICAA_VALUEOF_TEST([symbol],[symbol],,[headers])
— Autoconf Macro: NAUSICAA_DEFINE_VALUE (symbol, headers)

Wrapper for NAUSICAA_VALUEOF_TEST which expands to: 

          NAUSICAA_VALUEOF_TEST([symbol],[symbol],,[headers])
