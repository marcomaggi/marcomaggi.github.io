12.2.1 Overview of lexical tokens

Lexical token records are meant to store informations about a token from the input stream; they should be produced by the lexer and consumed by the parser. For example, when the lexer tokenises the number 123 from the string ‘"123"’, it can build a <lexical-token> object with: 

     (import (nausicaa)
       (prefix (nausicaa parser-tools lexical-tokens)   lt.))
     
     (lt.<lexical-token>
        ((lt.category: 'NUMBER)
         (lt.value:    123)
         (lt.length:   3)))

while when tokenising a string it can do: 

     (lt.<lexical-token>
        ((lt.category: STRING)
         (lt.value:    "some text")
         (lt.length:   9)))

When the lexer has to signal the end–of–input, it must return a token with category ‘*eoi*’, and continue to return it if it is called again and again; it can do: 

     (lt.<end-of-input-token> ())

When the lexer closure has to signal a lexer error (an invalid character from the input stream), it must return a token with category ‘*lexer-error*’: 

     (lt.<lexer-error> ((lt.error-message: "wrong input")))
