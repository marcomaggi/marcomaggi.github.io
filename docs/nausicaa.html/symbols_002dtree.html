24 Search tree for lists of symbols

The library (nausicaa language symbols-tree) handles search trees in which keys are lists of Scheme symbols and values can be anything; the tree has the structure of nested alists. Inserting the following keys and values (in this order): 

     (a b1 c1 d1)   1
     (a b1 c1 d2)   2
     (a b1 c2 d1)   1
     (a b1 c1 d3)   3
     (a b2 c1)      4
     (a b1 c2 d2)   2
     (a b2 c2)      5
     (a b1 c2 d3)   3
     (a b2 c3)      6
     (a b2)         7
     (a b1 c2)      8

yields the following tree: 

     ((a . ((b2 . ((#f . 7)
                   (c3 . ((#f . 6)))
                   (c2 . ((#f . 5)))
                   (c1 . ((#f . 4)))))
            (b1 . ((c2 . ((#f . 8)
                          (d3 . ((#f . 3)))
                          (d2 . ((#f . 2)))
                          (d1 . ((#f . 1)))))
                   (c1 . ((d3 . ((#f . 3)))
                          (d2 . ((#f . 2)))
                          (d1 . ((#f . 1))))))))))

notice that the keys are stored as sequences ending with a pair having #f as key, and such pairs are kept at the beginning of the alist.

Storing a key/value pair whose key already exists causes the old value to be overwritten.

— Function: tree-cons key value tree
— Function: tree-cons key value tree overwrite?

Add the key/value pair to tree and return the new tree. key must be null or a list of symbols, value can be anything, tree must be null or a symbols tree.

If overwrite? is true and the key already exists: the old value is overwritten with value, else it is left unchanged. overwrite? defaults to #f.

— Function: treeq key tree default

Search for key in tree and return the associated value; return default if key was not found. key must be null or a list of symbols, tree must be null or a symbols tree.

— Function: make-tree-iterator tree

Return an iterator closure upon tree. Whenever evaluated, the iterator returns three values: a boolean, false if the iteration is over; a list of symbols representing the reversed key for this tree element; this element's value.

Iterating over a tree means; composing a sequence of keys and values; for the following simple tree: 

          ((#f . 1)
           (a . ((#f . 2)))
           (b . ((c1 . ((#f . 3)))
                 (c2 . ((#f . 4))))))

iterating means generating the following sequence of key/value couples: 

          ()      1
          (a)     2
          (b c1)  3
          (b c3)  4

for implementation reasons the iterator returns the keys as reversed lists: 

          ()      1
          (a)     2
          (c1 b)  3
          (c3 b)  4

but this is not a big problem.

— Function: tree-merge dst src
— Function: tree-merge dst src overwrite?

Merge the src tree into the dst tree without mutating src; dst is mutated.

If overwrite? is true and a key from src already exists in dst: the old value in dst is overwritten with the new value from src, else it is left unchanged. overwrite? defaults to #f.
