72.2.3 Processing exact integers

The following bindings are exported by the (nausicaa standard-numbers luhn low) library and reexported by the (nausicaa standard-numbers luhn) library.

— Function: number->luhn-check-digit integer

Compute and return the check digit for the exact integer number integer. 

          (number->luhn-check-digit 54328)        ⇒ 0
          (number->luhn-check-digit 4992739871)   ⇒ 6
          (number->luhn-check-digit 302)          ⇒ 0
          (number->luhn-check-digit 312)          ⇒ 9
— Function: number->luhn-envelope integer

Compute the check digit for the exact integer number integer and return a new exact integer number representing integer with the checksum added: 

          (+ (* 10 integer) check)

Examples: 

          (number->luhn-envelope 54328)           ⇒ 543280
          (number->luhn-envelope 4992739871)      ⇒ 49927398716
          (number->luhn-envelope 302)             ⇒ 3020
          (number->luhn-envelope 312)             ⇒ 3129
— Function: luhn-number-envelope? integer

Return #t if the exact integer number integer is valid according to the Luhn checksum. 

          (luhn-number-envelope? 543280)          ⇒ #t
          (luhn-number-envelope? 543281)          ⇒ #f
