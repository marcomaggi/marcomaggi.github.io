5.17.2 Input and output port labels

— Builtin Label: <input-port>

Conventional type for objects satisfying input-port?. It is derived from <port>.

— Builtin Label: <output-port>

Conventional type for objects satisfying output-port?. It is derived from <port>.

— Builtin Label: <input/output-port>

Conventional type for objects satisfying input/output-port?. It is derived from <port>.

Safe accessors and mutators
— Immutable virtual field of <input-port>: <boolean> eof?
— Immutable virtual field of <input/output-port>: <boolean> eof?

Accessing this field is like applying to the port the function port-eof? from (rnrs io ports (6)).

— Mutable virtual field of <output-port>: <symbol> buffer-mode
— Mutable virtual field of <input/output-port>: <symbol> buffer-mode

Accessing and mutating this field is like applying to the port, respectively, the functions output-port-buffer-mode, from (rnrs io ports (6)), and set-port-buffer-mode!, from (vicare).

Input ports methods

For all the following input methods: if EOF is found and no data is available: return the EOF object; if the underlying device is in non–blocking mode and no data is available: return the would–block object.

— Method on <input-port>: get-single
— Method on <input/output-port>: get-single

Read, consume and return a single data unit from the port. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <input-port>: lookahead-single
— Method on <input/output-port>: lookahead-single

Read, without consuming and return a single data unit from the port. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <input-port>: get-multi-n count
— Method on <input/output-port>: get-multi-n count

Read, consume and return count data units from the port. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <input-port>: get-multi-n! storage start count
— Method on <input/output-port>: get-multi-n! storage start count

Read and consume count data units from the port and store them in the storage starting at offset start. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <input-port>: get-multi-some
— Method on <input/output-port>: get-multi-some

Read, consume and return multiple data units from the port. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <input-port>: get-multi-all
— Method on <input/output-port>: get-multi-all

Read, consume and return all the data from the port. It is the responsibility of subclasses to provide an implementation for this method.

Output ports methods
— Method on <output-port>: flush
— Method on <input/output-port>: flush

Calling this method is like applying to the port the function flush-output-port from (rnrs io ports (6)).

— Method on <output-port>: put-single storage
— Method on <input/output-port>: put-single storage

Write storage to the port. It is the responsibility of subclasses to provide an implementation for this method.

— Method on <output-port>: put-multi storage
— Method on <output-port>: put-multi storage start
— Method on <output-port>: put-multi storage start count
— Method on <input/output-port>: put-multi storage
— Method on <input/output-port>: put-multi storage start
— Method on <input/output-port>: put-multi storage start count

Write count units of data from storage to the port, starting at offset start. It is the responsibility of subclasses to provide an implementation for this method.
