3.17 Accessors and mutators

The following syntaxes provide a way to access concrete and virtual fields of class instances and labeled values; they also allow the extraction of accessor and mutator functions.

— Syntax: slot-ref ?expr ?field-name ?tag
— Auxiliary Syntax: <>

Expand to the slot accessor of ?field-name applied to ?expr, which can be any expression evaluating to an instance of ?tag, a class or label type. ?field-name can be either the name of a concrete field or the name of a virtual field.

When the auxiliary syntax <> is used in place of ?expr: the macro expands to the accessor function itself; the following forms are equivalent: 

          (slot-ref o car <pair>)
          ((slot-ref <> car <pair>) o)

this is useful whenever the accessor has to be used as function argument, for example for map or for-all. The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language oopp).

— Syntax: slot-set! ?obj-expr ?field-name ?tag ?val-expr
— Auxiliary Syntax: <>

Expand to the slot mutator of ?field-name applied to ?obj-expr and ?val-expr. ?obj-expr can be any expression evaluating to an instance of ?tag, a class or label type; ?val-expr can be any expression evaluating to the new field value. ?field-name can be either the name of a concrete field or the name of a virtual field.

When the auxiliary syntax <> is used in place of both ?obj-expr and ?val-expr: the macro expands to the mutator function itself; the following forms are equivalent: 

          (slot-set! o car <pair> 1)
          ((slot-set! <> car <pair> <>) o 1)

this is useful whenever the mutator has to be used as function argument. The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language oopp).
