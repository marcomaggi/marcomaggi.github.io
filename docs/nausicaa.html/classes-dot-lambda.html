24.7.3 Function bindings

The following macros integrate the facilities of the with-class macro with the function creation facilities of the lambda and case-lambda syntaxes from (rnrs base (6)).

NOTE It is possible to use the facilities of the (nausicaa contracts) library to assert that the type of arguments to the functions is the one selected by the class identifiers. Merging the features of contracts along with the function construction macros may result in inflexible code or bloated code after the expansion; so, for the time being, this merging is not implemented.
— Macro: lambda/with-class formals body

Like lambda but encloses the body into a with-class syntax defining identifier syntaxes to access the fields and virtual fields of the arguments. formals must have the same form of the formals for lambda, but each variable argument can be an identifier, as in lambda, or a list of the form: 

          (variable tag ...)

in which case the variable argument is meant to be of type tag and the corresponding identifier syntaxes are defined. When an argument as no class specifiers: we can use <top> as its conventional class name, or let lambda/with-class select <top> automatically. 

          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (mutable numerator)
                            (mutable denominator)))
          
          (define <fraction>-numerator
            (lambda/with-class ((o <fraction>))
              (numerator o.number)))
          
          (define <fraction>-numerator-set!
            (lambda/with-class ((o <fraction>) v)
              (set! o.number (/ v (denominator o.number)))))
          
          (define <fraction>-denominator
            (lambda/with-class ((o <fraction>))
              (denominator o.number)))
          
          (define <fraction>-denominator-set!
            (lambda/with-class ((o <fraction>) v)
              (set! o.number (/ (numerator o.number) v))))
          
          (let/with-class (((o <fraction>) (make <fraction> 2/3)))
            o.numerator)
          ⇒ 2

— Macro: define/with-class variable expression
— Macro: define/with-class variable

Like define but when used in the first and second forms it expands to the use of lambda/with-class. 

          (define-class <fraction>
            (fields (mutable number))
            (virtual-fields (mutable numerator)
                            (mutable denominator)))
          
          (define/with-class (<fraction>-numerator (o <fraction>))
            (numerator o.number))
          
          (define/with-class (<fraction>-numerator-set! (o <fraction>) v)
            (set! o.number (/ v (denominator o.number))))
          
          (define/with-class (<fraction>-denominator (o <fraction>))
            (denominator o.number))
          
          (define/with-class (<fraction>-denominator-set! (o <fraction>) v)
            (set! o.number (/ (numerator o.number) v)))
          
          (let/with-class (((o <fraction>) (make <fraction> 2/3)))
            o.numerator)
          ⇒ 2
— Macro: Syntax case-lambda/with-class case-lambda-clause ...

The natural extension of lambda/with-class.

— Macro: receive/with-class formals expression . body

Like receive but expands to lambda/with-class rather than lambda: 

          (define-syntax receive/with-class
            (syntax-rules ()
              ((_ ?formals ?expression ?form0 ?form ...)
               (call-with-values
                   (lambda () ?expression)
                 (lambda/with-class ?formals ?form0 ?form ...)))))
