11.1.8 Miscellaneous definitions

The following bindings are exported by the library (nausicaa uri ip).

— Label: <ipvfuture-version-flag>

Type of exact integers representing version flags for future representations of IP addresses. It is derived from <fixnum>. The version flag must be in the range [0, 15].

— Function: make-host-object type (ascii <bytevector>) data

Build and return a new instance of a specialised object of class <ip-address> representing the host component of a URI. The possible classes of the returned object are: <reg-name-address>, <ipv4-address>, <ipv6-address>, <ipvfuture-address>. As a special case if host.type is #f: the other arguments are ignored and the return value is the value unspecified from (nausicaa).

type a symbol among: reg-name, ipv4-address, ipv6-address, ipvfuture.

ascii must be a bytevector representing the ASCII representation of the host component.

data must be auxiliary data representing the host component:

  • For reg-name: data is undefined and unused.
  • For ipv4-address: data must be a vector of 4 exact integers representing the address components.
  • For ipv6-address: data must be a vector of 8 exact integers representing the address components.
  • For ipvfuture: data must be an exact integer representing the version number of the IP address literal representation.

The arguments are modeled after the 3 return values of parse-host from the library (nausicaa parser-tools uri).
