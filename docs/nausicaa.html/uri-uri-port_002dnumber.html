11.2.4 URI port–number objects

The following bindings are exported by the (nausicaa uri uri) library.

— Label: <port-number>

Label used to tag fixnums representing the port-number component of URIs; it is derived from <nonnegative-fixnum>.

— Constructor on <port-number>: constructor obj

Just return the given argument. If tagged argument validation is enabled: validate the argument with the predicate of <port-number>.

— Maker on <port-number>: <port-number> (?obj)

Filter the given argument using the label constructor.

— Predicate on <port-number>: predicate obj

Return #t if obj is a non–negative fixnum in the range [0, 65535].

Fields
— Immutable virtual field of <port-number>: <ascii-bytevector> bytevector

A bytevector representing the port-number component in the format requested by RFC 3986.

— Immutable virtual field of <port-number>: <ascii-string> string

A string representing the port-number component in the format requested by RFC 3986.

Methods
— Method on <port-number>: put-bytevector (port <binary-output-port>)

Write to port a bytevector representing the port-number component in the format requested by RFC 3986.
