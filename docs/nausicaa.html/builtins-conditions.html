5.20 Condition labels

— Builtin Label: <condition>

Conventional type for objects satisfying condition?.

— Common constructor on <condition>: <condition> constructor condition ...

Return the result of applying condition to the arguments condition.

Safe accessors and mutators
— Immutable virtual field of <condition>: message

Accessing this field is like applying condition-message to the condition object.

— Immutable virtual field of <condition>: who

Accessing this field is like applying condition-who to the condition object.

— Immutable virtual field of <condition>: irritants

Accessing this field is like applying condition-irritants to the condition object.

Methods
— Method on <condition>: simple

Return the result of applying simple-conditions to the instance.

OOPP syntaxes
— Maker on <condition>: <condition> (?condition ...)

Build and return a new condition object applying the common constructor to the result of evaluating the ?condition expressions.

— Getter on <condition>: instance (?condition-type)

Value getter. Equivalent to applying hashtable-ref to instance; return the value associated to ?key. If ?default is used: it must be an expression whose value is returned when the requested key is not in the table.
