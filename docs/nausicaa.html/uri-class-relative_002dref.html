70.6.2 The <relative-ref> class

The following bindings are exported by the (nausicaa uri) library.

— Class: <relative-ref>

Represents a parsed relative-ref component.

— Mutable field of <relative-ref>: authority

The authority component as a bytevector, or false if there is no authority component. Percent–encoded characters are not decoded in this field.

— Mutable field of <relative-ref>: userinfo

The userinfo component as a percent–decoded bytevector not containing the trailing at character, or false if there is no userinfo component.

— Mutable field of <uri>: host-type

A Scheme symbol among: reg-name, ipv4-address, ipv6-address, ipvfuture; when the host is empty: this value is reg-name.

— Mutable field of <uri>: host

Host data represented as the second return value from parse-host and associated to the symbol above. When host-type is reg-name: the value is a percent–decoded bytevector.

— Mutable field of <relative-ref>: port

The port component as a bytevector, or false if no port is present. Remember that port does not contain percent–encoded characters.

— Mutable field of <relative-ref>: path-type

A Scheme symbol among: path-abempty, path-empty, path-noscheme, path-absolute; represents the type of the path field.

— Mutable field of <relative-ref>: path

A possibly empty list holding percent–decoded bytevectors representing the path segments.

— Mutable field of <relative-ref>: query

The query component as a percent–decoded bytevector not containing the leading question mark, or false if there is no query component.

— Mutable field of <relative-ref>: fragment

The fragment component as a percent–decoded bytevector not containing the leading number–sign, or false if there is no fragment component.

— Immutable virtual field of <relative-ref>: bytevector

A bytevector representing the whole URI.

— Immutable virtual field of <relative-ref>: string

A string representing the whole URI; it is built from the bytevector field with the to-string function.

— Syntax on <relative-ref>: make* <relative-ref> ?source-bytevector

Build and return a new <relative-ref> instance holding fields from parsing source-bytevector; it relies on parsing with parse-uri.

— Syntax on <relative-ref>: make <relative-ref> ?clause ...
— Auxiliary Syntax on <relative-ref>: source-bytevector ?expr

Build and return a new <relative-ref> instance holding fields from parsing expr, which must be a bytevector; it relies on parsing with parse-uri.

— Syntax on <relative-ref>: is-a? expr <relative-ref>

Return true if expr evaluates to an instance of <relative-ref>.
