10.6.3 Queue accessors and mutators

The following bindings are exported by the library (nausicaa containers queues). The bindings whose name is prefixed with $ are unsafe operations: they do not validate their arguments before accessing them.

— Method on <queue>: front
— Method on <queue>: $front

Return the object at the front of the queue. Raise an assertion violation if the instance is empty.

— Method on <queue>: rear
— Method on <queue>: $rear

Return the object at the rear of the queue. Raise an assertion violation if the instance is empty.

— Method on <queue>: push! obj
— Method on <queue>: $push! obj

Push obj on the <queue>.

— Method on <queue>: pop!
— Method on <queue>: $pop!

Remove the object at the front of the <queue> and return it. Raise an assertion violation if queue is empty.
