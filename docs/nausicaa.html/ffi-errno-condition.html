28.4.4 Raising exceptions

— Condition: &errno

Type of condition objects that represent an error in a foreign function signaled with the ‘errno’ variable. It is derived from &error.

— Condition Constructor: make-errno-condition errno-numeric-value errno-symbolic-value
— Condition Predicate: errno-condition? object
— Field Accessor: errno-numeric-value object
— Field Accessor: errno-symbolic-value object

Functions to handle foreign system error condition objects.

Fake example: 

          (let ((errno ---))
            (raise
              (condition
                (make-who-condition 'read)
                (make-message-condition (strerror errno))
                (make-errno-condition errno
                  (errno->symbol/or-error errno)))))
— Function: make-errno-condition* errno-numeric-value

Wrapper for make-errno-condition that builds an &errno condition from the numeric value only.

— Function: raise-errno-error who errno-numeric-value
— Function: raise-errno-error who errno-numeric-value irritants

Raise a non–continuable exception representing a foreign function system error signaled through the ‘errno’ variable. If irritants is not given, the condition object has no &irritants component.

The following form: 

          (let ((errno ---)
                (pathname ---))
            (raise-errno-error 'write errno pathname))

is equivalent to: 

          (raise
            (condition (make-who-condition 'write)
                       (make-message-condition (strerror errno))
                       (make-errno-condition* errno)
                       (make-irritants-condition pathname)))
— Function: raise-errno-error/continuable who errno-numeric-value
— Function: raise-errno-error/continuable who errno-numeric-value irritants

Like raise-errno-error, but raise a continuable exception. This is occasionally useful with particular error codes like ‘E2BIG’.
