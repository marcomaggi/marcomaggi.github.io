46.7 Date object

A date is a timestamp on the UTC scale expressed in the proleptic gregorian calendar coordinates; it is defined by the tuple: year, month, day, hours, minutes, seconds, nanoseconds, time zone offset in seconds accounting also for Daylight Saving Time, a boolean value signaling if the Daylight Saving Time is on.

<date> objects are immutable once created. Notice that to do operations with dates, like the arithmetic ones, we have to convert the <date> objects to <time> objects.

The following bindings are exported by the (nausicaa times-and-dates) library.

— Class: <date>

Type representing a timestamp in the proleptic gregorian calendar coordinates, on the UTC scale.

— Immutable Field of <date>: year

An exact integer representing the year in the local time. There is almost no limit to the range of allowed values, but the number must be encoded according to the following rules:

  • Years after the conventional birth of Christ are positive.
  • Years before the conventional birth of Christ are negative.
  • The value zero is forbidden because the between 1 Annus Dominis and 1 Before Christ there is no year zero.

Also notice that to convert a <date> instance to a <time> instance, the year must be greater than or equal to -4714, that is past 4714 Before Christ.

— Immutable Field of <date>: month

An exact integer in the range [1, 12] representing the month in the local time; in which 1 means January, 2 February, and so on.

— Immutable Field of <date>: day

An exact integer in one of the ranges:

[1, 31]
In January (1), March (3), May (5), July (7), August (8), October (10), December (12).
[1, 30]
In April (4), June (6), September (9), November (11).
[1, 28]
In February (2) of non–leap years.
[1, 29]
In February (2) of leap years.

representing the day number in the local time.

— Immutable Field of <date>: hour

An exact integer in the range [0, 23] representing the hours count in the local time.

— Immutable Field of <date>: minute

An exact integer in the range [0, 59] representing the minutes count in the local time.

— Immutable Field of <date>: second

An exact integer usually in the range [0, 59], but in the range [0, 60] when the day has a leap second inserted, representing the seconds count in the local time.

— Immutable Field of <date>: nanosecond

An exact integer in the range [0, 999999999] representing the nanoseconds count in the local time.

— Immutable Field of <date>: zone-offset

An exact integer representing both the number of seconds East of GMT for the local time zone and the offset introduced by Daylight Saving Time. To compute the timestamp in time zone zero without Daylight Saving Time: this offset must be subtracted from the other fields.

— Immutable Field of <date>: daylight-saving-time?

A boolean, true if Daylight Saving Time is on.

Other fields and methods
— Immutable Virtual Field of <date>: year-day

Return the ordinal day of the year of this date; January 1 is 1, January 2 is 2, February 1 is 32, and so on.

— Immutable Virtual Field of <date>: week-day

Return the day of the week of this date; Sunday is 0, Monday is 1, and so on.

— Method on <date>: week-number day-of-week-starting-week

Return the ordinal week of the year which holds this date, ignoring a first partial week. day-of-week-starting-week is the integer corresponding to the day of the week which is to be considered the first day of the week (Sunday=0, Monday=1, etc.).

— Method on <date>: leap-year?

Return true if the year in the date is a leap year.

— Method on <date>: number-of-days-since-year-beginning

Return the number of days since the beginning of the year.

Traditional record API
— Function: make-<date> nanosecond second minute hour day month year zone-offset

Creates a date object.

— Function: <date>? obj

Return true if obj is a <date> instance.
