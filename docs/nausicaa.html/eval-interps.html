64.1 Evaluating code as in an interpreter

The (nausicaa eval interps) library exports an interface to (rnrs eval (6)) which allows evaluation of code in chunks with retention of global variables; (nausicaa eval interps) uses the facilities of (nausicaa language classes). Evaluation of code is handled by “interpreter” objects; examples: 

     (import (nausicaa)
       (nausicaa eval interps))
     
     (let (((o <interp>) (make <interp>
                           (imports: '((rnrs))))))
       (o.eval '(begin
                  (define a 1)
                  (define b 2)
                  (+ a b))))
     ⇒ 3
     
     (let (((o <interp>) (make <interp>
                           (imports: '((rnrs))))))
       (o.eval '(begin
                  (define-global a 1)
                  (values)))
       (o.eval '(+ a 2)))
     ⇒ 3
