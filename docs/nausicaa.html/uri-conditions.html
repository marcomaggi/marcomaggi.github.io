69.2 Condition object types

The following bindings are exported by the (nausicaa uri conditions) library and reexported by the (nausicaa uri) library; it is suggested to prefix the bindings when importing the library.

— Condition Type: &parser-error

Tag a compound condition object as being an URI parser error; it is derived from &error. It has the following fields:

offset
Zero–based offset of the byte which caused the error from the beginning of the input. Whether it is the offset from the beginning of a full URI or URI reference or other input depends on the function raising the exception.

— Function: make-parser-error offset

Build and return a new condition object of type &parser-error.

— Function: parser-error-condition? obj

Return true if obj is a condition object of type &parser-error.

— Function: condition-parser-error/offset obj

Getter for the field of condition objects of type &parser-error.
