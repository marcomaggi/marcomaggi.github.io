14 Parameter objects

The following bindings are exported by the (nausicaa language parameters) library.

— Function: make-parameter init
— Function: make-parameter init converter

Return a new parameter object which is associated to a mutable storage location containing the value init. The optional argument converter is meant to be a procedure accepting one argument and returning one value; when not given: the identity procedure is used. converter is not applied to init when initialising the location.

The parameter object is a procedure which accepts zero, one or two arguments:

  • When it is called with no argument, the content of location in the current dynamic environment is returned.
  • When it is called with one argument arg, the content of the location in the current dynamic environment is set to the result of the call (converter arg), and an unspecified value is returned.
  • When it is called with two arguments, the first being arg and the second being bool, the content of the location in the current dynamic environment is set to:
    • When bool is false: arg itself.
    • When bool is true: the result of the call (converter arg).

    an unspecified value is returned.

Examples: 

          (define radix
            (make-parameter 10))
          
          (radix)         ⇒ 10
          (radix 2)
          (radix)         ⇒ 2
— Syntax: parameterize ((expr1 expr2) ...) . body
— Syntax: parameterise ((expr1 expr2) ...) . body
— Syntax: parametrise ((expr1 expr2) ...) . body

The expressions expr1 and expr2 are evaluated in an unspecified order. The value of the expr1 expressions must be parameter objects. For each expr1 expression and in an unspecified order, the local dynamic environment is extended with a binding of the parameter object expr1 to a new location whose content is the result of the call (converter val), where val is the value of expr2 and converter is the conversion procedure of the parameter object.

The resulting dynamic environment is then used for the evaluation of the forms in body. The result(s) of the parameterize form are the result(s) of the body.

This macro takes care of applying the converter procedures only to the new values of the parameters; when the old values are restored: the converter is not applied to the old value.

With reference to the example above: 

          (radix)
          ⇒  2
          
          (parameterize ((radix 16))
            (radix))
          ⇒  16
          
          (radix)
          ⇒  2
          
          (define (f n)
            (number->string n (radix)))
          
          (f 10)
          ⇒  "1010"
          
          (parameterize ((radix 8))
            (f 10))
          ⇒  "12"
