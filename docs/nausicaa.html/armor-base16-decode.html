66.2.3 Decoding byte streams with base16

— Class: <base16-decode-ctx>

Context descriptor for base16 decoding. Nothing in this record depends on the input or output data: records of this type can be reused for different streams, both in sequence and in parallel.

— Immutable Field of <base16-decode-ctx>: encoding-case

A Scheme symbol among: lower, mixed, upper. Select the case of the alphabetic characters in the encoded input data.

— Immutable Field of <base16-decode-ctx>: table

A Scheme vector of exact integers representing the selected decoding's table.

— Function: make-<base16-decode-ctx> encoding-case

Build and return a new record of type <base16-decode-ctx>.

encoding-case must be a Scheme symbol selecting the expected case of the input string: lower, upper, mixed.

— Function: <base16-decode-ctx>? obj

Return true if obj is a record of type <base16-decode-ctx>.

— Function: <base16-decode-ctx>-encoding-case ctx
— Function: <base16-decode-ctx>-table ctx

Field accessors for records of type <base16-decode-ctx>.

— Function: base16-decode-update-length number-of-chars

Return the minimum number of bytes required in the output bytevector when base16-decode-update! is applied to number-of-chars ASCII characters of input. Notice that the function will consume input in 2-bytes blocks only and produce output in single–byte blocks.

— Function: base16-decode-final-length number-of-chars
— Function: base16-decode-final-length number-of-chars padding?

Return the minimum number of bytes required in the output bytevector when base16-decode-final! is applied to number-of-chars ASCII characters of input. number-of-chars must be a multiple of 2, else the return value is #f.

— Function: base16-decode-length number-of-chars
— Function: base16-decode-length number-of-chars padding?

Return the minimum number of bytes required in the output bytevector when base16-decode-final! is applied to number-of-chars ASCII characters of input. This is also the number required when both base16-decode-update! and base16-decode-final! are used. number-of-chars must be a multiple of 2, else the return value is #f.

— Constant: base16-decode-block-length

The length of the input block: 2.

— Function: base16-decode-update! ctx dst-bv dst-start src-bv src-start src-past

Decode ASCII characters from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store binary data bytes in the bytevector dst-bv starting at dst-start (included); decoding is performed according to the specification in the context ctx, which must be a record of type <base16-decode-ctx>.

Return three values:

  1. A boolean always false. This value exists to make this API equal to the one of base64.
  2. The index of the next non–written byte in dst-bv; if dst-bv is filled to the end, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the characters from src-bv are consumed, this value is src-past.

ASCII characters are read from src-bv in chunks of 2 bytes; binary data bytes are written to dst-bv in single-byte blocks. If not enough room exists in dst-bv to store a block, available input data is not consumed.

— Function: base16-decode-final! ctx dst-bv dst-start src-bv src-start src-past

Decode ASCII characters from the bytevector src-bv starting at index src-start (included) up to index src-past (excluded); store binary data bytes in the bytevector dst-bv starting at dst-start (included); decoding is performed according to the specification in the context ctx, which must be a record of type <base16-decode-ctx>.

Return three values:

  1. A boolean: true if success, false if the output bytevector was filled before flushing all the bytes.
  2. The index of the next non–written byte in dst-bv; if dst-bv is filled to the end, this value is the length of dst-bv.
  3. The index of the next non–read byte in src-bv; if all the bytes from src-bv are consumed, this value is src-past.

ASCII characters are read from src-bv in chunks of 2 bytes; binary data bytes are written to dst-bv in single–byte blocks. If not enough room exists in dst-bv to store a block, available input data is not consumed.
