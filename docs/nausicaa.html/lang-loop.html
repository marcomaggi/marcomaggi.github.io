9.3 Loop constructs

— Syntax: do* ((<variable1> <init1> <step1>) ...) (<test> <expr> ...) <command> ...

Like do, but bind the variables as in let* rather than as in let. The <step> form is optional.

— Syntax: while <test> . <body>
— Syntax: until <test> . <body>
— Auxiliary Syntax: break
— Auxiliary Syntax: continue

For the while syntax: if <test> is true evaluate the <body> forms; then loop, if <test> is true...

For the until syntax: if <test> is false evaluate the <body> forms; then loop, if <test> is false...

In the body, we can use the auxiliary syntax break to jump out of the loop to the continuation of while or until. NOTE break breaks out of the innermost while or until loop.

In the body, we can use the auxiliary syntax continue to jump immediately to the next iteration skipping the following forms; NOTE continue jumps to the start of the innermost while or until loop.

Both the break and continue identifiers must appear in the same lexical context of the while or until identifier.

— Syntax: do-while <test> . <body>
— Syntax: do-until <test> . <body>
— Auxiliary Syntax: break
— Auxiliary Syntax: continue

For do-while: evaluate the <body> forms; then if <test> is true loop: evaluate the <body> ...

For do-until: evaluate the <body> forms; then if <test> is false loop: evaluate the <body> ...

In the body, we can use the auxiliary syntax break to jump out of the loop to the continuation of do-while or do-until; NOTE break breaks out of the innermost do-while or do-until loop.

In the body, we can use the auxiliary syntax continue to jump immediately to the next test step skipping the following forms; NOTE continue jumps to the end of the innermost do-while or do-until loop.

Both the break and continue identifiers must appear in the same lexical context of the do-while or do-until identifier.

— Syntax: let/cc <kont> . <body>

Wrapper for call/cc defined as: 

          (define-syntax let/cc
            (syntax-rules ()
              ((_ ?kont . ?body)
               (call/cc (lambda (?kont) . ?body)))))
— Syntax: dotimes (varname exclusive-count) form ...
— Syntax: dotimes (varname exclusive-count result) form ...

Loop evaluating the forms and assigning varname to an integer from zero inclusive to exclusive-count. Return result, or #f if result is not given.

Example: 

          (dotimes (i 100)
            (display i)
            (newline))

prints the integers in the range [0, 99].

— Syntax: dolist (varname list) form ...
— Syntax: dolist (varname list result) form ...

Loop evaluating the forms and assigning varname to the elements from list. Return result, or #f if result is not given.

Example: 

          (dolist (elm '(1 2 3))
            (display elm)
            (newline))

prints 1, 2, 3.

— Syntax: loop-upon-list (varname list) (break-when test) form ...
— Syntax: loop-upon-list (varname list result) (break-when test) form ...

Loop evaluating the forms and assigning varname to the elements from list. Return result, or #f if result is not given. The loop is broken if test evaluates to true.

Example: 

          (loop-upon-list (i '(1 2 3 4) 'retval)
              (break-when (= i 3))
            (display i)
            (newline))

prints 1 and 2, then returns retval.

— Syntax: recursion formals body

Allow the simple and non–imperative construction of self–referential expressions. The implementation is: 

          (define-syntax recursion
            (syntax-rules ()
              ((_ (?name . ?variables) . ?body)
               (letrec ((?name (lambda ?variables . ?body))) ?name))
              ((_ ?name ?expr)
               (letrec ((?name ?expr)) ?name))))

recursion generates an inner lambda using formals and body; the first identifier in formals is meant to be bound to the inner lambda itself. The return value is a function that invokes the inner lambda with the inner lambda and the given arguments.

For example the following: 

          (define (dummy n)
            (let loop ((n n))
              (if (zero? n)
                  1
                (* n (loop (- n 1))))))
          
          (dummy 5)

can be rewritten: 

          (letrec ((dummy (lambda (n)
                            (if (zero? n)
                                1
                              (* n (dummy (- n 1)))))))
            (dummy 5))
          ⇒ 120

which can be written: 

          ((recursion (loop n)
             (if (zero? n)
                 1
               (* n (loop (- n 1)))))
           5)
          ⇒ 120
