61.3.2 Combinators results

— Record Type: <result>

Base type for the return value of the combinator functions. It is an “abstract” type, meaning that it is incorrect to instantiate it directly. Fields description follows:

‘state’
Immutable field holding a <state> record. If the record is a <success>, this state is the next, still unparsed, one; if the record is an <error>, this state is the one that generated the error.

— Record Type: <success>

Record type derived from <result>; it represents the successful application of a combinator to a <state> record. Fields description follows:

‘value’
An immutable field holding the semantic value generated by the combinator application.

— Record Type: <error>

Record type derived from <result>; it represents the failed application of a combinator to a <state> record. Fields description follows:

‘message’
An immutable field holding a Scheme string describing the error.

— Function: make-<success> next-state value
— Function: make-<error> state error-message

Record constructors.

— Function: <result>? obj
— Function: <success>? obj
— Function: <error>? obj

Return #t if obj is a record of type <result>, <success> or <error>.

— Function: <result>-state result
— Function: <success>-value success
— Function: <error>-message error

Field accessors.
