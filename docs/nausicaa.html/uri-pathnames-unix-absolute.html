11.3.3.2 Absolute Unix pathname objects

The following bindings are exported by the library (nausicaa uri pathnames unix).

— Class: <absolute-unix-pathname>

Type of absolute Unix pathnames; it is derived from <absolute-pathname> exported by (nausicaa uri pathnames abstract).

Makers, constructors, predicates
— Common constructor on <absolute-unix-pathname>: constructor pathname

Build and return a new Unix pathname record from pathname, which must be a string or bytevector representing an absolute Unix pathname according to the predicates pathname? and absolute? defined by (vicare parser-tools unix-pathnames).

pathname is converted to bytevector with the function string/bytevector->pathname-bytevector and normalised with the function normalise-segments, both defined by (vicare parser-tools unix-pathnames).

— Maker on <absolute-unix-pathname>: <absolute-unix-pathname> (?pathname)

Build and return a new instance of <absolute-unix-pathname> using the common constructor.

Methods
— Method on <absolute-unix-pathname>: extension

Return a bytevector representing the extension of the last segment of the instance pathname, dot separator not included. This method is implemented with the function extension defined by (vicare parser-tools unix-pathnames), extension.

— Method on <absolute-unix-pathname>: dirname

Return an <absolute-unix-pathname> object representing the directory part of the instance pathname. This method is implemented with the function dirname defined by (vicare parser-tools unix-pathnames), dirname.

— Method on <absolute-unix-pathname>: tailname

Return a <relative-unix-pathname> object representing the last segment of the instance pathname. This method is implemented with the function tailname defined by (vicare parser-tools unix-pathnames), tailname.

— Method on <absolute-unix-pathname>: rootname

Return an <absolute-unix-pathname> object representing the instance pathname with the extension of the last segment stripped, dot separator not included. This method is implemented with the function rootname defined by (vicare parser-tools unix-pathnames), rootname.

— Method on <absolute-unix-pathname>: split

Return 2 values: a boolean, #t if the instance pathname is absolute; a list of bytevectors each representing a segment of the instance pathname. This method is implemented with the function split defined by (vicare parser-tools unix-pathnames), split.

— Method on <absolute-unix-pathname>: prefix? (ptn <absolute-unix-pathname>)
— Method on <absolute-unix-pathname>: prefix? (ptn <relative-unix-pathname>)

Return #t if the instance pathname is a prefix of ptn, otherwise return #f. This method is implemented with the function prefix? defined by (vicare parser-tools unix-pathnames), prefix?.

— Method on <absolute-unix-pathname>: suffix? (ptn <absolute-unix-pathname>)
— Method on <absolute-unix-pathname>: suffix? (ptn <relative-unix-pathname>)

Return #t if the instance pathname is a suffix of ptn, otherwise return #f. This method is implemented with the function suffix? defined by (vicare parser-tools unix-pathnames), suffix?.

— Method on <absolute-unix-pathname>: prepend (ptn <absolute-unix-pathname>)

Return a new pathname object representing the instance pathname prepended to ptn. This method is implemented with the function prepend defined by (vicare parser-tools unix-pathnames), prepend.

— Method on <absolute-unix-pathname>: replace-extension (ext <bytevector>)
— Method on <absolute-unix-pathname>: replace-extension (ext <string>)

Return an <absolute-unix-pathname> object representing the instance pathname with the extension of the last segment replaced by ext. This method is implemented with the function replace-extension defined by (vicare parser-tools unix-pathnames), replace-extension.
