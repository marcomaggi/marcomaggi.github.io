10.4.4.1 Specifying the coordinates of elements

The elements' positions in arrays are represented by built–in Scheme vectors holding exact integers. The following API is compatible with the one documented in containers arrays proc position.

Type, constructor, predicate
— Label: <position>

Label to tag Scheme vectors used to select an element in a multidimensional array; it is derived from <xvector> from (nausicaa containers vectors).

— Syntax on <position>: make <position> index0 index ...
— Syntax on <position>: make* <position> index0 index ...

Build and return a new Scheme vector holding the index arguments. The indexes are checked to be exact integers. The number of arguments is meant to be equal to the number of dimensions of the array with which the position value will be used.

— Syntax on <position>: is-a? obj <position>

Return #t if obj is a Scheme vector holding exact integers.

Label fields
— Immutable Field of <position>: string

A string representation of the instance, specialised for array positions.

Port output
— Method on <position>: display
— Method on <position>: display port

Write to port a human readable string representation of the instance. port defaults to the current output port.

— Method on <position>: write
— Method on <position>: write port

Write to port a string representation of the instance such that it can be read back with read. port defaults to the current output port.
