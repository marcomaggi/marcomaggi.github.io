42.5.2 Byte conversion error condition type

The following bindings are exported by the (nausicaa pathname conditions) library; it is suggested to add a prefix when importing the library.

— Condition Type: &byte

Condition type representing a byte–to–char or char–to–byte conversion error: it is used when converting to string a bytevector representing a pathname, or vice versa; it is derived from &pathname.

It has the following fields:

sequence
The original string or bytevector.
offset
The offset of the byte or character who triggered the error.

— Function: make-byte-condition sequence offset

Build and return a new &byte condition value.

— Function: byte-condition? obj

Return #t if obj is a condition value of type &byte.

— Function: condition-byte/sequence obj
— Function: condition-byte/offset obj

Accessors for the fields of a condition value of type &byte.

— Syntax: raise-byte-error who message sequence offset

Raise an exception with compound condition object of types &byte, &who, &message.
