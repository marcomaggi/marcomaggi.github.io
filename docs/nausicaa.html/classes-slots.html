24.5 Accessors and mutators

The following syntaxes provide a way to access concrete and virtual fields of class instances; they also allow the extraction of accessor and mutator functions. We want to remember that the cut and cute macros from (nausicaa language extensions) provide us with alternative ways to create closures for field access and mutation through slot-ref and slot-set!; lang lang for details.

The syntaxes slot-ref and slot-set! work only when the requested accessor and mutator are proper functions; when the accessor or mutator of a virtual field is itself implemented as a syntax, an attempt to use slot-ref and slot-set! will result in a syntax violation exception.

— Syntax: slot-ref expr field-name class
— Auxiliary Syntax: <>

Expand to the slot accessor of field-name applied to expr, which can be any expression evaluating to an instance of class. field-name can be either the name of a concrete field or the name of a virtual field.

This syntax works with both classes and labels.

When the auxiliary syntax <> is used in place of expr: the macro expands to the accessor function itself; the following forms are equivalent: 

          (slot-ref o car <pair>)
          ((slot-ref <> car <pair>) o)

this is useful whenever the accessor has to be used as function argument, for example for map or for-all. The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language classes).

— Syntax: slot-set! obj-expr field-name class val-expr
— Auxiliary Syntax: <>

Expand to the slot mutator of field-name applied to obj-expr and val-expr. obj-expr can be any expression evaluating to an instance of class; val-expr can be any expression evaluating to the new field value. field-name can be either the name of a concrete field or the name of a virtual field.

This syntax works with both classes and labels.

When the auxiliary syntax <> is used in place of both obj-expr and val-expr: the macro expands to the mutator function itself; the following forms are equivalent: 

          (slot-set! o car <pair> 1)
          ((slot-set! <> car <pair> <>) o 1)

this is useful whenever the mutator has to be used as function argument, for example for map or for-all. The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language classes).
