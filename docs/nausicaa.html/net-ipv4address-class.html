65.1.4 IPv4 address and address prefix class objects

The following bindings are exported by the (net ipv4-addresses) library.

— Class: <ipv4-address>

Represents an IPv4 address. Its constructor accepts as single argument the list of numbers returned by the IPv4 address parser, already validated. 

          (let (((o <ipv4-address>)
                (make <ipv4-address>
                  (ipv4-address-parse "1.2.3.4"))))
            (list o.third o.second o.first o.zeroth))
          ⇒ (1 2 3 4)
— Immutable Field of <ipv4-address>: zeroth
— Immutable Field of <ipv4-address>: first
— Immutable Field of <ipv4-address>: second
— Immutable Field of <ipv4-address>: third

The numeric components of the address, each representing 8 bits. zeroth is the rightmost component, first is the second from the right, etc.

— Immutable Virtual Field of <ipv4-address>: bignum

A single exact integer of 32 bits representing the whole address number.

— Immutable Virtual Field of <ipv4-address>: string

The string representation of the address.

— Immutable Virtual Field of <ipv4-address>: private?

True if the address' prefix is 10.0.0.0/8 or 172.16.0.0/12 or 192.168.0.0/16.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: loopback?

True if the address' prefix is 127.0.0.0/8.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: localhost?

True if the address is 127.0.0.1.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: link-local?

True if the address' prefix is 169.254.0.0/16.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: reserved?

True if the address' prefix is 192.0.0.0/24 or 240.0.0.0/4.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: test-net-1?

True if the address' prefix is 192.0.2.0/24.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: six-to-four-relay-anycast?

True if the address' prefix is 192.88.99.0/24.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: benchmark-tests?

True if the address' prefix is 198.18.0.0/15.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: test-net-2?

True if the address' prefix is 198.51.100.0/24.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: test-net-3?

True if the address' prefix is 203.0.113.0/24.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: multicast?

True if the address' prefix is 224.0.0.0/4.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Immutable Virtual Field of <ipv4-address>: limited-broadcast?

True if the address prefix is 255.255.255.255.

See Section 3 “Global and Other Specialized Address Blocks” of RFC 5735.

— Class: <ipv4-address-prefix>

Represents an IPv4 address prefix; it is derived from <ipv4-address>. Its constructor accepts two arguments: the list of numbers returned by the IPv4 address parser, already validated; a number specifying how many of the leftmost contiguous bits of the address comprise the prefix.

— Immutable Field of <ipv4-address-prefix>: prefix-length

A number specifying how many of the leftmost contiguous bits of the address comprise the prefix.

— Immutable Virtual Field of <ipv4-address-prefix>: string

The string representation of the address prefix.
