46.8.1 Type predicates and assertions

Before using the assertions proposed here consider using directly the predicate with assert: the only advantage of these assertions is an error message a bit more explicative. The following bindings are exported by the (nausicaa times-and-dates types) library.

Type predicates
— Macro: years-type? obj

Return true if obj is a valid value for a count of years.

— Macro: month-type? obj

Return true if obj is a valid value for a month index.

— Macro: day-type? obj

Return true if obj is a valid value for a day index.

— Macro: hours-type? obj

Return true if obj is a valid value for a count of hours.

— Macro: minutes-type? obj

Return true if obj is a valid value for a count of minutes.

— Macro: seconds-type? obj

Return true if obj is a valid value for a count of seconds.

— Macro: milliseconds-type? obj

Return true if obj is a valid value for a count of milliseconds.

— Macro: microseconds-type? obj

Return true if obj is a valid value for a count of microseconds.

— Macro: nanoseconds-type? obj

Return true if obj is a valid value for a count of nanoseconds.

Range predicates
— Macro: date-years-range? obj

Return true if obj, which must be a number satisfying years-type?, is in the range allowed for a count of years.

— Macro: date-month-range? obj

Return true if obj, which must be a number satisfying month-type?, is in the range allowed for a month index.

— Macro: date-day-range? obj month-index

Return true if obj, which must be a number satisfying day-type?, is in the range allowed for a the month whose index is month-index in a non–leap year.

If month-index is invalid: raise an assertion violation.

— Macro: date-day-range?/leap-year obj month-index

Return true if obj, which must be a number satisfying day-type?, is in the range allowed for a the month whose index is month-index in a leap year.

If month-index is invalid: raise an assertion violation.

— Macro: date-hours-range? obj

Return true if obj, which must be a number satisfying hours-type?, is in the range allowed for a count of hours.

— Macro: date-minutes-range? obj

Return true if obj, which must be a number satisfying minutes-type?, is in the range allowed for a count of minutes.

— Macro: date-seconds-range? obj

Return true if obj, which must be a number satisfying seconds-type?, is in the range allowed for a count of seconds excluding leap seconds.

— Macro: date-seconds-range?/with-leap obj

Return true if obj, which must be a number satisfying seconds-type?, is in the range allowed for a count of seconds including leap seconds.

— Macro: date-milliseconds-range? obj

Return true if obj, which must be a number satisfying milliseconds-type?, is in the range allowed for a count of milliseconds.

— Macro: date-microseconds-range? obj

Return true if obj, which must be a number satisfying microseconds-type?, is in the range allowed for a count of microseconds.

— Macro: date-nanoseconds-range? obj

Return true if obj, which must be a number satisfying nanoseconds-type?, is in the range allowed for a count of nanoseconds.

Type assertions

In the following macros, the raised value is a compound condition object of type: &assertion, &who, &message, &irritants. The who argument is used for the &who condition.

— Macro: assert-years-type who obj

Raise an assertion violation if obj is not a valid value for a count of years.

— Macro: assert-month-type who obj

Raise an assertion violation if obj is not a valid value for a month index.

— Macro: assert-day-type who obj

Raise an assertion violation if obj is not a valid value for a day index.

— Macro: assert-hours-type who obj

Raise an assertion violation if obj is not a valid value for a count of hours.

— Macro: assert-hours-type who obj

Raise an assertion violation if obj is not a valid value for a day index.

— Macro: assert-minutes-type who obj

Raise an assertion violation if obj is not a valid value for a count of minutes.

— Macro: assert-seconds-type who obj

Raise an assertion violation if obj is not a valid value for a count of seconds.

— Macro: assert-milliseconds-type who obj

Raise an assertion violation if obj is not a valid value for a count of milliseconds.

— Macro: assert-microseconds-type who obj

Raise an assertion violation if obj is not a valid value for a count of microseconds.

— Macro: assert-nanoseconds-type who obj

Raise an assertion violation if obj is not a valid value for a count of nanoseconds.

Range assertions

In the following macros, the raised value is a compound condition object of type: &assertion, &who, &message, &irritants. The who argument is used for the &who condition.

— Macro: assert-date-years-range who obj

Raise an assertion violation if obj, which must be a number satisfying years-type?, is not in the range allowed for a count of years.

— Macro: assert-date-month-range who obj

Raise an assertion violation if obj, which must be a number satisfying month-type?, is not in the range allowed for a month index.

— Macro: assert-date-day-range who obj month-index

Raise an assertion violation if obj, which must be a number satisfying day-type?, is not in the range allowed for a day index in the month whose index is month-index in a non–leap year.

If month-index is invalid: raise an assertion violation.

— Macro: assert-date-day-range/leap-year who obj month-index

Raise an assertion violation if obj, which must be a number satisfying day-type?, is not in the range allowed for a day index in the month whose index is month-index in a leap year.

If month-index is invalid: raise an assertion violation.

— Macro: assert-date-hours-range who obj

Raise an assertion violation if obj, which must be a number satisfying hours-type?, is not in the range allowed for a count of hours.

— Macro: assert-date-minutes-range who obj

Raise an assertion violation if obj, which must be a number satisfying minutes-type?, is not in the range allowed for a count of minutes.

— Macro: assert-date-seconds-range who obj

Raise an assertion violation if obj, which must be a number satisfying seconds-type?, is not in the range allowed for a count of seconds excluding the leap second.

— Macro: assert-date-seconds-range/with-leap who obj

Raise an assertion violation if obj, which must be a number satisfying seconds-type?, is not in the range allowed for a count of seconds including the leap second.

— Macro: assert-date-milliseconds-range who obj

Raise an assertion violation if obj, which must be a number satisfying milliseconds-type?, is not in the range allowed for a count of milliseconds.

— Macro: assert-date-microseconds-range who obj

Raise an assertion violation if obj, which must be a number satisfying microseconds-type?, is not in the range allowed for a count of microseconds.

— Macro: assert-date-nanoseconds-range who obj
