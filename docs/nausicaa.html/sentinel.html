16 The sentinel value

The (nausicaa language sentinel) library defines a unique value and a predicate to test it. The sentinel value can be used as terminator for sequences or as value for uninitialised storage locations.

— Constant: sentinel

The sentinel value.

— Function: sentinel? obj

Return #t if obj is the sentinel value.

As usage example, the following form makes use of the sentinel as terminator for a list: 

     (let ((ell (list 1 2 3 4 5 sentinel)))
       (let loop ((ell ell)
                  (res '()))
         (if (sentinel? (car ell))
             res
           (loop (cdr ell) (cons (car ell) res)))))
     ⇒ (5 4 3 2 1))

the following form defines an iterator closure using the sentinel to signal the end of the iteration: 

     (let* ((ell  '(1 2 3 4 5))
            (iter (lambda ()
                    (let ((ell ell))
                      (if (null? ell)
                          sentinel
                        (begin
                          (car ell)
                          (set! ell (cdr ell))))))))
       (let loop ((res '()))
         (let ((v (iter)))
           (if (sentinel? v)
               res
             (loop (cons v res))))))
     ⇒ (5 4 3 2 1)
— Function: make-sentinel

Return a new sentinel value. sentinel? will return #t when applied to such a value, but new sentinel values are meant to be checked with eq?.

New sentinels can be used in situations where a unique terminator is needed: When we need to distinguish ‘sentinel’ from a temporary sentinel we need for a specific operation.

— Constant: undefined

The undefined value.

— Function: undefined? obj

Return true if obj is eq? to undefined.

— Constant: undefined

The unspecified value.

— Function: unspecified? obj

Return true if obj is eq? to unspecified.
