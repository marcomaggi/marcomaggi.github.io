11.1.1 Multimethods for IP address objects

The following bindings are exported by the library (nausicaa uri ip).

— Multimethod: ip-address->string ip-address

Given as argument an object type subclassed from <ip-address> build and return a string representation as defined by RFC 3986; every character in the string can be directly converted to an ASCII encoded character, some sequences of characters may represent a percent–encoded character.

Notice that some subclasses of <ip-address> have string representation that is invalid as percent–encoded string as defined by RFC 3986.

— Multimethod: ip-address->bytevector ip-address

Given as argument an object type subclassed from <ip-address> build and return a bytevector representation as defined by RFC 3986; every octet in the bytevector represents an ASCII encoded character, some sequences of octets may represent a percent–encoded character.

Notice that some subclasses of <ip-address> have bytevector representation that is invalid as percent–encoded bytevector as defined by RFC 3986.

— Multimethod: ip-address->bignum ip-address

Given as argument an object type subclassed from <ip-numeric-address> build and return an exact integer representation.
