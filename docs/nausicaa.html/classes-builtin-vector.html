24.8.4 Vector type

— Foreign Class: <vector>

Conventional type for objects satisfying vector?.

— Immutable Virtual Field of <vector>: length

Access the length of the vector.

— Immutable Virtual Field of <vector>: empty?

True if the vector length is zero.

— Method on <vector>: subvector begin
— Method on <vector>: subvector begin end

Build and return a new vector holding the selected subvector of the subject. When end is not used: it defaults to the length of the vector.

When begin and end are zero or positive: this method behaves like subvector from (nausicaa language common). If either begin and/or end are negative: they are first normalised with: 

          (+ begin/end (vector-length vector))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selected the penultimate value and so on.

— Method on <vector>: map proc . vectors

Apply vector-map to the instance vector and the optional vectors.

— Method on <vector>: for-each proc . vectors

Apply vector-for-each to the instance vector and the optional vectors.

— Method on <vector>: for-all proc . vectors

Apply vector-for-all to the instance vector and the optional vectors.

— Method on <vector>: exists proc . vectors

Apply vector-exists to the instance vector and the optional vectors.

— Method on <vector>: fill! fill

Apply vector-fill! to the instance vector and fill.

— Method on <vector>: ref index

When index is zero or positive: this method behaves like vector-ref from (rnrs base (6)). If index is negative: it is first normalised with: 

          (+ index (vector-length vector))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selected the penultimate value and so on.

— Method on <vector>: set! index value

Store value at index in the vector. When index is zero or positive: this method behaves like vector-set! from (rnrs vectors (6)). If index is negative: it is first normalised with: 

          (+ index (vector-length vector))

so that negative indices are counted from the end of the vector: -1 selects the ultimate value, -2 selected the penultimate value and so on.

— Syntax on <vector>: getf (o index)

Return the value at index from the vector o; index is an expression. It behaves like the ref method.

— Syntax on <vector>: setf (o index) value

Store value at index of the vector o; index is an expression. It behaves like the set! method.
