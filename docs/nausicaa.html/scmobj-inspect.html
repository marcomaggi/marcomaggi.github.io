50.5 Inspecting classes and instances

— Function: class? obj

Does its best to determine if obj is a valid class object.

— Function: instance? obj

Does its best to determine if obj is a valid instance object.

— Function: class-of class/instance

Return the class of the argument. This function can be applied to both class objects and instance objects. If this function is not able to determine a class, it returns #f; this should never happen, though.

— Function: is-a? class/instance class

Return true if class/instance is an object of class class. This function can be applied to both class objects and instance objects.

— Function: subclass? class-1 class-2

Return true if class-1 is a subclass of class-2, false otherwise.

— Function: class-definition-name class

Return the class definition name for class. Note that the definition name is set only for classes instantiated with define-class, for classes instantiated with make-class the return value is the symbol :uninitialised.

— Function: class-precedence-list class

Return a list of classes representing the class precendence list, used for method dispatching. Note that class itself is not part of the class precedence list.

— Function: class-slots class

Return a list of symbols representing the slot names of class. The symbol :class is excluded.

— Function: class-direct-slots class

Return a list of symbols representing the slot names of class, excluding the slots of its superclasses.
