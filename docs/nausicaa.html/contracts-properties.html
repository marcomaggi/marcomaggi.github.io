18.4 Contract identifier properties

The following bindings are exported by the library (nausicaa language contracts properties) and they are meant to be used for the expand phase; it is suggested to prefix them when importing the library, as in: 

     (import (nausicaa)
       (for (prefix (nausicaa language contracts properties)
                    prop.) expand))

Contract identifier properties are values (disjoint from all the other Scheme values) associated to the keyword identifier of contracts and available for inspection at expand time.

— Function: contract-properties-define identifier properties-value

Associate properties-value to identifier as a contract property. identifier must be an identifier representing the keyword of a contract; properties-value must be the return vlaue of a call to make-applicable-contract or make-variable-contract.

Usually, user code has no need to define property values.

— Function: contract-properties-ref identifier

If identifier is an identifier representing the keyword of a contract: return a properties value, else return false.

— Function: make-variable-contract subject contract

Build and return a new value representing the properties of a contract keyword. subject must be an identifier representing the subject of the contract. contract must be a syntax object representing the contract specification for a variable.

Usually, user code has no need to build property values.

— Function: make-applicable-contract subject contract

Build and return a new value representing the properties of a contract keyword. subject must be an identifier representing the subject of the contract. contract must be a syntax object representing the contract specification for a function or macro.

Usually, user code has no need to build property values.

— Function: contract? obj
— Function: applicable-contract? obj
— Function: variable-contract? obj

Return true if obj is a value representing the properties of a contract keyword, applicable or variable.

— Function: contract-subject prop
— Function: variable-contract-subject prop
— Function: applicable-contract-subject prop

If prop is a properties value of a contract keyword: return an identifier representing the subject of the contract.

— Function: contract-spec prop
— Function: variable-contract-spec prop
— Function: applicable-contract-spec prop

If prop is a properties value of a contract keyword: return a syntax object representing the specification of the contract.
