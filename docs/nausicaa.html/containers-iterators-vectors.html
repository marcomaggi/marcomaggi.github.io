10.8.5 Iterating over vectors

The following bindings are exported by the library (nausicaa containers iterators).

— Class: <vector-iterator>

An iterator for Scheme vectors; it is derived from <iterator>.

— Common constructor on <vector-iterator>: constructor the-vector start past stride

Build and return a new instance of <vector-iterator> using the-vector as vector upon which to iterate, start as index of the first item, past as index of the item past the last one to return and stride as stride.

The stride is the number of items in the vector to jump forward or backward to when iterating; it must be +1 or -1 when we want to visit all the items.

start and past must be fixnums greater than, or equal to, -1; stride must be a non–zero fixnum. If stride is positive, it must be: 

          0 <= start <= past <= (vector-length the-vector)

if stride is negative, it must be: 

          -1 <= past <= start < (vector-length the-vector)
— Maker on <vector-iterator>: <vector-iterator> (?clauses ...)
— Mandatory clause on <vector-iterator>: subject: ?the-vector
— Optional clause on <vector-iterator>: start: ?start
— Optional clause on <vector-iterator>: past: ?past
— Optional clause on <vector-iterator>: stride: ?stride

Build and return a new <vector-iterator> instance using the common constructor.

?stride defaults to +1. If the stride is positive: ?start defaults to zero and ?past defaults to the length of the vector. If the stride is negative: ?start defaults one less than the length of the vector and ?past defaults to -1.
