24.1.11 Defining classes having custom setters and getters

— Auxiliary Syntax: setter ?setter-name
— Auxiliary Syntax: setter ?setter-expression
— Auxiliary Syntax: getter ?getter-name
— Auxiliary Syntax: getter ?getter-expression

Define a setter or getter binding to be used by the setf or getf syntax. ?setter-name and ?getter-name are identifiers bound to a function or macro which is invoked by setf and getf as: 

          (?setter-name variable value key0 key ...)
          (?getter-name variable key0 key ...)

where: variable is the identifier to which the class instance is bound, key are the setter and getter keys, value is the new value. Notice the peculiar order of values for the setter form: putting the value first allows easy processing of an arbitrary number of key arguments. The number of keys to be used by the setter and getter is unspecified: it is enforced by the setter and getter functions or macros.

When the non–identifier ?setter-expression or ?getter-expression argument is used: it must be an expression evaluating to the setter or getter function. The setter and getter with a non–identifier argument are equivalent to the clauses: 

          (setter Setter-name)
          (getter Getter-name)

and the associated definitions: 

          (define Setter-name ?setter-expression)
          (define Getter-name ?getter-expression)

where Setter-name and Getter-name are automatically generated identifiers built by appending a dash and setf or getf to the class or label name.

An example with an identifier as setter selection: 

          #!r6rs
          (import (nausicaa))
          
          (define-class <alpha>
            (fields (mutable a))
            (setter <alpha>-setf))
          
          (define/with-class (<alpha>-setf (o <alpha>) value key0 key1)
            (set! o.a (list key0 key1 value)))
          
          (let/with-class (((o <alpha>) (make <alpha> 1)))
            (setf (o 2 3) 4)
            o.a)
          ⇒ (2 3 4)

the same code but with an inline expression as setter: 

          #!r6rs
          (import (nausicaa))
          
          (define-class <alpha>
            (fields (mutable a))
            (setter (lambda/with-class ((o <alpha>) value key0 key1)
                      (set! o.a (list key0 key1 value)))))
          
          (let/with-class (((o <alpha>) (make <alpha> 1)))
            (setf (o 2 3) 4)
            o.a)
          ⇒ (2 3 4)

another example with a macro as setter: 

          #!r6rs
          (import (nausicaa))
          
          (define-label <vector>
            (predicate vector?)
            (virtual-fields (immutable length vector-length))
            (setter vector-setf)
            (getter vector-ref))
          
          (define-inline (vector-setf ?vector ?value ?index)
            (vector-set! ?vector ?index ?value))
          
          (let/with-class (((o <vector>) (vector 0 1 2 3 4)))
            (setf (o 2) #\c)
            (getf (o 2)))
          ⇒ #\c
— Auxiliary Syntax: setter-syntax ?setter-name
— Auxiliary Syntax: setter-syntax ?setter-expression
— Auxiliary Syntax: getter-syntax ?getter-name
— Auxiliary Syntax: getter-syntax ?getter-expression

These clauses work exactly like setter and getter, but when the argument is a non–identifier the associated definition is: 

          (define-syntax Setter-name ?setter-expression)
          (define-syntax Getter-name ?getter-expression)

that is: it uses define-syntax rather than define.
