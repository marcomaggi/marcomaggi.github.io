29.2.3.4 Dealing with block portions

The following functions handle blocks and portions with the meaning displayed in this picture: 

                     block
     |-----------------------------------|
     |---------------|
           head       |------------------|
                              tail
— Function: memblock-head block head-size

Return a new <memblock> record referencing a portion of head-size bytes anchored to the beginning of block.

The returned block has alloc-size set to #f.

— Function: memblock-tail block tail-size

Return a new <memblock> record referencing a portion of tail-size bytes anchored to the end of block.

The returned block has alloc-size set to #f.

— Function: memblock-head? block head

Return true if head references a portion of block anchored to its beginning.

— Function: memblock-tail? block tail

Return true if tail references a portion of block anchored to its end.

— Function: memblock&head-tail block head

Both block and head must be <memblock> records; interpret head as a reference to a portion of block anchored to its beginning; return a new memory block representing the tail of block non–overlapping with head.

The returned block has alloc-size set to #f.

— Function: memblock&tail-head block tail

Both block and tail must be <memblock> records; interpret tail as a reference to a portion of block anchored to its end; return a new memory block representing the head of block non–overlapping with tail.

The returned block has alloc-size set to #f.
