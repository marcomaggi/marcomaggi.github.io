44.1 Introduction to messages catalogs

Messages translation is simply handled by a procedure applied to a string: the return value is meant to be the translated string. (nausicaa msgcat) defines a “standard” translator procedure accepting one or two arguments:

  • When applied to one argument: the argument must be the string to translate; the return value must be the translated string or the input string itself if no translation was found.
  • When applied to two arguments: the first argument must be the string to translate, the second argument must be a default value; the return value must be the translated string or the default value if no translation was found.

A translator procedure can do anything, for example it can query a set of other translation procedures in search of a translation. The template for a translation procedure is the following: 

     (define catalog
       (make-hashtable string-hash string=?))
     
     (define translator
       (case-lambda
         ((message)
          (translator message message))
         ((message default)
          (hashtable-ref catalog message default))))

an implementation querying a set of translator procedures is as follows: 

     (define translator-1 ---)
     (define translator-2 ---)
     (define translator-3 ---)
     
     (define translator
       (case-lambda
         ((message)
          (translator message message))
         ((message default)
          (or (find (lambda (T)
                      (T message #f))
                    (list translator-1 translator-2 translator-3))
              default))))

A catalog file is a textual file holding a single S–expression which can be read with the read function from (rnrs io simple (6)); the S–expression must match the following template: 

     (msgcat (<version0> <version>...)
       <catalog name>
       <data> ...)

where: the <version> datums are exact integers representing the format version of the catalog; the <catalog name> datum must be a symbol representing the catalog name; the <data> datums depend on the catalog format version.

Message catalog version 1

The S–expression of version 1 must match the following template: 

     (msgcat (<version0> <version>...)
       <catalog name>
       (<message> <translated message>)
       ...)

where the <message> datums are the message strings in the en_GB locale and the <translated message> datums are the translated message strings.
