51.2.4 Logic matching

— Function: sexp-or pattern-token ...
— Syntax: sexp-or* <pattern-token> ...

Return an operator which attempts to match the car of the form with each pattern-token, halting at the first positive match. The match of the operator is positive if at least one of the alternatives does match.

sexp-or* behaves like sexp-or, but quotes each of its arguments. 

          (sexp-match `(,(sexp-or 'a 'b 'c)) '(a)) ⇒ '()
          (sexp-match `(,(sexp-or 'a 'b 'c)) '(b)) ⇒ '()
          (sexp-match `(,(sexp-or 'a 'b 'c)) '(c)) ⇒ '()
          
          (sexp-match `(,(sexp-or* a b c)) '(a)) ⇒ '()
          (sexp-match `(,(sexp-or* a b c)) '(b)) ⇒ '()
          (sexp-match `(,(sexp-or* a b c)) '(c)) ⇒ '()

Notice that the arguments of sexp-or* must be pattern tokens, not pattern sexps: 

          (sexp-match `(,(sexp-or* (a) (b) (c))) '((a))) ⇒ '()
          (sexp-match `(,(sexp-or* (a) (b) (c))) '((b))) ⇒ '()
          (sexp-match `(,(sexp-or* (a) (b) (c))) '((c))) ⇒ '()

If sexp-or invoked with no arguments, it returns an operator which matches nothing and always raises a sexp mismatch error; this is consistent with (or) returning #f in R6RS.

— Function: sexp-and pattern-token ...
— Syntax: sexp-and* <pattern-token> ...

Return an operator which attempts to match the car of the form with each pattern-token, halting at the first mismatch. The match of the operator is positive if all the alternatives do match.

sexp-and* behaves like sexp-and, but quotes each of its arguments.

If sexp-and is invoked with no arguments, it returns an operator which always matches a single token; this is consistent with (and) returning #t in R6RS. 

          (sexp-match `(,(sexp-and* a)) '(a))      ⇒ '()
          (sexp-match `(,(sexp-and) b)  '(a b))    ⇒ '())
          
          (let-sexp-variables ((?v 0))
            (sexp-match `(,(sexp-and (sexp-pred integer?)
                                     (sexp-var ?v)))
                        '(123)))
          ⇒ ((#<record sexp-variable ?v> . 123))
— Function: sexp-any pattern-token
— Syntax: sexp-any* <pattern-token>

Return an operator which attempts to match every sexp in the form with pattern-token, halting at the first mismatch. It is fine if pattern-token does not match the car of the form.

sexp-any* behaves like sexp-any, but quotes its argument. 

          (sexp-match `(,(sexp-any* a)) '())      ⇒ '()
          (sexp-match `(,(sexp-any* a)) '(a))     ⇒ '()
          (sexp-match `(,(sexp-any* a)) '(a a))   ⇒ '()
          (sexp-match `(,(sexp-any* a)) '(a a a)) ⇒ '()
          
          (sexp-match `(,(sexp-any* a) b) '(b))       ⇒ '()
          (sexp-match `(,(sexp-any* a) b) '(a b))     ⇒ '()
          (sexp-match `(,(sexp-any* a) b) '(a a b))   ⇒ '()
          (sexp-match `(,(sexp-any* a) b) '(a a a b)) ⇒ '()
— Function: sexp-one pattern-token
— Syntax: sexp-one* <pattern-token>

Like sexp-any, but pattern-token must match at least the car of the form.
