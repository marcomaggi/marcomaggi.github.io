24.8.7 Character and string types

String type
— Foreign Class: <string>

Conventional type for immutable objects satisfying string?.

— Immutable Virtual Field of <string>: length

Accessing this field is like applying string-length to the string.

— Immutable Virtual Field of <string>: empty?

True if the string length is zero.

— Immutable Virtual Field of <string>: upcase

Accessing this field is like applying string-upcase to the string.

— Immutable Virtual Field of <string>: downcase

Accessing this field is like applying string-downcase to the string.

— Immutable Virtual Field of <string>: titlecase

Accessing this field is like applying string-titlecase to the string.

— Immutable Virtual Field of <string>: foldcase

Accessing this field is like applying string-foldcase to the string.

— Method on <string>: substring begin
— Method on <string>: substring begin end

Build and return a new string holding the selected substring of the subject. When end is not used: it defaults to the length of the string.

When begin and end are zero or positive: this method behaves like substring from (rnrs base (6)). If either begin and/or end are negative: they are first normalised with: 

          (+ begin/end (string-length string))

so that negative indices are counted from the end of the string: -1 selects the ultimate character, -2 selected the penultimate character and so on.

— Method on <string>: append string ...

Build and return a new string whose characters form the concatenation of the subject and given strings.

— Method on <string>: list

Return a list holding all the characters in the string.

— Method on <string>: for-each proc

Apply proc to the characters of the string.

— Method on <string>: copy

Build and return a new string holding the same characters.

— Method on <string>: ref index

When index is zero or positive: this method behaves like string-ref from (rnrs base (6)). If index is negative: it is first normalised with: 

          (+ index (string-length string))

so that negative indices are counted from the end of the string: -1 selects the ultimate character, -2 selected the penultimate character and so on.

— Syntax: getf (o index)

Character getter; it behaves like the ref method.

Character type
— Foreign Class: <char>

Conventional type for objects satisfying char?.

— Immutable Virtual Field of <char>: upcase

Accessing this field is like applying char-upcase to the char.

— Immutable Virtual Field of <char>: downcase

Accessing this field is like applying char-downcase to the char.

— Immutable Virtual Field of <char>: titlecase

Accessing this field is like applying char-titlecase to the char.

— Immutable Virtual Field of <char>: foldcase

Accessing this field is like applying char-foldcase to the char.
