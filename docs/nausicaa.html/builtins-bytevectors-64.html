5.13.5 Bytevector labels: 64-bit words

— Builtin Label: <bytevector-u64l>
— Builtin Label: <bytevector-s64l>
— Builtin Label: <bytevector-u64b>
— Builtin Label: <bytevector-s64b>
— Builtin Label: <bytevector-u64n>
— Builtin Label: <bytevector-s64n>

Conventional type for objects satisfying bytevector? whose contents must be accesses as 64-bit words. The suffix ‘l’ stands for little–endian; the suffix ‘b’ stands for big–endian; the suffix ‘n’ stands for native endianness.

OOPP syntaxes
— Getter on <bytevector-u64l>: instance (?index)
— Getter on <bytevector-u64b>: instance (?index)
— Getter on <bytevector-u64n>: instance (?index)
— Getter on <bytevector-s64l>: instance (?index)
— Getter on <bytevector-s64b>: instance (?index)
— Getter on <bytevector-s64n>: instance (?index)

64-bit word getter. ?index is the zero–based index of the selected word.

— Setter on <bytevector-u64l>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-u64b>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-u64n>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-s64l>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-s64b>: set!/tags (instance (?index)) ?item
— Setter on <bytevector-s64n>: set!/tags (instance (?index)) ?item

64-bit word setter. ?index is the zero–based index of the selected word.
