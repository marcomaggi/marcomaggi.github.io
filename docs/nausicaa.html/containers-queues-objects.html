10.6.1 Queue objects

The following bindings are exported by the library (nausicaa containers queues).

— Label Type: <queue>

Label type to be used to handle queue objects as defined by the library (vicare containers queues). In this documentation <queue> object arguments to functions are indicated as queue.

— Default constructor on <queue>: constructor obj ...

Build and return a <queue> object holding the given objects, which are pushed on the queue from lefto to right.

— Syntax: <queue> (obj ...)

Build and return a <queue> object using the default constructor.

Object properties
— Method on <queue>: putprop key value
— Method on <queue>: $putprop key value

Add a new property key to the property list of the <queue>; key must be a symbol. If key is already set: the old entry is mutated to reference the new value.

— Method on <queue>: getprop queue key
— Method on <queue>: $getprop queue key

Return the value of the property key in the property list of the <queue>; if key is not set: return #f. key must be a symbol.

— Method on <queue>: remprop queue key
— Method on <queue>: $remprop queue key

Remove the property key from the property list of the <queue>; if key is not set: nothing happens. key must be a symbol.

— Method on <queue>: property-list queue
— Method on <queue>: $property-list queue

Return a new association list representing the property list of the <queue>. The order of the entries is the same as the property creation order.

Other operations
— Method on <queue>: hash queue
— Method on <queue>: $hash queue

Return an exact integer to be used as hashtable key for the <queue>. Hashtables having a <queue> as key can be instantiated as follows: 

          (make-hashtable (lambda ((S <queue>))
                            (S hash))
                          eq?)
Arguments validation

The following bindings are meant to be used with the facilities of the library (nausicaa arguments validation).

— Validation Clause: <queue> obj

Succeed if obj is a <queue> object.
