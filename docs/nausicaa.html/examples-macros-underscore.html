B.1.2 About the underscore

It is stated that:

Underscores, _, also match arbitrary input subforms but are not pattern variables and so cannot be used to refer to those elements. Multiple underscores may appear in a ?pattern.

See syntax-rules. It means that the following script will evaluate successfully: 

     (import (rnrs) (nausicaa checks))
     (check-set-mode! 'report-failed)
     
     (check
      (let ()
        (define-syntax doit
          (syntax-rules ()
            ((doit _) 1)))
     
        (list (doit '(a b c))
              (doit 2)
              (doit '(a (b (c)) d (e)))))
      => '(1 1 1))
     
     (check
      (let ()
        (define-syntax doit
          (syntax-rules ()
            ((doit _ _) 1)))
     
        (list (doit '(a b c) '(d e f))
              (doit 1 2)))
      => '(1 1))
     
     (check-report)

The use of _ allows us not to invent pattern variable names for subpatterns that we do not care about in a ?syntax rule. In the following: 

     (define-syntax doit
       (syntax-rules ()
         ((doit ?arg)
          1)))

we do not care about the actual value of ?arg, so why should we be forced to invent a name for it? Better to write: 

     (define-syntax doit
       (syntax-rules ()
         ((doit _)
          1)))

It is customary to use _ in the pattern of ?syntax rules as a placeholder for the macro use name: 

     (define-syntax doit
       (syntax-rules ()
         ((_ ?arg)
          (list ?arg))))

but there is nothing special about this: it is convenient because one has only one char to type. Notice that when writing syntax-case forms using _ as first subpattern of patterns, while still possible, may need to be avoided because there is the need to use the macro use identifier, and it is not possible to do it with _.
