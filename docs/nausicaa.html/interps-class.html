71.2 Interpreter objects

The following bindings are exported by the (nausicaa interps) library.

— Class: <interp>

Hold the state of subsequent code evaluations.

— Immutable Field of <interp>: table-of-variables

A hash table mapping Scheme symbols to values; one for each persistent variable currently in the interpreter.

— Immutable Field of <interp>: eval-environment

The environment object to be used when evaluating code in the interpreter.

— Syntax: make <interp> clause ...
— Auxiliary Syntax: imports: import-specs

This syntax is a maker, Makers. Expand to an expression which evaluates to a new <interp> object. The following clauses are recognised:

imports: import-specs
import-specs must be an expression evaluating to a list of library import specifications; the list is used to build the environment in which code will be evaluated. The environment cannot be modified. The default value causes the (rnrs) library to be imported.

— Syntax: make* <interp> import-specs

Expand to an expression which evaluates to a new <interp> object.

import-specs must be an expression evaluating to a list of library import specifications; the list is used to build the environment in which code will be evaluated. The environment cannot be modified.

— Method on <interp>: eval sexp

Evaluate sexp in the environment of the interpreter. Return the return value of the expression; sexp can return any number of values. sexp is evaluated as if it is the first form in the body of a let form, so it can contain definitions.

— Method on <interp>: variable-ref varname default

Retrieve the value of variable name varname, a Scheme symbol; return default if the variable is not defined.

— Method on <interp>: variable-set! varname obj

Set obj as new value for variable varname, a Scheme symbol. If such a variable does not exist: it is created.

— Method on <interp>: clone
— Method on <interp>: clone filter

Build and return a clone of the interp; the clone's table of variables is populated with elements from the interp's table of variables according to filter.

If filter is not used: all the variables are duplicated into the new interp; keys and values from the source table are stored in the new table as is (there is no deep copy of the values).

When it is used, filter must accept 3 arguments: a symbol representing a variable name; a value representing the current variable's value; an object, unique according to eq?.

  • If the variable must be stored in the new table: filter must return the variable's value, which can be the argument's value itself.
  • If the variable must be discarded: filter must return the unique object.

The following example of filter causes the table to be transparently duplicated: 

          (lambda (key val avoid)
            val)

the following example of filter discards all the variables having a list as value and accepts all the others: 

          (lambda (key val avoid)
            (if (list? val)
                avoid
              val))

the following example of filter duplicates the spine of all the list values: 

          (lambda (key val avoid)
            (if (list? val)
                (list-copy val)
              val))
