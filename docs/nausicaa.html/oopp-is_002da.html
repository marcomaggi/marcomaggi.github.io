3.16 Testing a value's type

— Syntax: is-a? ?expression ?tag
— Syntax: is-a? <> ?tag
— Auxiliary Syntax: <>

In the first form: expand to the application of the class predicate of ?tag, a class or label type, to the result of evaluating ?expression, which can be any expression and is evaluated only once. In the second form: returns the predicate function of ?tag.

For example, given: 

          (define-class <alpha>
            (fields a b c))

the following statement: 

          (is-a? x <alpha>)

is equivalent to: 

          (<alpha>? x)

Another example, the following two forms are equivalent: 

          (is-a? x <list>)
          ((is-a? <> <list>) x)

this is especially useful when using map or for-all: 

          (for-all (is-a? <> <list>) obj)

The <> auxiliary syntax is exported by the library (nausicaa language auxiliary-syntaxes) and reexported by (nausicaa language oopp).
