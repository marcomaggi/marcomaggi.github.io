61.5.4.1 Introduction to error recovery

(nausicaa lalr) implements a very simple error recovery strategy. A production can be of the form: 

     (NON-TERMINAL
        ---                  ;right-hand side rules
        (error TERMINAL) : semantic-clause)

If, while parsing the right–hand side rules for ‘NON-TERMINAL’, the lookahead returns an invalid category: The parser resets the state associated with the parsing of the rule and skips all the tokens returned by the lexer until one with category ‘TERMINAL’ is found. The ‘TERMINAL’ token is also discarded and parsing continues with a new lookahead.

The symbol ‘error’ is reserved as error recovery directive, it cannot be used as terminal or non–terminal.

There can be several such right–hand side rules for a single non–terminal: For a C–like language, we could “synchronise” on semicolons and closing curly brackets by writing error rules like these: 

     (statement
        (expression SEMICOLON)        : ---
        (LBRACKET statement RBRACKET) : ---
        (error SEMICOLON)    ;empty semantic clause
        (error RBRACKET))    ;empty semantic clause

if an error occurs, the parser will discard its internal state up to, and including, the first token that started the ‘statement’ non–terminal (‘LBRACKET’ or the first of ‘expression’).
