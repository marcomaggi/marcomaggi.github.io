5.7 Pair label

— Builtin Label: <pair>

Conventional type of objects satisfying pair?. This label is meant to be used for immutable pairs.

— Builtin Label: <mutable-pair>

Conventional type of objects satisfying pair?; it is derived from <pair>.

— Common constructor on <pair>: <pair> constructor car cdr
— Common constructor on <mutable-pair>: <mutable-pair> constructor car cdr

Return the result of applying cons to car and cdr.

Safe accessors and mutators
— Immutable virtual field of <pair>: <top> car

Access the car of the pair using car.

— Immutable virtual field of <pair>: <top> cdr

Access the cdr of the pair using cdr.

— Mutable Virtual Field of <mutable-pair>: <top> car

Access and mutate the car of the pair using car and set-car!.

— Mutable Virtual Field of <mutable-pair>: <top> cdr

Access and mutate the cdr of the pair using cdr and set-cdr!.

Unsafe accessors and mutators
— Immutable virtual field of <pair>: <top> $car

Access the car of the pair using the unsafe operation $car from (vicare system $pairs).

— Immutable virtual field of <pair>: <top> $cdr

Access the cdr of the pair using the unsafe operation $cdr from (vicare system $pairs).

— Mutable Virtual Field of <mutable-pair>: <top> $car

Access and mutate the car of the pair using the unsafe operations $car and $set-car! from (vicare system $pairs).

— Mutable Virtual Field of <mutable-pair>: <top> $cdr

Access and mutate the cdr of the pair using the unsafe operations $cdr and set-cdr! from (vicare system $pairs).

OOPP syntaxes
— Maker on <pair>: <pair> (?car ?cdr)
— Maker on <pair>: <mutable-pair> (?car ?cdr)

Return the result of applying cons to the results of evaluating the expressions ?car and ?cdr.
