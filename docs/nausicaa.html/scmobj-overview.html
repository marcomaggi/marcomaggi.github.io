50.1 Overview of the library

(scmobj) objects, that is classes and instances, are association lists: Proper lists of pairs whose cars are symbols and whose cdrs can be any value.

In this document: each pair is called slot; the symbol is called slot name; the value is called slot value. By convention the slot name symbols must have name starting with a colon :.

A class object is an alist whose first slot's name is :class, and the other slot names are: 

     :class-definition-name
     :class-precedence-list
     :slots

An instance object is an alist whose first slot's name is :class, and the other slot names are configured by the user.

— Slot: :class

It must be the first slot for both class and instance objects.

For an instance object: The value must be the class of the instance.

For a class object: The value must be the predefined <class> object. So a class object is a special case of instance object.

— Slot: :class-definition-name

A class object's slot whose value is the “public name” of the class. By convention it must be a symbol whose name is enclosed in angular parentheses. It is useful to print debug messages.

— Slot: :class-precedence-list

A class object's slot whose value is a list representing the hierarchy of superclasses for the instance objects of the class, or #f if the class has no precedence list. It is used for dispatching of multimethods.

Notice that the value of this slot does not hold the class object itself.

— Slot: :slots

A class object's slot whose value is the list of slot names for the instances of the class, or #f if instances have no slots. For the predefined <class> object, the list is: 

          (:class-definition-name :class-precedence-list :slots)

Notice that the symbol :class is not included.
