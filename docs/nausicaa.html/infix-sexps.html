77.2 Converting S–expressions

— Function: infix->prefix sexp-atom ...

Convert the infix S–expression composed of the sequence of sexp-atom arguments into a prefix S–expression. The S–expression can be used directly for some symbolic interpretation or evaluated with eval. Evaluation

The simplest way to use the function is to apply it to a single S–expression: 

          (infix->prefix '(1 + 2 + 3))    ⇒ '(+ (+ 1 2) 3)
          (infix->prefix '(1 + 2 - 3))    ⇒ '(- (+ 1 2) 3)
          (infix->prefix '(1 + (2 + 3)))  ⇒ '(+ 1 (+ 2 3))
          (infix->prefix '(1 + (2 - 3)))  ⇒ '(+ 1 (- 2 3))
          
          (infix->prefix '(cos (sin (1.1))))
          ⇒ (cos (sin 1.1))
          
          (infix->prefix '(cos (sin (1.1) + 4)))
          ⇒ (cos (+ (sin 1.1) 4))
          
          (infix->prefix '(fun (1.1 2)))
          ⇒ (fun 1.1 2)
          
          (infix->prefix '(fun (1 2 3 4)))
          ⇒ (fun 1 2 3 4)
          
          (infix->prefix '(a * (b / a ? b : c)))
          ⇒ (* a (if (/ b a) b c))

while applying it to multiple arguments looks like: 

          (infix->prefix 1 '+ 2 '+ 3)     ⇒ '(+ (+ 1 2) 3)

Some interpretation rules:

  • The following infix operators are supported, in order of increasing precedence from left to right: 
                   + - * / // % ^ < > <= >= =

    remember that the Scheme comparison operators are meant to return a boolean value, rather than a number object. ‘//’ stands for div, ‘%’ stands for mod, ‘^’ stands for expt.

  • The if–then–else statement involves the special symbols ? and : which cannot be variable symbols; this statement has the least precedence.
  • The elements of the S–expression can be numbers, symbols, procedures, all “the other” values.
  • Symbols which are not operators are interpreted as variable references or function calls. If a symbol is followed by a list, it is a function call.
  • Function calls with one or more arguments require the arguments to be grouped in a list. There is no special separator in the list of arguments.
  • Procedures are treated like non–operator symbols.
  • Numbers and all “the other” values are just put there.
