77.4.6.1 Representing library forms

The bindings documented in this section are exported by the (nausicaa parser-tools r6rs libraries) library.

— Class: <library>

Record type describing a library form as defined by R6RS.

— Immutable field of <library>: <library-name> name

A record representing this library's name.

— Immutable field of <library>: <library-reference> reference

A <library-reference> record exactly referencing the name of this this library.

— Immutable field of <library>: <list> exports

List of renamings representing the export list.

— Immutable field of <library>: <list> import-specs

List of <import-spec> records representing the import sets of this library. Notice that an imported library can appear in more import sets.

— Immutable field of <library>: <list> import-references

List of <library-reference> records representing the list of imported libraries.

— Immutable field of <library>: <sexp> sexp

The full symbolic expression of the library: this is the library form. Mutating this symbolic expression results in undefined behaviour.

— Immutable field of <library>: <sexp> name-sexp

The library name's symbolic expression object being the library name from the library form.

— Immutable field of <library>: <sexp> exports-sexp

The raw content of the export clause; it is null if the library exports nothing.

— Immutable field of <library>: <sexp> imports-sexp

The raw content of the import clause; the import clause is not allowed to be null, so this field cannot be null either.

— Immutable field of <library>: <sexp> body-sexp

The body of the library; it is null if the library has empty body.

— Common constructor on <library>: constructor sexp

Build a new <library> record from the symbolic expression sexp which must be an R6RS compliant library form, else an assertion violation is raised.
