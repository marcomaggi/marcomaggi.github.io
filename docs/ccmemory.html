1 Overview of the package

This document describes version 0.1.0-devel.0 of CCMemory, a C11 language library implementing an interface to custom memory allocators.

This package supports POSIX platforms. This package depends on the package CCExceptions, at least version 0.8.0. To run the test suite: this package depends upon the package CCTests, at least version 0.2.0.

CCMemory installs the single header file ccmemory.h. All the function names in the API are prefixed with ccmem_; all the constant names are prefixed with CCMEM_; all the type names are prefixed with ccmem_ and suffixed with _t.

• overview linking:  Linking code with the library.

Up: overview   [Contents][Index]

1.1 Linking code with the library

This package installs a data file for pkg-config, so when searching for the installed library with the GNU Autotools, we can add the following macro use to configure.ac: 

PKG_CHECK_MODULES([CCMEMORY],[ccmemory >= 0.1])

which will set the variables CCMEMORY_LIBS and CCMEMORY_CFLAGS. To avoid problems with pkg-config, we need to remember that:

  • The macro file defining PKG_CHECK_MODULES is installed by the package pkg-config; such file usually is in one of the locations:
    /usr/share/aclocal/pkg.m4
    /usr/local/share/aclocal/pkg.m4
  • The macro file is located by aclocal, which in turn is installed by the package GNU Automake; aclocal is used by autoreconf. We must make sure that aclocal finds the installed macro files; for this purpose we can use the environment variable ACLOCAL_PATH; under a Bourne shell we can do this with: 
    export ACLOCAL_PATH=/usr/local/share/aclocal:/usr/share/aclocal

Alternatively we can use the raw GNU Autoconf macros: 

AC_CHECK_LIB([ccmemory],[ccmem_version_string],,
  [AC_MSG_FAILURE([test for CCMemory library failed])])
AC_CHECK_HEADERS([ccmemory.h],,
  [AC_MSG_FAILURE([test for CCMemory header failed])])

3.1 Type definitions

Struct Typedef: ccmem_allocator_t

Type of struct representing a memory allocator. It has the following fields:

ccmem_allocator_methods_t const * const methods

Pointer to a table of methods representing the behaviour of this allocator.

Instances of this type are meant to be wrapped in other struct to add state to the allocator. For example: 

typedef struct my_allocator_t   my_allocator_t;

struct my_allocator_t {
  ccmem_allocator_t     base;
  void *                state;
}
Struct Typedef: ccmem_allocator_methods_t

Type of struct representing the methods of a memory allocator. It has the following fields:

ccmem_malloc_fun_t * const malloc

Pointer to function that allocates memory in a way similar to the standard malloc().

ccmem_realloc_fun_t * const realloc

Pointer to function that reallocates memory in a way similar to the standard realloc().

ccmem_calloc_fun_t * const calloc

Pointer to function that allocates memory in a way similar to the standard calloc().

ccmem_free_fun_t * const free

Pointer to function that releases memory in a way similar to the standard free().

Function: void * ccmem_malloc_fun_t (cce_destination_t L, ccmem_allocator_t const * const A, size_t size)

Type of function that allocates memory in a way similar to the standard malloc(). The function uses the allocator A. If an error occurs: an exception is raised by jumping to the location L. Functions of this type never return NULL.

Function: void * ccmem_realloc_fun_t (cce_destination_t L, ccmem_allocator_t const * const A, void * ptr, size_t newsize)

Type of function that reallocates memory in a way similar to the standard realloc(). The function uses the allocator A. Only memory blocks allocated by A can be reallocated by this function. If an error occurs: an exception is raised by jumping to the location L. Functions of this type never return NULL.

Function: void * ccmem_calloc_fun_t (cce_destination_t L, ccmem_allocator_t const * const A, size_t count, size_t eltsize)

Type of function that allocates memory in a way similar to the standard calloc(). The function uses the allocator A. If an error occurs: an exception is raised by jumping to the location L. Functions of this type never return NULL.

Function: void ccmem_free_fun_t (cce_destination_t L, ccmem_allocator_t const * const A, void * ptr)

Type of function that releases memory in a way similar to the standard free(). The function uses the allocator A. Only memory allocated using the allocator A can be released with this function. If an error occurs: an exception is raised by jumping to the location L.

3.2 Programming interface

Function: void * ccmem_malloc (cce_destination_t L, ccmem_allocator_t const * const A, size_t size)

Allocate memory in a way similar to the standard malloc(), using the allocator A. If an error occurs: an exception is raised by jumping to the location L. This function never returns NULL.

Function: void * ccmem_realloc (cce_destination_t L, ccmem_allocator_t const * const A, void * ptr, size_t newsize)

Reallocate memory in a way similar to the standard realloc(), using the allocator A. If an error occurs: an exception is raised by jumping to the location L. This function never returns NULL. Only blocks of memory allocated using A can be reallocated using A.

Function: void * ccmem_calloc (cce_destination_t L, ccmem_allocator_t const * const A, size_t count, size_t eltsize)

Allocate memory in a way similar to the standard calloc(), using the allocator A. If an error occurs: an exception is raised by jumping to the location L. This function never returns NULL.

Function: void ccmem_free (cce_destination_t L, ccmem_allocator_t const * const A, void * ptr)

Release memory in a way similar to the standard free(), using the allocator A. If an error occurs: an exception is raised by jumping to the location L.

5.1 Type definitions

Struct Typedef: ccmem_block_t

Type of struct representing a block of memory. It has the following fields:

size_t len

The number of octets referenced by this block.

uint8_t * ptr

Pointer to the first octet in this block.

Blocks are so small (two machine words) that they can be used as arguments to function and return values with a very small overhead.

Next: blocks handlers, Previous: blocks typedefs, Up: blocks   [Contents][Index]

5.2 Constructors and destructors

Example of allocating and releasing a memory block: 

cce_location_t        L[1];

if (cce_location(L)) {
  cce_run_error_handlers_final(L);
} else {
   ccmem_block_t  B =
     ccmem_block_new(L, ccmem_standard_allocator, 4096);

  ccmem_block_delete(L, ccmem_standard_allocator, B);
  cce_run_clean_handlers(L);
}
Function: ccmem_block_t ccmem_block_new (cce_destination_t L, ccmem_allocator_t const * A, size_t len)

Allocate memory using A, then build and return a new block. If an error occurs: raise an exception by jumping to the location L.

Function: void ccmem_block_delete (cce_destination_t L, ccmem_allocator_t const * A, ccmem_block_t B)

Release memory using A. Only blocks allocated by A can be released by A. If an error occurs: raise an exception by jumping to the location L.

5.3 Exception handlers for memory blocks

Example of allocating and releasing a memory block with exception handlers: 

cce_location_t                L[1];
ccmem_block_clean_handler_t   B_H[1];

if (cce_location(L)) {
  cce_run_error_handlers_final(L);
} else {
  ccmem_block_t  B = ccmem_block_new(L,
                       ccmem_standard_allocator, 4096);
  ccmem_block_register_clean_handler(L, B_H,
    ccmem_standard_allocator, B);

  /* ... do something with B ... */
  cce_run_clean_handlers(L);
}
Struct Typedef: ccmem_block_clean_handler_t
Struct Typedef: ccmem_block_error_handler_t

Type of struct used to handle memory blocks release whenever an exception is raised in the local context. These types are wrappers for cce_clean_handler_t and cce_error_handler_t as defined by the library CCExceptions.

Function: void ccmem_block_register_clean_handler (cce_destination_t L, ccmem_block_clean_handler_t * H, ccmem_allocator_t const * A, ccmem_block_t B)
Function: void ccmem_block_register_error_handler (cce_destination_t L, ccmem_block_error_handler_t * H, ccmem_allocator_t const * A, ccmem_block_t B)

Register the allocator A and the block B in the handler H in the context of L. The block is released using the allocator in case of cleaning or error.

NOTE If an error occurs releasing memory: the registered handler will discard the raised exception.

5.4 Guarded constructors

Example of allocating and releasing a memory block with a guarded constructor: 

cce_location_t                L[1];
ccmem_block_clean_handler_t   B_H[1];

if (cce_location(L)) {
  cce_run_error_handlers_final(L);
} else {
  ccmem_block_t  B = ccmem_block_new_guarded(L, B_H,
                       ccmem_standard_allocator, 4096);

  /* ... do something with B ... */
  cce_run_clean_handlers(L);
}
Function: ccmem_block_t ccmem_block_new_guarded_clean (cce_destination_t L, ccmem_block_clean_handler_t * H, ccmem_allocator_t const * A, size_t len)

Allocate memory using A, then build and return a new block. If an error occurs: raise an exception by jumping to the location L. The allocator and the block are registered in the exception handler H, so that the block is released in case of cleaning.

NOTE If an error occurs releasing memory: the registered handler will discard the raised exception.

Function: ccmem_block_t ccmem_block_new_guarded_error (cce_destination_t L, ccmem_block_error_handler_t * H, ccmem_allocator_t const * A, size_t len)

Allocate memory using A, then build and return a new block. If an error occurs: raise an exception by jumping to the location L. The allocator and the block are registered in the exception handler H, so that the block is released in case of error.

NOTE If an error occurs releasing memory: the registered handler will discard the raised exception.

Preprocessor Macro: ccmem_block_t ccmem_block_new_guarded (cce_destination_t L, ccmem_block_clean_handler_t * H, ccmem_allocator_t const * A, size_t len)
Preprocessor Macro: ccmem_block_t ccmem_block_new_guarded (cce_destination_t L, ccmem_block_error_handler_t * H, ccmem_allocator_t const * A, size_t len)

This macro uses _Generic to dispatch the expansion according to the type of H. Allocate memory using A, then build and return a new block. If an error occurs: raise an exception by jumping to the location L. If the argument H is of type:

ccmem_block_clean_handler_t *

The allocator and the block are registered in the exception handler H, so that the block is released in case of cleaning.

ccmem_block_error_handler_t *

The allocator and the block are registered in the exception handler H, so that the block is released in case of error.

5.6 Operating on memory blocks

Function: void ccmem_block_memset (ccmem_block_t B, uint8_t octet)

Set to octet all the octets referenced by B.

Function: void ccmem_block_clean (ccmem_block_t B)

Set to zero all the octets referenced by B.

Function: void ccmem_block_nullify (ccmem_block_t * BP)

Nullify the block by setting the pointer to NULL and the number of octets to zero.

Function: ccmem_block_t ccmem_block_shift (ccmem_block_t B, ssize_t offset, size_t dim)

Imagine that B is referencing a block of memory holding objects of size dim; build and return a new block referencing the object at offset objects forwards or backwards. 

#include <complex.h>

cce_location_t  L[1];

ccmem_block_t  B = ccmem_block_new(L, ccmem_standard_allocator,
  3 * sizeof(complex));

ccmem_block_t  X = ccmem_block_shift(B, +2, sizeof(complex));
ccmem_block_t  Y = ccmem_block_shift(X, -1, sizeof(complex));
Function: ccmem_block_t ccmem_block_difference (ccmem_block_t A, ccmem_block_t B)

Build and return a new block representing the difference A - B. If C is the returned block, this means: 

C.ptr = A.ptr
C.len = A.len - B.len

A and B must have the same pointer and A must reference a number of octets greater than, or equal to, the one referenced by B, as in the following picture: 

                     A
|---------------------------------------|
|-----------------------|---------------|
           B                    C

6 Utility definitions

Macro: CCMEM_PC (POINTER_TYPE, POINTER_NAME, EXPRESSION)

Preprocessor macro used to define pointer variables initialised with a cast expression. It expands to: 

POINTER_TYPE * POINTER_NAME = (POINTER_TYPE *) (EXPRESSION)

For example, the macro use: 

CCMEM_PC(ccmem_descriptor_test_failure_t, C, _C);

expands into: 

ccmem_descriptor_test_failure_t * C = \
   (ccmem_descriptor_test_failure_t *) (_C);
Macro: CCMEM_UNUSED

Preprocessor macro that is meant to expand into an attribute declaration. If the symbol __GNUC__ is defined, the expansion is: 

__attribute__((__unused__))

otherwise the expansion is the empty string.
