9.2.2 Handling errno codes

Vicare makes use of the platform's own errno codes; they are negated to make them different from file descriptors, and so distinguishable when used as single return values from C functions, and converted to fixnums. Example: if the C language code for EPERM is the integer 1, the corresponding Scheme language value is the fixnum -1.

— Function: ikptr ik_errno_to_code (void)

Negate the current errno value and convert the result into a fixnum, return the fixnum. When an error occurs: many functions call this function and return its return value.

— Function: ikptr ikrt_strerror (ikptr ec, ikpcb * pcb)

Interface to the C function strerror(), strerror. Convert a fixnum representing an encoded errno code into a bytevector holding the string message which describes it; return a reference to the bytevector; if ec is invalid: return false.

— Function: ikptr ikrt_last_errno (ikpcb * pcb)

Return a fixnum representing the negated value of errno right after the last call to a foreign function through the FFI.
