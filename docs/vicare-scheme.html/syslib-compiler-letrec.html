12.20.5 Optimisation of letrec forms

For an introduction to processing letrec and letrec* syntaxes, and specifically to understand the code, we must read the following paper:

Oscar Waddell, Dipanwita Sarkar, R. Kent Dybvig. “Fixing Letrec: A Faithful Yet Efficient Implementation of Scheme's Recursive Binding Construct”.

then we can move to the following paper, which describes the SCC transformation used by default:

Abdulaziz Ghuloum, R. Kent Dybvig. “Fixing Letrec (reloaded)”. Workshop on Scheme and Functional Programming '09.

The following bindings are exported by the library (ikarus system $compiler).

— Function: $optimize-letrec input

Perform transformations to convert the recordized representation of libary-letrec*, LETREC and letrec* forms into low level binding constructs and assignments. The result is such that the order of bindings in a single binding construct can be changed at will.

— Parameter: $check-for-illegal-letrec

If set to true: $optimize-letrec will perform an additional recordized code pass to check for illegal binding references in the right–hand sides of letrec, letrec* and library-letrec* syntaxes, raising a syntax violation error if one is found. This is the default.

If set to false: the validating pass is not performed, faithfully hoping for no problems in code.

— Parameter: $current-letrec-pass

Select the transformations performed by $optimize-letrec; possible values are the symbols:

basic
The transformations are equivalent to: 
               (letrec* ((?var ?init) ...) . ?body)
               ==> (let ((?var (void)) ...)
                     (set! ?var ?init) ... . ?body)
               
               (library-letrec* ((?var ?loc ?init) ...) . ?body)
               ==> (let ((?var (void)) ...)
                     (set! ?var ?init) ... . ?body)
               
               (letrec ((?var ?init) ...) . ?body)
               ==> (let ((?var (void)) ...)
                     (let ((?tmp ?init) ...)
                       (set! ?var ?tmp) ... . ?body))

waddell
Perform a transformation similar (but no equal) to the one described in the “Fixing Letrec” paper by Waddell et al.
scc
This is the default.
