1 Overview of the package

Vicare Scheme is an R6RS compliant Scheme language implementation in the form of a native compiler for x86 architectures, officially supporting GNU+Linux systems. Vicare/GSASL is a distribution of C language and Scheme language libraries for Vicare Scheme; it provides bindings for the GSASL C language library.

The last time the author bothered to update this paragraph, he had tested GSASL version 1.8.0; the GSASL library is available at:

http://www.gnu.org/software/gsasl/

The package installs a C language library implementing wrapper C functions for the C language GSASL library; on top of this, the Scheme library (vicare crypto gsasl) exports one binding for each GSASL public function; additionally the library (vicare crypto gsasl constants) exports one binding for each constant value defined in the C language header gsasl.h.

The following Scheme libraries are installed:

(vicare crypto gsasl)

It exports one binding for each GSASL public function. All the Scheme function names are directly derived from the C function names by replacing underscore characters ‘_’ with dash characters ‘-’; so gsasl_init() becomes gsasl-init.

(vicare crypto gsasl unsafe-capi)

It exports one syntax binding for each GSASL public function. All the Scheme syntax names are directly derived from the C function names by replacing underscore characters ‘_’ with dash characters ‘-’; so gsasl_init() becomes gsasl-init.

These syntaxes expand to a direct call to the C language wrapper functions in the Vicare/GSASL library; they are not meant to be called in normal usage of the package.

(vicare crypto gsasl constants)

It exports one binding for each constant value defined in the C language header gsasl.h.

(vicare crypto gsasl features)

It exports one identifier syntax binding for each HAVE_ constant defined by the configure script, expanding to #t or #f.

Scheme libraries are installed under the directory:

$(libdir)/vicare-scheme

This document contains only a brief description of the functions: refer to the GSASL’s documentation for details.

2 GSASL version informations

The installed C library follows version numbering as established by the GNU Autotools. For an explanation of interface numbers as managed by GNU Libtool See Libtool’s versioning system in Shared library support for GNU.

The following bindings are exported by the library (vicare crypto gsasl).

Function: vicare-gsasl-version-interface-current
Function: vicare-gsasl-version-interface-revision
Function: vicare-gsasl-version-interface-age

Return a fixnum representing a version number.

Function: vicare-gsasl-version

Return a Scheme string representing the version number.

Function: gsasl-check-version required-version

Check that the loaded GSASL library conforms with the version specified by required-version. If successful return a Scheme string representing the locaded version of GSASL, else return #f.

required-version must be a general C string; (vicare-scheme)Introduction to generalised C strings. Typically this argument is the constant GSASL_VERSION.

3.1 PLAIN mechanism, GSASL_PASSWORD, clear passwords, ASCII strings

The following example shows mock server and client performing login authorisation with the PLAIN mechanism, by exchanging ASCII strings through a socket pair. The server performs authentication with the GSASL_PASSWORD mechanism. 

#!r6rs
(import (vicare)
  (vicare crypto gsasl)
  (vicare crypto gsasl constants)
  (prefix (vicare ffi) ffi.)
  (prefix (vicare posix) px.)
  (prefix (vicare platform constants) px.)
  (vicare syntactic-extensions))

(define-constant SASL_MECHANISM
  "PLAIN")

(module (main client.log server.log)

  (define (main username password message)
    ;;Main  demo function.   Fork a  process, start  the
    ;;server in the parent  process, start the client in
    ;;the  child  process.   Connect server  and  client
    ;;through a socket pair.
    ;;
    (receive (server.sock client.sock)
        (px.socketpair px.AF_LOCAL px.SOCK_DGRAM 0)
      (let ((P (make-sock-port server.sock "server"))
            (Q (make-sock-port client.sock "client")))
        (px.fork (lambda (child-pid)
                   (server child-pid P))
                 (lambda ()
                   (client username password message Q))))))

  (define (server child-pid port)
    ;;Server driver.   Start the server, run  a session,
    ;;wait for the child process to terminate.
    ;;
    (guard (E (else
               (pretty-print E (current-error-port))))
      (receive (server.run-session server.stop)
          (server.start)
        (unwind-protect
            (server.run-session port)
          (server.stop))))
    (server.log "waiting child")
    (let ((status (px.waitpid child-pid 0)))
      (server.log "child terminated, exit status ~a"
                  (and (px.WIFEXITED   status)
                       (px.WEXITSTATUS status))))
    (server.log "exiting"))

  (define (client username password message port)
    ;;Client driver.   Start the client, run  a session,
    ;;terminate the child process.
    ;;
    (guard (E (else (pretty-print E (current-error-port))))
      (receive (client.run-session client.stop)
          (client.start)
        (unwind-protect
            (client.run-session port username password message)
          (client.stop))))
    (client.log "exiting")
    (exit 0))

  (define (make-sock-port sock id)
    ;;Wrap a  textual input/output port around  a system
    ;;socket descriptor.
    ;;
    (let ((P (make-textual-socket-input/output-port
              sock id (native-transcoder))))
      (set-port-buffer-mode! P (buffer-mode line))
      P))

  (define (server.log template . args)
    (log "server" template args))

  (define (client.log template . args)
    (log "client" template args))

  (define (log which template args)
    (fprintf (current-error-port)
             "~a *** ~a\n" which
             (apply format template args)))

  #| end of module: main |# )

(module (server.start)

  (define (server.start)
    ;;Start the  server, create and configure  the GSASL
    ;;context; return 2 functions:
    ;;
    ;;1. Run a server session.
    ;;
    ;;2. Stop the server.
    ;;
    (import SERVER.AUTHENTICATION)
    (server.log "starting server")
    (let* ((ctx (gsasl-init))
           (cb  (make-gsasl-callback-function server.callback)))
      (gsasl-callback-set ctx cb)
      (values
       (lambda (port)
         ;;Run  a  server  session.   Create  the  GSASL
         ;;session, perform  authentication, receive the
         ;;message from  the client, close  the session.
         ;;Return unspecified values.
         ;;
         (unwind-protect
             (let ((session (gsasl-server-start ctx SASL_MECHANISM)))
               (unwind-protect
                   (parametrise ((server.port port))
                     (server.run-session session))
                 (gsasl-finish session)))
           (close-port port)))
       (lambda ()
         ;;Stop the server.  Return unspecified values.
         ;;
         (server.log "stopping server")
         (gsasl-done ctx)
         (ffi.free-c-callback cb)))))

  (define (server.run-session session)
    ;;Run a server session.  Exchange greetings, perform
    ;;authentication, receive message, receive quitting.
    ;;
    (server.send "HELO client")
    (server.recv) ;receive greetings
    (server.send "OK")
    (server.recv) ;receive AUTH PLAIN
    (server.send "OK")
    (when (server.auth-loop session (server.recv))
      (server.recv) ;receive message
      (server.send "OK")
      (server.recv))) ;receive QUIT

  (define (server.auth-loop session line)
    ;;Actually   perform   the  client   authentication.
    ;;Recursive function; the first LINE argument should
    ;;be the client's credentials.
    ;;
    ;;If  the  authentication  succeeds: send  "OK"  and
    ;;return  true; if  the  authentication fails:  send
    ;;"ERROR" and return false.
    ;;
    (define who 'server.auth-loop)
    (receive (code text)
        (gsasl-step/string session line)
      ($case-integers code
        ((GSASL_NEEDS_MORE)
         (server.send text)
         (server.auth-loop session (server.recv)))
        ((GSASL_OK)
         (server.log "authorisation success")
         (server.send "OK")
         #t)
        ((GSASL_AUTHENTICATION_ERROR)
         (server.log "authorisation failed")
         (server.send "ERROR")
         #f)
        (else
         (error who (gsasl-strerror code))))))

  (define server.port
    (make-parameter #f))

  (define (server.send str)
    (fprintf (current-error-port)
             "server send> ~s\n" str)
    (display str (server.port))
    (display #\newline (server.port)))

  (define (server.recv)
    (let ((line (read-line (server.port))))
      (fprintf (current-error-port)
               "server recv> ~s\n" line)
      line))

  #| end of module: server |# )

(module (client.start)

  (define (client.start)
    ;;Start  the  client,   create  the  GSASL  context.
    ;;Return 2 procedures:
    ;;
    ;;1. Run a client session.
    ;;
    ;;2. Stop the client.
    ;;
    (import CLIENT.AUTHENTICATION)
    (client.log "starting client")
    (let* ((ctx (gsasl-init))
           (cb  (make-gsasl-callback-function client.callback)))
      (gsasl-callback-set ctx cb)
      (values
       (lambda (port username password message)
         ;;Create    the    GSASL    session,    request
         ;;authorisation,  send   the  message.   Return
         ;;unspecified values.
         ;;
         (unwind-protect
             (let ((session (gsasl-client-start ctx SASL_MECHANISM)))
               (unwind-protect
                   (begin
                     (session.register-credentials session username password)
                     (parametrise ((client.port port))
                       (client.run-session session message)))
                 (gsasl-finish session)))
           (close-port port)))
       (lambda ()
         ;;Stop the client.  Return unspecified values.
         ;;
         (client.log "stopping client")
         (gsasl-done ctx)
         (ffi.free-c-callback cb)))))

  (define (session.register-credentials session username password)
    ;;Register the username and  password strings in the
    ;;custom data associated to the session data struct;
    ;;also register an  appropriate custom destructor to
    ;;release the values.  Return unspecified values.
    ;;
    (gsasl-session-hook-set
     session (register-to-avoid-collecting
              (cons username password)))
    (set-gsasl-session-custom-destructor!
     session (lambda (session)
               (forget-to-avoid-collecting
                (gsasl-session-hook-get session)))))

  (define (client.run-session session message)
    ;;Run a client session.  Exchange greetings, perform
    ;;authentication, send message, send quitting.
    ;;
    (client.recv) ;receive greetings
    (client.send "HELO server")
    (client.recv) ;receive OK
    (client.send "AUTH PLAIN")
    (client.recv) ;receive OK
    (when (client.auth-loop session "")
      (client.send message)
      (client.recv) ;receive OK
      (client.send "QUIT")))
   (define (client.auth-loop session line)
    ;;Actually   perform   the  client   authentication.
    ;;Recursive function; the first LINE argument should
    ;;be the empty string.
    ;;
    ;;If  the authentication  succeeds: return  true; if
    ;;the authentication fails: return false.
    ;;
    (define who 'client.auth-loop)
    (receive (code text)
        (gsasl-step/string session line)
      ($case-integers code
        ((GSASL_NEEDS_MORE)
         (client.send text)
         (client.auth-loop session (client.recv)))
        ((GSASL_OK)
         (client.send text)
         (case-strings (client.recv)
           (("OK")
            (client.log "authorisation success")
            #t)
           (else
            (client.log "authorisation failed")
            #f)))
        (else
         (error who (gsasl-strerror code))))))

  (define client.port
    (make-parameter #f))

  (define (client.send str)
    (fprintf (current-error-port)
             "client send> ~s\n" str)
    (display str (client.port))
    (display #\newline (client.port)))

  (define (client.recv)
    (let ((line (read-line (client.port))))
      (fprintf (current-error-port)
               "client recv> ~s\n" line)
      line))

  #| end of module: client.start |# )

(module SERVER.AUTHENTICATION
  (server.callback)
  ;;This  module organises  matching client  credentials
  ;;with  account informations  stored  in the  password
  ;;database.  Everything  happens through calls  to the
  ;;server  callback  registered  in  the  server  GSASL
  ;;context.

  (define (server.callback context session property)
    ;;This is  the callback for the  server context.  It
    ;;is expected to return one among:
    ;;
    ;;  GSASL_OK
    ;;  GSASL_AUTHENTICATION_ERROR
    ;;  GSASL_NO_CALLBACK
    ;;
    (define (authid)
      (gsasl-property-get/string session GSASL_AUTHID))
    (define (passwd pwd)
      (gsasl-property-set session GSASL_PASSWORD pwd))
    (guard (E (else
               (pretty-print E (current-error-port))
               GSASL_NO_CALLBACK))
      (server.log "callback requested property ~a"
                  (gsasl-property->symbol property))
      ($case-integers property
        ((GSASL_VALIDATE_SIMPLE)
         GSASL_NO_CALLBACK)
        ((GSASL_PASSWORD)
         (cond ((assoc (authid) PASSWORDS-DATABASE)
                => (lambda (database-row)
                     (passwd (cdr database-row))))
               (else
                ;;Unknown username.
                GSASL_AUTHENTICATION_ERROR)))
        (else
         GSASL_NO_CALLBACK))))

  (define-constant PASSWORDS-DATABASE
    ;;Passwords are  stored in  clear in  this database.
    ;;Ha! Ha! Ha!
    ;;
    '(("marco" . "ciao")
      ("mark"  . "hello")
      ("sora"  . "ohayo")))

  #| end of module: server.callback |# )

(module CLIENT.AUTHENTICATION
  (client.callback)
  ;;This  module  organises sharing  client  credentials
  ;;with  the  GSASL  session used  for  authentication.
  ;;Everything  happens  through  calls  to  the  client
  ;;callback registered in the client GSASL context.
  ;;
  (define (client.callback context session property)
    ;;This is  the callback for the  client context.  It
    ;;is    expected     to    return     GSASL_OK    or
    ;;GSASL_NO_CALLBACK.
    ;;
    (define (authid str)
      (gsasl-property-set session GSASL_AUTHID str))
    (define (passwd str)
      (gsasl-property-set session GSASL_PASSWORD str))
    (define (credentials)
      (retrieve-to-avoid-collecting
       (gsasl-session-hook-get session)))
    (define (credentials.username)
      (car (credentials)))
    (define (credentials.password)
      (cdr (credentials)))
    (guard (E (else
               (pretty-print E (current-error-port))
               GSASL_NO_CALLBACK))
      (client.log "callback requested property ~a"
                  (gsasl-property->symbol property))
      ($case-integers property
        ((GSASL_AUTHID)
         (authid (credentials.username)))
        ((GSASL_PASSWORD)
         (passwd (credentials.password)))
        (else
         GSASL_NO_CALLBACK))))

  #| end of module: client.callback |# )

  (server.callback)
  ;;This  module organises  matching client  credentials
  ;;with  account informations  stored  in the  password
  ;;database.  Everything  happens through calls  to the
  ;;server  callback  registered  in  the  server  GSASL
  ;;context.

  (define (server.callback context session property)
    ;;This is  the callback for the  server context.  It
    ;;is expected to return one among:
    ;;
    ;;  GSASL_OK
    ;;  GSASL_AUTHENTICATION_ERROR
    ;;  GSASL_NO_CALLBACK
    ;;

      (gsasl-property-get/string session GSASL_AUTHID))
    (define (passwd)
      (gsasl-property-get/string session GSASL_PASSWORD))
    (guard (E (else
               (pretty-print E (current-error-port))
               GSASL_NO_CALLBACK))
      (server.log "callback requested propert ~a"
                  (gsasl-property->symbol property))
      ($case-integers property
        ((GSASL_VALIDATE_SIMPLE)
         (cond ((assoc (authid) PASSWORDS-DATABASE)
                => (lambda (database-row)
                     (if (bytevector=? (gsasl-sha1 (passwd))
                                       (cdr database-row))
                         GSASL_OK
                       ;;Wrong password.
                       GSASL_AUTHENTICATION_ERROR)))
               (else
                ;;Unknown username.
                GSASL_AUTHENTICATION_ERROR)))
        (else
         GSASL_NO_CALLBACK))))

    ;;Passwords  are  stored in  as  SHA1  sums in  this
    ;;database;  notice that  the sums  are bytevectors,
    ;;not strings.
    ;;

           (cons (car entry)
                 (gsasl-sha1 (cdr entry))))
      '(("marco" . "ciao")
        ("mark"  . "hello")
        ("sora"  . "ohayo"))))

3.8 DIGEST-MD5 mechanism, GSASL_PASSWORD, clear passwords, ASCII strings

The following example shows mock server and client performing login authorisation with the DIGEST-MD5 mechanism, by exchanging ASCII strings through a socket pair. The server performs authentication with the GSASL_PASSWORD mechanism.

(module (server.start)

  (define (server.start)
    ;;Start the  server, create and configure  the GSASL
    ;;context; return 2 functions:
    ;;
    ;;1. Run a server session.
    ;;
    ;;2. Stop the server.
    ;;
    (import SERVER.AUTHENTICATION)

         (gsasl-done ctx)
         (ffi.free-c-callback cb)))))

  (define (server.run-session session)
    ;;Run a server session.  Exchange greetings, perform
    ;;authentication, receive message, receive quitting.
    ;;
    (server.send "HELO client")
    (server.recv) ;receive greetings
    (server.send "OK")
    (server.recv) ;receive AUTH DIGEST-MD5

      (server.recv) ;receive OK
      (server.send "OK")
      (server.recv) ;receive message
      (server.send "OK")
      (server.recv))) ;receive QUIT

    ;;Actually   perform   the  client   authentication.
    ;;Recursive function; the first LINE argument should
    ;;be the empty string.
    ;;
    ;;If  the  authentication  succeeds: send  "OK"  and
    ;;return  true; if  the  authentication fails:  send
    ;;"ERROR" and return false.

        (gsasl-step/string session line)

        ((GSASL_NEEDS_MORE)
         (server.send text)
         (server.auth-loop session (server.recv)))
        ((GSASL_OK)
         (server.log "authorisation success")
         (server.send text)
         #t)
        ((GSASL_AUTHENTICATION_ERROR)
         (server.log "authorisation failed")
         (server.send "ERROR")
         #f)

(module SERVER.AUTHENTICATION
  (server.callback)
  ;;This  module organises  matching client  credentials
  ;;with  account informations  stored  in the  password
  ;;database.  Everything  happens through calls  to the
  ;;server  callback  registered  in  the  server  GSASL
  ;;context.

  (define (server.callback context session property)
    ;;This is  the callback for the  server context.  It
    ;;is expected to return one among:
    ;;
    ;;  GSASL_OK
    ;;  GSASL_AUTHENTICATION_ERROR
    ;;  GSASL_NO_CALLBACK
    ;;

      (gsasl-property-get/string session GSASL_AUTHID))
    (define (passwd pwd)
      (gsasl-property-set session GSASL_PASSWORD pwd))
    (guard (E (else
               (pretty-print E (current-error-port))
               GSASL_NO_CALLBACK))

                  (gsasl-property->symbol property))

                => (lambda (database-row)
                     (passwd (cdr database-row))))

                GSASL_AUTHENTICATION_ERROR)))

         GSASL_NO_CALLBACK))))

         (authid (credentials.username)))
        ((GSASL_PASSWORD)
         (passwd (credentials.password)))
        ((GSASL_SERVICE)
         (service CLIENT_SERVICE))
        ((GSASL_HOSTNAME)
         (hostname CLIENT_HOSTNAME))
        ((GSASL_REALM)
         (realm CLIENT_REALM))

         GSASL_NO_CALLBACK))))

3.9 DIGEST-MD5 mechanism, GSASL_DIGEST_MD5_HASHED_PASSWORD, hashed passwords, ASCII strings

The following example shows mock server and client performing login authorisation with the DIGEST-MD5 mechanism, by exchanging ASCII strings through a socket pair. The server performs authentication with the GSASL_DIGEST_MD5_HASHED_PASSWORD mechanism. Passwords are stored as hash checksums on the server.

Notice that when GSASL asks the server’s callback the value for the GSASL_DIGEST_MD5_HASHED_PASSWORD property, the server must:

  1. Compose the string: <user>:<realm>:<password>.
  2. Hash such string with the function gsasl-md5.
  3. Convert the checksum’s bytevector to a string of hex digits (for example with the function bytevector->string-hex).
  4. Convert the hex string to lowercase.
  5. Set the lowercase string as property value.

3.10 SCRAM-SHA-1 mechanism, GSASL_PASSWORD, clear passwords, ASCII strings

The following example shows mock server and client performing login authorisation with the SCRAM-SHA-1 mechanism, by exchanging ASCII strings through a socket pair. The server performs authentication with the GSASL_password mechanism.

The server initialises both the GSASL_SCRAM_ITER and GSASL_SCRAM_SALT properties. Notice that:

  1. The property GSASL_SCRAM_ITER wants as value a string representing a number of iterations.
  2. The property GSASL_SCRAM_SALT wants as value a string random data encoded in base64. 
#!r6rs
(import (vicare)
  (vicare crypto gsasl)
  (vicare crypto gsasl constants)
  (prefix (vicare ffi) ffi.)
  (prefix (vicare posix) px.)
  (prefix (vicare platform constants) px.)
  (vicare syntactic-extensions))

(define-constant SASL_MECHANISM
  "SCRAM-SHA-1")

(module (main client.log server.log)

  (define (main username password message)
    ;;Main  demo function.   Fork a  process, start  the
    ;;server in the parent  process, start the client in
    ;;the  child  process.   Connect server  and  client
    ;;through a socket pair.
    ;;
    (receive (server.sock client.sock)
        (px.socketpair px.AF_LOCAL px.SOCK_DGRAM 0)
      (let ((P (make-sock-port server.sock "server"))
            (Q (make-sock-port client.sock "client")))
        (px.fork (lambda (child-pid)
                   (server child-pid P))
                 (lambda ()
                   (client username password message Q))))))

  (define (server child-pid port)
    ;;Server driver.   Start the server, run  a session,
    ;;wait for the child process to terminate.
    ;;
    (guard (E (else
               (pretty-print E (current-error-port))))
      (receive (server.run-session server.stop)
          (server.start)
        (unwind-protect
            (server.run-session port)
          (server.stop))))
    (server.log "waiting child")
    (let ((status (px.waitpid child-pid 0)))
      (server.log "child terminated, exit status ~a"
                  (and (px.WIFEXITED   status)
                       (px.WEXITSTATUS status))))
    (server.log "exiting"))

  (define (client username password message port)
    ;;Client driver.   Start the client, run  a session,
    ;;terminate the child process.
    ;;
    (guard (E (else
               (pretty-print E (current-error-port))))
      (receive (client.run-session client.stop)
          (client.start)
        (unwind-protect
            (client.run-session port username password message)
          (client.stop))))
    (client.log "exiting")
    (exit 0))

    ;;Wrap a  textual input/output port around  a system
    ;;socket descriptor.

      (set-port-buffer-mode! P (buffer-mode line))

(module (server.start)

  (define (server.start)
    ;;Start the  server, create and configure  the GSASL
    ;;context; return 2 functions:
    ;;
    ;;1. Run a server session.
    ;;
    ;;2. Stop the server.
    ;;
    (import SERVER.AUTHENTICATION)
    (server.log "starting server")
    (let* ((ctx (gsasl-init))
           (cb  (make-gsasl-callback-function server.callback)))
      (gsasl-callback-set ctx cb)
      (values
       (lambda (port)
         ;;Run  a  server  session.   Create  the  GSASL
         ;;session, perform  authentication, receive the
         ;;message from  the client, close  the session.
         ;;Return unspecified values.
         ;;
         (unwind-protect
             (let ((session (gsasl-server-start ctx
                               SASL_MECHANISM)))
               (unwind-protect
                   (parametrise ((server.port port))
                     (server.run-session session))
                 (gsasl-finish session)))
           (close-port port)))
       (lambda ()
         ;;Stop the server.  Return unspecified values.
         ;;
         (server.log "stopping server")
         (gsasl-done ctx)
         (ffi.free-c-callback cb)))))

  (define (server.run-session session)
    ;;Run a server session.  Exchange greetings, perform
    ;;authentication, receive message, receive quitting.
    ;;
    (server.send "HELO client")
    (server.recv) ;receive greetings
    (server.send "OK")
    (server.recv) ;receive AUTH SCRAM-SHA-1
    (server.send "OK")

      (server.send "OK")
      (server.recv))) ;receive QUIT

    (define who 'server.auth-loop)

        (gsasl-step/string session line)

        ((GSASL_NEEDS_MORE)
         (server.send text)
         (server.auth-loop session (server.recv)))
        ((GSASL_OK)
         (server.log "authorisation success")
         (server.send text)
         #t)

(module (server.start)

  (define (server.start)
    ;;Start the  server, create and configure  the GSASL
    ;;context; return 2 functions:
    ;;
    ;;1. Run a server session.
    ;;
    ;;2. Stop the server.
    ;;
    (import SERVER.AUTHENTICATION)
    (server.log "starting server")
    (let* ((ctx (gsasl-init))
           (cb  (make-gsasl-callback-function server.callback)))
      (gsasl-callback-set ctx cb)
      (values
       (lambda (port)
         ;;Run  a  server  session.   Create  the  GSASL
         ;;session, perform  authentication, receive the
         ;;message from  the client, close  the session.
         ;;Return unspecified values.
         ;;
         (unwind-protect
             (let ((session (gsasl-server-start ctx
                               SASL_MECHANISM)))
               (unwind-protect
                   (parametrise ((server.port port))
                     (server.run-session session))
                 (gsasl-finish session)))
           (close-port port)))
       (lambda ()
         ;;Stop the server.  Return unspecified values.
         ;;
         (server.log "stopping server")
         (gsasl-done ctx)
         (ffi.free-c-callback cb)))))

  (define (server.run-session session)
    ;;Run a server session.  Exchange greetings, perform
    ;;authentication, receive message, receive quitting.
    ;;
    (server.send "HELO client")
    (server.recv) ;receive greetings
    (server.send "OK")
    (server.recv) ;receive AUTH SECURID
    (server.send "OK")
    (when (server.auth-loop session (server.recv))
      (server.recv) ;receive message
      (server.send "OK")
      (server.recv))) ;receive QUIT

  (define (server.auth-loop session line)
    ;;Actually   perform   the  client   authentication.
    ;;Recursive function; the first LINE argument should
    ;;be the client's credentials.
    ;;
    ;;If  the  authentication  succeeds: send  "OK"  and
    ;;return  true; if  the  authentication fails:  send
    ;;"ERROR" and return false.
    ;;
    (define who 'server.auth-loop)
    (receive (code text)
        (gsasl-step/string session line)
      ($case-integers code
        ((GSASL_NEEDS_MORE)
         (server.send text)
         (server.auth-loop session (server.recv)))
        ((GSASL_OK)
         (server.log "authorisation success")
         (server.send "OK")
         #t)
        ((GSASL_AUTHENTICATION_ERROR)
         (server.log "authorisation failed")
         (server.send "ERROR")
         #f)
        (else
         (error who (gsasl-strerror code))))))

    (make-parameter #f))

    (fprintf (current-error-port)
             "server send> ~s\n" str)
    (display str (server.port))
    (display #\newline (server.port)))

  (define (server.recv)
    (let ((line (read-line (server.port))))
      (fprintf (current-error-port)
               "server recv> ~s\n" line)

  ;;with   account  informations   stored  in   the  PIN
  ;;database.  Everything  happens through calls  to the
  ;;server  callback  registered  in  the  server  GSASL

    ;;This is  the callback for the  server context.  It
    ;;is expected to return one among:
    ;;
    ;;  GSASL_OK
    ;;  GSASL_AUTHENTICATION_ERROR
    ;;  GSASL_NO_CALLBACK
    ;;
    (define (authid)
      (gsasl-property-get/string session GSASL_AUTHID))

      (gsasl-property-get/string session GSASL_PASSCODE))

               (pretty-print E (current-error-port))

                       GSASL_AUTHENTICATION_ERROR)))

                ;;Unknown username.
                GSASL_AUTHENTICATION_ERROR)))
        (else
         GSASL_NO_CALLBACK))))

        ((GSASL_AUTHID)
         (authid (credentials.username)))
        ((GSASL_PASSCODE)
         (let ()
           (import SecurID)
           (passcode (generate-passcode (credentials.pin)))))
        (else
         GSASL_NO_CALLBACK))))

4.1 GSASL contexts

The following bindings are exported by the library (vicare crypto gsasl).

Struct Type: gsasl-context

An opaque Scheme structure holding a pointer object referencing the C language type Gsasl; it references an authentication context.

When instances of this type are used as arguments to functions: this documentation identifies them as context.

There are two categories of gsasl-context instances: those who own the underlying GSASL context and those who merely reference it.

  • gsasl-context instances returned by gsasl-init do own the session. When instances of this category are garbage collected or when gsasl-done is applied to them: the context is closed, all the associated data is finalised; any error is ignored in this procedure.
Function: gsasl-context? obj

Return #t if obj is an instance of gsasl-context; otherwise return #f.

Function: gsasl-context?/alive obj

Return #t if obj is an instance of gsasl-context and the Gsasl it represents is open; otherwise return #f.

Function: gsasl-context-custom-destructor context
Function: set-gsasl-context-custom-destructor! context func

Retrieve or set a destructor function associated to the context.

Whenever the context is closed, either explicitly with gsasl-done or implicitly by the garbage collector, func is applied to context before the internal state of context is finalised.

Notice that func can do anything, not only destroy some state associated to context; if we do: 

(define context ...)
(set-gsasl-context-custom-destructor! context
  (let ((state (make-some-data-struct)))
    (case-lambda
      ((sex)
       (destroy state))
      (()
       state))))

(let ((state ((gsasl-context-custom-destructor context))))
  (do-something-with context state))

we can easily associate values to context.
