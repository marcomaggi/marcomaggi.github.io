4.3 Parsing from a string

All the following symbols are defined in the header file semver.h.

Function: char semver_comp_read (semver_comp_t * compp, const char * input_buf, size_t input_len, size_t * input_offp)

Parse an input string in ASCII format which is meant to hold a comparator specification. When successful: the return value is 0. If an error occurs: the return value is 1.

The results of parsing are stored in the already allocated structure referenced by compp; the structure is first initialised with semver_comp_ctor().

In both the successful and unsuccessful cases: every call to this function must be matched by the application of semver_comp_dtor() to compp.

The input string is referenced by input_buf; the length of the input string is input_len characters. The argument input_offp is both an input and output parameter:

  • Upon entering the function: the offset of the first character to parse in input_buf is acquired from the variable referenced by input_offp.
  • When the function returns successfully: the offset of the first non–parsed character is stored in the variable referenced by input_offp; further parsing operations can use such value as input.
  • When the function returns unsuccessfully: the value stored in the variable is invalid.

If the string is empty (input_len is zero): the function returns unsuccessfully.

Internally, this function allocates memory with malloc(); if memory allocation fails: the return value is 1.

Function: char semver_comp_and (semver_comp_t * compp, const char * input_buf, size_t input_len, size_t * input_offp)

Parse an input string in ASCII format which is meant to hold a comparator specification. When successful: the return value is 0. If an error occurs: the return value is 1.

The results of parsing are appended to the already allocated and built structure referenced by compp.

In the unsuccessful case: the structure referenced by compp is left untouched.

The meaning of the arguments input_buf, input_len and input_offp is the same as in semver_comp_read().

If the string is empty (input_len is zero): the function returns unsuccessfully.

Internally, this function allocates memory with malloc(); if memory allocation fails: the return value is 1.

Example: parsing a comparator

The following example parses the comparator specification ‘<=1.2.3’: 

static const char     input_str[] = "<=1.2.3";
semver_comp_t comp;
size_t        offset = 0;
char          rv;

rv = semver_comp_read(&comp, \
        input_str, strlen(input_str), &offset);
if (0 == rv) {
  printf("operation=%s, ", semver_op_string(comp.op));
  semver_fwrite(&comp.version, stdout);
  printf("\n");
}
semver_comp_dtor(&comp);

and it prints: 

operation=SEMVER_OP_LE, 1.2.3

Example: extending a comparator

The following example parses the comparator specification ‘<=1.2.3’ and then extends it with ‘<3.0.0’: 

static const char     input_str_1[] = ">=1.2.3";
static const char     input_str_2[] = "<3.0.0";
semver_comp_t comp;
size_t        offset_1 = 0;
size_t        offset_2 = 0;
char          rv;

rv = semver_comp_read(&comp, \
        input_str_1, strlen(input_str_1), &offset_1);
if (0 == rv) {
  rv = semver_comp_and(&comp, \
          input_str_2, strlen(input_str_2), &offset_2);
  if (0 == rv) {
    for (semver_comp_t * it = &comp; it; it = it->next) {
      printf("operation=%s, ", semver_op_string(it->op));
      semver_fwrite(&(it->version), stdout);
      printf("\n");
    }
    printf("comparator: ");
    semver_comp_fwrite(&comp, stdout);
    printf("\n");
  }
}
semver_comp_dtor(&comp);

and it prints: 

operation=SEMVER_OP_GE, 1.2.3
operation=SEMVER_OP_LT, 3.0.0
comparator: >=1.2.3 <3.0.0
