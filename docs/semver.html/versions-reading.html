3.2 Parsing from a string

All the following symbols are defined in the header file semver.h.

Function: char semver_read (semver_t * versionp, const char * input_buf, size_t input_len, size_t * input_offp)

Parse an input string in ASCII format which is meant to hold a version specification. When successful: the return value is 0. If an error occurs: the return value is 1.

The results of parsing are stored in the already allocated structure referenced by versionp; the structure is first initialised with semver_ctor().

In both the successful and unsuccessful cases: every call to this function must be matched by the application of semver_dtor() to versionp.

The input string is referenced by input_buf; the length of the input string is input_len characters. The argument input_offp is both an input and output parameter:

  • Upon entering the function: the offset of the first character to parse in input_buf is acquired from the variable referenced by input_offp.
  • When the function returns successfully: the offset of the first non–parsed character is stored in the variable referenced by input_offp; further parsing operations can use such value as input.
  • When the function returns unsuccessfully: the value stored in the variable is invalid.

If the string is empty (input_len is zero): the function returns unsuccessfully.

Internally, this function allocates memory with malloc(); if memory allocation fails: the return value is 1.

Example: parsing a numeric version

The following example parses the numeric version ‘1.2.3’: 

const char    input_str[] = "1.2.3";
semver_t      version;
size_t        offset = 0;
char          rv;

rv = semver_read(&version, input_str, strlen(input_str), &offset);
if (0 == rv) {
  printf("major=%d, minor=%d, patch=%d, ",
         version.major, version.minor, version.patch);
  printf("prerelease=");
  semver_id_fwrite(&version.prerelease, stdout);
  printf(", ");
  printf("build=");
  semver_id_fwrite(&version.build, stdout);
  printf("\n");
}
semver_dtor(&version);

and it prints: 

major=1, minor=2, patch=3, prerelease=, build=

Example: parsing a tagged version

The following example parses the version ‘1.2.3-alpha.7’, which has a prerelease tag: 

static const char     input_str[] = "1.2.3-alpha.7";
semver_t      version;
size_t        offset = 0;
char          rv;

rv = semver_read(&version, input_str, strlen(input_str), &offset);
if (0 == rv) {
  printf("major=%d, minor=%d, patch=%d, ",
         version.major, version.minor, version.patch);
  printf("prerelease=");
  semver_id_fwrite(&version.prerelease, stdout);
  printf(", ");
  printf("build=");
  semver_id_fwrite(&version.build, stdout);
  printf("\n");
}
semver_dtor(&version);

and it prints: 

major=1, minor=2, patch=3, prerelease=alpha.7, build=

Example: parsing a full version

The following example parses the version ‘1.2.3-alpha.7+x86-64’, which has both a prerelease tag and build metadata: 

static const char     input_str[] = "1.2.3-alpha.7+x86-64";
semver_t      version;
size_t        offset = 0;
char          rv;

rv = semver_read(&version, input_str, strlen(input_str), &offset);

         version.major, version.minor, version.patch);

  semver_id_fwrite(&version.build, stdout);

}

and it prints: 

major=1, minor=2, patch=3, prerelease=alpha.7, build=x86-64
