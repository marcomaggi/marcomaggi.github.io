7.3 Writing to a string

All the following symbols are defined in the header file semver.h.

Function: int semver_id_pwrite (const semver_id_t * idp, char * output_buf, size_t output_len)

Serialise the identifier idp as ASCII string in the memory block referenced by output_buf, which has at least output_len characters available. If the buffer has not enough room to hold the full identifier serialisation: the string is truncated to output_len characters.

Like snprintf(): return the number of characters needed to hold the full serialisation, excluding the trailing null; if the return value is greater than or equal to output_len, the string has been truncated.

Preprocessor Macro: int semver_id_write (semver_id_t id, char * output_buf, size_t output_len)

Like semver_id_pwrite(), but accept the identifier structure as argument, rather than a pointer to it.

Example: serialising an identifier

The following example serialises a compound identifier: 

static const char     input_str[] = "1.2.3-alpha.7";
size_t        buffer_len = 32;
char          buffer_ptr[buffer_len];
semver_id_t   id;
size_t        offset = 0;
char          rv;

memset(buffer_ptr, 0, buffer_len);

rv = semver_id_read(&id, input_str, strlen(input_str), &offset);
if (0 == rv) {
  size_t      needed_count;
  size_t      actual_count;
  needed_count = (size_t)semver_id_write(id, \
      buffer_ptr, buffer_len);
  if (0 < needed_count) {
    actual_count = (needed_count < buffer_len)? \
       needed_count : buffer_len;
    printf("len=%lu, actual_count=%lu, id=",
      strlen(input_str), actual_count);
    fwrite(buffer_ptr, sizeof(char), actual_count, stdout);
    printf("\n");
  }
}
semver_id_dtor(&id);

the code prints: 

len=13, actual_count=13, id=1.2.3-alpha.7
