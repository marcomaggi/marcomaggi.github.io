5.2 Parsing from a string

All the following symbols are defined in the header file semver.h.

Function: char semver_range_read (semver_range_t * rangep, const char * input_buf, size_t input_len, size_t * input_offp)

Parse an input string in ASCII format which is meant to hold a range specification. When successful: the return value is 0. If an error occurs: the return value is 1.

The results of parsing are stored in the already allocated structure referenced by rangep.

In both the successful and unsuccessful cases: every call to this function must be matched by the application of semver_range_dtor() to rangep.

The input string is referenced by input_buf; the length of the input string is input_len characters. The argument input_offp is both an input and output parameter:

  • Upon entering the function: the offset of the first character to parse in input_buf is acquired from the variable referenced by input_offp.
  • When the function returns successfully: the offset of the first non–parsed character is stored in the variable referenced by input_offp; further parsing operations can use such value as input.
  • When the function returns unsuccessfully: the value stored in the variable is invalid.

If the string is empty (input_len is zero): the function returns unsuccessfully.

Internally, this function allocates memory with malloc(); if memory allocation fails: the return value is 1.

Example: parsing a range

The following example parses the range specification ‘1.2.7 || >=1.2.9 <2.0.0’: 

static const char  input_str[] = "1.2.7 || >=1.2.9 <2.0.0";
semver_range_t     range;
size_t             offset = 0;
char               rv;

rv = semver_range_read(&range, \
        input_str, strlen(input_str), &offset);
if (0 == rv) {
  printf("range: ");
  semver_range_fwrite(&range, stdout);
  printf("\n");
}
semver_range_dtor(&range);

and it prints: 

range: 1.2.7 || >=1.2.9 <2.0.0
