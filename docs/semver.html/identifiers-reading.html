7.2 Parsing from a string

All the following symbols are defined in the header file semver.h.

Function: char semver_id_read (semver_id_t * idp, const char * input_buf, size_t input_len, size_t * input_offp)

Parse an input string in ASCII format which is meant to hold an identifier. When successful: the return value is 0. If an error occurs: the return value is 1.

The results of parsing are stored in the already allocated structure referenced by idp; the structure is first initialised with semver_id_ctor().

In both the successful and unsuccessful cases: every call to this function must be matched by the application of semver_id_dtor() to idp.

The input string is referenced by input_buf; the length of the input string is input_len characters. The argument input_offp is both an input and output parameter:

  • Upon entering the function: the offset of the first character to parse in input_buf is acquired from the variable referenced by input_offp.
  • When the function returns successfully: the offset of the first non–parsed character is stored in the variable referenced by input_offp; further parsing operations can use such value as input.
  • When the function returns unsuccessfully: the value stored in the variable is invalid.

If the string is empty (input_len is zero): the function returns unsuccessfully.

Internally, this function allocates memory with malloc(); if memory allocation fails: the return value is 1.

Example: parsing a numeric identifier

The following example parses the numeric identifier ‘123’: 

static const char     input_str[] = "123";
semver_id_t   id;
size_t        offset = 0;
char          rv;

rv = semver_id_read(&id, input_str, strlen(input_str), &offset);
if (0 == rv) {
  printf("numeric?=%d, num=%d, len=%lu, raw=%s, next=%p\n",
         (int)id.numeric, id.num, id.len, id.raw,
         (void *)id.next);
}
semver_id_dtor(&id);

and it prints: 

numeric?=1, num=123, len=3, raw=123, next=(nil)

Example: parsing an alphabetic identifier

The following example parses the alphabetic identifier ‘alpha’: 

static const char     input_str[] = "alpha";
semver_id_t   id;

if (0 == rv) {

         (void *)id.next);
}
semver_id_dtor(&id);

and it prints: 

numeric?=0, num=0, len=5, raw=alpha, next=(nil)

Example: parsing a compound identifier

The following example parses the compound identifier ‘1.2.3-alpha.7’: 

static const char     input_str[] = "1.2.3-alpha.7";
semver_id_t   id;
size_t        offset = 0;
char          rv;

rv = semver_id_read(&id, input_str, strlen(input_str), &offset);
if (0 == rv) {
  for (semver_id_t * iter = &id; iter; iter = iter->next) {
    printf("numeric?=%d, num=%d, len=%lu, raw=%s\n",
           (int)iter->numeric, iter->num, iter->len, iter->raw);
  }
}
semver_id_dtor(&id);

and it prints: 

numeric?=1, num=1, len=1, raw=1.2.3-alpha.7
numeric?=1, num=2, len=1, raw=2.3-alpha.7
numeric?=0, num=0, len=7, raw=3-alpha.7
numeric?=1, num=7, len=1, raw=7

Example: parsing a compound identifier with build metadata

The following example parses the compound identifier ‘1.2.3-alpha.7+x86-64’:

and it prints:

numeric?=1, num=7, len=1, raw=7+x86-64
