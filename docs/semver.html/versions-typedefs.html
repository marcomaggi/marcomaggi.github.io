3.1 Type definitions

All the following symbols are defined in the header file semver.h.

Struct Typedef: semver_t

Type of data structures representing parsed versions. Instances of this structure are initialised with semver_ctor() or semver_read(). Instances of this structure are finalised with semver_dtor().

It has the following public fields:

int major

An integer representing the major version number.

int minor

An integer representing the minor version number.

int patch

An integer representing the patch level.

semver_id_t prerelease

An identifier representing the prerelease tag.

semver_id_t build

An identifier representing the build metadata.

size_t len

The number of characters in the input string that where parsed to form this version.

const char * raw

Pointer to the character in the input string representing the first character of this component. This pointer is valid as long as the input string handed to semver_read() is not mutated.

Function: void semver_ctor (semver_t * versionp)

Initialise to empty an already allocated data structure representing a parsed version. Every call to this function must be matched by a call to semver_dtor(): 

semver_t        version;

semver_ctor(&version);
{
  /* Do something with "version" here. */
}
semver_dtor(&version);
Function: void semver_dtor (semver_t * versionp)

Finalise to empty a data structure representing a parsed version; calls to this function must match a previous call to semver_ctor() or semver_read().

All the resources associated to versionp are released, but the structure itself. The following code is correct: 

semver_t        version;

semver_ctor(&version);
{
  /* Do something with "version" here. */
}
semver_dtor(&version);

the following code is also correct: 

semver_t *      versionp;

versionp = malloc(sizeof(semver_t));
assert(NULL != versionp);
{
  semver_ctor(versionp);
  {
    /* Do something with "versionp" here. */
  }
  semver_dtor(versionp);
}
free(versionp);
