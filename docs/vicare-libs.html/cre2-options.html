A.3 Matching configuration

The following bindings are exported by the (vicare cre2) library.

— Function: make-options

Build and return a configuration option object initialised with default values. The returned object must be finalised with delete-options; this operation is automatically performed when the object is garbage collected.

— Function: delete-options opt

Finalise a configuration option object releasing all the associated resources. This function can be safely applied multiple times to the same opt object: the first time the object is finalised, the subsequent times nothing happens.

— Function: options? obj

Return #t if obj is a configuration options object, return #f otherwise.

All the following functions are setters and getters for matching options; see the original documentation for details.

— Function: set-posix-syntax! opt bool
— Function: set-longest-match! opt bool
— Function: set-log-errors! opt bool
— Function: set-literal! opt bool
— Function: set-never-nl! opt bool
— Function: set-case-sensitive! opt bool
— Function: set-perl-classes! opt bool
— Function: set-word-boundary! opt bool
— Function: set-one-line! opt bool

Configuration option setters. opt must be configuration options object. bool must be true or false.

— Function: posix-syntax opt
— Function: longest-match opt
— Function: log-errors opt
— Function: literal opt
— Function: never-nl opt
— Function: case-sensitive opt
— Function: perl-classes opt
— Function: word-boundary opt
— Function: one-line opt

Configuration option setters; return #t if the option is set, #f otherwise. opt must be configuration options object.

— Function: set-max-mem! opt dimension
— Function: max-mem opt

Setter and getter for the maximum memory configuration option. opt must be a configuration options object. dimension must be a positive exact integer fitting in a platform's int C language data type.
