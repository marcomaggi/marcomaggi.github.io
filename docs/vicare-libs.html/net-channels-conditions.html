17.8 Condition object types

Whenever an error condition happens: the functions of the library (vicare net channels) react by raising non–continuable exceptions whose raised objects are compound conditions of the following types.

The following bindings are exported by the library (vicare net channels).

Channel condition type
— Condition Type: &channel

Condition object containing an instance of channel; it is derived from &condition.

— Function: make-channel-condition chan

Build and return an instance of &channel wrapping the channel chan.

— Function: channel-condition? obj

Return #t if obj is a condition object of type &channel or a compound condition object with a component of type &channel.

— Function: condition-channel cond

Given an instance of &channel: return the channel instance it contains.

Delivery timeout error
— Condition Type: &delivery-timeout-expired

Condition object used to tag exceptions as of type “message delivery timeout expired”; it is derived from &error.

— Function: make-delivery-timeout-expired-condition

Build and return an instance of &delivery-timeout-expired.

— Function: delivery-timeout-expired-condition? obj

Return #t if obj is a condition object of type &delivery-timeout-expired or a compound condition object with a component of type &delivery-timeout-expired.

Message size error
— Condition Type: &maximum-message-size-exceeded

Condition object used to tag exceptions as of type “maximum message size exceeded; it is derived from &error.

— Function: make-maximum-message-size-exceeded-condition

Build and return an instance of &maximum-message-size-exceeded.

— Function: maximum-message-size-exceeded-condition? obj

Return #t if obj is a condition object of type &maximum-message-size-exceeded or a compound condition object with a component of type &maximum-message-size-exceeded.
