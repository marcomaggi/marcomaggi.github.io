1.6 Numeric increment syntaxes

The library (vicare language-extensions increments) exports syntaxes to ease incrementing and decrementing of numeric values. The bindings whose name is prefixed with $ are unsafe operations: they do not validate their arguments before accessing them.

— Syntax: incr! ?expr
— Syntax: incr! ?expr ?step
— Syntax: $incr! ?expr
— Syntax: $incr! ?expr ?step

Increment ?expr by ?step and return the result. Both ?expr and ?step must be expressions evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

As special case, when ?expr is an identifier: the syntax expands as follows: 

          (incr! ?id ?step)
          ==> (begin
                (set! ?id (+ ?id ?step))
                ?id)
— Syntax: pre-incr! ?expr
— Syntax: pre-incr! ?expr ?step
— Syntax: $pre-incr! ?expr
— Syntax: $pre-incr! ?expr ?step

Aliases for incr! and $incr!.

— Syntax: post-incr! ?expr
— Syntax: post-incr! ?expr ?step
— Syntax: $post-incr! ?expr
— Syntax: $post-incr! ?expr ?step

Increment ?expr by ?step and return the result. Both ?expr and ?step must be expressions evaluating to a number object; when ?step is not given: it defaults to the fixnum 1.

As special case, when ?expr is an identifier: the syntax expands as follows: 

          (post-incr! ?id ?step)
          ==> (receive-and-return (v)
                  ?id
                (set! ?id (+ ?id ?step)))
— Syntax: decr! ?expr
— Syntax: decr! ?expr ?step
— Syntax: $decr! ?expr
— Syntax: $decr! ?expr ?step

As special case, when ?expr is an identifier: the syntax expands as follows: 

          (decr! ?id ?step)
          ==> (begin
                (set! ?id (- ?id ?step))

— Syntax: pre-decr! ?expr ?step
— Syntax: $pre-decr! ?expr
— Syntax: $pre-decr! ?expr ?step

Aliases for decr! and $decr!.

— Syntax: post-decr! ?expr ?step
— Syntax: $post-decr! ?expr
— Syntax: $post-decr! ?expr ?step
