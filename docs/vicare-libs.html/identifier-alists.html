1.22 Utilities for alists of identifiers

The library (vicare language-extensions identifier-alists) exports an interface to association lists having identifiers as keys; the identifiers are compared using free-identifier=?.

— Function: identifier-alist-cons key value table

Given the alist table: store key and value in the alist and return the new alist. Does no checks for duplicate key in table.

— Function: identifier-alist-new key value table

Given the alist table: store key and value in the alist and return the new alist. Check if key is already in table, if it is: raise an assertion violation.

— Function: identifier-alist-cons-and-replace key value table

Given the alist table: store key and value in the alist and return the new alist. All the previously existent occurrences of key in table are removed.

— Function: identifier-alist-ref table key default

Given the alist table: look for the identifier key and return its value. If key is not present: return default.

— Function: identifier-alist-set! table key new-value

Given the alist table: look for the identifier key and replace its value with new-value; return table. If key is not present: raise an assertion violation.

— Function: identifier-alist-remove table key

Given the alist table: remove the occurrences of key and return the new alist.

— Function: identifier-alist-exists table key

Given the alist table: return the first pair in the list having key as car. If key is not present: return #f.
