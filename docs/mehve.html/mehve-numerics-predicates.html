2.4.1 Numeric predicates generic functions

The component library (nausicaa mehve language numerics predicates) defines the generic functions behind the numeric predicates; we need to import it only if we desire to add specialised methods to the numeric predicates. The numeric predicate functions are the following: 

     = < > <= >=
     zero? positive? negative? non-negative? non-positive?
     odd? even? finite? infinite? nan?

As example we look at how to specialise the comparison predicates =, <, >, <= and >= to operate on lists of items: 

     #!r6rs
     (import (nausicaa mehve)
       (prefix (nausicaa mehve language numerics predicates)
               mehve.))
     
     (define-method (mehve.equal-predicate-1 (o <spine>))
       (o for-all =))
     
     (define-method
         (mehve.equal-predicate-2 (a <spine>) (b <spine>))
       (a for-all = b))
     
     (define-method
         (mehve.less-than-predicate-1 (o <spine>))
       (o for-all <))
     
     (define-method
         (mehve.less-than-predicate-2 (a <spine>) (b <spine>))
       (a for-all < b))
     
     (define-method
         (mehve.greater-than-predicate-1 (o <spine>))
       (o for-all >))
     
     (define-method
         (mehve.greater-than-predicate-2 (a <spine>) (b <spine>))
       (a for-all > b))
     
     (define-method
         (mehve.less-than-or-equal-to-predicate-1 (o <spine>))
       (o for-all <=))
     
     (define-method
         (mehve.less-than-or-equal-to-predicate-2
             (a <spine>) (b <spine>))
       (a for-all <= b))
     
     (define-method
         (mehve.greater-than-or-equal-to-predicate-1 (o <spine>))
       (o for-all >=))
     
     (define-method
         (mehve.greater-than-or-equal-to-predicate-2
             (a <spine>) (b <spine>))
       (a for-all >= b))
     
     (= '(1 2 3) '(1 2 3))  ⇒ (#t #t #t)
     (= '(1 2 3) '(1 4 3))  ⇒ (#t #f #t)
     (< '(1 2 3) '(1 2 3))  ⇒ (#f #f #f)
     (< '(1 2 3) '(4 5 6))  ⇒ (#t #t #t)
     (< '(1 5 3) '(4 2 6))  ⇒ (#t #f #t)

— Generic Function: equal-predicate-1 A
— Generic Function: equal-predicate-2 A B

Used when = is invoked with zero, one or two arguments.

— Method: equal-predicate-0
— Method: equal-predicate-1 (A <top>)

Even and odd predicates generic functions

The generic functions odd? and even? exported by (nausicaa mehve numeric-predicates), and reexported by (nausicaa mehve), generalise the corresponding functions exported by (rnrs base (6)).

— Generic Function: odd? A
— Method: odd? (A <fixnum>)
— Method: odd? (A <flonum>)
— Method: odd? (A <real>)

Return #t if the argument is odd. Apply to the argument the function fxodd?, flodd? or odd? as appropriate for the argument.

— Generic Function: even? A
— Method: even? (A <fixnum>)
— Method: even? (A <flonum>)
— Method: even? (A <real>)

Return #t if the argument is even. Apply to the argument the function fxeven?, fleven? or even? as appropriate for the argument.

Infinity and not–a–number predicates generic functions

We have to remember that, according to R6RS, the only values satisfying the predicate real? and not the predicate rational? are ‘+inf.0’, ‘-inf.0’ and ‘+nan.0’.

— Generic Function: finite? A
— Method: finite? (A <fixnum>)
— Method: finite? (A <flonum>)
— Method: finite? (A <complex>)

Return #t if the argument is finite and not not–a–number.

— Generic Function: infinite? A
— Method: infinite? (A <fixnum>)
— Method: infinite? (A <flonum>)
— Method: infinite? (A <complex>)

Return #t if the argument is infinite.

— Generic Function: nan? A
— Method: nan? (A <fixnum>)
— Method: nan? (A <flonum>)
— Method: nan? (A <complex>)

Return #t if the argument is not–a–number.
