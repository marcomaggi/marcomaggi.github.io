2.4.4 Transcendental generic functions

The auxiliary library (nausicaa mehve language numerics transcendental) defines the generic functions behind the transcendental operations; we need to import it only if we desire to add specialised methods to transcendental functions. The transcendental functions and the other functions exported by the library are the following: 

     expt sqrt cbrt square cube exp log
     sin cos tan asin acos atan
     sinh cosh tanh asinh acosh atanh

As example we look at how to specialise the transcendental functions sin, cos and tan to operate on lists of items: 

     #!r6rs
     (import (nausicaa mehve)
       (prefix (nausicaa mehve language numerics transcendental)
               mehve.))
     
     (define-method (mehve.sin (o <spine>))
       (o map sin))
     
     (define-method (mehve.cos (o <spine>))
       (o map cos))
     
     (define-method (mehve.tan (o <spine>))
       (o map tan))
     
     (sin '(1.1 2.2 3.3))
     ⇒ (0.8912073600614354 0.8084964038195901 -0.1577456941432482)
     
     (cos '(1.1 2.2 3.3))
     ⇒ (0.4535961214255773 -0.5885011172553458 -0.9874797699088649)
     
     (tan '(1.1 2.2 3.3))
     ⇒ (1.9647596572486523 -1.3738230567687946 0.15974574766003222)

notice how the methods recursively call the generic functions in their body, this way the lists can hold any of the value types for which there are methods.

Exponentiation and logarithm generic functions
— Generic Function: expt A B
— Method: expt (A <flonum>) (B <flonum>)
— Method: expt (A <complex>) (B <complex>)

Compute A raised to the power of B. Use expt or one of its type–specific variants as appropriate for the argument.

— Generic Function: sqrt A
— Method: sqrt (A <flonum>)
— Method: sqrt (A <complex>)

Compute square root of A. Use sqrt or one of its type–specific variants as appropriate for the argument.

— Generic Function: cbrt A
— Method: cbrt (A <flonum>)
