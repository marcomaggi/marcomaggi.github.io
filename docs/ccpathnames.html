1 Overview of the package

This document describes version 0.1.0-beta.0 of CCPathnames, a C11 language library implementing Unix pathnames manipulation functions.

This package supports POSIX platforms. This package depends on the package CCExceptions, at least version 0.6.4-devel.3.

CCPathnames installs the single header file ccpathnames.h. All the function names in the API are prefixed with ccptn_; all the preprocessor symbol names are prefixed with CCPTN_; all the type names are prefixed with ccptn_ and suffixed with _t.

Notice that, before using the library, we need to call the global initialisation function ccptn_init().

• overview linking:  Linking code with the library.

Up: overview   [Contents][Index]

1.1 Linking code with the library

This package installs a data file for pkg-config, so when searching for the installed library with the GNU Autotools, we can add the following macro use to configure.ac: 

PKG_CHECK_MODULES([CCPATHNAMES],[ccpathnames >= 0.1])

which will set the variables CCPATHNAMES_LIBS and CCPATHNAMES_CFLAGS. To avoid problems with pkg-config, we need to remember that:

  • The macro file defining PKG_CHECK_MODULES is installed by the package pkg-config; such file usually is in one of the locations:
    /usr/share/aclocal/pkg.m4
    /usr/local/share/aclocal/pkg.m4
  • The macro file is located by aclocal, which in turn is installed by the package GNU Automake; aclocal is used by autoreconf. We must make sure that aclocal finds the installed macro files; for this purpose we can use the environment variable ACLOCAL_PATH; under a Bourne shell we can do this with: 
    export ACLOCAL_PATH=/usr/local/share/aclocal:/usr/share/aclocal

Alternatively we can use the raw GNU Autoconf macros: 

AC_CHECK_LIB([ccpathnames],[ccptn_version_string],,
  [AC_MSG_FAILURE([test for CCPathnames library failed])])
AC_CHECK_HEADERS([ccpathnames.h],,
  [AC_MSG_FAILURE([test for CCPathnames header failed])])

5 Custom memory allocation

CCPathnames exposes an API to allow the use of a custom allocator for the pathnames.

Struct Typedef: ccptn_allocator_t

The type of a custom memory allocator. It has the following public fields:

ccptn_malloc_fun_t * malloc

Pointer to a function used to allocate memory.

ccptn_free_fun_t * free

Pointer to a function used to release allocated memory.

Function Pointer: void * ccptn_malloc_fun_t (ccptn_allocator_t const * A, cce_destination_t L, size_t size)

Type of function used to allocate memory. Basically it works like the standard malloc(), but if allocation fails: it raises an exception by performing a non–local exit to L. The argument A is a pointer to the allocator itself.

Function Pointer: void ccptn_free_fun_t (ccptn_allocator_t const * A, void * ptr)

Type of function used to release allocated memory. Basically it works like the standard free(). The argument A is a pointer to the allocator itself.

Constant: ccptn_allocator_t const * const ccptn_default_allocator

Default allocator predefined by CCPathnames. At the core it uses the standard malloc() and free().

Variable: ccptn_allocator_t const * ccptn_current_allocator

Currently selected custom memory allocator. It is initialised with ccptn_default_allocator.

Function: void * ccptn_malloc (cce_destination_t L, size_t size)

Allocate memory using the currently selected memory allocator.

Function: void ccptn_free (void * ptr)

Release allocated memory using the currently selected memory allocator.

6 Type definitions

Struct Typedef: ccptn_t

Structure representing a Unix pathname. All the fields are opaque. All the instances of this type are immutable.

The pathname is internally represented with an ASCIIZ string. All the octets in the range [1, 255] are valid for the pathname. A pathname is never empty: it always contains at least one octet.

Instances of this type include a pointer to function referencing a finalisation function appropriate for the internal data. We can always release the resources associated to an instance of ccptn_t by applying to it the function ccptn_delete().

7.1 Pathnames from ASCIIZ strings

Function: ccptn_t * ccptn_new_nodup_asciiz (cce_destination_t L, char const * pathname)

Allocate a new ccptn_t instance initialising it with data from the ASCIIZ string pathname, return a pointer to it. If an error occurs: raise an exception by performing a non–local exit to to L.

The data is not duplicated: the returned ccptn_t references pathname itself. Memory is allocated with ccptn_malloc(). The finalisation function registered in the instance will release the struct itself, but not the data buffer.

As usage example see the following code: 

cce_location_t        L[1];

if (cce_location(L)) {
  cce_run_error_handlers_final(L);
} else {
  char const *  pathname = "/path/to/file.ext";
  ccptn_t *     P;

  P = ccptn_new_nodup_asciiz(L, pathname);
  printf("%s\n", ccptn_asciiz(P));
  ccptn_delete(P);
  cce_run_cleanup_handlers(L);
}

12.2 Normalising pathnames

A normalised pathname: has no multiple slashes; has no useless terminating slash; has no useless single–dot components; has the double–dot components removed as much as possible. Examples: 

/                       ⇒ /
file.ext                ⇒ file.ext
/path/to/file.ext       ⇒ /path/to/file.ext
//                      ⇒ /
/path///to///file.ext   ⇒ /path/to/file.ext
/path/to/dir/           ⇒ /path/to/dir/
/path/to/././file.ext   ⇒ /path/to/file.ext
/path/to/.              ⇒ /path/to
/.                      ⇒ /
.                       ⇒ .
/path/to/../file.ext    ⇒ /path/file.ext
/path/../../to/file.ext ⇒ ../to/file.ext
..                      ⇒ ..
/..                     error→ invalid pathname

Some notes on the normalisation:

  • We do not want to remove an ending slash octet, unless the whole pathname is ./ or ../. This is because a pathname like a/b/ is meant to represent a directory, while a/b can represent a non–directory too.
  • We do not want to fully remove a directory part. This is because ./file.ext may represent an executable file in the current directory, while file.ext may represent an executable pathname in any of the directories selected by the value in the environment variable PATH. So we want the following normalisations: 
    ./file.ext              ⇒ ./file.ext
../path/file.ext        ⇒ ./file.ext

13.1 Extracting pathname extensions

An extension is the last portion of the last segment in an input pathname following the last dot, but only if it is not the whole segment. Examples: 

"/path/to/file.ext"     => ".ext"
"/path/to/file"         => ""
"/path/to/dir/"         => ""
"/path/to/dir.ext/"     => ".ext"
"file.ext"              => ".ext"
".fvwmrc"               => ""
"."                     => ""
".."                    => ""
"/"                     => ""

Extension extraction is possible only for an input pathname that has been previously normalised; if the input is not normalised: the functions below will raise an exception of type ccptn_condition_normalised_pathname_t.

Function: ccptn_extension_t ccptn_extension (cce_destination_t L, ccptn_t const * P)

Find the extension of the last segment in the input pathname referenced by P and return an instance of ccptn_extension_t representing it; the returned value references the section of data in the input pathname and so it is valid data only while the input pathname exists. If an error occurs: raise an exception by performing a non–local exit to L.

If the input pathname has no extension: the returned value represents the empty string by having the length field to zero, while the pointer field references the terminating zero in the ASCIIZ representation of the input pathname.

13.2 Extracting pathname rootnames

The rootname of an input pathname is a pathname representing the input with the extension of the last segment stripped. Examples: 

"/path/to/file.ext"     => "/path/to/file"
"/path/to/file"         => "/path/to/file"
"/path/to/dir/"         => "/path/to/dir"
"file.ext"              => "file.ext"
".fvwmrc"               => ".fvwmrc"
"."                     --> error
".."                    --> error
"/"                     --> error

It is an error to attempt rootname extraction from an input pathname that has no rootname; in this case the functions below will raise an exception of type ccptn_condition_no_rootname_t.

15 Manipulating segments

Struct Typedef: ccptn_segment_t

This struct represents a segment in a pathname; this struct does not own the referenced memory. Given the pathname: 

/path/to/file.ext

the segments are the strings: 

path
to
file.ext

notice that the slash separators are excluded. This type has the following fields:

size_t len

The number of octets in the ptr array representing a segment of pathname.

char const * ptr

Pointer to the first octet in an ASCII string representing a pathname’s segment. In general, it is not an ASCIIZ string: it is not zero terminated.

Function: bool ccptn_segment_is_empty (ccptn_segment_t S)

Return true if S is empty; otherwise return false.

Function: bool ccptn_segment_is_dot (ccptn_segment_t S)

Return true if S represents a single dot: ‘.’; otherwise return false.

Function: bool ccptn_segment_is_double_dot (ccptn_segment_t S)

Return true if S represents a double dot: ‘..’; otherwise return false.

Function: ccptn_segment_t ccptn_segment_next (char const * ptr, size_t len)

Given a pointer ptr to an ASCII string of at least len octets: return a segment struct referencing the octects up to, but not including, the first slash character or the end of the string. The returned segment does not include a terminating zero octet.

Function: size_t ccptn_segment_size_of_next (char const * in, size_t len)

Given a pointer in to an ASCII string of at least len octets: return the number of octets between the one referenced by in (included) and the first slash octet (excluded); return len itself if no slash is found.

Function: void ccptn_segment_print (cce_destination_t L, FILE * stream, ccptn_segment_t S)

Print the segment to stream; do not flush the stream. If an error occurs: raise an exception by performing a non–local exit to L.

16 Manipulating extensions

Struct Typedef: ccptn_extension_t

This struct represents the extension of a segment in a pathname; this struct does not own the referenced memory. Given the pathname:

This type has the following fields:

size_t len

char const * ptr

Pointer to the first octet in an ASCII string representing a pathname’s segment’s extension. In general this string is not zero terminated.

Function: bool ccptn_extension_is_empty (ccptn_extension_t E)

17 Utility definitions

Macro: CCPTN_PC (POINTER_TYPE, POINTER_NAME, EXPRESSION)

Preprocessor macro used to define pointer variables initialised with a cast expression. It expands to: 

POINTER_TYPE * POINTER_NAME = (POINTER_TYPE *) (EXPRESSION)

For example, the macro use: 

CCPTN_PC(ccpathnames_descriptor_test_failure_t, C, _C);

expands into: 

ccpathnames_descriptor_test_failure_t * C = \
   (ccpathnames_descriptor_test_failure_t *) (_C);
Macro: CCPTN_UNUSED

Preprocessor macro that is meant to expand into an attribute declaration. If the symbol __GNUC__ is defined, the expansion is: 

__attribute__((__unused__))

otherwise the expansion is the empty string.
