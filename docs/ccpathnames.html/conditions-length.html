4.2 Maximum pathname length exceeded

The exceeded length exceptional–condition object–type has the purpose of describing an exceptional condition caused by attempting an operation that would generate a too long pathname.

There should be only one exceeded length exceptional–condition object: the one predefined by CCPathnames. Subtyping from this object–type is allowed.

All the following definitions are accessible from the header file ccpathnames.h.

Struct Typedef: ccptn_descriptor_exceeded_length_t

Structure type representing the descriptor for exceeded length exceptional–condition objects. This descriptor has cce_descriptor_runtime_error_t as parent descriptor. It has the following public fields:

cce_descriptor_t descriptor

Core values for the exceptional–condition object–type.

Struct Typedef: ccptn_condition_exceeded_length_t

Structure type representing an exceeded length exceptional–condition object. It has the following public fields:

cce_condition_runtime_error_t runtime_error

Core values for the exceptional–condition object.

Function: void ccptn_condition_init_exceeded_length (ccptn_condition_exceeded_length_t * C)

Initialise an already allocated exceptional–condition object. This function exists to allow sub–typing from ccptn_condition_exceeded_length_t.

Function: cce_condition_t const * ccptn_condition_new_exceeded_length (void)

Return a pointer to the statically allocated structure representing the single exceeded length exceptional–condition object.

Function: bool ccptn_condition_is_exceeded_length (cce_condition_t const * C)

Return true if C is an exceeded length exceptional–condition object; otherwise return false.
