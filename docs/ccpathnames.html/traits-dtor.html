13.2 The dtor trait

The dtor trait offers a common API for the destruction of struct types; (ccstructs)The dtor trait.

Function: ccstructs_dtor_T ccname_trait_new(ccstructs_dtor_T, ccptn_t) (ccptn_t const * P)

Build and return an instance of dtor trait which we can use to destroy both embedded and standalone instances of ccptn_t referenced by P. It is usually better to use the handlers API or, even better, the guarded constructors.

We can use the dtor trait with the generic handlers API defined by CCStructs, (ccstructs)Exception handlers. Code sample: 

cce_location_t             L[1];
ccstructs_clean_handler_t  H[1];
ccmem_allocator_t const *  A = ccmem_standard_allocator;

if (cce_location(L)) {
  cce_run_catch_handlers_final(L);
} else {
  char const        *input = "/path/to/file.ext";
  ccptn_t           P[1];
  ccstructs_dtor_T  ID;

  ccname_init(ccptn_t, pointer)(L, A, input);
  ID = ccname_trait_new(ccstructs_dtor_T, ccptn_t)(P);
  ccstructs_init_and_register_handler(L, H, ID);

  cce_run_body_handlers(L);
}
