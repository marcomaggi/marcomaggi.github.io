13.2 The destructor interface

The destructor interface offers a common API for the destruction of struct types; (ccstructs)The destructor interface.

Function: ccstructs_dtor_I ccname_iface_new(ccstructs_dtor_I, ccptn_t) (ccptn_t const * P)

Build and return an instance of destructor interface which we can use to destroy both embedded and standalone instances of ccptn_t. It is usually better to use the handlers API or, even better, the guarded constructors.

We can use the destructor interface with the generic handlers API defined by CCStructs, (ccstructs)Exception handlers. Code sample: 

cce_location_t             L[1];
ccstructs_clean_handler_t  H[1];
ccmem_allocator_t const *  A = ccmem_standard_allocator;

if (cce_location(L)) {
  cce_run_catch_handlers_raise(L, upper_L);
} else {
  char const        *input = "/path/to/file.ext";
  ccptn_t           P[1];
  ccstructs_dtor_I  ID;

  ccname_init(ccptn_t, pointer)(L, A, input);
  ID = ccname_iface_new(ccstructs_dtor_I, ccptn_t)(P);
  ccstructs_handler_init(L, H, ID);

  cce_run_body_handlers(L);
}
