4.6 Directory part is missing

The no dirname exceptional–condition object–type has the purpose of describing an exceptional condition caused by attempting an operation on a pathname that has no directory part, when this is required. The dirname is the pathname representing the input pathname with the last segment stripped. Examples: 

"/path/to/file.ext"     ⇒ "/path/to"
"/path/to/dir/"         ⇒ "/path/to"
"file.ext"              error→ no dirname
"."                     ⇒ ".."
".."                    error→ no dirname

There should be only one no dirname exceptional–condition object: the one predefined by CCPathnames. Subtyping from this object–type is allowed.

All the following definitions are accessible from the header file ccpathnames.h.

Struct Typedef: ccptn_descriptor_no_dirname_t

Structure type representing the descriptor for no dirname exceptional–condition objects. This descriptor has ccptn_descriptor_invalid_pathname_t as parent descriptor. It has the following public fields:

cce_descriptor_t descriptor

Core values for the exceptional–condition object–type.

Struct Typedef: ccptn_condition_no_dirname_t

Structure type representing an no dirname exceptional–condition object. It has the following public fields:

ccptn_descriptor_invalid_pathname_t invalid_pathname

Core values for the exceptional–condition object.

Function: void ccptn_condition_init_no_dirname (ccptn_condition_no_dirname_t * C)

Initialise an already allocated exceptional–condition object. This function exists to allow sub–typing from ccptn_condition_no_dirname_t.

Function: cce_condition_t const * ccptn_condition_new_no_dirname (void)

Return a pointer to the statically allocated structure representing the single no dirname exceptional–condition object.

Function: bool ccptn_condition_is_no_dirname (cce_condition_t const * C)

Return true if C is a no dirname exceptional–condition object; otherwise return false.
