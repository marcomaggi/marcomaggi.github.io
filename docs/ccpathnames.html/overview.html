1 Overview of the package

This document describes version 0.3.0-devel.1 of CCPathnames, a C11 language library implementing Unix file system pathnames manipulation functions.

This package supports POSIX platforms. This package depends on the packages: CCExceptions, at least version 0.9.0; CCMemory, at least version 0.2.1; CCStructs, at least version 0.3.0. To run the test suite: this package depends upon the package CCTests, at least version 0.4.1.

CCPathnames installs the multiple header files; the whole API is available by including the single file ccpathnames.h. All the function names in the API are prefixed with ccptn_; all the preprocessor symbol names are prefixed with CCPTN_; all the type names are prefixed with ccptn_ and suffixed with _t.

Notice that, before using the library, we need to call the global initialisation function ccptn_library_init().
