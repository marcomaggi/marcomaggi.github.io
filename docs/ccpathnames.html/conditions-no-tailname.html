4.8 Tailname is missing

The no tailname exceptional–condition object–type has the purpose of describing an exceptional condition caused by attempting an operation on a pathname that has no tailname part, when this is required. The tailname is the relative pathname representing the last segment. Examples: 

"/path/to/file.ext"     ⇒ "file.ext"
"/path/to/dir/"         ⇒ "dir/"
"file.ext"              ⇒ "file.ext"
"."                     ⇒ "."
".."                    ⇒ ".."
"/"                     error→ no tailname

There should be only one no tailname exceptional–condition object: the one predefined by CCPathnames. Subtyping from this object–type is allowed.

All the following definitions are accessible from the header file ccpathnames.h.

Struct Typedef: ccptn_descriptor_no_tailname_t

Structure type representing the descriptor for no tailname exceptional–condition objects. This descriptor has ccptn_descriptor_invalid_pathname_t as parent descriptor. It has the following public fields:

cce_descriptor_t descriptor

Core values for the exceptional–condition object–type.

Struct Typedef: ccptn_condition_no_tailname_t

Structure type representing an no tailname exceptional–condition object. It has the following public fields:

ccptn_descriptor_invalid_pathname_t invalid_pathname

Core values for the exceptional–condition object.

Function: void ccptn_condition_init_no_tailname (ccptn_condition_no_tailname_t * C)

Initialise an already allocated exceptional–condition object. This function exists to allow sub–typing from ccptn_condition_no_tailname_t.

Function: cce_condition_t const * ccptn_condition_new_no_tailname (void)

Return a pointer to the statically allocated structure representing the single no tailname exceptional–condition object.

Function: bool ccptn_condition_is_no_tailname (cce_condition_t const * C)

Return true if C is a no tailname exceptional–condition object; otherwise return false.
