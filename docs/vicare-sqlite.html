1 Overview of the package

Vicare Scheme is an R6RS compliant Scheme language implementation in the form of a native compiler for x86 architectures, officially supporting GNU+Linux systems. Vicare/SQLite is a distribution of C language and Scheme language libraries for Vicare Scheme; it provides bindings for the SQLite SQL database engine. The last time the maintainer bothered to update this paragraph, he had tested SQLite version 3.13.0.

SQLite is a in–process library that implements a self–contained, serverless, zero–configuration, transactional SQL database engine. The code for SQLite is in the public domain and is thus free for use for any purpose, commercial or private.

Vicare/SQLite installs a C language library implementing wrapper C functions for the C language SQLite library; on top of this, a set of Scheme libraries exports an adapted API. This document contains only a brief description of the functions: refer to the SQLite’s documentation for details:

http://www.sqlite.org/c3ref/intro.html

The following Scheme libraries are installed:

(vicare databases sqlite3)

It exports one binding for each SQLite public function. All the Scheme function names are directly derived from the C function names by replacing underscore characters ‘_’ with dash characters ‘-’; so sqlite3_exec() becomes sqlite3-exec.

(vicare databases sqlite3 unsafe-capi)

It exports one syntax binding for each SQLite public function. All the Scheme syntax names are directly derived from the C function names by replacing underscore characters ‘_’ with dash characters ‘-’; so sqlite3_exec() becomes sqlite3-exec.

These syntaxes expand to a direct call to the C language wrapper functions in the Vicare/SQLite library; they are not meant to be called in normal usage of the package.

(vicare databases sqlite3 constants)

It exports one binding for each constant value defined in the C language header sqlite3.h.

(vicare databases sqlite3 features)

It exports one keyword syntactic binding for each HAVE_ constant defined by the configure script, expanding to #t or #f.

2.1 Library initialisation and finalisation

The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-initialize

Initialise the SQLite library. If successful return SQLITE_OK, else return an error code.

Function: sqlite3-shutdown

Deallocate any resources that were allocated by sqlite3-initialize. If successful return SQLITE_OK, else return an error code.

Function: sqlite3-os-init

Do operating–system specific initialization of the SQLite library. If successful return SQLITE_OK, else return an error code.

Function: sqlite3-os-end

Undo the effect of sqlite3-os-init. If successful return SQLITE_OK, else return an error code.

Function: sqlite3-config option-identifier arg ...

Make global configuration changes to SQLite in order to tune SQLite to the specific needs of the application. When successful return SQLITE_OK, else return a SQLITE_ error code.

option-identifier must be a fixnum representing a SQLITE_CONFIG_ constant. If the identifier is not recognised: the return value is SQLITE_ERROR.

The optional arg depend on the selected option; if the number and type of arguments is invalid: the return value is SQLITE_ERROR.

SQLITE_CONFIG_SINGLETHREAD

Accept no additional arguments.

SQLITE_CONFIG_MULTITHREAD

Accept no additional arguments.

SQLITE_CONFIG_SERIALIZED

Accept no additional arguments.

SQLITE_CONFIG_MALLOC

Accept a pointer object as single argument.

SQLITE_CONFIG_GETMALLOC

Accept a pointer object as single argument.

SQLITE_CONFIG_SCRATCH

Accept 3 arguments: a pointer, a fixnum, a fixnum.

SQLITE_CONFIG_PAGECACHE

Accept 3 arguments: a pointer, a fixnum, a fixnum.

SQLITE_CONFIG_HEAP

Accept 3 arguments: a pointer, a fixnum, a fixnum.

SQLITE_CONFIG_MEMSTATUS

Accept a value interpreted as boolean.

SQLITE_CONFIG_MUTEX

Accept a pointer object as single argument.

SQLITE_CONFIG_GETMUTEX

Accept a pointer object as single argument.

SQLITE_CONFIG_LOOKASIDE

Accept 3 arguments: a fixnum, a fixnum.

SQLITE_CONFIG_PCACHE
SQLITE_CONFIG_GETPCACHE

These options are obsolete: using them causes no action to be taken.

SQLITE_CONFIG_LOG

Accept a pointer object referencing a callback function as single argument. Always set to NULL the custom void * argument to the callback.

NOTE The log callback can be registered and unregistered only before initialising the SQLite library; so either we set it before doing anything else or we call sqlite3-shutdown and then sqlite3-config.

SQLITE_CONFIG_URI

Accept a value interpreted as boolean.

SQLITE_CONFIG_PCACHE2

Accept a pointer object as single argument.

SQLITE_CONFIG_GETPCACHE2

Accept a pointer object as single argument.

Function: sqlite3-memory-used

Return an exact integer representing number of bytes of memory currently outstanding (malloced but not freed).

Function: sqlite3-memory-highwater reset

Return an exact integer representing the maximum value of sqlite3-memory-used since the high–water mark was last reset. reset is interpreted as boolean: if true reset the value, if #f do not.

Function: sqlite3-enable-shared-cache bool

Enable or disable the sharing of the database cache and schema data structures between connections to the same database; sharing is enabled if the argument is true and disabled if the argument is false. If successful return SQLITE_OK, else return a SQLITE_ error code.

Function: sqlite3-release-memory number-of-bytes

Attempt to free number-of-bytes bytes of heap memory by deallocating non–essential memory allocations held by the database library. Return an exact integer representing the number of bytes actually freed, which might be more or less than the amount requested.

Function: sqlite3-soft-heap-limit64 limit

Set and/or query the soft limit on the amount of heap memory that may be allocated by SQLite. Return a non–negative exact integer representing the size of the soft heap limit prior to the call, or a negative exact integer in the case of an error.

limit must be an exact integer:

  • When negative: no change is made and the returned value represents the current limit.
  • When zero: the limit is disabled.
  • When positive: limit represents the new limit.
Function: sqlite3-soft-heap-limit limit

Deprecated version of sqlite3-soft-heap-limit.

Function: sqlite3-status opcode
Function: sqlite3-status opcode reset?

Retrieve informations about runtime status of the performance of SQLite, and optionally reset various highwater marks. Return 3 values, when successful:

  1. An exact integer representing SQLITE_OK.
  2. An exact integer representing the current value.
  3. An exact integer representing the maximum registered value.

else return a SQLITE_ error code, #f and #f.

opcode must be an exact integer representing one of the SQLITE_STATUS_ constants. When reset? is true the recorded value is reset; when not given it defaults to #f.

2.4.1 Data structure representing connections

The following bindings are exported by the (vicare databases sqlite3) library.

Struct Type: sqlite3

An opaque Scheme structure holding a reference to an instance of the C language type sqlite3, it references a connection to a SQLite database. (vicare-scheme)Structures and type descriptors

When instances of this type are used as arguments to functions: this documentation identifies them as connection.

There are two categories of sqlite3 instances: those who own the underlying database connection and those who merely reference it.

  • sqlite3 instances returned by sqlite3-open-v2 (and similar) and by sqlite3-db-handle do own the database connection. When instances of this category are garbage collected or when sqlite3-close is applied to them: the database connection is closed, all the associated sqlite3-stmt instances are finalised and all the associated sqlite3-blob instances are closed; any error is ignored in this procedure.
  • sqlite3 instances returned by sqlite3-context-db-handle do not own the database connection. When instances of this category are garbage collected or when sqlite3-close is applied to them: the database connection left open, all the associated sqlite3-stmt instances are finalised and all the associated sqlite3-blob instances are closed; any error is ignored in this procedure.
Function: sqlite3? obj

Return #t if obj is an instance of sqlite3.

Function: sqlite3?/open obj

Return #t if obj is an instance of sqlite3 and the connection it represents is open.

Function: sqlite3-destructor connection
Function: set-sqlite3-destructor! connection func

Retrieve or set a destructor function associated to the connection.

Whenever the connection is closed, either explicitly with sqlite3-close or implicitly by the garbage collector, func is applied to connection before the internal state of connection is finalised.

Notice that func can do anything, not only destroy some state associated to connection; If we do: 

(define connection ...)
(set-sqlite3-destructor! connection
  (let ((state (make-some-data-struct)))
    (case-lambda
      ((conn)
       (destroy state))
      (()
       state))))

(let ((state ((sqlite3-destructor connection))))
  (do-something-with connection state))

we can easily associate values to connection.

2.4.2 Opening and closing connections

The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-close connection

Close the database connection referenced by connection, which must be an instance of sqlite3. Return an exact integer representing a SQLITE_ constant.

It is safe to apply multiple times this function to the same connection: the first time the connection is closed, the subsequent times nothing happens and the return value is SQLITE_OK.

This function finalises all the statements associated to connection and closes all the BLOBs associated to connection.

Function: sqlite3-open pathname
Function: sqlite3-open16 pathname

Open a new database connection to the selected file; if successful return a new instance of sqlite3, else return a SQLITE_ constant.

pathname must be a string, bytevector, pointer object or memory-block; the expected encoding is UTF-8 or UTF-16 in native encoding; strings are converted to bytevectors with string->utf8 or string->utf16n.

NOTE It appears that SQLite’s idea of zero–terminated UTF-16 array means that the array must end with 2 zero bytes. So to obtain a suitable UTF-16 bytevector pathname from a string we should do: 

(bytevector-append (string->utf16n pathname)
                   '#vu8(0 0))

This is exactly what sqlite3-open16 does when pathname is a string.

The following example shows how to open and close a database: 

#!r6rs
(import (vicare)
  (vicare databases sqlite3))

(let ((conn (sqlite3-open "sqlite.test.db")))
  (unwind-protect
      (sqlite3?/open conn)
    (when (sqlite3? conn)
      (sqlite3-close conn))))
⇒ #t ;if successful
Function: sqlite3-open-v2 pathname flags
Function: sqlite3-open-v2 pathname flags vfs-module

Open a new database connection to the selected file; if successful return a new instance of sqlite3, else return a SQLITE_ constant.

pathname must be a string, bytevector, pointer object or memory-block; the expected encoding is UTF-8 or UTF-16 in native encoding; strings are converted to bytevectors with string->utf8 or string->utf16n.

To create a private, temporary, in–memory database we must use :memory: as pathname.

flags must be an exact integer representing database configuration flags.

The optional vfs-module must be #f or a string, UTF-8 bytevector, pointer object or memory-block representing the name of a VFS module; when #f or not given: the default module is used; when a string: it is converted to bytevector with string->utf8. 

#!r6rs
(import (vicare)
  (vicare databases sqlite3))

(let ((conn (sqlite3-open-v2 "sqlite.test.db"
                             (fxior SQLITE_OPEN_READWRITE
                                    SQLITE_OPEN_CREATE))))
  (unwind-protect
      (sqlite3?/open conn)
    (when (sqlite3? conn)
      (sqlite3-close conn))))
⇒ #t ;if successful

2.4.4 Setting a connection–busy handler

The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-busy-handler connection
Function: sqlite3-busy-handler connection callback

Set a callback function that might be invoked whenever an attempt is made to open a database table that another thread or process has locked; if successful return SQLITE_OK, else return a SQLITE_ error code.

The optional callback must be a pointer object referencing a callback produced by make-sqlite3-busy-handler-callback; when #f or not given: no callback is used and NULL is handed to sqlite3_busy_handler().

Function: make-sqlite3-busy-handler-callback user-scheme-callback

Wrap a user supplied Scheme function and return a pointer object referencing a callback function suitable to be used as callback in calls to sqlite3-busy-handler; the returned pointer must be released with a call to free-c-callback from (vicare ffi).

user-scheme-callback must accept a single argument: an exact integer representing the number of times that the busy handler has been invoked for this locking event.

user-scheme-callback must return a single value which interpreted as answer to the question “retry to access the database?”; when true: another attempt is made to open the database for reading and the cycle repeats; when #f no additional attempts are made to access the database.

user-scheme-callback is wrapped in an internally generated Scheme function which does the following:

  • Block and discard all the raised exceptions using guard from (vicare); when an exception is catched: it is discarded and the callback returns 0 to the caller. user-scheme-callback should take care of handling its own exceptions.
  • If user-scheme-callback returns true: the wrapping callback returns 1 to the caller, else it returns 0.
Function: sqlite3-busy-timeout connection milliseconds

Set a busy handler that sleeps for a specified amount of time when a table is locked; if successful return SQLITE_OK else return a SQLITE_ error code.

milliseconds must be a fixnum representing the timeout span in milliseconds.

2.4.8 Controlling access to data

The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-set-authorizer connection callback

Register an authorizer callback with a particular database connection; if successful return SQLITE_OK, else return an SQLITE_ error code.

The authorizer callback is called while a statement is being compiled and must return a value to authorize or deny specific operations.

callback must be #f or a a pointer object referencing a callback produced by make-sqlite3-authorizer-callback; when #f: the previously registered callback, if any, is unregistered from the state of connection.

Function: make-sqlite3-authorizer-callback callback

Wrap a user supplied Scheme function and return a pointer object referencing a callback function suitable to be used as second argument in calls to sqlite3-set-authorizer; the returned pointer must be released with a call to free-c-callback from (vicare ffi).

user-scheme-callback must accept 5 arguments:

  1. An exact integer, in the range of the C language type int, representing an authorizer action code (SQLITE_INSERT, SQLITE_READ, SQLITE_SELECT, …).
  2. #f or a pointer object referencing a zero–terminated string in UTF-8 format; see SQLite documentation for details.
  3. #f or a pointer object referencing a zero–terminated string in UTF-8 format; see SQLite documentation for details.
  4. #f or a pointer object referencing a zero–terminated string in UTF-8 format; the string should be the database name (for example main).
  5. #f or a pointer object referencing a zero–terminated string in UTF-8 format; see SQLite documentation for details.

user-scheme-callback must return an exact integer, in the range of the C language type int, representing an authorizer return code: SQLITE_OK, SQLITE_DENY, SQLITE_IGNORE.

user-scheme-callback is wrapped in an internally generated Scheme function which does the following:

  • Block and discard all the raised exceptions using guard from (vicare). user-scheme-callback should take care of handling its own exceptions.
  • When an exception is catched: it is discarded and the callback returns SQLITE_DENY.
Function: sqlite3-authorizer-action-code->symbol code

Convert an exact integer, in the range of the C language type int, representing a SQLite authorizer action code to the corresponding symbol. 

(sqlite3-authorizer-action-code->symbol SQLITE_TRANSACTION)
⇒ SQLITE_TRANSACTION ;;symbol
Function: sqlite3-authorizer-return-code->symbol code

Convert an exact integer, in the range of the C language type int, representing a SQLite authorizer callback return code to the corresponding symbol. 

(sqlite3-authorizer-return-code->symbol SQLITE_DENY)
⇒ SQLITE_DENY ;;symbol

The following meaningless example shows the mechanism of registering an authorizer: 

#!r6rs
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants)
  (prefix (vicare ffi) ffi.))

(define-syntax with-connection
  (syntax-rules ()
    ((_ (?connect-var) . ?body)
     (let ((?connect-var (sqlite3-open ":memory:")))
       (unwind-protect
           (let () . ?body)
         (when (sqlite3? ?connect-var)
           (sqlite3-close ?connect-var)))))))

(define (authorizer action a b c d)
  (define-inline (cstr->str s)
    (and (not (pointer-null? s))
         (cstring->string s)))
  (printf "Authorizer code: ~a\n"
          (sqlite3-authorizer-action-code->symbol action))
  (printf "Authorizer info: ~a, ~a, ~a, ~a\n"
          (cstr->str a) (cstr->str b)
          (cstr->str c) (cstr->str d))
  SQLITE_OK)

(define sql-preparation
  "create table Stuff
     (A TEXT, B TEXT, C TEXT);
   insert into Stuff (A, B, C)
     values ('one', 'two', 'three');
   insert into Stuff (A, B, C)
     values ('four', 'five', 'six');")

(define sql-statement
  "select * from Stuff;")

(let ((auth-cb (make-sqlite3-authorizer-callback authorizer)))
  (unwind-protect
      (with-connection (conn)
        (sqlite3-exec conn sql-preparation)
        (sqlite3-set-authorizer conn auth-cb)
        (let-values (((code stmt end-offset)

2.5 Simple execution of SQL snippets

The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-exec connection sql-snippet
Function: sqlite3-exec connection sql-snippet each-row-callback

Execute the sql-snippet in the context of the database connection; when given, the each-row-callback is called to process the result. Return two values; when successful return SQLITE_OK and #f, else return an exact integer representing an error code and a string representing the associated error message.

sql-snippet must be a string or UTF-8 bytevector or pointer object or memory-block representing or referencing a snippet of SQL code; the string is converted to bytevector with string->utf8.

each-row-callback must be a pointer object referencing a callback produced by make-sqlite3-exec-callback; when not given, no callback is used.

Function: sqlite3-exec* arg ...

Like sqlite3-exec, but return a single value being the error code; the error message is discarded.

Function: make-sqlite3-exec-callback user-scheme-callback

Wrap a user supplied Scheme function and return a pointer object referencing a callback function suitable to be used as each-row-callback in calls to sqlite3-exec; the returned pointer must be released with a call to free-c-callback from (vicare ffi).

user-scheme-callback must accept 3 arguments:

  1. An exact integer representing the number of elements in the result.
  2. A vector of #f or UTF-8 bytevectors representing texts resulting from the SQL command; its length equals the value of the first argument.
  3. A vector of #f or UTF-8 bytevectors representing names resulting from the SQL command; its length equals the value of the first argument.

user-scheme-callback must return a single value which is the answer to the question “stop the query?”; if the returned value is true the call to sqlite3-exec returns immediately; if the returned value is #f the execution of SQL code goes on.

user-scheme-callback is wrapped in an internally generated Scheme function which does the following:

  • Block and discard all the raised exceptions using guard from (vicare); when an exception is catched: it is discarded and the callback returns SQLITE_ABORT to sqlite3_exec(). user-scheme-callback should take care of handling its own exceptions.
  • Check that the number of elements in the result can be represented with a fixnum; the maximum length of vectors in Vicare is the value returned by greatest-fixnum. If the number of elements cannot be represented with a fixnum: the callback returns SQLITE_ABORT to sqlite3_exec().
  • If user-scheme-callback returns true: the wrapping callback returns SQLITE_ABORT to sqlite3_exec(), else it returns SQLITE_OK.

The following example shows how to use the sqlite3-exec function with a simple callback that stores the results in global variables. 

#!r6rs
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants)
  (prefix (vicare ffi) ffi.))

(define database-pathname "sqlite.proof.db")
(define connection
  (sqlite3-open database-pathname))

(define sql-snippet
  "create table accounts
     (id       INTEGER PRIMARY KEY,
      nickname TEXT,
      password TEXT);
   insert into accounts (nickname, password)
     values ('ichigo', 'abcde');
   insert into accounts (nickname, password)
     values ('rukia', '12345');
   insert into accounts (nickname, password)
     values ('chad', 'fist');
   select * from accounts;")

(define result-names #f)
(define result-texts '())

(define each-row-callback
  (make-sqlite3-exec-callback
   (lambda (number-of-rows texts names)
     (unless result-names
       (set! result-names
             (map utf8->string (vector->list names))))
     (set! result-texts
           (cons (map utf8->string (vector->list texts))
                 result-texts))
     #f)))

(let-values (((rv error-message)
              (sqlite3-exec connection sql-snippet
                            each-row-callback)))
  (if (= SQLITE_OK rv)
      (begin
        (printf "names: ~a\n" result-names)
        (printf "texts: ~a\n" (reverse result-texts)))
    (printf "error: ~a\n" error-message)))
-| names: ("id" "nickname" "password")
-| texts: (("1" "ichigo" "abcde")
           ("2" "rukia" "12345")
           ("3" "chad" "fist"))

(ffi.free-c-callback each-row-callback)
(sqlite3-close connection)

2.7.1 Precompiled SQL statement example

The following example shows how to insert some data into a database and then extract it using a prepared statement: 

#!r6rs
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants))

(set-port-buffer-mode! (current-output-port)
                       (buffer-mode line))

(define database-pathname ":memory:")

(define sql-preparation
  "create table accounts
     (id       INTEGER PRIMARY KEY,
      nickname TEXT,
      password TEXT);
   insert into accounts (nickname, password)
     values ('ichigo', 'abcde');
   insert into accounts (nickname, password)
     values ('rukia', '12345');
   insert into accounts (nickname, password)
     values ('chad', 'fist');")

(define sql-query
  "select * from accounts;")

(let ((conn (sqlite3-open database-pathname)))
  (when (sqlite3?/open conn)
    (let-values (((rv error-message)
                  (sqlite3-exec conn sql-preparation)))
      (when (= rv SQLITE_OK)
        (let-values (((code stmt end-offset)
                      (sqlite3-prepare-v2 conn sql-query)))
          (when (= code SQLITE_OK)

                (do ((rv (sqlite3-step stmt) (sqlite3-step stmt))
                     (row 0 (+ 1 row)))

                         (not (= rv SQLITE_ROW))))

                    (printf "row ~a: ~a, ~a\n" row
                       (sqlite3-column-text/string stmt 1)
                       (sqlite3-column-text/string stmt 2))))
              (sqlite3-finalize stmt))))))
    (sqlite3-close conn)))

2.8 BLOBs for incremental input/output

This API accesses the data area of a BLOB in chunks, rather than as a whole memory block; this allows SQLite not to load in memory the whole contents of a BLOB, making some operations more efficient. The following bindings are exported by the (vicare databases sqlite3) library.

Struct Type: sqlite3-blob

A Scheme structure holding a reference to an instance of the C language type sqlite3_blob, it references a BLOB used for incremental input/output. (vicare-scheme)Structures and type descriptors

It has the following public fields:

connection

The instance of sqlite3 to which this BLOB is associated.

database

A string representing the name of the database in which this BLOB is contained.

table

A string representing the name of the table in which this BLOB is contained.

column

A string representing the name of the column in which this BLOB is contained.

rowid

An exact integer representing the id of the row in which this BLOB is contained.

write-enabled?

A boolean, #t if this BLOB can be written to.

When instances of this type are used as arguments to functions: this documentation identifies them as blob.

When instances of this type are garbage collected: sqlite3-blob-close is automatically applied to them.

Function: sqlite3-blob? obj

Return #t if obj is an instance of sqlite3-blob.

Function: sqlite3-blob?/open obj

Return #t if obj is an instance of sqlite3-blob and the BLOB it represents is open.

Function: sqlite3-blob-connection blob
Function: sqlite3-blob-database blob
Function: sqlite3-blob-table blob
Function: sqlite3-blob-column blob
Function: sqlite3-blob-rowid blob
Function: sqlite3-blob-write-enabled? blob

Accessors for the fields of sqlite3-blob instances.

Function: sqlite3-blob-destructor blob
Function: set-sqlite3-blob-destructor! blob func

Retrieve or set a destructor function associated to the blob.

Whenever the blob is closed, either explicitly with sqlite3-blob-close or implicitly by the garbage collector, func is applied to blob before the internal state of blob is finalised.

Notice that func can do anything, not only destroy some state associated to blob; if we do: 

(define blob ...)
(set-sqlite3-blob-destructor! blob
  (let ((state (make-some-data-struct)))
    (case-lambda
      ((blob)
       (destroy state))
      (()
       state))))

(let ((state ((sqlite3-blob-destructor blob))))
  (do-something-with blob state))

we can easily associate values to blob.

Function: sqlite3-blob-open connection database-name table-name column-name rowid write-enabled?

Open a handle to the BLOB located in row rowid, column column-name, table table-name in database database-name. If successful return 2 values: SQLITE_OK and an instance of sqlite3-blob; else return 2 values: a SQLITE_ error code and #f.

database-name must be a string, UTF-8 bytevector, pointer object or memory-block representing the database name; a string is converted to bytevector with string->utf8. The main database is called main.

table-name must be a string, UTF-8 bytevector, pointer object or memory-block representing the table name; a string is converted to bytevector with string->utf8.

column-name must be a string, UTF-8 bytevector, pointer object or memory-block representing the column name; a string is converted to bytevector with string->utf8.

rowid must be an exact integer in the range of the C language type sqlite3_int64 representing the row id.

If write-enabled? is true: the returned BLOB is opened for reading and writing; if it is #f: the returned BLOB is opened for reading only.

Move an existing blob handle so that it points to a different row of the same database table. If successful return SQLITE_OK else return a SQLITE_ error code.

rowid must be an exact integer in the range of the C language type sqlite3_int64 representing the row id.

Close the BLOB. If successful return SQLITE_OK, else return a SQLITE_ error code.

It is safe to apply multiple times this function to the same blob: the first time the BLOB is closed, the subsequent times nothing happens and the return value is SQLITE_OK.

Function: sqlite3-blob-bytes blob

Return an exact integer representing the size in bytes of the blob.

Function: sqlite3-blob-read src.blob src.offset dst.buffer dst.offset number-of-bytes

Read data from an open src.blob handle into a caller–supplied buffer. If successful return SQLITE_OK, else return a SQLITE_ error code.

  • When src.buffer is a pointer object: src.offset must be in the range of the C language type int.
  • When src.buffer is a bytevector: src.offset must be a fixnum.
  • When src.buffer is a memory-block: src.offset must be in the range of the C language type int.

#!vicare
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants))

(define-syntax with-connection
  (syntax-rules ()
    ((_ (?connect-var) . ?body)
     (let ((pathname "sqlite.test.db"))
       (unwind-protect
           (let ((?connect-var (sqlite3-open pathname)))
             (unwind-protect
                 (begin . ?body)
               (when (sqlite3? ?connect-var)
                 (sqlite3-close ?connect-var))))
         (when (file-exists? pathname)
           (delete-file pathname)))))))

(with-connection (conn)
  (sqlite3-exec conn "create table stuff \
     (id INTEGER PRIMARY KEY, data TEXT);")
  (let-values
       (((code stmt end-offset1)
         (sqlite3-prepare-v2 conn
            "insert into stuff (id, data) \
               values (?1, ?2);")))
    (unwind-protect
        (begin
          (sqlite3-bind-int64    stmt 1 1)
          (sqlite3-bind-zeroblob stmt 2 4)
          (sqlite3-step stmt))
      (when (sqlite3-stmt?/valid stmt)
        (sqlite3-finalize stmt))))
  (let-values
      (((rv blob)
        (sqlite3-blob-open conn "main" "stuff" "data" 1 #t)))
    (unwind-protect
        (begin
          (sqlite3-blob-write blob 0 '#ve(ascii "ciao") 0 4)
          (let ((buffer (make-bytevector 4)))
            (sqlite3-blob-read blob 0 buffer 0 4)
            (printf "~a\n" (utf8->string buffer))))
      (sqlite3-blob-close blob))))

  (let* ((x (vector-ref args 0))

2.13.1.2 The column sum aggregate

The following example shows how to implement and use the SQL aggregate function mysum, which sums values from a column of flonums; notice how the aggregate stores the accumulated sum in a block of raw memory allocated with sqlite3-aggregate-context and such block is reset to zero by SQLite, which is fine as starting value for a sum function: 

#!r6rs
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants)
  (prefix (vicare ffi) ffi.)
  (prefix (vicare words) words.))

(define-syntax with-connection
  (syntax-rules ()
    ((_ (?connect-var) . ?body)
     (let ((?connect-var (sqlite3-open ":memory:")))
       (unwind-protect
           (let () . ?body)
         (when (sqlite3? ?connect-var)
           (sqlite3-close ?connect-var)))))))

(define (mysum.data context)
  (sqlite3-aggregate-context context words.SIZEOF_DOUBLE))

(define (mysum.accumulated-sum-ref data.ptr)
  (pointer-ref-c-double data.ptr 0))

(define (mysum.accumulated-sum-set! data.ptr sum)
  (pointer-set-c-double! data.ptr 0 sum))

(define (mysum.step context args)
  (define (%error message)
    (sqlite3-result-error-code context SQLITE_ERROR)
    (sqlite3-result-error context message))
  (guard (E (else
             (%error (if (message-condition? E)
                         (condition-message E)
                       "internal error while executing \
                        mysum function"))))
    (let* ((P (mysum.data context))
           (x (vector-ref args 0))
           (T (sqlite3-value-numeric-type x)))
      (if (or (= T SQLITE_INTEGER)
              (= T SQLITE_FLOAT))
          (let* ((S (mysum.accumulated-sum-ref P))
                 (A (vector-ref args 0))
                 (X (sqlite3-value-double A))
                 (S (+ X S)))
            (mysum.accumulated-sum-set! P S))
        (mysum.accumulated-sum-set! P +nan.0)))))

(define (mysum.final context)
  (define (%error message)
    (sqlite3-result-error-code context SQLITE_ERROR)
    (sqlite3-result-error context message))
  (guard (E (else
             (%error (if (message-condition? E)
                         (condition-message E)
                       "internal error while executing \
                        mysum function"))))
    (let* ((P (mysum.data context))
           (S (mysum.accumulated-sum-ref P)))
      (if (nan? S)
          (%error "expected numeric argument for \
                   mysum function")
        (sqlite3-result-double context S)))))

(define (exec-cb number-of-cols texts names)
  (printf "~a: ~a\n"
     (utf8->string (vector-ref names 0))
     (string->number (utf8->string (vector-ref texts 0))))
  #f)

2.13.1.3 The column max aggregate

The following example shows how to implement and use the SQL aggregate function mymax, which selects the maximum value from a column of flonums; notice that the aggregate stores the current maximum in a block of raw memory allocated with sqlite3-aggregate-context along with an initialisation flag:

(define mymax.context-size
  (+ words.SIZEOF_LONG words.SIZEOF_DOUBLE))

(define (mymax.context-pointer context)
  (sqlite3-aggregate-context context mymax.context-size))

(define-inline (mymax.initialised! pointer)
  (pointer-set-c-unsigned-long! pointer 0 1))

(define-inline (mymax.initialised? pointer)
  (not (zero? (pointer-ref-c-unsigned-long pointer 0))))

(define-inline (mymax.max-set! pointer flonum)
  (pointer-set-c-double! pointer words.SIZEOF_LONG flonum))

(define-inline (mymax.max-ref pointer)
  (pointer-ref-c-double  pointer words.SIZEOF_LONG))

(define (mymax.step context args)
  (define (%error message)
    (sqlite3-result-error-code context SQLITE_ERROR)
    (sqlite3-result-error context message))
  (guard (E (else
             (%error (if (message-condition? E)
                         (condition-message E)
                       "internal error while executing \
                        mysum function"))))
    (let* ((P (mymax.context-pointer context))
           (A (vector-ref args 0))
           (T (sqlite3-value-numeric-type A)))
      (if (or (= T SQLITE_INTEGER)
              (= T SQLITE_FLOAT))
          (let ((X (sqlite3-value-double A))
                (M (mymax.max-ref P)))
            (if (mymax.initialised? P)
                (unless (nan? M)
                  (mymax.max-set! P (max X M)))
              (begin
                (mymax.initialised! P)
                (mymax.max-set! P X))))
        (begin
          (unless (mymax.initialised? P)
            (mymax.initialised! P))
          (mymax.max-set! P +nan.0))))))

(define (mymax.final context)
  (define (%error message)
    (sqlite3-result-error-code context SQLITE_ERROR)
    (sqlite3-result-error context message))
  (guard (E (else
             (%error (if (message-condition? E)
                         (condition-message E)
                       "internal error while executing \
                        mysum function"))))
    (let ((P (mymax.context-pointer context)))
      (if (mymax.initialised? P)
          (let ((M (mymax.max-ref P)))
            (if (nan? M)
                (%error "expected numeric argument for \
                         mymax function")
              (sqlite3-result-double context M)))
        -inf.0))))

(define (regexp context args)

                  (%compile-rex context rex))))
    (sqlite3-result-int context
                        (if (glibc.regexec cre str 0)
                            1
                          0))))

2.13.2 Creating new SQL functions

The following functions are used to add SQL scalar functions or aggregate functions or to redefine the behavior of existing SQL scalar functions or aggregate functions. The following bindings are exported by the (vicare databases sqlite3) library.

Function: sqlite3-create-function connection function-name arity text-encoding custom-data func step final

Create a new SQL scalar or aggregate function, or override an existing SQL scalar or aggregate function, in the context of connection. If successful return SQLITE_OK, else return a SQLITE_ error code.

function-name must be a string or UTF-8 bytevector or pointer object or memory-block representing the name of the new SQL function; when a string, it is converted to bytevector with string->utf8; its length in bytes must not exceed 255 bytes.

arity must be an exact integer representing the number of arguments to the SQL function.

text-encoding must be an exact integer representing one of the constants: SQLITE_UTF8, SQLITE_UTF16LE, SQLITE_UTF16BE, SQLITE_UTF16, SQLITE_ANY, SQLITE_UTF16_ALIGNED. It specifies which text encoding this SQL function prefers for its parameters; when no preference is needed, we should use SQLITE_ANY.

custom-data must be #f or a pointer object; #f is internally converted to a NULL pointer. When a pointer object: this argument must reference custom data for the SQL function, which can be retrieved with sqlite3-user-data.

func must be #f or a pointer object representing NULL or a pointer object referencing a callback function implementing a SQL scalar function; when a non-NULL pointer, it should be the return value of a call to make-sqlite3-function.

step must be #f or a pointer object representing NULL or a pointer object referencing a callback function implementing the step of a SQL aggregate function; when a non-NULL pointer, it should be the return value of a call to make-sqlite3-aggregate-step.

final must be #f or a pointer object representing NULL or a pointer object referencing a callback function implementing the finalisation of a SQL aggregate function; when a non-NULL pointer, it should be the return value of a call to make-sqlite3-aggregate-final.

Function: sqlite3-create-function16 connection function-name arity text-encoding custom-data func step final

Like sqlite3-create-function, but function-name must be a string or UTF-16 bytevector (in native byte order) or pointer object or memory-block representing the name of the new SQL function; when a string, it is converted to bytevector with string->utf16n; its length in bytes must not exceed 255 bytes.

Function: sqlite3-create-function-v2 connection function-name arity text-encoding custom-data func step final destroy

Like sqlite3-create-function, but accept the additional argument destroy.

destroy must be #f or a pointer object representing NULL or referencing a callback function:

2.15 Defining collating sequences

Collation is the act of bringing things together and comparing them; SQLite allows applications to define comparison functions to be used through the COLLATE SQL clause. For example, whenever we select rows from a table: 

create table Stuff (alpha TEXT);
insert into Stuff (alpha) values ('ciao');
insert into Stuff (alpha) values ('hello');
insert into Stuff (alpha) values ('salut');
select * from Stuff;

select * from Stuff order by alpha collate ThisWay;

#!r6rs
(import (vicare)
  (vicare databases sqlite3)
  (vicare databases sqlite3 constants)
  (prefix (vicare ffi) ffi.))

(define-syntax with-connection
  (syntax-rules ()
    ((_ (?connect-var) . ?body)
     (let ((?connect-var (sqlite3-open ":memory:")))
       (unwind-protect
           (let () . ?body)
         (when (sqlite3? ?connect-var)
           (sqlite3-close ?connect-var)))))))

(define sql-snippet
  "create table Stuff (alpha TEXT);
   insert into Stuff (alpha) values ('hello');
   insert into Stuff (alpha) values ('salut');
   insert into Stuff (alpha) values ('ciao');
   select * from Stuff order by alpha collate ThisWay;")

(define (collation.comparison S1 S2)
  (cond ((string<? S1 S2)   -1)
        ((string=? S1 S2)   0)
        (else               +1)))

(define (collation.callback custom-data len1 ptr1 len2 ptr2)
  (let ((S1 (cstring->string ptr1 len1))
        (S2 (cstring->string ptr2 len2)))
    (collation.comparison S1 S2)))

(define collation-cb
  (make-sqlite3-collation-callback collation.callback))

(define (needed-callback custom-data conn encoding name)
  (cond ((string=? name "ThisWay")
         (sqlite3-create-collation conn "ThisWay" encoding
                                   #f collation-cb))))

(define (exec-callback nrows texts names)
  (printf "~a: ~a\n"
          (utf8->string (vector-ref names 0))
          (utf8->string (vector-ref texts 0)))
  #f)

(let ((needed-cb (make-sqlite3-collation-needed-callback
                    needed-callback))
      (exec-cb   (make-sqlite3-exec-callback
                    exec-callback)))
  (unwind-protect
      (with-connection (conn)
        (sqlite3-collation-needed conn #f needed-cb)
        (sqlite3-exec* conn sql-snippet exec-cb))
    (ffi.free-c-callback needed-cb)
    (ffi.free-c-callback exec-cb)
    (ffi.free-c-callback collation-cb)))

-| alpha: hello
-| alpha: salut

2.16.1 URI parameters

SQLite lets us specify configuration values, when opening a database, as URI parameters; for example we could do: 

(sqlite3-open-v2 "file:///file.db?mode=ro"
                 SQLITE_OPEN_CREATE)

to open a database in read–only mode. Application–defined virtual file systems are allowed to define their configuration parameters; such parameters can be read by the VFS xOpen() method with the functions documented here.

Notice that the split-uri argument is not the plain URI handed to the sqlite3-open-v2 function, rather it is a preprocessed version in which zero bytes are used to split the string in its fields (at least in SQLite version 3.7.13); if the original string is: 

"file:///file.db?mode=ro"

the split-uri argument is the string: 

"file:///file.db\x0;mode\x0;ro"
