Abstract

This is the home page of Vicare an implementation of the Scheme language compliant with the R6RS standard. It is a fork of Ikarus Scheme by Abdulaziz Ghuloum.

The project aims at being a native compiler for R6 Scheme producing single threaded programs running on Intel x86 processors; it is to be considered alpha quality. Development is hosted at Github and as backup at Gitorious.

Introduction

Scheme is a statically scoped and properly tail-recursive dialect of the Lisp programming language, invented by Guy Lewis Steele Jr. and Gerald Jay Sussman. It was designed to have an exceptionally clear and simple semantics and few different ways to form expressions.

The Revised6 Report on the Algorithmic Language Scheme gives a defining description of the programming language Scheme. The report is the work of many people in the course of many years. Revision 6 was edited by Michael Sperber, R. Kent Dybvig, Matthew Flatt and Anton Van Straaten.

Ikarus Scheme is an almost R6RS compliant implementation of the Scheme programming language; it is the creation of Abdulaziz Ghuloum, which retired from deveoping it in early 2010.

Vicare (pronounced the etruscan way) is an R6RS compliant fork of Ikarus Scheme, aiming at being a native compiler for R6 Scheme producing single threaded programs running on Intel x86 32-bit processors, with experimental support for 64-bit machines. The documentation is available on line.

Here is a list of other known R6RS implementations: Chez Scheme, Larceny, Mosh, Racket, Ypsilon.

Download

The package archive (tarball) can be downloaded from the download pages at GitHub or Google Code.

The development revisions can be downloaded directly from the GitHub project page. Users should especially track the master and devel branches.

GNU+Linux systems are the only platform supported so far; however it should be possible to run Vicare on Mac OS X, FreeBSD, NetBSD, and Microsoft Windows XP using the Cygwin environment.

Community

This project has a mailing list for announcements and discussions. Bug reports should be registerd at Github's Issue Tracker. It is suggested to use Vicare along with Nausicaa, a large collection of libraries for R6 Scheme.

Development

The R6 Scheme language is fully implemented. The code base can be partitioned in:

  • Executable run-time support program written in C.
  • Scheme code implementing the assembler and the compiler.
  • Scheme libraries implementing the language.
  • Multilayer interface to the operative system services.

the current development goal is to put the code in such a shape that: the run-time support program and the language implementation will not change anymore and are just subject to maintenance (mostly bug fixes); only the assembler, compiler and interface to the OS should see further significant work.
